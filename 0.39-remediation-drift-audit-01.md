# v0.39 / v0.39b Remediation Drift Audit 01

Date: 2026-05-27

Scope: final post-remediation audit after implementing the findings from
`0.39-impl-audit-03.md`, reconciled against `0.39-impl-audit-02.md`, current
plans, roadmap intent, and the implemented code.

## Result

Automated remediation is complete and release-gate clean. The branch is ready
for operator manual validation before release tagging.

The v0.39 intent is now represented end to end:

- First-run onboarding is evidence-aware in CLI and workspace surfaces.
- Operators see exact next commands, current provider/model evidence, doctor
  targets, channel credential state, and Active Memory readiness.
- Channel selection no longer claims configuration success; Telegram/email
  choices are recorded as `selected` until configured, while `none` is skipped.
- Provider doctor diagnostics use the ADR 0047 fixed catalog.
- Local-first defaults are consistent across resolved settings and schema
  metadata.

The v0.39b intent is now represented end to end:

- Identity memory is a system memory namespace, not an app.
- Active Memory retrieves only reviewed `:kept` entries.
- Long entries are split into deterministic byte windows before scoring.
- Returned windows are complete bounded chunks and do not append ellipses.
- Direct-answer retrieval uses the request-start timestamp when available.
- Trace metadata remains body-free.

## Final Drift Found And Fixed

The final audit found one additional consistency issue after the milestone
commits: `Settings.defaults()` had been changed to `local`, but
`Settings.schema()` still advertised `fast` as the default for
`intent.model_profile` and `intent.direct_answer_model_profile`.

Fix:

- Updated both schema metadata entries to `local`.
- Added settings tests that pin resolved defaults and schema metadata together.

I also renamed the runtime-boundary-map column from `Planned boundary` to
`Boundary` so implemented v0.39/v0.39b rows no longer sit under stale wording.

## Manual Validation Checklist

Use a disposable Allbert Home:

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v039-release.XXXXXX)"
mix allbert.onboard
```

Then validate:

1. CLI onboarding prints evidence and next command for each current step.
2. Workspace onboarding at
   `http://localhost:4000/workspace?destination=workspace:onboard` renders
   without overlap in Chrome extension browser control.
3. Selecting Telegram/email records `selected` unless configured; choosing
   none records skipped.
4. `mix allbert.model doctor local` reports the local endpoint/model state with
   fixed ADR 0047 diagnostics and no auto-pull.
5. A credentialed remote doctor path reports only redacted host and fixed
   diagnostics.
6. Identity memory smoke:

   ```sh
   mkdir -p "$ALLBERT_HOME/memory/identity"
   printf '%s\n' '# Persona' 'Prefers concise release reports.' > "$ALLBERT_HOME/memory/identity/persona.md"
   mix allbert.memory review "$ALLBERT_HOME/memory/identity/persona.md" --user local --status kept --note "Operator-authored identity"
   mix allbert.memory retrieve --user local --query "concise release reports"
   ```

7. Direct-answer trace shows an `Active Memory` section when model mode is
   enabled, and trace metadata does not include memory body text.
8. Disabled model mode skips direct-answer model execution safely.
9. Classifier/intent paths do not consume memory bodies.
10. Replay remains deterministic for the same request timestamp and settings.
