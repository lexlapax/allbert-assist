# v0.43 Remediation Plan From Audits 01 And 02

Date: 2026-06-01
Inputs:

- `v0.43-impl-audit-01.md`
- `v0.43-impl-audit-02.md`

Purpose: consolidate the two post-implementation audits into one followable
remediation plan for closing the v0.43 browser/web-research gaps.

## Decision

v0.43 must ship with actual local Playwright control. The remediation path is
not "document the placeholder"; it is to replace the placeholder with a working
local Chromium driver, prove it with a real external smoke, and then align the
release record with that behavior.

A doc-only downgrade is not an acceptable v0.43 remediation path. If the bridge
is deferred, v0.43 must not be described as operational browser research.

## Playwright Research Notes

Current Playwright Node documentation supports the intended local sidecar shape:

- Use `require("playwright").chromium` for Chromium automation.
- Launch with `chromium.launch(...)`, headless by default.
- Use isolated, non-persistent browser contexts via `browser.newContext(...)`.
- Open a page with `context.newPage()`.
- Navigate with `page.goto(url, {waitUntil: "domcontentloaded", timeout})`.
- Extract HTML with `page.content()`.
- Extract visible text with a bounded `page.evaluate(() =>
  document.body?.innerText || "")`.
- Capture PNG screenshots with `page.screenshot(...)`, returning a buffer or
  writing a path.
- Handle downloads through Playwright download events and then inspect/save or
  delete the downloaded artifact under browser cache control.
- Always close page/context/browser resources with `context.close()` and
  `browser.close()`.

Implementation implication: the bridge should use non-persistent contexts, not
operator profiles or authenticated persistent browser state. One browser process
may serve multiple bounded sessions, but each Allbert session should own an
isolated Playwright context and page so close/timeout behavior is reliable.

## Audit Agreement

The audits agree on the important release verdict:

- v0.43 has a strong security and architecture substrate.
- The release gates are green and the deterministic browser/security tests are
  valuable.
- The real operator browser runtime is not complete.
- `AllbertBrowser.Driver.Playwright` is a placeholder and cannot launch,
  navigate, extract, screenshot, fill, click, or download.
- The current external-smoke story does not prove real local Chromium behavior.
- Operator-facing docs and release wording overstate what v0.43 can do today.
- A corrective pass is needed before calling browser research operational.

The audits also agree that these shipped foundations should be preserved:

- registered browser actions through the action runner
- browser-specific operation classes and resource URIs
- conservative permission floors
- fill and download denied by default
- browser content treated as evidence, not authority
- deterministic stub-driver test coverage
- release.v043 gate and security eval inventory
- cache metadata sidecars and workspace panel contribution

## Reconciled Differences

### External smoke wording

Audit 01 says `external-smoke -- browser_research` runs against the placeholder
Playwright path and produces no meaningful smoke evidence. Audit 02 found that
the v0.43 browser security eval setup pins `AllbertBrowser.Driver.Stub`, so the
named external-smoke path can also pass without real Playwright.

Resolution:

- Treat both as the same release blocker: there is no true real-browser external
  smoke.
- The remediation must add a separate external smoke that requires
  `AllbertBrowser.Driver.Playwright`, launches Chromium, navigates to a local
  fixture, extracts content, captures a screenshot, and closes the session.

### Doctor live-verify wording

Audit 01 says doctor live-verify wiring exists. Audit 02 says doctor cannot
perform ADR-required live verification because the Playwright driver is a
placeholder.

Resolution:

- The doctor orchestration and persisted result shape exist.
- The live verification is not functionally complete until the real driver can
  launch Chromium, navigate to `about:blank`, close, and report capabilities.

### Milestone completeness wording

Audit 01 says the per-milestone substrate acceptance is mostly met. Audit 02
marks M1/M3/M4 incomplete for operator release readiness.

Resolution:

- The substrate milestone work is largely implemented.
- The operator-runtime acceptance is incomplete.
- The remediation plan below treats runtime readiness, cache/session bounds, and
  smoke evidence as release blockers.

## Remediation Scope

### Release blockers

These must be fixed before v0.43 is considered operational:

1. Implement the real local Playwright bridge.
2. Make `browser_doctor` perform a true live verification through that bridge.
3. Replace/augment the current external smoke with a real-browser smoke.
4. Enforce session max lifetime and idle timeout.
5. Contribute the browser cache sweep job automatically and idempotently.
6. Enforce `browser.cache.max_bytes`.
7. Reconcile operator docs, plan docs, request-flow docs, ADR text, roadmap,
   README, and changelog with the corrected runtime.

### Important follow-ups in the same remediation pass

These are lower risk than the blockers but should be done before closeout to
avoid a fourth audit pass:

1. Decide and implement `browser.navigation.allowed_domains` semantics.
2. Implement or explicitly descope `mix allbert.browser research`.
3. Align workspace panel documentation and implementation naming/visibility.
4. Replace unsafe `elem(Settings.get(...), 1)` settings reads.
5. Reconcile the action count in the plan: 12 actions, not 10.
6. Clarify PDF extraction scope as text-layer-only and fail-closed for
   encrypted, scanned, or malformed PDFs.
7. Clarify session lifecycle permission taxonomy or document the current
   permission reuse.

### R1 - Real Playwright Driver And Doctor

Goal: replace the production placeholder with a local, policy-bounded browser
driver.

Implementation:

- Add the reviewed local Playwright bridge under the browser plugin.
- Keep release tests free of package installation and external network needs.
- Use a small JSON-over-stdio or equivalent local protocol between Elixir and
  the Node bridge. Keep the command set explicit:
  - `verify`
  - `start_session`
  - `navigate`
  - `extract`
  - `screenshot`
  - `click`
  - `fill`
  - `download`
  - `close_session`
  - `shutdown`
- Model each Allbert browser session as an isolated Playwright
  `BrowserContext` plus one active `Page`.
- Use `chromium.launch` in headless mode.
- Use `browser.newContext` rather than persistent profiles.
- Use `context.newPage` for page creation.
- Use `page.goto(..., waitUntil: "domcontentloaded")` for navigation.
- Use `page.content` and bounded `page.evaluate` for extraction.
- Use `page.screenshot` for PNG screenshots.
- Use Playwright download events only inside the already-confirmed download
  action path.
- Close contexts on session close, idle timeout, max lifetime, and bridge
  shutdown.
- Implement `AllbertBrowser.Driver.Playwright` callbacks:
  - `verify/1`
  - `start_session/1`
  - `navigate/3`
  - `extract/3`
  - `screenshot/2`
  - `click/3`
  - `fill/3`
  - `download/3`
  - `close/1`
- Preserve existing action-level policy. The driver must not grant permission
  or bypass confirmations.
- Make `verify/1` launch an ephemeral Chromium context, navigate to
  `about:blank`, close cleanly, and return capability/version data.
- Make `Doctor.run/1` persist enough data for operator troubleshooting:
  driver kind, Playwright/Chromium availability, live check status,
  last verified timestamp, bridge version, and error category.
- Ensure `browser_start_session` still fails closed when doctor is missing,
  stale, or not `:ok`.

Focused validation:

- Existing browser action tests with stub driver.
- New focused Playwright-driver unit tests using a fake bridge boundary where
  possible.
- `MIX_ENV=test mix test ../../plugins/allbert.browser/test` or the closest
  existing browser plugin focused lane.
- `MIX_ENV=test mix allbert.test release.v043` after R1 if the bridge changes
  action/doctor behavior.

External smoke:

- Add a real smoke that is opt-in and local:
  - set `ALLBERT_BROWSER_EXTERNAL_SMOKE=1`
  - run `browser_doctor`
  - start a session
  - navigate to a local fixture HTTP page
  - extract text
  - capture a screenshot
  - close the session
- The smoke must use `AllbertBrowser.Driver.Playwright`, not
  `AllbertBrowser.Driver.Stub`.
- For release handoff, Playwright/Chromium unavailability is a failure. A generic
  developer machine may report an environment skip only outside release
  handoff.

Development lane:

- Serial barrier. Other work can prepare tests/docs in parallel, but no release
  closeout should proceed until this proves real local browser behavior.

Commit point:

- Commit after focused tests and external smoke are green.

### R2 - Session And Cache Bounds

Goal: make the local-resource safety claims true.

Implementation:

- Add session `last_activity_at` state.
- Schedule max-lifetime termination from
  `browser.session.max_lifetime_ms`.
- Schedule idle termination from `browser.session.idle_timeout_ms`.
- Refresh idle timeout after navigate, click, fill, download, extract, and
  screenshot.
- On timeout or normal close, call `Driver.close/1`.
- Keep `browser.session.max_concurrent` enforcement.
- Wire `AllbertBrowser.Cache.ensure_sweep_job/0` into a safe, idempotent plugin
  registration/setup path.
- Keep the sweep job paused by default unless
  `browser.cache.sweep.schedule == "operator_approved"`.
- Enforce `browser.cache.max_bytes` with deterministic oldest-first eviction.
- Keep age-based eviction from `browser.cache.max_age_ms`.

Focused validation:

- Session closes automatically on max lifetime.
- Session closes automatically on idle timeout.
- Driver `close/1` is called on timeout.
- Dead sessions disappear from list output.
- Cache sweep job is contributed on plugin setup/registration.
- Cache age eviction still works.
- Cache size eviction removes oldest artifacts and preserves metadata
  consistency.

Development lane:

- Parallelizable after R1 driver interface stabilizes:
  - session bounds lane
  - cache bounds/job lane
- Rejoin before validation because both touch browser runtime behavior.

Commit point:

- Commit after focused session/cache tests pass.

### R3 - Policy, CLI, Surface, And Small Correctness Gaps

  [--extract-format=text|markdown|html|pdf]`, or remove it from v0.43 docs.

- Clarify session lifecycle permissions:

- Update plan action table to include `browser_sweep_cache` and

- Update plan/ADR PDF language to text-layer-only, fail-closed for encrypted,
  scanned, and malformed PDFs.

- Workspace surface/provider focused tests, if code changes.

## External Smoke Definition

The remediation is not complete until this smoke exists and is exercised:

1. Configure a temporary `ALLBERT_HOME` and test database path.
2. Enable browser support through Settings Central.
3. Run `browser_doctor` with `AllbertBrowser.Driver.Playwright`.
4. Assert doctor status is `:ok` and `live_check_status` is `:ok`.
5. Start a browser session.
6. Navigate to a local fixture URL served by the test harness.
7. Extract text and assert fixture content appears.
8. Capture a screenshot and assert a non-empty PNG cache artifact is written.
9. Close the session.
10. Assert no live session remains.
11. Record evidence path and driver/browser versions.

This smoke must not use remote websites, user profiles, saved cookies, real
secrets, or the operator's real `~/.allbert`.

## Non-Goals For This Remediation

Do not expand the v0.43 remediation into these future browser features:

- persistent or authenticated browser profiles
- headed mode
- multi-tab or multi-window orchestration
- arbitrary JavaScript evaluation actions
- OCR for scanned PDFs
- broad document parsing beyond the bounded v0.43 extraction scope
- Windows/WSL2 driver support unless it falls out naturally and is explicitly
  tested

## If The Bridge Cannot Ship

If the real Playwright bridge cannot be implemented in this remediation pass,
stop release closeout. Do not ship or describe v0.43 as operational browser
research.

The only acceptable non-bridge outcome is a non-release disclosure package:

1. Add a known-limitation section to `CHANGELOG.md`.
2. Downgrade `docs/operator/browser-and-research.md` to say v0.43 contains only
   the policy/action/test substrate.
3. Add "Playwright bridge implementation" to the roadmap deferred list and
   future-features plan.
4. Make `mix allbert.test external-smoke -- browser_research` fail with a
   structured message saying the bridge is not implemented.
5. Keep the label:
   "v0.43 browser architecture/security foundation; real browser research is
   not shipped."

That path is a release stop, not a successful remediation.
