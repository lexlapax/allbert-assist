### D4 — Doctor `error_category` field not present (P3)

Audit-03 R1 specified: "Make `Doctor.run/1` persist enough data for
operator troubleshooting: driver kind, Playwright/Chromium
availability, live check status, last verified timestamp, bridge
version, and **error category**."

`doctor.ex:22-28` failure path:

```elixir
{:error, reason} ->
  %{
    status: :error,
    live_check_status: :unavailable,
    checked_at: DateTime.to_iso8601(checked_at),
    error: inspect(reason)
  }
```

The `inspect(reason)` carries the category implicitly (e.g.
`":playwright_bridge_unavailable"`, `":start_session_timeout"`) as a
string. There's no separate structured `error_category` atom field. An
operator parsing `state.json` to choose a remediation path has to
string-match the inspect output.

**Why this matters:** small. Operators get the information they need;
they just have to grep the string instead of reading a clean atom.
v0.43.x or v0.43.1 can tighten this without ceremony.

**Fix (deferred):** add `error_category` atom field on the failure
path (e.g., `:playwright_bridge_unavailable`, `:bridge_timeout`,
`:chromium_launch_failed`, `:bridge_protocol_error`). Update operator
doc with the enum.
