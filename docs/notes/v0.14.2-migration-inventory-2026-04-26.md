# v0.14.2 Migration Inventory

Status: Stub. Populate during v0.14.2 M0 before moving code.

This document is the checked-in handoff artifact for retiring `allbert-kernel` and moving to direct `allbert-kernel-core` / `allbert-kernel-services` imports.

## Generation Contract

M0 must generate the initial inventory from the current `crates/allbert-kernel/src/lib.rs` `pub mod` and `pub use` surface, then reconcile it by hand where a symbol's destination is ambiguous.

Required sources:

- `crates/allbert-kernel/src/lib.rs`
- every public module under `crates/allbert-kernel/src/`
- workspace imports in CLI, daemon, jobs, and tests
- current Cargo feature declarations, especially `integration-trainer`

## Release-Exit Checks

- No workspace Rust source imports `allbert_kernel`.
- No Cargo manifest or `Cargo.lock` contains package `allbert-kernel`.
- `crates/allbert-kernel` is removed from the workspace.
- Core/services import fixtures compile.
- Self-improvement path fixtures recognize `crates/allbert-kernel-core` and `crates/allbert-kernel-services`.
