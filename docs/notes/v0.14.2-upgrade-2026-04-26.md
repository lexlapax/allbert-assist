# v0.14.2 Upgrade Notes

v0.14.2 is a structural release after v0.14.1. It keeps the operator-visible product surface steady while making the source tree more honest and enforceable.

## What Changed

- The old `allbert-kernel` crate is retired.
- Runtime contracts live in `allbert-kernel-core`.
- Concrete behavior and default runtime wiring live in `allbert-kernel-services`.
- Workspace crates import those two crates directly; there is no compatibility facade for the old Rust `allbert_kernel` API.
- Release validation now enforces kernel size, crate graph, import migration, and dependency compactness checks.
- Default-parallel daemon integration validation no longer relies on the earlier serial-test workaround for local socket boot reliability.

## Operator Impact

No profile migration, protocol bump, storage migration, or command migration is required for normal source-based users.

Fresh profiles still default to local Ollama with `gemma4` and the TUI. Existing profiles keep their previous UI mode and settings.

## Contributor Impact

Source changes that previously imported `allbert_kernel` should now choose:

- `allbert_kernel_core` for config, paths, policy, provider/tool DTOs, shared errors, and other runtime contracts.
- `allbert_kernel_services` for concrete runtime entrypoints, memory implementations, skills, adapters, self-diagnosis, self-improvement, scripting, replay, local utilities, and concrete providers.

The release gate remains strict: `allbert-kernel-services/src/` must stay below 30,000 LOC, and production service code must stay under `src/` so it counts against that gate. Large test-only unit suites may live in crate-local test support only when they remain compiled by `cargo test`.

## Validation

The v0.14.2 release validation path is:

```bash
cargo fmt --check
env -u RUSTC_WRAPPER cargo clippy --workspace --all-targets -- -D warnings
env -u RUSTC_WRAPPER cargo test -q
env -u RUSTC_WRAPPER cargo run -q -p allbert-cli -- --help
env -u RUSTC_WRAPPER cargo doc --workspace
tools/check_doc_reality.sh
tools/check_kernel_size.sh --enforce
tools/check_kernel_crate_graph.sh --enforce
tools/check_kernel_import_migration.sh --enforce
tools/check_kernel_dependency_compactness.sh --enforce
```
