# v0.14.1 Upgrade Notes

v0.14.1 is a repair release for the v0.13/v0.14 surfaces that had drifted ahead of the running code.

## What Changed

- Daemon adapter protocol v5 messages now bridge to the same local adapter store and training runner used by the CLI.
- Adapter training now resolves production trainers from `learning.adapter_training.default_backend` or the protocol request backend override, and fails closed when training is disabled or the backend is missing, unknown, disallowed, or blocked by exec policy.
- The tool-call parser accepts the local-default Ollama/Gemma4 variants for `input`, `arguments`, nested `function`, and direct `program`/`args` calls, then authorizes direct spawns through the active tool catalog, skill `allowed-tools`, and exec policy.
- `diagnose run --remediate` now records candidate/fallback provenance. Daemon-started remediation uses the attached provider when available; offline CLI remediation records `fallback:offline_no_provider`.
- A doc-reality check now runs in the workspace test suite so future plan/docs overclaims have to be marked honestly.

## Operator Notes

Existing profiles load with defaults for the new config keys:

```toml
[intent]
tool_call_retry_enabled = true

[self_diagnosis]
remediation_provider_max_tokens = 4096
```

No protocol bump is required. v0.14.1 completes existing v5 adapter and v6 diagnosis surfaces.

## Validation
