# v0.11 Upgrade Notes

v0.11 is the TUI and adaptive-memory release. It adds a richer terminal UI, daemon-owned telemetry, configurable memory routing, explicit episode/fact recall tiers, an optional semantic-retrieval seam, and a review-first personality digest seam.

## What Changes

- Fresh profiles default to the TUI.
- Existing upgraded profiles keep classic REPL unless you opt into TUI.
- `/telemetry`, `/statusline`, `/memory routing`, and `/memory stats` are available from both TUI and classic REPL.
- `allbert-cli telemetry --json` prints the same kernel-owned telemetry used by the TUI status line.
- `memory-curator` is always eligible by default, but its full body is activated only by routing policy.
- Session journals are indexed as explicit `episode` recall. They remain working history, not approved durable memory.
- Staged/promoted memory may carry temporal fact metadata. Fact recall remains review-first and durable only after promotion.
- Semantic retrieval remains disabled by default. v0.11 ships the fake deterministic semantic provider for validation; real embedding adapters are future additive work.
- `PERSONALITY.md` is optional and absent by default. It is a lower-authority learned overlay, not a replacement for `SOUL.md`.
- `personality-digest` is disabled by default and writes drafts before any accepted install.

## Operator Action

To try the TUI once:

```bash
cargo run -p allbert-cli -- repl --tui
```

To keep using classic REPL:

```bash
cargo run -p allbert-cli -- repl --classic
```

To persist either choice:

```bash
cargo run -p allbert-cli -- repl --tui --save
cargo run -p allbert-cli -- repl --classic --save
```

Inspect new runtime state:

```bash
cargo run -p allbert-cli -- telemetry --json
cargo run -p allbert-cli -- memory stats
cargo run -p allbert-cli -- memory routing show
```

Preview the personality digest corpus without provider calls or writes:

```bash
cargo run -p allbert-cli -- learning digest --preview
```

Enable the bundled digest job template only if you want scheduled draft generation:

```bash
cargo run -p allbert-cli -- jobs template enable personality-digest
```

## Compatibility

- `SessionStatus` remains available for old status clients.
- Existing memory files stay markdown ground truth.
- The memory index schema changes; first launch after upgrade may rebuild derived indexes from markdown.
- If `model.context_window_tokens = 0`, context percentage renders as unknown.
- `SOUL.md`, `USER.md`, `IDENTITY.md`, `TOOLS.md`, `AGENTS.md`, `HEARTBEAT.md`, and `BOOTSTRAP.md` keep their existing roles. `PERSONALITY.md` joins only when present.

## SOUL.md And PERSONALITY.md

`SOUL.md` is seeded, operator-owned, and high authority. The digest must never generate or overwrite it.

`PERSONALITY.md` is optional learned guidance. Digest output installs there only after explicit acceptance and loses conflicts to current instructions, `SOUL.md`, profile files, policy, and tool/security rules.

## Release Smoke Commands

Use a temporary profile for release checks:

```bash
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v011-smoke.XXXXXX)"
cargo run -q -p allbert-cli -- --help
cargo run -q -p allbert-cli -- telemetry --json
cargo run -q -p allbert-cli -- memory stats
cargo run -q -p allbert-cli -- memory routing show
cargo run -q -p allbert-cli -- learning digest --preview
cargo run -q -p allbert-cli -- jobs template enable personality-digest
cargo run -q -p allbert-cli -- jobs template disable personality-digest
```

For daemon smoke, use a completed temp setup flow and then run:

```bash
cargo run -q -p allbert-cli -- daemon start
cargo run -q -p allbert-cli -- daemon status
cargo run -q -p allbert-cli -- telemetry --json
cargo run -q -p allbert-cli -- memory stats
cargo run -q -p allbert-cli -- daemon stop
```

Do not run release smokes against a real long-lived `~/.allbert` profile.
