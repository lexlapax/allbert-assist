# v0.10 Upgrade Notes

v0.10 expands Allbert's provider layer and changes the fresh-profile default to local Ollama with `gemma4`.

## What changes

- Fresh profiles default to `provider = "ollama"`, `model_id = "gemma4"`, and `base_url = "http://127.0.0.1:11434"`.
- Existing profiles keep their configured provider/model unless you rerun setup or switch with `/model`.
- Anthropic and OpenRouter remain supported.
- OpenAI and Gemini are now direct hosted-provider options.
- Ollama is keyless and runs against the local Ollama API.

## Operator action

For the default local provider path, install Ollama separately and run:

```bash
ollama run gemma4
```

Allbert does not install Ollama, pull `gemma4`, or manage local model storage.

For hosted providers, export the relevant API key before your first live turn:

```bash
export ANTHROPIC_API_KEY=...
export OPENROUTER_API_KEY=...
export OPENAI_API_KEY=...
export GEMINI_API_KEY=...
```

## Model switching

Use `/model` to inspect the current session model.

Use `/model <anthropic|openrouter|openai|gemini|ollama> <model_id> [api_key_env]` to switch the attached session only.

Examples:

```text
/model ollama gemma4
/model openai gpt-5.4-mini OPENAI_API_KEY
/model gemini gemini-2.5-flash GEMINI_API_KEY
```

## Compatibility

Old TOML configs with `api_key_env = "ANTHROPIC_API_KEY"` or `api_key_env = "OPENROUTER_API_KEY"` continue to load. The config shape now also allows `api_key_env` to be omitted and `base_url` to be set for local/custom endpoints.
