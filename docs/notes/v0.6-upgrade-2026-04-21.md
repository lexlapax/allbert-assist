# v0.6 Upgrade Note

This note is for operators upgrading an existing v0.5 profile to the shipped v0.6 foundation-hardening release.

## What changed in v0.6

v0.6 does not introduce a new product surface area like v0.5 did. It hardens the existing daemon, memory, and operator path:

- active sessions now persist as markdown journals under `~/.allbert/sessions/`
- `allbert-cli daemon resume --list` and `allbert-cli daemon resume [--session <id>]` expose restart-durable session recovery
- `allbert-cli daemon forget <session-id>` moves a resumable session into `sessions/.trash/`
- `limits.daily_usd_cap` is now a real kernel-enforced turn gate
- `/cost --override <reason>` is the one-turn REPL escape hatch when the daily cap blocks work
- `allbert-cli memory status` now reports manifest/index health, last reconcile time, and whether daemon reconciliation is active
- `allbert-cli memory verify` is the scriptable daemon-off reconciliation check
- the bundled `memory-compile` template is now staging-first and safe to preselect on fresh profiles that explicitly opt into recurring jobs

## New files you may notice

- `~/.allbert/sessions/<session-id>/turns.md`
- `~/.allbert/sessions/<session-id>/meta.json`
- `~/.allbert/memory/index/reconcile.json`

These are working-state or rebuildable metadata artifacts. They are not a bypass of staged durable memory.

## Recommended post-upgrade checks

1. Run `cargo run -p allbert-cli -- daemon status`.
2. Run `cargo run -p allbert-cli -- daemon resume --list`.
3. Run `cargo run -p allbert-cli -- memory status`.
4. Run `cargo run -p allbert-cli -- memory verify`.
5. If you use recurring jobs, inspect `~/.allbert/jobs/templates/memory-compile.md` and confirm it now stages candidates rather than writing durable notes directly.

## What did not change

- durable memory still lives under `~/.allbert/memory/notes/`
- staged learnings still require explicit promotion
- sub-agent nesting is still bounded to one level
- boot-time OS service install is still out of scope
- the daemon is still local-user and local-IPC only
