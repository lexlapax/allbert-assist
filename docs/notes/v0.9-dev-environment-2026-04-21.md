# v0.9 developer-environment closeout — 2026-04-21

v0.9 closes a repo/workflow gap for contributors. It is packaged as the v0.9.0 source-based end-user release, carrying forward the shipped v0.8 runtime surface without adding a new profile migration.

What shipped:

- the current end-user release identity for the source tree
- a pinned Rust contributor toolchain via `rust-toolchain.toml`
- a repo-level contributor guide in `DEVELOPMENT.md`
- a repo-level `AGENTS.md` for coding agents, explicitly distinct from runtime `~/.allbert/AGENTS.md`
- `.env.example` for optional live verification
- a clear Tier A vs Tier B validation split
- documented temp-profile smoke recipes for local shells and Codex-style ephemeral workspaces

This release makes the Allbert source tree reproducible on macOS, Linux, and Codex Web style workspaces. Allbert remains a local-first daemon for end users; v0.9 packages the existing end-user runtime posture with a clearer, reproducible contributor contract. For operator-facing upgrade guidance, see `v0.9-upgrade-2026-04-24.md`.
