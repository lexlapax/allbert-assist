# v0.14 upgrade notes

v0.14 adds bounded self-diagnosis, daemon-owned diagnosis/utility protocol surfaces, and an opt-in local-utilities manifest for structured Unix-style composition.

Reality note: optional remediation routing shipped in v0.14, and concrete candidate/fallback provenance is reconciled in v0.14.1.

## What changes

- Protocol v6 daemons accept v2, v3, v4, v5, and v6 clients. Older clients keep working and do not receive diagnosis or utility-only messages.
- Existing profiles load in-memory defaults for `[self_diagnosis]` and `[local_utilities]`; they are not forced through setup again.
- Fresh profiles see an optional local-utility discovery prompt, but no utility is enabled by default.
- The first-party `self-diagnose` skill is seeded for fresh profiles and uses the bounded `self_diagnose` tool.
- Diagnosis reports are session artifacts under `sessions/<session-id>/artifacts/diagnostics/`.
- `utilities/enabled.toml` is host-specific and excluded from profile export/sync by default.

## Minimal checks

```bash
cargo run -p allbert-cli -- diagnose list --offline
cargo run -p allbert-cli -- utilities discover --offline
cargo run -p allbert-cli -- settings show self_diagnosis
cargo run -p allbert-cli -- settings show local_utilities
```

## Diagnosis posture

`diagnose run` reads bounded, redacted trace bundles and writes a markdown report. It does not apply fixes by default.

Optional remediation requires:

- `self_diagnosis.allow_remediation = true`
- an explicit command such as `diagnose run --remediate code --reason <text>`
- an existing review path: `patch-approval`, skill quarantine, or staged memory

Telegram remains structural-only for remediation in v0.14.

## Local utilities

Enable utilities deliberately:

```bash
cargo run -p allbert-cli -- utilities enable rg
cargo run -p allbert-cli -- utilities doctor
```

`utilities enable` does not edit `security.exec_allow`. If the enabled binary still requires exec approval, `unix_pipe` refuses until the operator updates exec policy.

## Unix pipe

`unix_pipe` is direct-spawn pipeline composition over enabled utility ids. It is not `sh -c` and does not parse shell strings, globs, redirects, or env overrides. Text I/O is bounded; invalid UTF-8 is rendered lossily with a warning marker.

See [Self-diagnosis and local utilities](../operator/self-diagnosis-and-utilities.md) for the full workflow.
