# v0.12.1 upgrade notes

v0.12.1 is the operator-legibility point release for the v0.12 self-improvement surface.

## Upgrade Behavior

- Existing profiles load without migration.
- Protocol v3 daemons accept v2 and v3 clients. v2 clients do not receive v3-only activity messages or fields.
- Fresh and upgraded configs get new defaults through normal config loading. Supported settings can be changed through `allbert-cli settings` or `/settings` with path-preserving TOML edits.
- `config.toml.last-good` is written after a successful daemon config load/start.
- Existing installed skills without `enabled` metadata load as enabled.
- New recovery directories are created lazily: `memory/trash/` and `memory/reject/`.

## Activity And TUI

Use live activity to see what the daemon says Allbert is doing:

```bash
cargo run -p allbert-cli -- activity
cargo run -p allbert-cli -- activity --json
```

In TUI and REPL:

```text
/activity
```

The TUI now redraws while a turn is running, keeps typed text as a next-turn draft, and keeps approval/input modal text separate from that draft.

## Settings

Use the typed settings hub instead of broad config rewrites:

```bash
cargo run -p allbert-cli -- settings list
cargo run -p allbert-cli -- settings show repl.tui.spinner_style
cargo run -p allbert-cli -- settings set repl.tui.spinner_style off
```

Unsupported keys, secrets, path escapes, and unsafe edits fail with remediation hints.

## Recovery

Durable memory can be soft-deleted and restored:

```bash
cargo run -p allbert-cli -- memory forget <path-or-query> --confirm
cargo run -p allbert-cli -- memory restore <id-or-path>
```

Rejected staged memory can be reconsidered while retained:

```bash
cargo run -p allbert-cli -- memory reject <staged-id> --reason "not durable"
cargo run -p allbert-cli -- memory reconsider <staged-id>
```

Installed skills can be disabled without deleting the directory:

```bash
cargo run -p allbert-cli -- skills disable <name>
cargo run -p allbert-cli -- skills enable <name>
```

Broken config edits can be rolled back after at least one successful daemon start:

```bash
cargo run -p allbert-cli -- config restore-last-good
```

The broken config is kept as `config.toml.broken-<timestamp>`.

## Review Surfaces

Approval prompts now include bounded context across TUI, REPL, CLI, and Telegram. Patch approvals may show a short diff preview, but the full diff remains a session artifact and installing a patch remains a separate explicit command.
