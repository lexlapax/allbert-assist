# v0.12 upgrade notes

v0.12 is the self-improvement release: source patches in isolated worktrees, review-first patch approvals, natural-language skill authoring, skill provenance, and an opt-in embedded Lua scripting seam.

## Upgrade Behavior

- Existing profiles load without migration.
- `self_improvement.*` keys take documented defaults when absent.
- `scripting.engine` defaults to `disabled`.
- `lua` is not added to `security.exec_allow`.
- Existing installed skills without `provenance` load as `external`.
- `skill-author` is seeded into `~/.allbert/skills/installed/skill-author/` on first run when absent, with first-party install metadata.
- `patch-approval` is additive over the existing inbox kinds.
- PROTOCOL_VERSION does not bump.

## Source Checkout Requirement

`rust-rebuild` requires a local source checkout. Configure it with:

```bash
cargo run -p allbert-cli -- self-improvement config set --source-checkout /path/to/allbert-assist
```

Check current state:

```bash
cargo run -p allbert-cli -- self-improvement config show
```

If no checkout is configured or discoverable, ordinary Allbert usage still works. Binary-drop users can still use `skill-author` and Lua scripting, but `rust-rebuild` refuses activation with a setup hint.

## Patch Approval

`patch-approval` is the fourth inbox kind alongside tool approvals, cost-cap overrides, and job approvals. Accepting a patch records operator review only:

```bash
cargo run -p allbert-cli -- inbox accept <approval-id> --reason "approved"
```

Install is a separate explicit command:

```bash
cargo run -p allbert-cli -- self-improvement install <approval-id>
```

Allbert never swaps its own binary. After install applies the patch to the source checkout, the operator still owns `cargo install --path crates/allbert-cli` and daemon restart.

## Skill Authoring

`skill-author` is the natural-language path for creating AgentSkills-format skills. Drafts land in:

```text
~/.allbert/skills/incoming/<draft-name>/
```

They carry `provenance: self-authored` and still go through install preview and confirmation. The CLI wizard remains available:

```bash
cargo run -p allbert-cli -- skills init <name>
```

## Lua Scripting

Lua is off by default. To enable embedded Lua scripts, both gates must be set:

```toml
[scripting]
engine = "lua"

[security]
exec_allow = ["bash", "python", "lua"]
```

The sandbox defaults to `string`, `math`, and `table` only; denies filesystem, process, module loading, debug, and coroutine surfaces; and enforces execution, memory, and output caps.
