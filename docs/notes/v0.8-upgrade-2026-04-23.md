# v0.8 upgrade notes

v0.8 is the continuity and sync release.

Main upgrades from v0.7:

- `~/.allbert/identity/user.md` becomes the cross-channel identity record for sender routing.
- Sessions now route by identity and can resume coherently across REPL, CLI, and Telegram surfaces.
- `allbert-cli inbox list|show|accept|reject` is now the shared approval-inbox surface rather than a files-only placeholder.
- `cost-cap-override` and `job-approval` are first-class approval kinds alongside `tool-approval`.
- `allbert-cli profile export|import` ships as the supported continuity workflow for moving a profile between devices.
- `~/.allbert/HEARTBEAT.md` now participates in both bootstrap context and daemon-side cadence decisions for `daily-brief`, `weekly-review`, and inbox nags.
- `allbert-cli daemon status` now reports lock ownership plus whether the configured API-key env var is visible to the running daemon.

Operator checks after upgrade:

1. Run `cargo run -p allbert-cli -- identity show` and confirm the seeded or migrated channel bindings look right.
2. Run `cargo run -p allbert-cli -- inbox list` and confirm pending approvals are visible from the shell.
3. Run `cargo run -p allbert-cli -- heartbeat show` and fix any validation warnings before relying on proactive check-ins.
4. If you use multiple devices, prefer `cargo run -p allbert-cli -- profile export ...` and `profile import ...` over copying `~/.allbert/` manually.
5. Recheck `cargo run -p allbert-cli -- daemon status` after restart so the lock owner and API-key visibility match expectations.

Important boundaries in v0.8:

- proactive heartbeat delivery is still Telegram-only
- daily cost caps are still per-device rather than globally aggregated
- file-level sync still needs explicit excludes for `run/`, `logs/`, `traces/`, `secrets/`, `costs.jsonl`, and `daemon.lock`
