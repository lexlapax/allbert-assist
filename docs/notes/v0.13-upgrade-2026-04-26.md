# v0.13 upgrade notes

v0.13 adds local, review-first personalization through small adapter training. Training is disabled by default, runs only on the operator's machine, and requires an explicit local trainer backend allowlist plus the existing exec-policy allowlist.

Reality note: daemon adapter handlers and production trainer selection from configured real backends are reconciled in v0.14.1.

## What changes

- Protocol v5 daemons accept v2, v3, v4, and v5 clients. Older clients keep working and do not receive adapter-only messages.
- Existing profiles get in-memory adapter-training defaults. On daemon start, if `config.toml` has no `[learning.adapter_training]` block, Allbert attempts a path-preserving default write and logs the result in `daemon.log`.
- Partial `[learning.adapter_training]` blocks are not rewritten. Missing keys load defaults and are visible with `/settings show learning.adapter_training`.
- Adapter training is local-only. Hosted providers ignore active adapters and show a one-shot notice per session.
- Accepting an `adapter-approval` installs the adapter but does not activate it. Activation is explicit with `allbert-cli adapters activate <id>`.

## Minimal opt-in

```bash
cargo run -p allbert-cli -- settings show learning.adapter_training
cargo run -p allbert-cli -- adapters training preview
```

To enable real training, use setup or settings to:

- set `learning.adapter_training.enabled = true`
- add one backend to `learning.adapter_training.allowed_backends`
- set `learning.adapter_training.default_backend`
- add the trainer binary to `security.exec_allow`

The fake backend is for tests and explicit developer validation. Normal setup only suggests real detected backends.

## Compute cap

`learning.compute_cap_wall_seconds` is a daily local-training wall-clock cap. The default is `7200` seconds. `0` or `null` disables enforcement; positive values are bounded to 60..86400 seconds.

## Corpus and privacy

The adapter corpus can include:

- `SOUL.md`
- accepted `PERSONALITY.md`
- approved durable memory notes
- approved fact frontmatter
- bounded episode summaries
- optional redacted v0.12.2 trace excerpts when `learning.adapter_training.capture_traces = true`

Staged memory is not included. Trace excerpts pass through the v0.12.2 secret redactor again during corpus assembly.

## Profile export

Adapter weights are derived and host-specific. `profile export` excludes `adapters/` by default. Use `--include-adapters` only when you deliberately want installed adapters plus `adapters/active.json` in the archive:

```bash
cargo run -p allbert-cli -- profile export profile.tgz --include-adapters
```

Runs, incoming adapters, runtime caches, and adapter history remain excluded.

## Review flow

Training creates an `adapter-approval` inbox item with eval summary, loss curve, and behavioral diff artifacts. Reject deletes the run directory by default; accepting installs the adapter under `adapters/installed/<id>/`. Activation, deactivation, removal, history, and GC are under `allbert-cli adapters`.

See [Personalization guide](../operator/personalization.md) for the full operator workflow.
