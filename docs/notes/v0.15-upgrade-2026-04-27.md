# v0.15 Upgrade Notes

v0.15 is the vector RAG, recall, and help foundation release. It adds a derived
SQLite RAG index, real local vectors through Ollama embeddings and `sqlite-vec`,
and daemon-owned maintenance/channel surfaces.

## What Changes

- RAG artifacts live under `~/.allbert/index/rag/rag.sqlite` and can be deleted
  and rebuilt from source truth.
- Lexical RAG works provider-free with SQLite FTS.
- Vector RAG is real, local, and optional. The default vector provider is
  Ollama with `embeddinggemma`.
- The normal turn flow can use tiny lexical pre-router hints, post-router RAG
  evidence for eligible help/meta/memory-query/local-context prompts, and one
  capped refresh after external tool evidence.
- Durable memory, facts, episodes, and session summaries can be retrieved
  through RAG. The existing memory synopsis remains in the prompt; duplicate
  Tantivy prompt snippets for RAG-owned memory sources are suppressed.
- The root model gets a read-only capped `search_rag` tool for a second
  retrieval pass.
- Protocol v7 adds daemon RAG status/search/rebuild/GC messages. v2-v6 clients
  can still attach but do not receive v7-only RAG messages.
- REPL/TUI expose `/rag status`, `/rag search <query>`, `/rag rebuild
  [--stale-only] [--vectors]`, and `/rag gc [--dry-run]`.
- Telegram exposes read-only `/rag status` and `/rag search <query>`.

## Operator Steps

Provider-free lexical rebuild:

```bash
allbert-cli rag rebuild --no-vectors
allbert-cli rag status
allbert-cli rag search "configure Telegram" --mode lexical
```

Optional vector setup:

```bash
ollama pull embeddinggemma
allbert-cli settings set rag.vector.enabled true
allbert-cli rag rebuild --vectors
allbert-cli rag doctor
```

## Maintenance

RAG maintenance is daemon-owned deterministic service work, not a
prompt-authored scheduled job. Startup missing-DB rebuilds and scheduled
stale-only rebuilds share the same rebuild lock as manual rebuilds. Automatic
maintenance records daemon log and status/run posture without injecting
unsolicited protocol messages into unrelated turns.

## Compatibility

- Existing profiles keep RAG enabled with lexical fallback and vectors disabled
  unless the operator enables `[rag.vector]`.
- Hosted embedding providers are not part of v0.15.
- Growth-loop ingestion, browser capture, and ambient activity ingestion remain
  parked in `docs/plans/future-plans.md`.

## Validation

Run the v0.15 release gate from `docs/onboarding-and-operations.md`. The local
Ollama vector smoke is optional for provider-free CI, but release readiness
should include it on a machine with Ollama and `embeddinggemma` available.
