# v0.7 upgrade notes

v0.7 adds the first shipped non-REPL channel surface and the operator tooling around it.

Main upgrades from v0.6:

- `allbert-cli daemon channels list|status|add|remove` is now the operator surface for channel setup.
- `allbert-cli approvals list|show` gives read-only inspection of pending async approvals from CLI and REPL.
- Telegram is the first shipped async channel.
- Telegram approvals resolve from Telegram itself with `/approve <approval-id>`, `/reject <approval-id>`, and `/override <reason>`.
- Telegram photo input now works for vision-capable models. Downloaded photos are stored as session artifacts under `~/.allbert/sessions/<session-id>/artifacts/`.
- Sub-agent depth is now budget-governed rather than fixed by nesting count.
- Explicit-intent web learning ships through `record_as` on `web_search` and `fetch_url`.

Operator checks after upgrade:

1. If you want Telegram enabled, add the bot token to `~/.allbert/secrets/telegram/bot_token`.
2. Add one allowlisted chat id per line to `~/.allbert/config/channels.telegram.allowed_chats`.
3. Run `cargo run -p allbert-cli -- daemon channels add telegram`.
4. Restart the daemon if it is already running.
5. Confirm state with `cargo run -p allbert-cli -- daemon channels status telegram`.

Important boundary in v0.7:

- CLI and REPL can inspect pending approvals, but they do not resolve Telegram-originated approvals.
- Cross-surface approval inbox behavior is deferred to v0.8.
