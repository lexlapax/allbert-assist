# v0.9 upgrade notes

v0.9 is a minor source-based release for end users.

It carries forward the shipped v0.8 runtime surface: continuity, approval inbox resolution, profile export/import, `HEARTBEAT.md`, Telegram photo input, memory verification, strict AgentSkills installs, and local-first daemon operation remain the user-facing product baseline.

There is no additional profile migration for v0.9. If you already upgraded through v0.8, keep using the same `~/.allbert` profile. If you are upgrading from v0.7 or earlier, apply the v0.8 continuity upgrade notes first.

What v0.9 adds around the source release:

- pinned Rust `1.94.0` contributor toolchain with `rustfmt` and `clippy`
- repo-level `DEVELOPMENT.md` and `AGENTS.md` contributor guidance
- `.env.example` for optional live-provider checks
- a provider-free Tier A validation path
- Codex Web / ephemeral workspace setup guidance

End-user posture is unchanged:

- source-based install
- terminal-first operation
- local daemon and local IPC only
- explicit workspace trust
- provider API keys supplied by the operator
- no hosted control plane

If you are validating the release from source, run:

```bash
cargo fmt --check
env -u RUSTC_WRAPPER cargo clippy --workspace --all-targets -- -D warnings
env -u RUSTC_WRAPPER cargo test -q
env -u RUSTC_WRAPPER cargo run -q -p allbert-cli -- --help
```
