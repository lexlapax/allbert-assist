# v0.5 Upgrade Note

This note is for operators upgrading an existing v0.4 profile to the shipped v0.5 curated-memory release.

## What changes on first v0.5 boot

Allbert performs a one-time curated-memory bootstrap the first time a v0.5 binary touches an older profile:

- creates `~/.allbert/memory/notes/`, `staging/`, `index/`, `.trash/`, and `migrations/` if missing
- imports legacy v0.4 durable buckets from:
  - `~/.allbert/memory/topics/`
  - `~/.allbert/memory/people/`
  - `~/.allbert/memory/projects/`
  - `~/.allbert/memory/decisions/`
- copies the imported notes into `~/.allbert/memory/notes/<bucket>/...`
- writes an import report to `~/.allbert/memory/migrations/v0.5-import.json`
- moves the original bucket directories under `~/.allbert/memory/.legacy-v04/`
- builds `~/.allbert/memory/manifest.json`
- rebuilds the Tantivy index under `~/.allbert/memory/index/`

This upgrade path is automatic. There are no manual migration commands.

## What does not change

- `MEMORY.md` is left in place.
- Existing bootstrap files and skill installs remain intact.
- Existing jobs and daemon config stay in the same profile.
- Curated retrieval still treats markdown as the source of truth; the manifest and index remain rebuildable artifacts.

## How to confirm the upgrade succeeded

Run:

```bash
cargo run -p allbert-cli -- memory status
```

You should see:

- the current profile version
- indexed document count
- staged entry counts
- index age
- last rebuild reason and elapsed time

If you want to force a rebuild:

```bash
cargo run -p allbert-cli -- memory rebuild-index --force
```

## New operator surfaces in v0.5

Curated memory is now operated through:

- `cargo run -p allbert-cli -- memory search "postgres"`
- `cargo run -p allbert-cli -- memory staged list`
- `cargo run -p allbert-cli -- memory staged show <id>`
- `cargo run -p allbert-cli -- memory promote <id> --confirm`
- `cargo run -p allbert-cli -- memory reject <id> --reason "not durable"`
- `cargo run -p allbert-cli -- memory forget <path-or-query> --confirm`

The shipped `memory-curator` skill is also available on fresh and upgraded profiles:

```bash
cargo run -p allbert-cli -- skills show memory-curator
```

## Troubleshooting

If memory search returns nothing after upgrade:

1. Run `cargo run -p allbert-cli -- memory status`.
2. Run `cargo run -p allbert-cli -- memory rebuild-index --force`.
3. Inspect `~/.allbert/memory/migrations/v0.5-import.json`.
4. Confirm imported notes now exist under `~/.allbert/memory/notes/`.

If something looks wrong, keep the `.legacy-v04/` copy in place until you confirm the new `notes/` tree is correct.
