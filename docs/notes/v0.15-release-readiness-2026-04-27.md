# v0.15 Release Readiness

v0.15.0 is ready as the current source-based operator release. The release was
tagged as `v0.15.0` after the M0-M7 implementation and closeout validation
completed.

## Shipped Scope

- SQLite RAG index under `~/.allbert/index/rag/rag.sqlite`, derived from source
  truth and safe to rebuild.
- Provider-free lexical retrieval through SQLite FTS.
- Real local vector retrieval through Ollama `embeddinggemma` embeddings and
  `sqlite-vec`.
- Turn-flow integration: tiny lexical pre-router hints, schema-bound router,
  terminal action precedence, post-router RAG evidence, labelled prompt
  rendering, root-model call, and at most one capped refresh after external
  evidence.
- Memory/RAG boundary cleanup: memory synopsis stays in prompt; durable memory,
  facts, episodes, and session summaries are retrieved through RAG for eligible
  turns without duplicate Tantivy snippet injection.
- Read-only capped `search_rag` root-model tool.
- Daemon-owned RAG maintenance, protocol v7 status/search/rebuild/GC messages,
  REPL/TUI `/rag ...`, and Telegram read-only RAG status/search.
- Schema-v2 collection model with system/user separation and manifest-backed
  user collection source truth.
- User collection ingestion from trusted local files/directories and explicit
  HTTP(S) URLs.
- First-party `rag` skill for explicit local-session collection lifecycle and
  attach/detach flows.

## Release Notes

- Fresh profiles remain source-based and local-first: Ollama/Gemma4 is the
  default assistant model, but Allbert does not install Ollama or pull models.
- RAG vectors are local-Ollama only in v0.15. Hosted embedding providers are
  deferred and must be cost/privacy-gated separately.
- User collections are explicit task/corpus context. They are never searched or
  injected by default, and attachment is session-scoped until detach, delete,
  or session reset.
- Local user-collection sources must be inside trusted filesystem roots.
- URL ingestion is exact-URL by default, with optional bounded same-origin
  expansion governed by config. It does not use browser state, credentials,
  JavaScript execution, authenticated sessions, or broad crawling.
- Telegram remains read-only for RAG status/search. Collection creation,
  ingestion, rebuild, delete, and attach/detach stay on local terminal/model
  sessions in v0.15.
- Dedicated protocol lifecycle mutation messages for collections and editing
  individual source URIs inside an existing collection are deferred.
