## M1 Locked Decisions

Each is recorded in `v0.47-plan.md` §"M1 Locked Decisions" with a
one-paragraph rationale.

1. Draft store — one unified reviewed-draft store, generalized from the v0.37
   `DynamicPlugins.Draft` lifecycle and its
   `<ALLBERT_HOME>/dynamic_plugins/drafts/` compatibility root, with
   `<ALLBERT_HOME>/drafts/` subroots for new non-code draft kinds; promotion
   routes to the existing live write path per kind.
2. Suggestion surface — generalize the v0.42 `Tools.Discovery.Suggestion`
   schema and reuse the `Workspace.DiscoverySuggestions` panel; one queue,
   one panel.
3. Discovery substrate — build a redaction-respecting trace index over
   `<ALLBERT_HOME>/memory/traces/` so repetition is queryable, alongside
   structured objective events, memory review decisions, and operator-marked
   examples.
4. Release scope — v0.47 ships discovery + skill/workflow/memory drafts;
   template/marketplace/delegate-plugin/capability-gap/objective drafts ship
   in v0.47b.
5. Discovery action shape — a registered `:read_only`, `:internal` action
   returning `{suggestions, diagnostics}`, modeled on `find_tools` +
   `Tools.Finder`.

## Main User/Operator Flows

- Operator (self-improvement + trace index enabled) asks Allbert what
  repeated work it could turn into a skill or workflow.
- Allbert runs the read-only `discover_patterns` action; it reads the trace
  index plus structured signals and proposes inert suggestions into the
  passive discovery queue (no message is pushed).
- Operator reviews suggestions in the `DiscoverySuggestions` panel or via
  `mix allbert.self_improvement list/inspect`.
- Operator discards, edits, or promotes a suggestion to an **inert draft** in
  the unified store (disabled/untrusted skill, workflow draft, or memory
  draft).
- Promoting a draft to live is a separate confirmed action that writes through
  the existing live path for that kind.

## Intent Phrase Corpus

Source of truth in `v0.47-plan.md` §"Planned Interfaces". M3 implements one

| Pattern | Routes to |
|---|---|
| `show self-improvement suggestions` | read-only `discover_patterns` (when enabled) |
| `what could you turn into a skill` | `discover_patterns`, skill-biased ranking |
| `what could you turn into a workflow` | `discover_patterns`, workflow-biased ranking |

Patterns match case-insensitively. When self-improvement is disabled, these
remain inert and must not read the trace index or write suggestions. The
corpus is M3-locked — additions require an explicit re-decision here and in
the plan.

## Data Flow

Self-improvement discovery is read-only with respect to live authority: it may
write advisory suggestion rows, but it must not write live settings, skills,
workflows, memory, plugins, code, permissions, or confirmations. Only draft
promotion writes a live artifact, and only through an existing confirmed action:

1. Intent frames a read-only `discover_patterns` request (`:read_only`,
   `:internal`).
2. `discover_patterns` reads `AllbertAssist.SelfImprovement.TraceIndex`
   (repeated prompts, action chains, corrections, failed intents) plus
   `objective_events`, `Memory.list_entries(review_status: ...)` review
   decisions, and operator-marked examples. It writes inert
   `Tools.Discovery.Suggestion` rows (self-improvement `suggestion_type`,
   self-improvement provenance) and returns `{suggestions, diagnostics}`.
   It grants no permission and enables nothing.
3. The `Workspace.DiscoverySuggestions` panel renders the new suggestions
   passively (no unprompted messaging); they expire per
   `self_improvement.suggestions.ttl_days`.
4. Suggestion lifecycle is fixed. Pending suggestions may expire. Discarding
   marks a suggestion dismissed. Promoting a suggestion accepts it and records
   `draft_id`; re-promoting the same accepted suggestion returns the same draft
   instead of creating a duplicate.
5. On operator promotion of a suggestion, `create_self_improvement_draft`
   writes an inert draft into the unified `Drafts.Store` with a `kind`
   (`skill`, `workflow`, `memory_promotion`, `memory_update`), provenance,
   and tier `draft`. Skill drafts are disabled/untrusted; workflow drafts
   validate against the v0.44 compile-time schema (`Actions.Registry` +
   `Step.kinds()`).
6. On operator promotion of a draft, the promotion action
   (`promote_skill_draft` / `promote_workflow_draft` /
   `promote_memory_draft`) routes to the existing live path under Security
   Central + durable confirmation: enable the skill, write a live
   `<ALLBERT_HOME>/workflows/<id>.yaml`, or call `Memory.append/1`. A denied
   confirmation writes nothing.
7. Trace/audit carry the actor, the discovery scope, the suggestion id, the
   draft id, and the promotion target; the trace index inherits the v0.40–
   v0.43 redaction; raw trace content stays in its source files.

## Authority Invariants

- **Pattern discovery is read-only.** `discover_patterns` is `:read_only`;
  it reads the index and structured signals and writes only inert advisory
  suggestions, never live authority or confirmations.
- **Suggestion packets are advisory metadata, not authority** (ADR 0021 §4 /
  ADR 0045). They do not alter agent behavior and do not auto-promote.
- **Drafts are inert.** A draft is a `Drafts.Store` record with no live
  effect. Skill drafts are disabled/untrusted; workflow drafts are
  declarative inputs; memory drafts are draft state, not memory entries.
- **Promotion is the only effectful boundary** and always requires a
  registered action, Security Central, durable confirmation, trace, and
  audit.
- **The trace index inherits trace redaction**; it records repetition and
  references, not new secrets, and grants nothing.
- **Repeated use never grants permission.** Frequency, score, and prior
  approvals are advisory; they never auto-approve a new suggestion or
  promotion (ADR 0045).
- **No new authority surface.** No new permission class, operation class, URI
  scheme, or memory namespace; the only new settings are an enable toggle and
  bounded caps.

## Handles / CLI Surfaces

- New module: `AllbertAssist.SelfImprovement.TraceIndex`.
- Generalized: `AllbertAssist.Tools.Discovery.Suggestion`
  (self-improvement `suggestion_type`s),
  `AllbertAssist.Drafts.Store` (from `DynamicPlugins.Draft`).
- Registered actions: `discover_patterns` (`:read_only`),
  `create_self_improvement_draft`, `promote_skill_draft`,
  `promote_workflow_draft`, `promote_memory_draft`.

- CLI: `mix allbert.self_improvement list`,
  `mix allbert.self_improvement inspect <id>`.
- Settings fragment: `self_improvement.*`.

## Smoke Checklist

- Confirm `self_improvement.enabled` and `trace_index.enabled` default
  `false`; `schema_version` is read-only; caps reject over-limit writes.
- Enable self-improvement + trace index; run a discovery scan.
- Confirm the scan is read-only and writes only suggestions (no enabled
  skill, setting, plugin, workflow, or code).
- Confirm suggestions render in the passive panel and expire per TTL.
- Confirm promoting a suggestion creates an inert draft (skill
  disabled/untrusted; workflow schema-valid + not live; memory draft state).
- Confirm discarding a suggestion or draft leaves no enabled capability.
- Confirm promoting a draft to live requires a confirmation and writes
  nothing when denied.
- Confirm the trace index leaks no raw content (redaction inherited).
- Confirm `mix allbert.test release.v047` runs green on a fresh checkout and
  writes reproducible evidence.

## Operator Validation

Status: planned. Per-milestone blocks are scenario sketches and fixture
sources for `mix allbert.test release.v047` (M6), not a promise that every
commented step is copy-paste runnable before the milestone code exists. The
implemented release gate must materialize each sketch with fixture traces,
suggestion ids, draft ids, and disposable `ALLBERT_HOME` setup; the exact
primary operator command is the M6 `mix allbert.test release.v047` block.

### M1 — trace index + settings

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v047-m1.XXXXXX)"
mix do ecto.create --quiet + ecto.migrate --quiet + run -e '
  IO.inspect(AllbertAssist.Settings.get("self_improvement.enabled"));
  IO.inspect(AllbertAssist.Settings.put("self_improvement.schema_version", 2, %{actor: "local", channel: :test}));
  IO.inspect(AllbertAssist.Settings.put("self_improvement.trace_index.min_repetitions", 1, %{actor: "local", channel: :test}));
'
```

Expected: `self_improvement.enabled` defaults `false`; the `schema_version`
write and the below-floor `min_repetitions` write are both rejected.

### M2 — generalized suggestion surface

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v047-m2.XXXXXX)"
mix do ecto.create --quiet + ecto.migrate --quiet + allbert.settings set self_improvement.enabled true
mix run -e '
  context = %{actor: "local", user_id: "local", channel: :cli}
  IO.inspect(AllbertAssist.Tools.Discovery.list_suggestions(status: "pending"), label: "queue")
'
```

Expected: the discovery queue accepts and lists a self-improvement
suggestion type; the panel renders it; it expires per TTL.

### M6 — deterministic release gate (primary)

```sh

# evidence: <ALLBERT_HOME>/release_evidence/v047/release-v047-<ts>.json
```

The gate runs against fixture traces/objectives with no live model and no
network. It exercises the full v0.47 surface: enable, read-only discovery,
advisory suggestions, inert drafts, draft-only facades,

settings/traces/index.
