## Main User/Operator Flows

- Operator asks Allbert to inspect repeated prompts, corrections, traces,
  marketplace metadata, or objective patterns.
- Allbert runs a read-only pattern-discovery action and records trace metadata.
- Allbert proposes one or more inert suggestions: skill draft, workflow YAML
  draft, template input, marketplace-backed suggestion, memory update/promotion
  draft, objective/workflow draft, or dynamic capability-gap draft.
- Operator reviews the suggestion in the workspace queue or CLI.
- Operator may discard, edit, park, or promote the draft into an existing
  reviewed creation path.
- Code-bearing drafts, if any, enter the v0.36/v0.37 sandbox/gate/integration
  path before any live authority can be requested.

## Data Flow

Inputs are traces, objective events, memory review decisions, marketplace-lite
metadata, operator-marked examples, and existing skill/action/workflow metadata.
The analysis output is a bounded suggestion packet. Promoted drafts are written
only to reviewed draft roots and remain disabled/untrusted until a later
confirmed action changes their state.

Team-channel traces are a later input. When Discord and Slack ship in v0.50,
their redacted traces can feed the same pattern-discovery action without
changing v0.47 authority.

## Authority Invariants

- Pattern discovery is read-only.
- Suggestion packets are advisory metadata, not authority.
- Marketplace metadata remains descriptive only.
- Repeated use never creates a permission grant.
- Draft creation does not enable execution.
- Memory and workflow facades write draft state only.
- Model output never bypasses registered actions, Security Central, Resource
  Access, sandbox/gate evidence, confirmations, traces, or audits.
- Operator confirmation is required for enablement, live integration,
  publishing, rollback, or marketplace submission.

## Smoke Checklist

- Pattern scan produces suggestions without mutating enabled skills, settings,
  plugins, workflows, or code.
- Marketplace metadata can appear in suggestions without granting authority.
- Discarding a suggestion leaves no enabled capability behind.
- Promoting an instruction-only skill draft creates a disabled/untrusted draft.
- Promoting a memory or workflow suggestion creates draft state only.
- Promoting a code-bearing draft requires the existing sandbox/gate path before
  any live integration confirmation appears.
- Repeated prior approvals do not auto-approve a new suggestion.
