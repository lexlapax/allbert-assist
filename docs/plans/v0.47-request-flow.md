## Main User/Operator Flows

- Operator configures WhatsApp, Signal, iMessage, or Matrix.
- Operator completes provider-specific pairing/setup.
- External message becomes runtime input.
- Allbert replies and renders confirmations through the provider affordance.

## Data Flow

Provider events pass through the shared channel event store, identity mapping,
runtime submission, response rendering, dedupe, and redacted provider status
updates. Pairing/session state is provider-specific Settings Central data.

## Authority Invariants

Phone number, device session, Matrix id, or iMessage account identity is not
permission. Channels do not approve their own work. SMS stays out of scope.
