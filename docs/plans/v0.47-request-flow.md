## Main User/Operator Flows

- Operator starts CLI voice mode.
- Operator uses the workspace microphone affordance.
- Telegram voice note is transcribed as text input.
- Allbert may emit TTS output through a registered provider action.

## Data Flow

Audio capture creates bounded audio resources, sends content only through the
configured STT provider path, emits transcribed text into normal runtime input,
and records redacted trace metadata. TTS output is a separate registered action
with provider/cost metadata.

## Authority Invariants

- Voice input is input modality only; it does not change action authority.
- Microphone capture requires explicit per-session confirmation.
- Audio content is redacted from traces by default.
- Channel adapters do not own STT/TTS authority.
