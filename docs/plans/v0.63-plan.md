# Allbert v0.63 Product RC & Integrated Validation

Request flow: `docs/plans/v0.63-request-flow.md`
ADR: no new authority ADR. This release consumes ADR 0046, ADR 0065, ADR 0069,
ADR 0074, ADR 0075, and ADR 0076 as accepted inputs and proves their combined
operator path before the v1.0 freeze.

Status: planned (pre-1.0 integrated product release candidate).

Source: post-v0.58 product-readiness review. v0.59 is engineering substrate;
v0.60-v0.62 add product capability. v0.63 exists because the first complete
install -> serve -> onboard -> first-use -> inspect -> export/import path should
not be validated for the first time during v1.0.

## Purpose

Run the integrated product release candidate after the web UX, packaging, and
onboarding/profile work has landed. v0.63 adds no features. It proves that a
technical prosumer can install Allbert, start it, complete guided onboarding,
reach first value, inspect safety/trust state, use at least one real integration
path, export/import or upgrade a Home, and uninstall/rollback without hidden
state, warning, security, or docs drift.

## Expected Inputs

- v0.59 hardening substrate accepted: export/import, settings migration,
  central action param contracts, and security sweep.
- v0.60 web UX professional pass accepted.
- v0.61 packaged binary, unified CLI, daemon, mix-free TUI, and OS vault
  accepted.
- v0.62 guided onboarding and user-category profiles accepted.
- Version-specific gates `release.v059` through `release.v062` passing.

## Operator-Visible Win

An operator can follow one precise validation script from a clean machine/home:
install Allbert without Elixir/OTP, start `allbert serve`, complete QuickStart,
ask a first useful question, inspect confirmations/security/events, exercise one
tool/channel/browser path under policy, export/import or upgrade the Home, and
uninstall cleanly. Evidence is redacted, durable, and ready for the v1.0 freeze.

## Planned Interfaces

- Product RC operator validation checklist.
- Cross-platform install/onboard/serve evidence template.
- Browser-controlled web smoke for `/`, `/workspace`, `/jobs`, `/objectives`,
  onboarding, settings/model, and operator panels.
- CLI/TUI smoke for grouped command help, `allbert serve`, `allbert onboard`,
  `allbert tui`, and admin reads.
- Upgrade/export/import smoke from a v0.58/v0.59-style Home into the packaged
  v0.63 shape.
- Secret and evidence leak scan.
- `mix allbert.test release.v063` deterministic gate.

## Security Posture

No new capability and no new authority. This is validation only. Failures found
in this release must be fixed in the owning capability area rather than deferred
past 1.0. Any exception must be documented as non-blocking with an explicit
rationale and user-visible workaround.

## Non-Goals

- No feature work.
- No native desktop GUI client.
- No new public protocol surface.
- No new profile category unless required to fix a v0.62 correctness issue.
- No contract freeze. v1.0 owns the freeze after this RC is green.

## Milestones

- M1: Product RC evidence harness and disposable-home validation script.
- M2: Cross-platform install/serve/onboard smoke: macOS, Linux, Windows/WSL2
  where supported by the packaged binary.
- M3: Browser-controlled web smoke across landing, workspace, jobs, objectives,
  onboarding, settings/model, and operator panels.
- M4: CLI/TUI smoke across grouped help, serve health, terminal onboarding, TUI
  admin reads, and operator/dev command split.
- M5: Integration smoke for one real policy-bounded path: MCP, browser research,
  or a configured channel, with explicit secret redaction.
- M6: Upgrade/export/import/uninstall validation and Allbert Home layout
  reconciliation.
- M7: Drift sweep, docs closeout, `release.v063`, docs gate, warning gate, and
  v1.0 handoff.

## Documentation Deliverables

- Product RC operator validation guide with exact commands and expected output.
- Cross-platform install and uninstall validation notes.
- Browser-control smoke checklist with expected pages/states.
- Upgrade/export/import evidence guide.
- Secret/evidence leak-scan command set.
- v1.0 handoff note listing the frozen acceptance matrix inputs.

## Definition Of Done

v0.63 is done when:

- the complete clean-machine/home install -> serve -> onboard -> first chat ->
  inspect -> integration smoke -> export/import or upgrade -> uninstall path is
  validated with redacted evidence;
- browser-controlled web smoke covers `/`, `/workspace`, `/jobs`,
  `/objectives`, onboarding, settings/model, and operator panels with no console
  errors or layout regressions;
- grouped CLI and TUI operator flows work without raw `mix`;
- no raw secret appears in logs, output, evidence, docs, screenshots, or release
  artifacts;
- `mix allbert.test release.v063`, docs, warning, and drift checks pass;
- all v1.0 acceptance-matrix inputs are current and linked;
- no known warning/security/doc/product-readiness drift remains.
