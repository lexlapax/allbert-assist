# Allbert v0.39 First-Run Onboarding, Provider Control, Identity Slot, And Active Memory

Request flow: `docs/plans/v0.39-request-flow.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Make Allbert usable from a fresh checkout before the broader v1.0 capability
arc. The milestone turns existing Settings Central, provider/model profiles,
Objective Runtime, channel setup, and reviewed memory retrieval into one guided
operator setup path.

## Expected Inputs

- v0.38 templated creation remains the active implementation predecessor.
- Existing Settings Central provider/model profile schema.
- Existing encrypted Settings Central secret store.
- Existing Objective Runtime and registered action runner.
- v0.21 reviewed-memory retrieval and memory review substrate.

## Operator-Visible Win

An operator can run one guided setup from CLI or `/workspace`, choose a local or
remote model, validate credentials, optionally connect a channel, write an
operator-editable identity memory, and get replies informed by reviewed memory.

## Planned Interfaces

- `mix allbert.onboard`
- `/workspace` onboarding destination.
- `mix allbert.model list`
- `mix allbert.model use PROFILE`
- `mix allbert.model doctor PROFILE`
- Workspace provider/model picker.
- Optional channel command equivalent for model/profile status.
- `identity` memory namespace under Allbert Home.

## Security Posture

Onboarding is a registered objective whose steps call registered actions.
Provider checks never print raw secrets. The identity/personality file is inert
memory content and grants no permission. Active Memory retrieval is read-only
and traceable; it does not promote, mutate, or infer durable memory by itself.

## Non-Goals

- No new provider backend.
- No hidden provider failover.
- No automatic channel enrollment without operator input.
- No memory distillation or small-model training.
- No autonomous skill creation from onboarding traces.

## Milestones

- M1: Onboarding objective skeleton and CLI entrypoint.
- M2: Provider/model list, use, and doctor commands plus workspace picker.
- M3: Optional identity memory namespace and onboarding step.
- M4: Deterministic Active Memory pre-reply retrieval and trace metadata.
- M5: Security evals, operator docs, and closeout.

## Definition Of Done

v0.39 is done when a disposable Allbert Home can finish guided onboarding,
select and validate a model profile, preserve redacted secret handling, create
or skip the identity memory, retrieve reviewed memory before replies, and pass
focused docs/security checks plus the release gate chosen by the milestone.
