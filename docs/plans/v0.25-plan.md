# Allbert v0.25 Full App Contract And Surface DSL

Formerly tracked as M-AppContract-Full.

## Purpose

v0.25 matures the minimal app registration contract into the full five-layer
`AllbertAssist.App` contract and adds the `AllbertAssist.Surface` DSL consumed
by v0.27 workspace/canvas work.

## Expected Inputs

- v0.15 minimal app contract exists.
- v0.17 plugin contract exists.
- v0.18 StockSage proves the lite contract as a plugin-contributed real app.
- ADR 0015 is accepted and updated with any implementation decisions.

## Expected Direction

- Extend `AllbertAssist.App` to include identity/OTP, agents/actions/signals,
  skills, UI surfaces, data/settings, and memory namespaces.
- Extend `AllbertAssist.App.Registry` with signal subscriptions, settings
  validation, and memory namespace registration.
- Keep plugin registration as the package/discovery layer. The app contract
  should expand app capabilities without turning `AllbertAssist.App.Registry`
  into the general plugin registry.
- Add `AllbertAssist.App.SurfaceProvider`.
- Add `AllbertAssist.Surface` with catalog validation over known node shapes.
- Add optional `AllbertAssist.Surface.Encoder.to_a2ui/1`; do not use it in the
  Phoenix LiveView path.
- Add `mix allbert.validate_app MyApp`.
- Update `StockSage.App` to implement the full contract.
- Add `docs/how-to-create-an-allbert-app.md`.

## Non-Goals

- No app generator; that is v0.29.
- No AG-UI/A2UI package dependency for the local web path.
- No canvas implementation; that is v0.27.
- No permission grants from app registration.

## Test Plan

- `mix allbert.validate_app StockSage.App` passes.
- `mix allbert.plugins show stocksage` or equivalent plugin inspection shows
  StockSage's app contribution without granting authority.
- Invalid callbacks or invalid surfaces fail validation.
- Surface DSL rejects model-invented component names and action bindings.
- App registration still does not bypass Security Central or action runner.

## Definition Of Done

v0.25 is done when StockSage implements the full contract, surface validation
exists, ADR 0015 is accepted, and v0.27 can build the workspace shell on these
contracts.
