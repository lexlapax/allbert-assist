## Purpose

Run the integrated product release candidate after the presentation overhaul,
packaging, onboarding/profile, trusted first-run, and local-knowledge work has
landed. v0.66 adds no new capability. It proves that a non-developer local-first
operator can install Allbert, start the persistent service, complete guided onboarding
through the consumer-default local model path, connect local files/notes, review agent
memory, reach first value, inspect safety/trust state, use existing implemented advanced
surfaces without regression, export/import or upgrade a Home, and uninstall/rollback
without hidden state, warning, security, or docs drift. BYOK/custom setup is validated as
the advanced and fallback path, not the primary no-docs path.

## Locked Decisions

1. **Two-layer verification (see Status).** `release.v066` GREEN means the
   contract/routing/boundary invariants hold and the evidence artifacts exist; the
   install/browser/model/cross-platform claims are carried by scripted smokes + human
   audit, reconciled at closeout. Every milestone/row/DoD bullet is tagged
   **gate-proved-contract** or **operator-attested-smoke**.
2. **`product-rc-*` rows are contract-level proxies.** Like the v0.59 sweep, gate rows
   assert capability exposure/permission/confirmation floors and routing decisions, not
   live behavior (e.g. `â€¦web-smoke-no-console-error` asserts LiveView render/dispatch, not
   a real browser console; `â€¦oneclick-model-no-key-first-chat` asserts the
   routing/state machine, not that a model answered). Each row states which it is.
3. **Item-11 usability audit is evidence, not a gate row.** It is a human,
   non-`AssertBinding` audit; the plan defines its rubric and evidence artifact (M3),
   and `product-rc-web-usability-audit-item11-001` is operator-attested.
4. **Gate-enforced doc sweep (M8).** The full doc-surface consistency sweep is not a
   one-time editorial promise: hardcoded "current as of vX" stamps and restated release
   blurbs are replaced with links to the canonical sources (CHANGELOG/roadmap), and a new
   docs-gate check fails `release.v066` on stale version pins, missing index entries, and
   orphaned docs â€” so drift cannot silently recur next release.
5. **No new authority, capability, permission class, or product surface** (validation
   only). Findings are fixed in the **owning capability area** before v1.0, never deferred
   into the freeze.

## Definition Of Done

v0.66 is done when both layers are satisfied (Locked Decision 1): the deterministic
`release.v066` gate is GREEN **and** the operator-attested evidence is recorded/reconciled
at closeout. The gate proves contract/routing/boundary invariants; it does **not** by
itself prove the install/browser/model/cross-platform outcomes â€” those are attested.

- the complete clean-machine/home install -> service start -> onboard -> first chat ->
  local files/notes/memory -> inspect -> advanced-surface regression smoke ->
  export/import or upgrade -> uninstall path is validated with redacted
  evidence;
- a fresh non-developer operator can complete the primary launch path without
  reading developer docs, including the consumer-default guided local-runtime setup if
  needed and one-click model download to first chat with no manual model CLI or API key,
  with observed stumbling blocks recorded as blocking defects or explicit v1.0 caveats;
- natural non-developer prompts route sensibly with no mis-route to a disabled or demo
  capability, and the web workspace passes the non-developer usability audit against the
  item-11 UX bar;
- browser-controlled web smoke covers `/`, `/workspace`, `/jobs`,
  `/objectives`, onboarding, settings/model, and operator panels with no console
  errors or layout regressions;
- grouped CLI and TUI operator flows work without raw `mix`;
- remote channels, MCP, browser research, plan approval, public protocol, and
  export/import surfaces have passing smoke/eval evidence;
- no raw secret appears in logs, output, evidence, docs, screenshots, or release
  artifacts;
- the item-11 non-developer usability audit report is recorded and its blocking defects
  resolved in their owning area;
- the full doc-surface consistency sweep (M11) is complete â€” README trimmed under target,
  operator + developer docs version-pin-swept, all indexes reconciled, no orphaned docs â€”
  and the docs staleness/index-completeness check passes as a `release.v066` step so drift
  cannot silently recur;
- `mix allbert.test release.v066`, docs, warning, and drift checks pass;
- all v1.0 acceptance-matrix inputs are current and linked;
- no known warning/security/doc/product-readiness drift remains.
