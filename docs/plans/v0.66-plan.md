# Allbert v0.66 Product RC & No-Docs Validation

Request flow: `docs/plans/v0.66-request-flow.md`
ADR: no new authority ADR. This release consumes ADR 0046, ADR 0065, ADR 0069,
ADR 0074, ADR 0075, ADR 0076, ADR 0077, and ADR 0078 as accepted inputs and
proves their combined operator path before the v1.0 freeze.

Status: planned (pre-1.0 integrated product release candidate).

Source: post-v0.58 product-readiness review. v0.59 is engineering substrate;
v0.60 is the product-experience design; v0.61-v0.63 add the first product
capability on that design; v0.64 adds trusted install plus repairable first-run
onboarding for non-developers; and v0.65 makes local files, notes, and reviewed
agent memory the launch-critical knowledge path. v0.66 validates the complete
pre-1.0 product, including existing advanced surfaces, so v1.0 is not the first
time the full install -> serve -> onboard -> local-knowledge -> first-use ->
inspect -> export/import path is tested end to end.

## Purpose

Run the integrated product release candidate after the presentation overhaul,
packaging, onboarding/profile, trusted first-run, and local-knowledge work has
landed. v0.66 adds no new capability. It proves that a non-developer local-first
operator can install Allbert, start it, complete guided onboarding through either
the local model path or BYOK hosted path, connect local files/notes, review agent
memory, reach first value, inspect safety/trust state, use existing implemented
advanced surfaces without regression, export/import or upgrade a Home, and
uninstall/rollback without hidden state, warning, security, or docs drift.

## Operator-Visible Win

An operator can follow one precise validation script from a clean machine/home:
install Allbert without Elixir/OTP, start `allbert serve`, complete QuickStart,
connect local files/notes, ask a first useful question grounded in local
knowledge, review or reject one memory candidate, inspect
confirmations/security/events, exercise implemented advanced surfaces under
policy, export/import or upgrade the Home, and uninstall cleanly. Evidence is
redacted, durable, and ready for the v1.0 freeze.

## Security Posture

No new capability and no new authority. This is validation only. Failures found
in this release must be fixed in the owning capability area rather than deferred
past 1.0. Any exception must be documented as non-blocking with an explicit
rationale and user-visible workaround.

## Non-Goals

- No feature work (no new capability, permission class, or product surface).
- No native desktop GUI client.
- No new public protocol surface.
- No new profile category unless required to fix a v0.63 correctness issue.
- No contract freeze. v1.0 owns the freeze after this RC is green.

## Milestones

- M1: Product RC evidence harness and disposable-home validation script.
- M2: Cross-platform install/serve/onboard smoke: macOS, Linux, Windows/WSL2
  where supported by the packaged binary.
- M3: Browser-controlled web smoke across landing, workspace, jobs, objectives,
  onboarding, settings/model, and operator panels — **plus a non-developer usability
  audit** of the web workspace against the acceptance-matrix item-11 UX bar (IA,
  navigation, composition, empty states, first-run legibility). This satisfies item 11 by
  audit, not by "v0.61 shipped"; findings are blocking defects or explicit v1.0 caveats.
- M4: CLI/TUI smoke across grouped help, serve health, terminal onboarding, TUI
  admin reads, and operator/dev command split.
- M5: Launch-path smoke for local files, notes, and reviewed agent memory:
  connect a notes root, search/read a note, confirm a note write, review one
  memory candidate, and recall the reviewed memory in a later chat.
- M5.0: Existing advanced-surface regression smoke for remote channels, MCP,
  browser research, plan approval, public protocols, and export/import. These
  are not the default first-run path, but implemented surfaces must work before
  v1.0.
- M5.2: Consumer-default first-chat + conversational routing quality. Prove the consumer
  default reaches a first useful chat with **no external tool and no API key** (the v0.64
  one-click local-model download). Then run a fixed set of natural non-developer prompts
  (chitchat, general questions, "help me with X", the onboarding-suggested starters) and
  assert they route to an answer or the correct action with **no mis-route to a disabled
  or demo capability** (the v0.63 F5 class). Confirm StockSage/demo intents do not lead
  the default first run.
- M5.1: Cross-surface security eval delta-sweep over the surfaces added since the
  v0.59 M4 sweep — landing/`/` (v0.61), the packaged binary / grouped CLI /
  daemon / OS secret-vault (v0.62), the onboarding wizard / masked-credential
  entry (v0.63), trusted installer verification / repairable first-run (v0.64),
  and local-knowledge/memory path (v0.65) — reusing the v0.59 M4 sweep
  methodology, not only the leak scan.
  The secret-vault and credential-entry surfaces are authority-adjacent and must
  be swept, not just leak-scanned, before the v1.0 freeze.
- M6: Upgrade/export/import/uninstall validation and Allbert Home layout
  reconciliation.
- M7: Drift sweep, docs closeout, `release.v066`, docs gate, warning gate, and
  v1.0 handoff.

## Definition Of Done

v0.66 is done when:

- the complete clean-machine/home install -> serve -> onboard -> first chat ->
  local files/notes/memory -> inspect -> advanced-surface regression smoke ->
  export/import or upgrade -> uninstall path is validated with redacted
  evidence;
- a fresh non-developer operator can complete the primary launch path without
  reading developer docs — including the consumer-default one-click model download to
  first chat with no external tool or API key — with observed stumbling blocks recorded
  as blocking defects or explicit v1.0 caveats;
- natural non-developer prompts route sensibly with no mis-route to a disabled or demo
  capability, and the web workspace passes the non-developer usability audit against the
  item-11 UX bar;
- browser-controlled web smoke covers `/`, `/workspace`, `/jobs`,
  `/objectives`, onboarding, settings/model, and operator panels with no console
  errors or layout regressions;
- grouped CLI and TUI operator flows work without raw `mix`;
- remote channels, MCP, browser research, plan approval, public protocol, and
  export/import surfaces have passing smoke/eval evidence;
- no raw secret appears in logs, output, evidence, docs, screenshots, or release
  artifacts;
- `mix allbert.test release.v066`, docs, warning, and drift checks pass;
- all v1.0 acceptance-matrix inputs are current and linked;
- no known warning/security/doc/product-readiness drift remains.
