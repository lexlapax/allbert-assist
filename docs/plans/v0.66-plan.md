# Allbert v0.66 Product RC & No-Docs Validation

Request flow: `docs/plans/v0.66-request-flow.md`
ADR: no new authority ADR. This release consumes ADR 0046, ADR 0065, ADR 0069,
ADR 0074, ADR 0075, ADR 0076, ADR 0077, ADR 0078, ADR 0079, and ADR 0080 as
accepted inputs and proves their combined operator path before the v1.0 freeze.

Status: planned (pre-1.0 integrated product release candidate); implementation-readiness
deepened 2026-07-11 against the shipped code (Current Code State + Code Seams below).

**Two-layer verification (Locked Decision 1).** v0.66 explicitly uses the proven v0.62
two-layer model, so "smoke/validation" is not one thing: (1) a checkout-bound
`mix allbert.test release.v066` gate that proves **contract, routing, and boundary
invariants** deterministically (no packaged binary, no browser, no network, no
cross-host); and (2) **operator-attested evidence** ŌĆö scripted host smokes against a
built binary, real-browser web smoke + the item-11 usability audit, cross-platform/WSL2
installs, and real-egress model/advanced-surface runs ŌĆö reconciled at closeout. Each
milestone, each gate-bound or mixed `product-rc-*` row, each attested evidence id, and
each Definition-of-Done bullet is tagged **gate-proved-contract** or
**operator-attested-smoke** so the gate never reads as if one `mix` command proves the
whole non-developer product path.

Source: post-v0.58 product-readiness review. v0.59 is engineering substrate;
v0.60 is the product-experience design; v0.60b locks the chosen visual language;
v0.61 and v0.61b deliver the presentation baseline and post-validation UX fixes;
v0.62-v0.63 add packaged entry points and guided onboarding; v0.64 adds trusted
install plus repairable first-run onboarding for non-developers; and v0.65 makes
local files, notes, and reviewed agent memory the launch-critical knowledge path.
v0.66 validates the complete pre-1.0 product, including existing advanced surfaces,
so v1.0 is not the first time the full install -> serve -> onboard ->
local-knowledge -> first-use -> inspect -> export/import path is tested end to end.

## Purpose

Run the integrated product release candidate after the presentation overhaul,
packaging, onboarding/profile, trusted first-run, and local-knowledge work has
landed. v0.66 adds no new capability. It proves that a non-developer local-first
operator can install Allbert, start the persistent service, complete guided onboarding
through the consumer-default local model path, connect local files/notes, review agent
memory, reach first value, inspect safety/trust state, use existing implemented advanced
surfaces without regression, export/import or upgrade a Home, and uninstall/rollback
without hidden state, warning, security, or docs drift. BYOK/custom setup is validated as
the advanced and fallback path, not the primary no-docs path.

## Locked Decisions

1. **Two-layer verification (see Status).** `release.v066` GREEN means the
   contract/routing/boundary invariants hold and the evidence artifacts exist; the
   install/browser/model/cross-platform claims are carried by scripted smokes + human
   audit, reconciled at closeout. Every milestone/row/DoD bullet is tagged
   **gate-proved-contract** or **operator-attested-smoke**.
2. **Gate-bound `product-rc-*` rows are contract-level proxies.** Like the v0.59
   sweep, gate rows assert capability exposure/permission/confirmation floors and
   routing decisions, not live behavior (e.g. `ŌĆ”web-smoke-no-console-error` asserts
   LiveView render/dispatch, not a real browser console; `ŌĆ”oneclick-model-no-key-first-chat`
   asserts the routing/state machine, not that a model answered). Attested-only
   `product-rc-*` ids are evidence ids in `docs/validation/v0.66/`, not
   `EvalInventory` rows.
3. **Item-11 usability audit is evidence, not a gate row.** It is a human,
   non-`AssertBinding` audit; the plan defines its rubric and evidence artifact (M3),
   and `product-rc-web-usability-audit-item11-001` is operator-attested.
4. **Gate-enforced doc sweep (M10).** The full doc-surface consistency sweep is not a
   one-time editorial promise: hardcoded "current as of vX" stamps and restated release
   blurbs are replaced with links to the canonical sources (CHANGELOG/roadmap), and a new
   docs-gate check fails `release.v066` on stale version pins, missing index entries, and
   orphaned docs ŌĆö so drift cannot silently recur next release. The automated scope is
   active documentation: `README.md`, `docs/README.md`, `docs/operator/`, `docs/developer/`,
   `docs/design/`, and active `docs/plans/` indexes/current-next plans. Archives, samples,
   generated release evidence, screenshots/binary design assets, and older historical plans
   are excluded unless an active index links them as current guidance.
5. **No new authority, capability, permission class, or product surface** (validation
   only). Findings are fixed in the **owning capability area** before v1.0, never deferred
   into the freeze.
6. **Advanced-surface live regression bar.** The gate proves every advanced-surface
   contract, but live/manual evidence must cover each implemented surface class:
   browser research (`browser_research` and `browser_research_delegate`), remote channels
   (all configured providers, with at least one outbound and one inbound class represented
   or an explicit blocking-not-configured record), public protocols (MCP HTTP/stdio,
   OpenAI-compatible HTTP, ACP stdio using the v0.51 command set), Plan/Build workflow
   approval (the v0.44 workflow smoke path), and export/import. If a whole class cannot be
   exercised, closeout records it as blocking unless the operator explicitly scopes it out.
