## M1 Locked Decisions

Each is recorded in `v0.46-plan.md` §"M1 Locked Decisions" with a
one-paragraph rationale.

1. Delivery shape — plugin-contributed delegate agent at
   `./plugins/allbert.research/`, NOT a core module.
2. Authority surface — zero new authority; orchestrate shipped v0.43
   browser actions plus pinned model-backed summarization or deterministic
   extractive fallback through `Actions.Runner.run/3`; only a small
   `research.*` settings fragment.
3. Research scope — read-only navigate + bounded extract + advisory
   summary; bounded source count; inherits v0.43 deny-by-default.
4. Contract goal — second delegate-agent consumer to harden the contract
   before the v1.0 freeze (ADR 0021 A21); not an abstraction, not a
   StockSage refactor, not operator no-code authoring.
5. Delegate command contract — the `delegate_agent` action resolves the
   agent and allowlists its `allowed_commands` (string or atom) at the
   existing boundary; no arbitrary command dispatch and no dynamic atom
   creation.
6. Command threading — `Objectives.Commands.execute/4` threads the step
   command from `action_params.command` into the action `command` arg
   (replacing the hard-coded `execute`) so the boundary in decision 5 is
   reached by the objective and Plan/Build paths. No Step-schema migration
   (ADR 0021 A3).

## Main User/Operator Flows

- Operator (browser + research enabled) asks Allbert to research a topic
  or URL.
- Intent frames a one-step (or Plan/Build) objective with a
  `kind: delegate_agent` step targeting `research.specialist`.
- The research specialist orchestrates the v0.43 browser sequence per
  source; each `browser_navigate` confirms (or applies a v0.43 remembered
  per-domain grant) and renders inline under the parent delegate step.
- The specialist returns an advisory report packet (summary + per-source
  evidence references); nothing auto-promotes to memory.
- Operator can compose the research delegate as a step inside an authored
  workflow YAML (v0.44).

## Intent Phrase Corpus

Source of truth in `v0.46-plan.md` §"Planned Interfaces". M3 implements
one routing test per phrase; `release.v046` covers each as a step.

| Pattern | Routes to |
|---|---|
| `research <topic>` | delegated `research` command (when research enabled) |
| `research <URL> and summarize` | delegated `summarize_url` command over the URL |
| `summarize the research on <topic>` | delegated `research` command |

Patterns match case-insensitively. When research is disabled, these remain
inert and must not execute browser actions; operators can use the
browser-specific page handoff prompts (`summarize the page`, screenshot,
render, extract) or enable research explicitly. The corpus is M3-locked —
additions require an explicit re-decision here and in the plan.

## Data Flow

Research requests flow through `Actions.Runner.run/3` at every effectful
hop:

1. Intent frames an objective with a `kind: :delegate_agent` step:
   `delegate_agent_id: "research.specialist"`, `command: "research"` for
   topic requests or `command: "summarize_url"` for URL-summary requests,
   `params: {topic | url, max_sources?}`.
2. The engine's `:execute_step` for a `delegate_agent` step threads the
   command from `action_params.command` (default `execute`) into the
   `delegate_agent` action's `command` arg and forwards
   `action_params.params` as `params` — replacing the prior hard-coded
   `command: "execute"`. The registered `delegate_agent` action
   (`:objective_write`) looks up `research.specialist` in
   `AllbertAssist.Objectives.AgentRegistry`, validates `"research"`
   against the agent's registered `allowed_commands` metadata, normalizes
   the command without dynamic atom creation, and dispatches it. Unknown
   commands return the existing `:invalid_delegate_command` error.
   Legacy direct delegate steps that have no nested `params` key continue
   to pass the whole `action_params` map as params under the default
   `execute` command.
3. `AllbertResearch.Agent` orchestrates, per source, through
   `Actions.Runner.run/3`:
   - `browser_start_session` (`:browser_session_start`
     `:needs_confirmation`) — one session-start confirmation;
   - `browser_navigate` (`:browser_navigate` `:needs_confirmation`) —
     per-domain confirmation or a v0.43 remembered grant;
   - `browser_extract` (`:browser_extract` `:allowed`) — bounded by
     `research.max_extract_bytes_per_source` and the v0.43 caps;
   - pinned model-backed summarization over the extracted evidence when a
     reusable runtime entry point exists; otherwise deterministic
     extractive fallback over bounded extracts;
   - `browser_close_session` — always closes.
4. Each browser confirmation pauses at the standard Approval Handoff
   seam; the confirmation renders **inline under the parent delegate
   step** (v0.44 subagent-delegation visibility).
5. The agent returns an advisory normalized response (ADR 0029; advisory
   status per ADR 0021 §4) with `output_data: {summary, sources, notes}`.
6. The engine records the delegate step result; RunProgress renders the
   completed step with summary + per-source evidence references.
7. Trace/audit carry the objective id, delegate agent id, command, and
   redacted per-source URLs; `AllbertAssist.Security.Redactor` applies the
   v0.43 browser redaction; raw page content lives in the v0.43 browser
   cache, not in traces; the research summary is advisory metadata.

## Authority Invariants

- **Delegation does not widen authority.** The `:delegate_agent` step is
  `:objective_write`; it authorizes delegation, not browsing. Every
  `browser_navigate` inside a research dispatch still evaluates
  `:browser_navigate` and confirms (or applies a v0.43 grant).
- **Advisory output, never authority.** The research report is
  descriptive (ADR 0021 §4); it does not alter agent behavior and does
  not auto-promote to memory.
- **No new authority surface.** No new registered action, permission
  class, operation class, URI scheme, or memory namespace; the only new
  settings are an enable toggle and bounded caps.
- **Command names are not authority.** `delegate_agent` accepts only
  `execute` plus commands declared in the resolved agent's metadata, such
  as `research` and `summarize_url` for `research.specialist`. A command
  string selects an advisory delegate flow; it never grants browser,
  memory, settings, or workflow authority. The engine threads the step
  command into this boundary (it no longer hard-codes `execute`), so the
  objective and Plan/Build paths are validated here, not only direct
  action calls.
- **Inherits v0.43 posture.** Per-domain grants, network policy,
  extraction caps, redaction, ephemeral profiles, and the doctor-gated
  session start all apply because the agent uses the v0.43 actions
  unchanged.
- **Agent isolation.** `research.specialist` cannot reach StockSage agent
  state and vice versa; each is a distinct `AgentRegistry` registrant
  reachable only through the `delegate_agent` action.
- **Bounded fan-out.** `research.max_sources` (default 3, cap 8) bounds a
  single research command; no recursive crawling.

## Handles / CLI Surfaces

- Delegate agent id: `research.specialist` (in `AgentRegistry`).
- Registered action boundary: `delegate_agent` (v0.24, `:objective_write`).
- Allowlisted delegate commands: `research`, `summarize_url`.
- Orchestrated actions (v0.43): `browser_start_session`,
  `browser_navigate`, `browser_extract`, `browser_close_session`.
- CLI: `mix allbert.research "<topic-or-url>" [--max-sources=N]` (M3).
- Plan/Build: `kind: delegate_agent` step with
  `delegate_agent_id: research.specialist` (M3 example fixture).

## Smoke Checklist

- Confirm `research.enabled` defaults to `false`; `summary.engine` and
  `schema_version` are read-only; `max_sources` rejects > 8.
- Enable browser + research; frame a delegated research objective.
- Confirm a `browser_navigate` inside the research dispatch produces a
  navigation confirmation (delegation does not pre-approve).
- Confirm per-source navigations render inline under the parent delegate
  step.
- Confirm the report packet is advisory and writes nothing to memory.
- Confirm unknown delegate commands reject with
  `:invalid_delegate_command` through the objective path (a
  `kind: delegate_agent` step), not only a direct action call.
- Confirm `research.max_sources` bounds the source count.
- Confirm the browser session is always closed after a research command.
- Confirm `research.specialist` and a StockSage agent cannot read each
  other's state.
- Confirm `mix allbert.test release.v046` runs green on a fresh checkout
  and writes reproducible evidence.
