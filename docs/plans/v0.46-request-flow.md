## Main User/Operator Flows

- Operator uploads or pastes an image.
- Operator captures a screenshot.
- Allbert analyzes image/screenshot content through a vision-capable model.
- Operator requests image generation and confirms provider/cost details.

## Data Flow

Image or screenshot input becomes a bounded resource, provider doctor confirms
model capability, analysis or generation runs through registered actions, media
renders in workspace, and traces keep redacted metadata.

## Authority Invariants

Media resources do not grant action authority. Image generation is an external
provider action with normal policy. Binary media is not copied into traces by
default.

## Handles / CLI Surfaces

- `image://capture/<id>`
- `screen://capture/<id>`
