## Purpose

Give Allbert a **Pi-mode coding surface**: a gated terminal coding loop that
feels as minimal as Pi (four tools, a sub-1000-token prompt+tool-defs budget,
streamed diffs, deliberate chunked-read context discipline) **while running on
the one authority spine** — the same `AllbertAssist.Actions.Runner.run/3`,
Security Central, trace, and markdown-first memory as every other surface.

Stated structurally (rationale §4): Allbert's cognitive runtime is the
seven-stage machine **Receive → Interpret → Frame/resume objective →
Propose/evaluate steps → Authorize step → Execute (`Actions.Runner.run/3`) →
Observe/advance.** Pi is `{Receive → Interpret(implicit) → Execute(YOLO) →
Observe}` — it deletes stages 3–5. v0.57 makes **stages 6–7 and the tool surface
Pi-minimal** (four boundary actions, streamed split-payload diffs, observable,
sub-1000-token prompt+tool-defs budget) **while leaving stages 3–5 fully intact**.
A Pi-mode session is one where the objective is "pair on this repo," step proposal
is trivial, and authorization runs under the **local-coding operator trust tier
at sandbox Level 1** for a single trusted local operator. The gates are *present
but cheap*, never *absent*.

This is a **channel/app under ADR 0016 on the existing spine** — NOT a sibling
runtime, NOT a new authority. Four things land:

1. Four boundary actions (`read`/`write`/`edit`/`bash`) routed through the action
   runner and Security Central; `bash` runs host processes at sandbox Level 1.
2. A sub-1000-token coding prompt+tool-defs budget + deliberate chunked-read
   context-gathering discipline (offset/limit reads + artifact hand-off, not
   whole-file ingestion).
3. A named **"local-coding operator"** trust tier (ADR 0056 lineage, running at
   ADR 0009 **Level 1**) — single trusted operator, main session, terminal
   channel; never the default, never for channel-originated or generated-code
   sessions. Not a sandbox level; never "level 0".
4. **Net-new** streamed diffs — a live-region substrate + progressive
   tool-argument rendering over the v0.55 static split payload — plus an operator
   affordance for mid-session model switch (`req_llm` history handoff), and the
   minimalism budget as a standing coding policy.

## Readiness Sweep
- **Contract fit.** The Pi-mode surface is a delivery+driver experience over the
  v0.55 `tui` channel; it owns no intent/Security Central/Resource Access/
  confirmation storage/conversation history. Coding effects reach the host *only*
  through registered actions executed by `Actions.Runner.run/3`. The runner +
  Security Central decide; the surface renders and dispatches.
- **Reuse fit.** Reuse `plugins/allbert.tui` (normalize → `Identity.resolve` →
  `Channels.derive_session_id` → `Runtime.submit_user_input` → `ChannelThread` →
  `channel_events`), the v0.55 **static** split-result contract
  (`AllbertAssist.Runtime.Response` `model_payload`/`surface_payload`, ADR 0067
  extending ADR 0029/0030) as the payload v0.57's **net-new** streaming renders
  over, the existing `actions/` boundary — specifically `run_shell_command`
  (argv-only Level 1) and `execution/skill_script_runner` (timeout + brutal-kill)
  — as the template for the four new coding actions and the `bash` policy, the
  ADR 0056 trust-tier convention + ADR 0009 Level 1 for the tier, the v0.03 skill
  loader (`AllbertAssist.Skills`) + `AGENTS.md` hierarchy for context, and
  req_llm message-history serialization for the model switch (cross-provider
  conversion is best-effort; confirm the shape via Context7 at M4 — it is **not**
  ADR 0047/0051, which are the provider doctor/capability contracts).
- **Upstream check.** Confirm v0.55 delivered the `tui` channel AND the **static**
  split tool-result payload (`model_payload`/`surface_payload`) through the typed
  response contract + renderer. v0.55 did **not** ship a live-region/streaming
  substrate; that is v0.57-owned (M1). If the `tui` channel or the static split is
  missing/regressed, stop and reconcile v0.55/v0.56 before implementing v0.57.
- **Authority check (AGENTS.md Non-Negotiables).** Advisory output is never
  authority: the coding model may propose `read`/`write`/`edit`/`bash` calls but
  may not bypass `Actions.Runner.run/3`, Security Central, or confirmations. The
  bash action is policy-bounded host execution behind a registered action — OTP
  supervision is **not** an OS security boundary. No model-decides-it's-done for
  effectful or generated-code work. No sub-agent-via-bash-spawn.
- **Library fit.** No new library is required; `owl` (v0.55, terminal render) and
  `req_llm` (provider neutrality + message-history serialization) cover the
  surface. The live-region render is built on `owl` in M1 (net-new, no new dep).
  Context7 confirms the req_llm message-history/serialization shape before the M4
  switch affordance, and the cross-provider conversion limits (best-effort).

## Locked Decisions
| # | Decision | Chosen value | Trade-off if changed |
|---|---|---|---|
| 1 | Surface authority | Pi-mode is a **channel/app on the one spine** (ADR 0016): every coding effect routes through `Actions.Runner.run/3` + Security Central. The surface has **no authority beyond any other channel**. | A sibling runtime or a direct-effect coding loop reintroduces Pi's YOLO threat model — the thing Allbert exists to reject. |
| 2 | Tool surface | Exactly **four default boundary actions** for coding — `read`, `write`, `edit`, `bash` — each a registered Jido action via `Actions.Registry`/`Runner`. Everything else is lazy-disclosed (MCP-first, ADR 0038/0039), not pre-loaded. | A wide default tool surface defeats the minimalism budget and the lazy-disclosure discipline Allbert already has from v0.03. |
| 3 | Trust tier | A named **"local-coding operator"** trust tier (ADR 0056 lineage) running at **sandbox Level 1** (ADR 0009): single trusted operator, main session, `tui` channel. **Never** the default; **never** for channel-originated inbound or generated-code sessions. The tier lowers only the *confirmation burden*; policy/redaction/trace/audit still run at Level 1. It is **not** a sandbox level and never "level 0". | An implicit or default tier turns a low-friction convenience into Pi-by-default and breaks the action boundary. Mislabeling it "level 0" collides with ADR 0009 (level 0 = inert/no-exec) and conflates caller-trust with isolation. |
| 4 | Acceptance | **Deterministic acceptance rules**, not model confidence. For effectful or generated-code work the model never "decides it's done"; evidence + confirmation rule (same posture Allbert took against Hermes). | Model-confidence completion contradicts "advisory output is never authority" and the codegen committee's deterministic gates. |
| 5 | Prompt + context | A **sub-1000-token prompt+tool-defs budget** (system prompt and the four tool definitions combined, against the configured tokenizer — Pi's actual bar). Context discipline follows **Pi's documented practice**: chunked reads (offset/limit) + separate context-gathering sessions + file artifacts, **not** whole-file ingestion. Reuse the `AGENTS.md` hierarchy + v0.03 skill loader. | A large default budget or whole-file context bloats the surface. (Note: full-file ingestion is *not* a Pi finding — Pi reads in bounded chunks and notes models resist full reads; do not attribute it to Pi.) |
| 6 | Diff rendering | **Net-new in v0.57:** a live-region substrate + progressive tool-argument rendering, built on the v0.55 **static** split payload (`model_payload` vs. `surface_payload`, ADR 0067) through the unified renderer (ADR 0030). The terminal renders the surface payload (streamed as tool-call arguments arrive) while the model sees the clean model payload. The streamed unit is the tool-argument stream, not the tool result. | Treating streaming as a v0.55 inheritance understates the work; v0.55 ships only the static split. A single conflated payload re-bloats model context. |
| 7 | Model switch | Mid-session model switch is an **operator affordance** over req_llm's canonical `Context` handoff (ADR 0047/0051), surfaced in the `tui` session; it changes the model, never the authority path. | Hiding the switch wastes a structurally-cheap req_llm capability Pi proved is worth surfacing. |
| 8 | Sub-agents | Delegation uses **restricted-registry delegate agents**, never an opaque `bash`-spawned sub-agent. | Self-spawn via bash is an unobservable, unrestricted child — the Hermes "narrow the child's tools" lesson, violated. |
| 9 | `bash` semantics | **Raw shell strings, but only at the local-coding operator tier** (Pi-faithful). For every other caller `bash` is unavailable or falls back to argv-only `run_shell_command`. Tier-restricted raw shell is bounded by cwd-confinement to the repo root, a wall-clock timeout + brutal-kill (reuse `skill_script_runner`), full trace/audit, and the stance that the host/container — not OTP — is the security boundary. | A general raw-shell action would break `run_shell_command`'s anti-injection (argv-only) posture for untrusted callers. Restricting raw shell to one trusted local operator keeps the convenience without widening the channel-originated attack surface. |

## Milestones

### M0: Contracts + ADR + Trust-Tier Definition  *(serial barrier)*

at sandbox **Level 1**) as the authority contract; confirm the v0.55 upstreams
(`tui` channel + the **static** split tool-result) exist or block. Define the
coding-action contract shape (the four actions' params/results, signals, and
Security Central policy hooks) and the trust-tier vocabulary as an ADR 0056
extension resolving at ADR 0009 Level 1. **No effectful code lands here** — this is
the barrier the parallel lanes rejoin against.
- **Implementation:**
  - **Modules/signatures:** `Security.Policy`/`Security.PermissionGate` gain a
    trust-tier resolver `coding_tier(context) :: :local_coding_operator | :none`
    keyed on `operator_id` + main-session `session_id` + `channel == "tui"` +
    not-channel-originated + not-generated-code; ADR 0068 + the ADR 0009 v0.57
    clarification + ADR 0056 lineage are the written contracts.
  - **Files:** `docs/adr/0068-*.md` (Accepted at M0), ADR 0009 clarification +
    ADR 0067 streaming-ownership correction referenced; contract notes in this
    plan + request-flow.
  - **Settings (declared here, populated later):** see `## Settings Central Keys`.
  - **Tests:** trust-tier resolution fixture (tier resolves only for trusted

  - **Verify:** ADR-fixture resolution table green; upstream check (`tui` +

### M1: Live-Region Substrate + Streamed-Diff Terminal Render  *(parallel lane A)*
**Net-new streaming.** Build the live-region render substrate the v0.55 static
split lacks: render the `surface_payload` diff via the unified renderer
(ADR 0030) / `owl`, while the model receives only the clean `model_payload`.
Stream `write`/`edit` diffs to the terminal by **progressively parsing tool-call
arguments** as they arrive (Pi's mechanism — the streamed unit is the argument
stream, not the tool result). Touches rendering only — no authority.
- **Implementation:**
  - **Modules:** a TUI live-region renderer (e.g. `Channels.TUI.LiveRegion`) over
    `owl`; a progressive tool-argument parser feeding partial diffs; consumes
    `AllbertAssist.Runtime.Response` `surface_payload`.
  - **Files:** `plugins/allbert.tui/lib/.../live_region.ex` (net-new),
    renderer wiring in the TUI adapter.
  - **Tests:** `tui_live_region_test.exs` — `surface_payload` renders to terminal
    with no UI payload in `model_payload`/memory; progressive-arg chunks render in
    order; **turn-complete fallback** when streaming is unavailable (degrades to
    the v0.55 static render).
  - **Verify:** a simulated streamed `edit` renders an incremental diff; the
    model context contains only the clean payload.
- Lane: parallel with M2 (renderer surface, no runner coupling).

### M2: Four Boundary Actions + Runner Routing  *(parallel lane B)*
Register `read`, `write`, `edit`, `bash` as Jido actions via `Actions.Registry`,
each executed through `Actions.Runner.run/3` and gated by Security Central. `bash`
is host execution at **sandbox Level 1** behind the registered action (not an OS
boundary; not a sub-agent spawn). Results emit the split payload (clean
`model_payload` + render `surface_payload` diff) for M1.
- **Implementation:**
  - **Modules/Actions:** `Actions.Coding.{Read,Write,Edit,Bash}` (names TBD at M0),
    each `use AllbertAssist.Action` with `permission:`/`exposure:`/`confirmation:`
    metadata. `Bash` models on `actions/intent/run_shell_command.ex` for the
    argv-only fallback path and `execution/skill_script_runner.ex` for
    timeout + brutal-kill; **raw shell only resolves under the M3 tier**.
  - **Files:** `apps/allbert_assist/lib/allbert_assist/actions/coding/{read,write,edit,bash}.ex`;
    registry entries in `actions/registry.ex`.
  - **Permissions:** new Security.Policy entries (e.g. `coding.read`,
    `coding.write`, `coding.edit`, `coding.bash`) with safety floors; `bash`
    `confirmation: :required` by default (the M3 tier makes it cheap, not absent).
  - **Settings:** `## Settings Central Keys` (`coding.bash.cwd_jail`,
    `coding.bash.timeout_ms`, `coding.bash.allow_raw_shell`, etc).
  - **Tests:** each action resolves via `Actions.Registry` + executes via
    `Runner.run/3`; `bash` refuses outside policy and (pre-M3) requires
    confirmation; `write`/`edit` produce a split result; no action bypasses
    Security Central; no bash-spawned sub-agent path exists; `read` is chunked
    (offset/limit).
  - **Verify:** `Actions.Runner.run/3` drives each of the four; `bash` honors
    cwd-jail + timeout.
- Lane: parallel with M1 (action boundary, no render coupling).

### M3: Local-Coding Operator Trust Tier  *(serial — gates M2's cheapness)*
Implement the **local-coding operator** tier from M0: Security Central resolves it
**only** for a single trusted operator in the main session on the `tui` channel,

argv-only `bash` for every other caller. The tier runs at **sandbox Level 1** —
policy, redaction, trace, and audit still execute. Channel-originated inbound,
scheduled jobs, and generated-code sessions **cannot** reach the tier.
- **Implementation:**
  - **Modules:** the `coding_tier/1` resolver from M0 wired into
    `Security.PermissionGate`'s confirmation-cost decision and into `Bash`'s
    raw-vs-argv selection.
  - **Files:** `apps/allbert_assist/lib/allbert_assist/security/{policy,permission_gate}.ex`.
  - **Settings:** `coding.trusted_operator_id`, `coding.bash.allow_raw_shell`
    (tier-gated), `coding.pi_mode.enabled`.
  - **Tests:** trusted-operator/main/`tui` → tier resolves, cheap gate + raw shell,
    full trace; channel-originated → full gate, argv-only, no tier escalation;
    generated-code session → full gate; tier never the default; tier name +
    resolution context auditable in the trace.
  - **Verify:** the resolution table holds; a simulated channel-originated caller
    cannot reach the tier or raw shell.
- Serial after M2 (the tier modulates how M2's gate runs); rejoins before M4.

## Settings Central Keys
All new settings register in the Settings Central schema (specs + defaults +
`safe_write_keys`); no module reads ad-hoc config. Defaults are conservative
(Pi-mode off; raw shell off until the tier resolves).

| Key | Default | Purpose |
|---|---|---|
| `coding.pi_mode.enabled` | `false` | Master enable for the Pi-mode coding surface. |
| `coding.trusted_operator_id` | `nil` | The single operator the local-coding tier resolves for. |
| `coding.bash.allow_raw_shell` | `false` | Permit raw shell strings — **honored only when the local-coding tier resolves**; argv-only otherwise. |
| `coding.bash.cwd_jail` | repo root | Directory the `bash`/`write`/`edit` actions are confined to. |
| `coding.bash.timeout_ms` | (e.g. `120000`) | Wall-clock timeout before brutal-kill (reuse `skill_script_runner`). |
| `coding.prompt.token_budget` | `1000` | Combined system-prompt + tool-defs ceiling (named tokenizer). |
| `coding.model_profile` | (router/coding default) | Model profile the Pi-mode session starts with. |
| `coding.read.default_limit` | (e.g. `2000`) | Default chunked-read line limit (offset/limit). |

## Non-Goals
*(from the Pi analysis §8 "what not to do")*
- **No YOLO-by-default.** The local-coding operator tier is a named, gated,
  non-default tier (sandbox Level 1) — never the default, never for
  channel-originated or generated-code sessions.
- **No weakening of the action boundary / Security Central / confirmations.** The
  gates are made *cheap at the tier*, not *absent*; every effect still routes
  through `Actions.Runner.run/3` at Level 1.
- **No "sandbox level 0" label / no isolation relaxation.** The tier changes the
  confirmation burden, not the sandbox level; host execution stays Level 1.
- **No model-decides-it's-done** for effectful or generated-code work;
  deterministic acceptance rules.
- **Keep MCP-first with lazy disclosure** (ADR 0038/0039); do not adopt Pi's
  no-MCP stance. The minimalism is in the *default* surface, not a ban.
- **No sub-agent-via-bash-spawn**; use restricted-registry delegate agents.
- **No sibling runtime**; Pi-mode is a channel/app on the one spine — same
  authority boundary, same trace, same memory.
- No new TUI renderer choice; reuse the v0.55 `owl` terminal surface (Pi's
  scrollback-native model stays reserved until evidence).

## Definition Of Done
v0.57 is done when:
- a Pi-mode coding surface runs as a channel/app on the `tui` channel, exposing
  exactly four boundary actions (`read`/`write`/`edit`/`bash`), each resolving
  through `Actions.Registry` and executing via `Actions.Runner.run/3` under
  Security Central;
- a named "local-coding operator" trust tier (ADR 0056 lineage, running at ADR
  0009 **Level 1**, ADR 0068) makes the gate present-but-cheap **only** for a
  single trusted operator in the main session on `tui` (and enables raw-shell
  `bash` only there), and provably cannot be reached by channel-originated,
  scheduled, or generated-code sessions, and is never the default;
- streamed diffs render in the terminal via the **net-new v0.57** live-region
  substrate over the v0.55 static split payload (ADR 0067/0029/0030) with no
  `surface_payload` leaking into model context;
- the coding prompt **+ tool definitions combined** are under the token budget,
  context gathering uses chunked reads + artifacts (not whole-file ingestion), the
  default turn exposes only the four actions, and a mid-session model switch
  preserves session history via req_llm without touching the authority path;
- acceptance for effectful/generated-code work is deterministic (evidence +
  confirmation), with no model-confidence completion and no bash-spawned sub-agent;
- `:v057` evals + `release.v057` + the full release gate pass;
- ADR 0068, the minimalism-budget coding policy, roadmap, vision, and
  agent-context-map agree, and `pi-integration-rethink.md` items 3–5 are delivered.
