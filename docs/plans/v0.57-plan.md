## Purpose

Give Allbert a **Pi-mode coding surface**: a gated terminal coding loop that
feels as minimal as Pi and as **intuitive as Claude Code / Codex / Gemini CLI** (a
tight tool surface — four effectful boundary actions plus read-only `grep`/`glob` —
a sub-1000-token prompt+tool-defs budget, streamed diffs **and** assistant-text
token streaming, coder-facing approval modes, Esc-to-cancel, a familiar slash set,
deliberate chunked-read context discipline) **while running on the one authority
spine** — the same `AllbertAssist.Actions.Runner.run/3`, Security Central, trace,
and markdown-first memory as every other surface.

Stated structurally (rationale §4): Allbert's cognitive runtime is the
seven-stage machine **Receive → Interpret → Frame/resume objective →
Propose/evaluate steps → Authorize step → Execute (`Actions.Runner.run/3`) →
Observe/advance.** Pi is `{Receive → Interpret(implicit) → Execute(YOLO) →
Observe}` — it deletes stages 3–5. v0.57 makes **stages 6–7 and the tool surface
Pi-minimal** (four effectful boundary actions + read-only `grep`/`glob`, streamed
diffs + token streaming, observable, sub-1000-token prompt+tool-defs budget)
**while leaving stages 3–5 fully intact**.
A Pi-mode session is one where the objective is "pair on this repo," step proposal
is trivial, and authorization runs under the **local-coding operator trust tier
at sandbox Level 1** for a single trusted local operator. The gates are *present
but cheap*, never *absent*.

This is a **channel/app under ADR 0016 on the existing spine** — NOT a sibling
runtime, NOT a new authority. What lands:

1. **Six default tools:** four effectful boundary actions
   (`read`/`write`/`edit`/`bash`) + two **read-only** search actions
   (`grep`/`glob`, ripgrep-backed, `.gitignore`-aware), all routed through the
   action runner and Security Central; `bash` runs host processes at sandbox
   Level 1; read-only search runs unprompted, as in every peer tool.
2. **Coder-facing approval modes** (`default` / `accept-edits` / `plan` / `tier`)
   plus persistent **"always allow this command"** per repo — a presentation +
   persistence layer over Security Central (Policy allowlist + confirmations),
   granting **no new authority**.
3. A named **"local-coding operator"** trust tier (ADR 0056 lineage, running at
   ADR 0009 **Level 1**) — single trusted operator, main session, terminal
   channel; never the default, never for channel-originated or generated-code
   sessions. Not a sandbox level; never "level 0".
4. **Net-new live rendering:** a coding-diff live-region renderer + progressive
   tool-argument streaming + **assistant-text token streaming** over the v0.55
   static split payload, plus **Esc-to-cancel** and a **queued-correction** steer.
5. A sub-1000-token prompt+tool-defs budget + chunked-read context discipline; a
   familiar coding **slash set** (`/help`, `/model`, `/clear`, `/init`, `/diff`,
   `/compact`); a mid-session model switch (`req_llm` history handoff); and the
   minimalism budget as a standing coding policy.

## Readiness Sweep
- **Contract fit.** The Pi-mode surface is a delivery+driver experience over the
  v0.55 `tui` channel; it owns no intent/Security Central/Resource Access/
  confirmation storage/conversation history. Coding effects reach the host *only*
  through registered actions executed by `Actions.Runner.run/3`. The runner +
  Security Central decide; the surface renders and dispatches.
- **Reuse fit.** Reuse `plugins/allbert.tui` (normalize → `Identity.resolve` →
  `Channels.derive_session_id` → `Runtime.submit_user_input` → `ChannelThread` →
  `channel_events`), the v0.55 **static** split-result contract
  (`AllbertAssist.Runtime.Response` `model_payload`/`surface_payload`, ADR 0067
  extending ADR 0029/0030) as the payload v0.57's **net-new** streaming renders
  over, the existing `actions/` boundary — specifically `run_shell_command`
  (argv-only Level 1) and `execution/skill_script_runner` (timeout + brutal-kill)
  — as the template for the four new coding actions and the `bash` policy, the
  ADR 0056 trust-tier convention + ADR 0009 Level 1 for the tier, the v0.03 skill
  loader (`AllbertAssist.Skills`) + `AGENTS.md` hierarchy for context, and
  req_llm message-history serialization for the model switch (cross-provider
  conversion is best-effort; at M4 try Context7 first, then verify against the
  vendored `deps/req_llm/usage-rules.md` and `deps/req_llm/lib/req_llm/context.ex`
  if Context7 has no `req_llm` package entry — it is **not** ADR 0047/0051, which
  are the provider doctor/capability contracts).
- **Upstream check.** Confirm v0.55 delivered the `tui` channel AND the **static**
  split tool-result payload (`model_payload`/`surface_payload`) through the typed
  response contract + renderer. v0.55 did **not** ship a progressive coding-diff
  renderer or streaming substrate; that is v0.57-owned (M1). If the `tui` channel
  or the static split is missing/regressed, stop and reconcile v0.55/v0.56 before
  implementing v0.57.
- **Authority check (AGENTS.md Non-Negotiables).** Advisory output is never
  authority: the coding model may propose `read`/`write`/`edit`/`bash` calls but
  may not bypass `Actions.Runner.run/3`, Security Central, or confirmations. The
  bash action is policy-bounded host execution behind a registered action — OTP
  supervision is **not** an OS security boundary. No model-decides-it's-done for
  effectful or generated-code work. No sub-agent-via-bash-spawn.
- **Library fit.** No new library is required; `owl` (v0.55, terminal render) and
  `req_llm` (provider neutrality + message-history serialization) cover the
  surface. The coding diff live-region renderer is built on `owl` in M1 (net-new,
  no new dep). M4 first tries Context7 for `req_llm` docs, then falls back to the
  vendored `deps/req_llm` docs/source if Context7 has no first-class package
  entry; the cross-provider conversion limit remains best-effort.

## Coding-Surface Ergonomics & CLI Parity
The surface must feel intuitive to a developer arriving from Claude Code, Pi,
Codex CLI, or Gemini CLI. The minimalism is in the *authority spine and default
surface*, **not** in the interaction model — the everyday coding ergonomics those
tools share are first-class deliverables here, expressed as Allbert primitives.

| Convention | Claude Code / Codex / Gemini / Pi | Allbert Pi-mode (v0.57) |
|---|---|---|
| **Tool surface** | read/write/edit/bash + Grep/Glob (Claude, Gemini); Codex/Pi = shell does search | **Six**: `read`/`write`/`edit`/`bash` + read-only `grep`/`glob` (registered actions). Beyond the six → MCP lazy disclosure. |
| **Code search** | first-class Grep/Glob, gitignore-aware, unprompted | `grep`/`glob` actions (ripgrep-backed, `.gitignore`-aware), `permission: :read_only`, run unprompted; `bash` `rg`/`find` also available at the tier. |
| **Approval UX** | named modes (acceptEdits/plan/yolo…) + persistent "always allow" | modes `default` / `accept-edits` / `plan` / `tier` + per-repo "always allow this command" — a view over Security Central, no new authority. |
| **Streaming** | assistant text streams token-by-token; diffs/output render live | **net-new v0.57**: assistant-text token streaming + progressive tool-argument diff streaming via the live-region renderer. |
| **Interrupt / steer** | Esc cancels; queued correction / Enter-to-steer | **Esc-to-cancel** a running turn (keep work so far) + **queued correction**; `bash` timeout/brutal-kill underneath. |
| **Slash set** | /help /model /clear /init /diff /compact … | `/help` (exists), `/model`, `/clear`, `/init`, `/diff`, `/compact` — coding-session slash allowlist (net-new beyond the v0.55.1 operator set). |
| **Context file** | AGENTS.md / CLAUDE.md / GEMINI.md auto-load + @file + /init | `AGENTS.md` hierarchy (Allbert already uses it — Codex made AGENTS.md the native default, so this is **parity**) + `@file` mentions + `/init`. |
| **Loop / "done"** | loop until model stops emitting tool calls, then return to prompt | same interactive REPL: the turn ends when the model stops calling tools. "Deterministic acceptance / model never decides done" applies to **effectful/generated-code acceptance gates**, not to ending a conversational turn. |

Two of these choices carry **runtime dependencies beyond rendering** and are flagged
in the milestones: assistant-text **token streaming** needs the runtime to emit
model token deltas (today's v0.55 response is turn-complete), and **interrupt**
needs clean turn cancellation (abort the in-flight model call + any running action
with a partial-turn trace; `bash` brutal-kill exists, model-call abort is net-new).

## Locked Decisions
| # | Decision | Chosen value | Trade-off if changed |
|---|---|---|---|
| 1 | Surface authority | Pi-mode is a **channel/app on the one spine** (ADR 0016): every coding effect routes through `Actions.Runner.run/3` + Security Central. The surface has **no authority beyond any other channel**. | A sibling runtime or a direct-effect coding loop reintroduces Pi's YOLO threat model — the thing Allbert exists to reject. |
| 2 | Tool surface | **Six default tools**: four effectful boundary actions (`read`, `write`, `edit`, `bash`) + two **read-only** search actions (`grep`, `glob`; ripgrep-backed, `.gitignore`-aware, `permission: :read_only`, run unprompted). Each a registered Jido action via `Actions.Registry`/`Runner`. Capabilities **beyond** the six are lazy-disclosed (MCP-first, ADR 0038/0039), not pre-loaded. | Omitting `grep`/`glob` reads as impoverished to Claude Code / Gemini users (2 of the 3 ship them first-class) and forces an MCP round-trip just to search a repo. A *wide* default surface beyond search would defeat the minimalism budget. |
| 3 | Trust tier | A named **"local-coding operator"** trust tier (ADR 0056 lineage) running at **sandbox Level 1** (ADR 0009): single trusted operator, main session, `tui` channel. **Never** the default; **never** for channel-originated inbound or generated-code sessions. The tier lowers only the *confirmation burden*; policy/redaction/trace/audit still run at Level 1. It is **not** a sandbox level and never "level 0". | An implicit or default tier turns a low-friction convenience into Pi-by-default and breaks the action boundary. Mislabeling it "level 0" collides with ADR 0009 (level 0 = inert/no-exec) and conflates caller-trust with isolation. |
| 4 | Acceptance | **Deterministic acceptance rules**, not model confidence. For effectful or generated-code work the model never "decides it's done"; evidence + confirmation rule (same posture Allbert took against Hermes). | Model-confidence completion contradicts "advisory output is never authority" and the codegen committee's deterministic gates. |
| 5 | Prompt + context | A **sub-1000-token prompt+tool-defs budget** (system prompt and the four tool definitions combined, against the configured tokenizer — Pi's actual bar). Context discipline follows **Pi's documented practice**: chunked reads (offset/limit) + separate context-gathering sessions + file artifacts, **not** whole-file ingestion. Reuse the `AGENTS.md` hierarchy + v0.03 skill loader. | A large default budget or whole-file context bloats the surface. (Note: full-file ingestion is *not* a Pi finding — Pi reads in bounded chunks and notes models resist full reads; do not attribute it to Pi.) |
| 6 | Live rendering | **Net-new in v0.57:** a coding-diff live-region renderer + progressive tool-argument rendering **and assistant-text token streaming**, built on the v0.55 **static** split payload (`model_payload` vs. `surface_payload`, ADR 0067) through the unified renderer (ADR 0030). The terminal renders the surface payload (diffs as tool-call arguments arrive; assistant prose token-by-token) while the model sees the clean model payload. Token streaming requires the runtime to emit model token deltas (today's response is turn-complete) — a runtime dependency, not just a renderer. | Treating streaming as a v0.55 inheritance understates the work; v0.55 ships only the static split. A turn-complete reply (no token streaming) feels dead next to every peer CLI. A single conflated payload re-bloats model context. |
| 7 | Model switch | Mid-session model switch is an **operator affordance** over req_llm's canonical `Context` handoff (verified through Context7 when available, otherwise through vendored `deps/req_llm` docs/source), surfaced in the `tui` session; it changes the model, never the authority path. ADR 0047/0051 remain provider doctor/capability contracts and do not own this handoff. | Hiding the switch wastes a structurally-cheap req_llm capability Pi proved is worth surfacing. |
| 8 | Sub-agents | Delegation uses **restricted-registry delegate agents**, never an opaque `bash`-spawned sub-agent. | Self-spawn via bash is an unobservable, unrestricted child — the Hermes "narrow the child's tools" lesson, violated. |
| 9 | `bash` semantics | **Raw shell strings, but only at the local-coding operator tier** (Pi-faithful). For every other caller `bash` is unavailable or falls back to argv-only `run_shell_command`. Tier-restricted raw shell is bounded by cwd-confinement to the repo root, a wall-clock timeout + brutal-kill (reuse `skill_script_runner`), full trace/audit, and the stance that the host/container — not OTP — is the security boundary. | A general raw-shell action would break `run_shell_command`'s anti-injection (argv-only) posture for untrusted callers. Restricting raw shell to one trusted local operator keeps the convenience without widening the channel-originated attack surface. |
| 10 | Approval modes | Coder-facing modes `default` / `accept-edits` / `plan` / `tier`, plus persistent **"always allow this command"** per repo. This is a **presentation + persistence layer over Security Central** (Policy allowlist + confirmations) — it changes *confirmation cost*, never the authority path; `plan` is read-only (propose, never execute). The `tier` mode is the local-coding operator tier (cheapest). | No coder-facing modes means a non-tier session prompts on every effect (permission fatigue — the exact thing Claude/Codex/Gemini solved with modes + remembered allowances). Modes that granted authority would violate "advisory output is never authority." |
| 11 | Interrupt / steer | **Esc cancels** a running turn (work done so far is kept + traced) and a **queued correction** lets the operator redirect without killing the turn. Cancellation cleanly aborts the in-flight model call and any running action; `bash` already supports timeout/brutal-kill. | No interrupt is jarring — Esc-to-cancel is a universal reflex. Cancellation is a runtime concern (model-call abort + partial-turn trace), so it is scoped as net-new work, not assumed from v0.55. |
| 12 | Slash set | A familiar coding **slash allowlist**: `/help` (exists), `/model`, `/clear`, `/init` (scaffold `AGENTS.md`), `/diff`, `/compact`, reusing the v0.55.1 slash-allowlist branch + `@file` mentions. Coding slash commands are non-routable (allowlist only), like the v0.55.1 operator set. | Only `/model` would feel bare; coders expect a baseline. Routable slash commands would re-introduce the operator/agent-surface boundary problem v0.56 M14a fixed. |

## Milestones

Lane summary: **M0** contracts (serial barrier) → **M1** six actions ∥ **M2**
live-region diff render → **M3** token streaming + interrupt/steer (after M2) ∥
**M4** trust tier + approval modes (serial, after M1) → **M5** prompt + context +
model switch + slash set → **M6** evals + closeout (serial barrier). Each
milestone's **Implementation** block lists Modules / Files / Settings / Tests /
Verify so any agent can build and verify it independently.

### M0: Contracts + ADR + Trust-Tier + Approval-Mode Definition  *(serial barrier)*
Land ADR 0068 (Pi-mode coding surface + the **local-coding operator** trust tier
at sandbox **Level 1**) as the authority contract; confirm the v0.55 upstreams
(`tui` channel + the **static** split tool-result) exist or block. Define the
contract shapes the parallel lanes rejoin against: the **six tools'**
params/results/signals + Security Central policy hooks, the trust-tier vocabulary
as an ADR 0056 extension resolving at ADR 0009 Level 1, and the **approval-mode**
vocabulary (`default`/`accept-edits`/`plan`/`tier`) as a confirmation-cost layer
over Security Central. **No effectful code lands here.**
- **Implementation:**
  - **Modules/signatures:** `Security.Policy`/`Security.PermissionGate` gain a
    trust-tier resolver `coding_tier(context) :: :local_coding_operator | :none`
    (keyed on `operator_id` + main-session `session_id` + `channel == "tui"` +
    not-channel-originated + not-generated-code) and an `approval_mode/1` reader;
    ADR 0068 + the ADR 0009 v0.57 clarification + ADR 0056 lineage are the
    written contracts.
  - **Files:** `docs/adr/0068-*.md` (Accepted at M0), ADR 0009 clarification +
    ADR 0067 streaming-ownership correction referenced; contract notes here +
    request-flow.
  - **Settings (declared here, populated later):** see `## Settings Central Keys`.
  - **Tests:** trust-tier resolution fixture (resolves only for trusted operator +
    main session + `tui`; rejects channel-originated / generated-code / non-`tui`);
    approval-mode vocabulary fixture (`plan` is read-only; no mode grants
    authority); no effectful path exists yet.
  - **Verify:** ADR-fixture resolution + mode tables green; upstream check (`tui`
    + static split) green.
- Gate: ADR 0068 accepted; upstream check green; tier + mode contracts reviewed.
- Rejoin point for M1 and M2.

### M1: Six Boundary + Search Actions + Runner Routing  *(parallel lane B)*
Register the four effectful actions `read`, `write`, `edit`, `bash` **and the two
read-only search actions `grep`, `glob`** as Jido actions via `Actions.Registry`,
each executed through `Actions.Runner.run/3` and gated by Security Central. `bash`
is host execution at **sandbox Level 1** behind the registered action (not an OS
boundary; not a sub-agent spawn). Effectful results emit the split payload (clean
`model_payload` + render `surface_payload` diff) for M2.
- **Implementation:**
  - **Modules/Actions:** `Actions.Coding.{Read,Write,Edit,Bash,Grep,Glob}` (names
    TBD at M0), each `use AllbertAssist.Action`. `Grep`/`Glob` are
    `permission: :read_only`, `confirmation: :not_required`, ripgrep-backed,
    `.gitignore`-aware, and run **unprompted**. `Bash` models on
    `actions/intent/run_shell_command.ex` (argv-only fallback) +
    `execution/skill_script_runner.ex` (timeout + brutal-kill); **raw shell only
    resolves under the M4 tier** via a dedicated `Coding.RawShellSpec`/runner path
    (name TBD at M0) so the existing `RunShellCommand` argv-only contract stays
    intact for all non-tier callers.
  - **Files:** `apps/allbert_assist/lib/allbert_assist/actions/coding/{read,write,edit,bash,grep,glob}.ex`;
    registry entries in `actions/registry.ex`.
  - **Permissions:** new Security.Policy entries (`coding.read`, `coding.write`,
    `coding.edit`, `coding.bash`; `coding.grep`/`coding.glob` read-only) with
    safety floors; `bash`/`write`/`edit` `confirmation: :required` by default (the
    M4 tier/modes make it cheap, not absent).
  - **Settings:** `## Settings Central Keys` (`coding.workspace.cwd_jail`,
    `coding.bash.timeout_ms`, `coding.bash.allow_raw_shell`, `coding.read.default_limit`).
  - **Tests:** each action resolves via `Actions.Registry` + executes via
    `Runner.run/3`; `grep`/`glob` are read-only, gitignore-aware, and never
    prompt; `bash` refuses outside policy and (pre-M4) requires confirmation; raw
    shell cannot be reached through `run_shell_command`; `read` is chunked
    (offset/limit); `write`/`edit` produce a split result and refuse path
    traversal, symlink escape, and writes outside the workspace cwd jail; an
    `edit` exact-match miss returns a clear failure (re-read/retry), not a silent
    no-op; no action bypasses Security Central; no bash-spawned sub-agent exists.
  - **Verify:** `Actions.Runner.run/3` drives all six; `grep` finds across the
    repo unprompted; `bash` honors cwd-jail + timeout; file effects cannot escape
    the cwd jail through normalized paths or symlinks.
- Lane: parallel with M2 (action boundary, no render coupling).

### M2: Live-Region Substrate + Streamed-Diff Terminal Render  *(parallel lane A)*
**Net-new rendering.** Build the live-region render substrate the v0.55 static
split lacks: render the `surface_payload` diff via the unified renderer
(ADR 0030) / `owl`, while the model receives only the clean `model_payload`.
Stream `write`/`edit` diffs to the terminal by **progressively parsing tool-call
arguments** as they arrive (Pi's mechanism — the streamed unit is the argument
stream, not the tool result). Touches rendering only — no authority.
- **Implementation:**
  - **Modules:** a TUI live-region renderer (e.g. `Channels.TUI.LiveRegion`) over
    `owl`; a progressive tool-argument parser feeding partial diffs; consumes
    `AllbertAssist.Runtime.Response` `surface_payload`.
  - **Files:** `plugins/allbert.tui/lib/.../live_region.ex` (net-new),
    renderer wiring in the TUI adapter.
  - **Tests:** `tui_live_region_test.exs` — `surface_payload` renders to terminal
    with no UI payload in `model_payload`/memory; progressive-arg chunks render in
    order; **turn-complete fallback** when streaming is unavailable (degrades to
    the v0.55 static render).
  - **Verify:** a simulated streamed `edit` renders an incremental diff; the model
    context contains only the clean payload.
- Lane: parallel with M1 (renderer surface, no runner coupling).

### M3: Token Streaming + Interrupt/Steer  *(after M2 — live-turn affordances)*
Make the live turn feel like a peer CLI: stream the **assistant's prose
token-by-token** into the live region (alongside M2's diff streaming), and let the
operator **Esc-cancel** a running turn (keeping + tracing work done so far) and
**queue a correction** to redirect without killing the turn. **Runtime
dependency:** this requires the runtime to emit model token deltas and to support
turn cancellation — today's v0.55 response is turn-complete and uninterruptible.
- **Implementation:**
  - **Modules:** a runtime token-delta channel (req_llm stream → `Runtime` →
    `Channels.TUI.LiveRegion`); a turn-cancellation path that aborts the in-flight
    model call + any running action and writes a **partial-turn trace**; the TUI
    input loop gains Esc-cancel + a queued-correction buffer.
  - **Files:** runtime streaming/cancel wiring; TUI input-loop + live-region
    extensions (net-new).
  - **Settings:** `coding.streaming.enabled` (default true), `coding.steer.enabled`.
  - **Tests:** assistant text renders token-by-token then reconciles to the final
    `model_payload`; Esc mid-turn cancels cleanly, keeps prior work, and records a
    partial-turn trace (no orphaned action/model call); a queued correction is
    applied on the next turn; `bash` brutal-kill still bounds a runaway command.
  - **Verify:** a simulated long turn streams tokens, cancels on Esc with a clean
    trace, and applies a queued correction.
- Depends on M2 (renders through the same live region); no authority change.

### M4: Local-Coding Trust Tier + Approval Modes  *(serial — gates M1's cheapness)*
Implement the **local-coding operator** tier from M0 **and the coder-facing
approval modes**. Security Central resolves the tier **only** for a single trusted
operator in the main session on `tui` (cheap gate + raw-shell `bash`), keeping the
full gate and argv-only `bash` for every other caller; the tier runs at **sandbox
Level 1** (policy/redaction/trace/audit still execute). On top, the operator picks
a **mode** — `default` (prompt effects), `accept-edits` (auto `write`/`edit`,
prompt `bash`), `plan` (read-only: `read`/`grep`/`glob` only, propose never
execute), `tier` (cheapest, trusted-local) — and may persist **"always allow this
command"** per repo. Modes and allowlist are a confirmation-cost/persistence layer
over Security Central; they grant **no authority**. Channel-originated, scheduled,
and generated-code sessions cannot reach the tier or a relaxed mode.
- **Implementation:**
  - **Modules:** the `coding_tier/1` resolver + `approval_mode/1` from M0 wired
    into `Security.PermissionGate`'s confirmation-cost decision and `Bash`'s
    raw-vs-argv selection; a per-repo command allowlist store
    (`coding.allowlist.*`, auditable) consulted before prompting.
  - **Files:** `apps/allbert_assist/lib/allbert_assist/security/{policy,permission_gate}.ex`;
    allowlist persistence module.
  - **Settings:** `coding.approval_mode` (default `default`),
    `coding.trusted_operator_id`, `coding.bash.allow_raw_shell` (tier-gated),
    `coding.pi_mode.enabled`, `coding.allowlist.store`.
  - **Tests:** trusted-operator/main/`tui` → tier resolves, cheap gate + raw
    shell; channel-originated/scheduled/generated-code → full gate, argv-only, no
    tier; `plan` mode refuses every effect (read-only) and never executes;
    `accept-edits` auto-approves `write`/`edit` but still gates `bash`; a persisted
    "always allow `<cmd>`" suppresses the prompt only for that repo + command and
    is auditable; no mode grants authority beyond its definition; tier + mode +
    allowlist decisions are in the trace.
  - **Verify:** the resolution + mode tables hold; a channel-originated caller
    cannot reach the tier, a relaxed mode, or raw shell; `plan` mode blocks a
    `write`.

## Security / Eval Rows (`:v057`)
- `pi-mode-effect-routes-through-runner-001` (every `read`/`write`/`edit`/`bash`
  resolves through `Actions.Registry` and executes via `Runner.run/3`; no direct
  effect path).
- `pi-mode-search-read-only-001` (`grep`/`glob` are `permission: :read_only`,
  gitignore-aware, run unprompted, have no effect path, and never mutate).
- `pi-mode-bash-policy-bounded-001` (`bash` refuses outside policy — cwd-jail +
  timeout enforced; OTP/BEAM is not treated as the security boundary; the
  host/container is).
- `pi-mode-bash-raw-shell-tier-only-001` (raw shell strings resolve **only** at the
  local-coding operator tier; every other caller is argv-only or refused).
- `pi-mode-write-edit-cwd-jail-001` (`write`/`edit` refuse normalized path escape,
  symlink escape, and writes outside `coding.workspace.cwd_jail`).
- `pi-mode-file-effects-tier-gated-001` (`write`/`edit` get the same
  trusted-tier/full-gate split as `bash`; channel-originated, scheduled, and
  generated-code callers cannot get cheap file effects).
- `local-coding-tier-trusted-only-001` (the tier resolves only for a single trusted
  operator + main session + `tui`, at sandbox Level 1).
- `local-coding-tier-not-default-001` (default resolution is never the tier).
- `local-coding-tier-rejects-channel-origin-001` (channel-originated inbound /
  scheduled / generated-code → full gate, no tier escalation, no raw shell).
- `pi-mode-deterministic-acceptance-001` (effectful/generated-code work is never
  accepted on model confidence; evidence + confirmation rule).
- `split-result-no-ui-leak-001` (the `surface_payload`/diff never leaks into
  `model_payload`; the model sees only the clean payload).
- `pi-mode-prompt-token-budget-001` (assembled system prompt **+ the six tool
  definitions combined** are < the configured budget against the named tokenizer).
- `pi-mode-context-discipline-chunked-001` (default context gathering uses chunked
  reads (offset/limit) + artifacts, not whole-file ingestion).
- `pi-mode-default-tool-surface-001` (a default coding turn exposes exactly the six
  tools — `read`/`write`/`edit`/`bash`/`grep`/`glob`; the rest are lazy-disclosed).
- `pi-mode-approval-mode-grants-no-authority-001` (`default`/`accept-edits`/`plan`/
  `tier` change only confirmation cost; `plan` is read-only and executes nothing; no
  mode grants authority beyond its definition).
- `pi-mode-allowlist-scoped-and-auditable-001` ("always allow `<cmd>`" suppresses the
  prompt only for that repo + command, persists, and is auditable; never a blanket
  bypass).
- `pi-mode-assistant-text-streams-001` (assistant prose renders token-by-token then
  reconciles to the final `model_payload`; degrades to turn-complete cleanly).
- `pi-mode-interrupt-clean-cancel-001` (Esc mid-turn aborts the in-flight model

- `pi-mode-coding-slash-non-routable-001` (`/help`/`/model`/`/clear`/`/init`/`/diff`/

  `agent_modules/0`).
- `pi-mode-model-switch-preserves-authority-001` (mid-session switch changes the
  model via req_llm message-history handoff, never the Security Central path).
- `pi-mode-no-bash-subagent-001` (no bash-spawned sub-agent; delegation only via
  restricted-registry delegate agents).
- `pi-mode-no-authority-001` (the surface cannot bypass Security Central, set the

## Non-Goals
*(from the Pi analysis §8 "what not to do")*
- **No YOLO-by-default.** The local-coding operator tier is a named, gated,
  non-default tier (sandbox Level 1) — never the default, never for
  channel-originated or generated-code sessions.
- **No weakening of the action boundary / Security Central / confirmations.** The
  gates are made *cheap at the tier*, not *absent*; every effect still routes
  through `Actions.Runner.run/3` at Level 1.
- **No "sandbox level 0" label / no isolation relaxation.** The tier changes the
  confirmation burden, not the sandbox level; host execution stays Level 1.
- **No model-decides-it's-done** for effectful or generated-code work;
  deterministic acceptance rules.
- **Keep MCP-first with lazy disclosure** (ADR 0038/0039) for capabilities
  **beyond the six default tools**; do not adopt Pi's no-MCP stance. The minimalism
  is in the *default* surface, not a ban — and code search (`grep`/`glob`) is part
  of that default surface, not an MCP round-trip.
- **No sub-agent-via-bash-spawn**; use restricted-registry delegate agents.
- **No sibling runtime**; Pi-mode is a channel/app on the one spine — same
  authority boundary, same trace, same memory.
- No new TUI renderer choice; reuse the v0.55 `owl` terminal surface (Pi's
  scrollback-native model stays reserved until evidence).

## Definition Of Done
v0.57 is done when:
- a Pi-mode coding surface runs as a channel/app on the `tui` channel, exposing
  exactly **six default tools** — four effectful boundary actions
  (`read`/`write`/`edit`/`bash`) + two read-only search actions (`grep`/`glob`) —
  each resolving through `Actions.Registry` and executing via
  `Actions.Runner.run/3` under Security Central;
- a named "local-coding operator" trust tier (ADR 0056 lineage, running at ADR
  0009 **Level 1**, ADR 0068) plus coder-facing approval modes
  (`default`/`accept-edits`/`plan`/`tier`) and per-repo "always allow" make the
  gate present-but-cheap **only** for a single trusted operator in the main session
  on `tui` (and enable raw-shell `bash` only there), grant no authority, and
  provably cannot be reached by channel-originated, scheduled, or generated-code
  sessions, and are never the default;
- the live region streams **assistant text token-by-token** and renders diffs via
  the **net-new v0.57** live-region substrate over the v0.55 static split payload
  (ADR 0067/0029/0030) with no `surface_payload` leaking into model context, and
  **Esc cancels** a running turn cleanly (partial-turn trace) with a queued
  correction available;
- the coding prompt **+ six tool definitions combined** are under the token budget,
  context gathering uses chunked reads + artifacts (not whole-file ingestion), the
  default turn exposes only the six tools, the coding slash set
  (`/help`/`/model`/`/clear`/`/init`/`/diff`/`/compact`) is non-routable, and a
  mid-session model switch preserves session history via req_llm without touching
  the authority path;
- acceptance for effectful/generated-code work is deterministic (evidence +
  confirmation), with no model-confidence completion and no bash-spawned sub-agent;
  the ordinary interactive turn ends when the model stops calling tools;
- `:v057` evals + `release.v057` + the full release gate pass;
- ADR 0068, the minimalism-budget coding policy, roadmap, vision, and
  agent-context-map agree, and `pi-integration-rethink.md` items 3–5 are delivered.
