## Purpose

Give Allbert a **Pi-mode coding surface**: a gated terminal coding loop that
feels as minimal as Pi and as **intuitive as Claude Code / Codex / Gemini CLI** (a
tight six-tool surface — read-only/sensitive `read`/`grep`/`glob` plus effectful
`write`/`edit`/`bash` — a sub-1000-token prompt+tool-defs budget, streamed diffs
**and** assistant-text token streaming, coder-facing approval modes,
Esc-to-cancel, a familiar slash set, deliberate chunked-read context discipline)
**while running on the one authority spine** — the same
`AllbertAssist.Actions.Runner.run/3`, Security Central, trace, and markdown-first
memory as every other surface.

Stated structurally (rationale §4): Allbert's cognitive runtime is the
seven-stage machine **Receive → Interpret → Frame/resume objective →
Propose/evaluate steps → Authorize step → Execute (`Actions.Runner.run/3`) →
Observe/advance.** Pi is `{Receive → Interpret(implicit) → Execute(YOLO) →
Observe}` — it deletes stages 3–5. v0.57 makes **stages 6–7 and the tool surface
Pi-minimal** (read/search/file/shell tools, streamed diffs + token streaming,
observable, sub-1000-token prompt+tool-defs budget)
**while leaving stages 3–5 fully intact**.
A Pi-mode session is one where the objective is "pair on this repo," step proposal
is trivial, and authorization runs under the **local-coding operator trust tier
at sandbox Level 1** for a single trusted local operator. The gates are *present
but cheap*, never *absent*.

This is a **channel/app under ADR 0016 on the existing spine** — NOT a sibling
runtime, NOT a new authority. What lands:

1. **Six default tools:** three **read-only but sensitive** tools
   (`read`/`grep`/`glob`) + three **effectful** tools (`write`/`edit`/`bash`),
   all routed through the action runner and Security Central. Read/search tools
   run without a confirmation prompt but still enforce cwd jail, ignore policy,
   output caps, redaction, trace, and audit; `bash` runs host processes at
   sandbox Level 1.
2. **Coder-facing approval modes** (`default` / `accept-edits` / `plan` / `tier`)
   plus persistent **"always allow this command"** per repo — a presentation +
   persistence layer over Security Central (Policy allowlist + confirmations),
   stored under Allbert Home by repo fingerprint, revocable and auditable,
   granting **no new authority**.
3. A named **"local-coding operator"** trust tier (ADR 0056 lineage, running at
   ADR 0009 **Level 1**) — single trusted operator, main session, terminal
   channel; never the default, never for channel-originated or generated-code
   sessions. Not a sandbox level; never "level 0".
4. **Net-new live rendering:** a coding-diff live-region renderer + progressive
   tool-argument streaming + **assistant-text token streaming** over the v0.55
   static split payload, plus **Esc-to-cancel** and a **queued-correction** steer.
5. A sub-1000-token prompt+tool-defs budget + chunked-read context discipline; a
   familiar coding **slash set** (`/help`, `/mode`, `/model`, `/clear`, `/init`,
   `/diff`, `/compact`) with an explicit effect matrix; a mid-session model switch
   (`req_llm` history handoff); and the minimalism budget as a standing coding
   policy.

## Readiness Sweep
- **Contract fit.** The Pi-mode surface is a delivery+driver experience over the
  v0.55 `tui` channel; it owns no intent/Security Central/Resource Access/
  confirmation storage/conversation history. Coding effects reach the host *only*
  through registered actions executed by `Actions.Runner.run/3`. The runner +
  Security Central decide; the surface renders and dispatches.
- **Reuse fit.** Reuse `plugins/allbert.tui` (normalize → `Identity.resolve` →
  `Channels.derive_session_id` → `Runtime.submit_user_input` → `ChannelThread` →
  `channel_events`), the v0.55 **static** split-result contract
  (`AllbertAssist.Runtime.Response` `model_payload`/`surface_payload`, ADR 0067
  extending ADR 0029/0030) as the payload v0.57's **net-new** streaming renders
  over, the existing `actions/` boundary — specifically `run_shell_command`
  (argv-only Level 1) and `execution/skill_script_runner` (timeout + brutal-kill)
  — as the template for the `bash` policy, the ADR 0056 trust-tier convention +
  ADR 0009 Level 1 for the tier, the v0.03 skill loader
  (`AllbertAssist.Skills`) + `AGENTS.md` hierarchy for context, and req_llm
  message-history serialization/streaming for the model switch and live turn.
  Cross-provider conversion is best-effort; M0/M4 must verify the concrete
  `ReqLLM.Context`, `ReqLLM.StreamResponse`, and `ReqLLM.StreamChunk` APIs against
  vendored `deps/req_llm` source because Context7 currently resolves `Req`, not a
  first-class `req_llm` package. ADR 0047/0051 remain provider doctor/capability
  contracts, not this handoff.
- **Upstream check.** Confirm v0.55 delivered the `tui` channel AND the **static**
  split tool-result payload (`model_payload`/`surface_payload`) through the typed
  response contract + renderer. v0.55 did **not** ship a progressive coding-diff
  renderer or streaming substrate; diff rendering is v0.57 M4 and token
  streaming/cancel is v0.57 M5. If the `tui` channel or the static split is
  missing/regressed, stop and reconcile v0.55/v0.56 before implementing v0.57.
- **Authority check (AGENTS.md Non-Negotiables).** Advisory output is never
  authority: the coding model may propose `read`/`write`/`edit`/`bash` calls but
  may not bypass `Actions.Runner.run/3`, Security Central, or confirmations. The
  bash action is policy-bounded host execution behind a registered action — OTP
  supervision is **not** an OS security boundary. No model-decides-it's-done for
  effectful or generated-code work. No sub-agent-via-bash-spawn.
- **Library fit.** No new library is required. `owl` (v0.55, terminal render)
  supports live-updating blocks, but its own docs position it as CLI/script UI
  tooling rather than a full-screen terminal framework; Esc handling and turn
  cancellation are Allbert TUI/input-loop work. `req_llm` covers provider-neutral
  context, streaming chunks, and cancellation hooks in local source, but M0/M4/M5
  must verify provider-specific tool-call streaming and cancellation semantics.

## Coding-Surface Ergonomics & CLI Parity
The surface must feel intuitive to a developer arriving from Claude Code, Pi,
Codex CLI, or Gemini CLI. The minimalism is in the *authority spine and default
surface*, **not** in the interaction model — the everyday coding ergonomics those
tools share are first-class deliverables here, expressed as Allbert primitives.

| Convention | Claude Code / Codex / Gemini / Pi | Allbert Pi-mode (v0.57) |
|---|---|---|
| **Tool surface** | read/write/edit/bash + Grep/Glob (Claude, Gemini); Codex/Pi = shell does search | **Six**: read-only/sensitive `read`/`grep`/`glob` + effectful `write`/`edit`/`bash` (registered actions). Beyond the six → MCP lazy disclosure. |
| **Code search** | first-class Grep/Glob, gitignore-aware, unprompted | `grep`/`glob` actions run without a confirmation prompt but still pass Runner/Security Central, cwd jail, ignore policy, output caps, trace, and redaction; `bash` `rg`/`find` also available at the tier. |
| **Approval UX** | named modes (acceptEdits/plan/yolo...) + persistent "always allow" | modes `default` / `accept-edits` / `plan` / `tier` + Allbert Home-rooted per-repo "always allow this command" — a view over Security Central, no new authority. |
| **Streaming** | assistant text streams token-by-token; diffs/output render live | **net-new v0.57**: explicit stream-event contract, assistant-text token streaming, progressive tool-argument diff streaming, and turn-complete fallback. |
| **Interrupt / steer** | Esc cancels; queued correction / Enter-to-steer | **Esc-to-cancel** is Allbert input-loop/runtime work (not Owl-provided): cancel model stream + running action best-effort, keep prior work, partial-turn trace, queued correction; `bash` timeout/brutal-kill underneath. |
| **Slash set** | /help /mode /model /clear /init /diff /compact ... | `/help`, `/mode`, `/model`, `/clear`, `/init`, `/diff`, `/compact` — slash-allowlisted and non-routable, with a per-command effect/action matrix. |
| **Context file** | AGENTS.md / CLAUDE.md / GEMINI.md auto-load + @file + /init | `AGENTS.md` hierarchy (Allbert already uses it — Codex made AGENTS.md the native default, so this is **parity**) + `@file` mentions + `/init`. |
| **Loop / "done"** | loop until model stops emitting tool calls, then return to prompt | same interactive REPL: the turn ends when the model stops calling tools. "Deterministic acceptance / model never decides done" applies to **effectful/generated-code acceptance gates**, not to ending a conversational turn. |

Two of these choices carry **runtime dependencies beyond rendering** and are flagged
in the milestones: assistant-text **token streaming** needs the runtime to emit
model token deltas (today's v0.55 response is turn-complete), and **interrupt**
needs clean turn cancellation (abort the in-flight model call + any running action
with a partial-turn trace; `bash` brutal-kill exists, model-call abort is net-new).

## Locked Decisions
| # | Decision | Chosen value | Trade-off if changed |
|---|---|---|---|
| 1 | Surface authority | Pi-mode is a **channel/app on the one spine** (ADR 0016): every coding effect routes through `Actions.Runner.run/3` + Security Central. The surface has **no authority beyond any other channel**. | A sibling runtime or a direct-effect coding loop reintroduces Pi's YOLO threat model — the thing Allbert exists to reject. |
| 2 | Tool surface | **Six default tools**: read-only/sensitive `read`, `grep`, `glob` + effectful `write`, `edit`, `bash`. Each is a registered Jido action via `Actions.Registry`/`Runner`. `read`/`grep`/`glob` use the `:coding_file_read` permission and run without a confirmation prompt, but still enforce cwd jail, symlink policy, ignore policy, binary handling, output caps, redaction, trace, and audit. `write`/`edit` use `:coding_file_write`; `bash` uses `:coding_shell_execute`. Dotted strings such as `permissions.coding_file_write` are Settings Central keys, not permission atoms. Capabilities **beyond** the six are lazy-disclosed (MCP-first, ADR 0038/0039), not pre-loaded. | Omitting `grep`/`glob` reads as impoverished to Claude Code / Gemini users (2 of the 3 ship them first-class) and forces an MCP round-trip just to search a repo. A *wide* default surface beyond search would defeat the minimalism budget. |
| 3 | Trust tier | A named **"local-coding operator"** trust tier (ADR 0056 lineage) running at **sandbox Level 1** (ADR 0009): single trusted operator, main session, `tui` channel. **Never** the default; **never** for channel-originated inbound or generated-code sessions. The tier lowers only the *confirmation burden*; policy/redaction/trace/audit still run at Level 1. It is **not** a sandbox level and never "level 0". | An implicit or default tier turns a low-friction convenience into Pi-by-default and breaks the action boundary. Mislabeling it "level 0" collides with ADR 0009 (level 0 = inert/no-exec) and conflates caller-trust with isolation. |
| 4 | Acceptance | **Deterministic acceptance rules**, not model confidence. For effectful or generated-code work the model never "decides it's done"; evidence + confirmation rule (same posture Allbert took against Hermes). | Model-confidence completion contradicts "advisory output is never authority" and the codegen committee's deterministic gates. |
| 5 | Prompt + context | A **sub-1000-token prompt+tool-defs budget** (system prompt and all six tool definitions combined, against the configured tokenizer). Context discipline follows **Pi's documented practice**: chunked reads (offset/limit) + separate context-gathering sessions + file artifacts, **not** whole-file ingestion. Reuse the `AGENTS.md` hierarchy + v0.03 skill loader; `@file` mentions resolve through the same bounded read action, never raw path ingestion. | A large default budget or whole-file context bloats the surface. (Note: full-file ingestion is *not* a Pi finding — Pi reads in bounded chunks and notes models resist full reads; do not attribute it to Pi.) |
| 6 | Live rendering | **Net-new in v0.57:** a coding-diff live-region renderer + progressive tool-argument rendering **and assistant-text token streaming**, built on the v0.55 **static** split payload (`model_payload` vs. `surface_payload`, ADR 0067) through the unified renderer (ADR 0030). M0 defines the stream events (`assistant_token_delta`, `tool_call_argument_delta`, `tool_call_argument_complete`, `tool_result_delta`, `turn_cancelled`, `turn_complete`). The terminal renders ephemeral deltas and reconciles to the final model/surface payloads; the model sees only the clean model payload. | Treating streaming as a v0.55 inheritance understates the work; v0.55 ships only the static split. A turn-complete reply (no token streaming) feels dead next to every peer CLI. A single conflated payload re-bloats model context. |
| 7 | Model switch | Mid-session model switch is an **operator affordance** over req_llm's canonical `Context` handoff (verified through Context7 when available, otherwise through vendored `deps/req_llm` docs/source), surfaced in the `tui` session; it changes the model, never the authority path. ADR 0047/0051 remain provider doctor/capability contracts and do not own this handoff. | Hiding the switch wastes a structurally-cheap req_llm capability Pi proved is worth surfacing. |
| 8 | Sub-agents | Delegation uses **restricted-registry delegate agents**, never an opaque `bash`-spawned sub-agent. | Self-spawn via bash is an unobservable, unrestricted child — the Hermes "narrow the child's tools" lesson, violated. |
| 9 | `bash` semantics | **Raw shell strings, but only at the local-coding operator tier** (Pi-faithful). For every other caller `bash` is unavailable or falls back to argv-only `run_shell_command`. Tier-restricted raw shell is bounded by cwd-confinement to the repo root, a wall-clock timeout + brutal-kill (reuse `skill_script_runner`), full trace/audit, and the stance that the host/container — not OTP — is the security boundary. | A general raw-shell action would break `run_shell_command`'s anti-injection (argv-only) posture for untrusted callers. Restricting raw shell to one trusted local operator keeps the convenience without widening the channel-originated attack surface. |
| 10 | Approval modes | Coder-facing modes `default` / `accept-edits` / `plan` / `tier`, plus persistent **"always allow this command"** per repo. This is a **confirmation-cost seam over Security Central** — implemented in `Security.Decision.build/1` so it drops only the interactive prompt while **preserving the `:needs_confirmation` decision, policy, risk, trace, and audit**; it changes confirmation cost, never the authority path. `plan` executes only read/search; it never executes `write`/`edit`/`bash`. "Always allow" **reuses the existing `Resources.Grants` remembered-grant lifecycle** (durable, expirable, revocable, audited, Allbert Home-rooted) with a command-scoped grant keyed by repo fingerprint + permission + cwd + canonical (normalized/redacted) command — **not** a parallel allowlist store. The `tier` mode is the local-coding operator tier (cheapest). | No coder-facing modes means a non-tier session prompts on every effect (permission fatigue — the exact thing Claude/Codex/Gemini solved with modes + remembered allowances). A second approval-memory store would drift from `Resources.Grants`. Modes that granted authority would violate "advisory output is never authority." |
| 11 | Interrupt / steer | **Esc cancels** a running turn (work done so far is kept + traced) and a **queued correction** lets the operator redirect without killing the turn. Cancellation cleanly aborts the in-flight model stream when the provider/ReqLLM path supports it and cancels or brutal-kills any running action by policy; failures degrade to bounded timeout + partial-turn trace, never silent orphaning. Owl provides live blocks, not the Esc semantics. | No interrupt is jarring — Esc-to-cancel is a universal reflex. Cancellation is a runtime concern (model-call abort + partial-turn trace), so it is scoped as net-new work, not assumed from v0.55. |
| 12 | Slash set | A familiar coding **slash allowlist**: `/help` (router-local), `/diff` + `@file` (read via `:coding_file_read`), `/init` (file write via `:coding_file_write`), and `/mode` / `/model` / `/clear` / `/compact` as **ungated session/model ops** (in-memory, no persistent or external effect, **no permission atom** — gating them would be ceremony no peer CLI imposes). All reuse the v0.55.1 slash-allowlist branch, are non-routable (allowlist only), and create no model turn. There is **no** `:coding_session_write` atom. | Only `/model` would feel bare; coders expect a baseline. Routable slash commands would re-introduce the operator/agent-surface boundary problem v0.56 M14a fixed. Permission-gating `/clear`/`/compact` would prompt-fatigue the coder for a no-op. |

### M1: Read/Search/Path Substrate  *(read-only but policy-bounded)*
Register `read`, `grep`, and `glob` as read-only/sensitive coding actions through
`Actions.Registry` and `Actions.Runner.run/3`. They run without a confirmation
prompt, but still enforce cwd jail, symlink policy, ignore policy, binary
handling, output caps, redaction, trace, and audit.
- **Implementation:** `Actions.Coding.{Read,Grep,Glob}` plus `Coding.PathPolicy`
  and `Coding.Search`. M1 uses direct Elixir file-walk/globbing so read/search
  stays inside the policy/read substrate; `bash`/process execution remains M3/M6
  scope.
- **Settings:** `coding.workspace.cwd_jail`, `coding.read.default_limit`,
  `coding.read.max_bytes`, `coding.search.max_results`,
  `coding.search.max_output_bytes`, `coding.search.respect_gitignore`,
  `coding.search.respect_allbertignore`, `permissions.coding_file_read`.
- **Implemented:** `Actions.Coding.{Read,Grep,Glob}` are `:agent` actions,
  `confirmation: :not_required`, permissioned by `:coding_file_read`, and return
  relative-path, redacted, bounded payloads. `Coding.PathPolicy` resolves the cwd
  jail, rejects path traversal and symlink escape, rejects binary reads, and exposes
  the bounded read helper that `/diff`/`@file` will consume in M8. `Coding.Search`
  honors `.gitignore`, `.allbertignore`, `.git/`, result caps, and output caps.
- **Tests:** all three actions resolve via Runner; shared bounded read path is
  tested for later `@file`; grep/glob honor cwd jail and ignore policy; no search
  path mutates files; output is redacted/truncated.
- Gate: read/search substrate green before M2/M4 consume it.

### M2: Write/Edit Actions + Split-Payload Diffs  *(file effects)*
Register `write` and `edit` as effectful coding actions with permission
`:coding_file_write`. `write` creates files; `edit` performs exact-match
replacement and returns a clear miss so the model re-reads and retries. Results
produce clean `model_payload` plus renderable `surface_payload` diff for M4.
- **Implementation:** `Actions.Coding.{Write,Edit}` plus
  `Coding.FileEffects`. Both actions are `:agent`, `confirmation: :required`,
  `resumable?: true`, resolve through `Actions.Registry`/`Actions.Runner.run/3`,
  check `:coding_file_write` before filesystem validation, and then create a
  durable confirmation through the existing confirmation/resume path. `write`
  refuses overwrites and only creates a new file whose existing parent resolves
  inside the cwd jail. `edit` reads an existing text file, requires a non-empty
  exact match, refuses over-broad replacements above the configured cap, and
  applies the replacement only after approval.
- **Settings:** `coding.workspace.cwd_jail`, `coding.write.max_bytes`,
  `coding.edit.max_replacements`, `permissions.coding_file_write`.
- **Implemented:** split payloads carry redacted, bounded diff previews with
  relative paths only. Confirmation output redacts raw write content and exact-match
  strings; the existing confirmation store also redacts secret-shaped resume
  params at persistence time, so the default confirmation path must not be used as
  a secret-write channel. Secrets belong in the existing Settings/secrets paths,
  not in pending file-edit confirmations.
- **Tests:** write/edit resolve only through Runner; default decision is
  confirmation-needed until M6 modes/tier make the prompt cheap; approval resumes
  and mutates ordinary content; permission denial blocks before filesystem
  validation; normalized path, symlink, overwrite, binary, exact-match-miss, and
  outside-jail escapes are refused; split payload contains no UI/chrome in
  `model_payload` and redacts secret-shaped content.
- Gate: M2 depends on M1 path policy; M4 consumes M2's diff payload contract.

### M3: Bash + Raw-Shell Runner Contract  *(host execution)*
Register Pi-mode `bash` as an effectful coding action with permission
`:coding_shell_execute`. Preserve existing `run_shell_command` argv-only semantics
for non-tier callers. Raw shell strings are accepted only when
`coding.bash.allow_raw_shell` is true **and** the local-coding operator tier
resolves; otherwise raw shell is denied before any command execution.
- **Implementation:** `Actions.Coding.Bash`, `Coding.BashSpec`, and the existing
  `Execution.CommandSpec` + `Execution.LocalRunner` Level 1 execution path.
  Argv-style commands normalize through `CommandSpec` with execution-policy
  allowed-command, cwd, timeout, output, root, and env allowlist checks. Raw shell
  normalizes to `/bin/sh -lc` only at the local-coding tier with the raw-shell
  setting enabled. Both paths run through `Actions.Runner.run/3`,
  Security Central, confirmation/resume, redaction, and trace surfaces; opaque
  sub-agent launch words (`codex`, `claude`, `gemini`, `opencode`, `cursor`,
  `antigravity`) are denied as a policy bypass.
- **Settings:** `coding.workspace.cwd_jail`, `coding.bash.timeout_ms`,
  `coding.bash.max_output_bytes`, `coding.bash.allow_raw_shell`,
  `permissions.coding_shell_execute`.
- **Implemented:** `bash` is registered as a resumable `:agent` action with
  permission `:coding_shell_execute`, `confirmation: :required`, and split
  `model_payload`/`surface_payload` summaries. Approval resumes the stored command
  via the generic confirmation path. Confirmation output redacts raw shell strings,
  argv args, and env payloads; output previews are bounded and redacted. The cwd
  jail is shared with M1/M2. Environment keys outside the execution policy
  allowlist are denied, not silently dropped. Raw shell remains off by default and
  unavailable to non-tier callers.
- **Tests:** bash resolves through Runner; cwd jail, timeout, output caps, env
  allowlist, and redaction are enforced; raw shell fails for non-tier callers;
  argv-only fallback remains intact; bash cannot spawn opaque sub-agents as an
  authority bypass.
- Gate: M3 depends on M1 path policy and precedes M6 raw-shell cheap gate.

### M4: Stream Event Pipeline + Live Renderer  *(rendering, no authority)*
Build the live-region render substrate the v0.55 static split lacks. It consumes
the M0 stream-event contract and M2 split diff payloads: assistant token deltas
render as live prose, tool-call argument deltas render as progressive diffs, and
final payloads reconcile to `model_payload`/`surface_payload`. Rendering grants no
authority and must degrade to the v0.55 turn-complete static render.
- **Implementation:** runtime stream-event emitter (req_llm stream -> Allbert
  runtime events), TUI live-region renderer over `owl`, and progressive
  tool-argument parser.
- **Settings:** `coding.streaming.enabled`, `coding.streaming.turn_complete_fallback`.
- **Implemented:** `Coding.StreamPipeline` converts provider-neutral
  `ReqLLM.StreamChunk` content/thinking/tool-call chunks into validated
  `Coding.StreamEvent` maps and emits turn-complete events from normalized runtime
  responses. `Coding.StreamRenderer` is the pure, authority-free state machine for
  assistant token deltas, progressive tool-call arguments, tool-result deltas,
  cancellation events, and final `surface_payload` reconciliation. The TUI
  renderer accepts `stream_events` and falls back to final turn-complete split
  payloads; `Channels.TUI.LiveRegion` wraps the existing `Owl.LiveScreen`
  add/update/await/flush pattern for live coding blocks. M4 does not create the
  async turn handle or Esc cancellation path; M5/M6 own those runtime controls.
- **Tests:** assistant tokens render incrementally and reconcile; progressive-arg
  chunks render in order; `surface_payload` never leaks into `model_payload` or
  memory; turn-complete fallback works.
- Gate: M4 depends on M0 event contract and M2 split diff payloads.

### M5: Async Turn-Execution Boundary  *(runtime prerequisite for cancellation)*
**Net-new runtime work, and the prerequisite the synchronous v0.55 runtime lacks.**
Today `Runtime.submit_user_input/1` calls the agent runner and `Actions.Runner.run/3`
**synchronously and blocking** (`runtime.ex:115`, `runner.ex:85`) — there is no
handle to an in-flight turn, so it cannot be cancelled. M5 moves a coding turn's
model call (and its cancellable actions) onto a **supervised Task** behind the
runtime, registered so it is referenceable and killable, with a clean partial-turn
trace on shutdown. This is what makes M6's true Esc-to-cancel possible (option a:
abort the in-flight model call, not merely abandon it at the channel). No authority
change — same runner, same Security Central, same trace.
- **Implementation:** a per-turn supervisor + turn registry (e.g.
  `Runtime.Turn`/`Coding.TurnSupervisor`) that runs the model call as a
  `Task` under a supervisor and tracks it by session/turn id; the coding loop awaits
  the task with a monitor; shutdown writes a partial-turn trace. Reuses
  `ReqLLM.StreamResponse` (the `cancel` field + `process_stream/2`) from M4 so the
  streamed turn is the cancellable unit.
- **Settings:** `coding.turn.max_ms` (hard ceiling), `coding.turn.supervised`
  (default true; falls back to synchronous turn-complete when false).
- **Implemented:** `AllbertAssist.Coding.TurnSupervisor` wraps explicitly marked
  coding turns in `AllbertAssist.TaskSupervisor`, registers the in-flight task in
  `AllbertAssist.Coding.TurnRegistry` by turn id, and converts task shutdown or
  timeout into a partial runtime response with `:cancelled`/`:timed_out` status.
  The partial response follows the ordinary runtime path: response signal,
  assistant-message persistence, trace recording, and turn-completed signal. Normal
  non-coding turns remain synchronous; `coding.turn.supervised=false` preserves the
  v0.55 turn-complete fallback. M5 exposes the addressable/shutdown-capable task
  boundary that M6 uses for Esc/provider-stream cancellation and queued steering.
- **Tests:** a coding turn runs under a supervisor and is addressable by turn id;
  killing the task yields a partial-turn trace and no orphaned model call/action;
  the synchronous fallback still produces the v0.55 turn-complete result; no
  authority path changes.
- Gate: M5 depends on M4 stream events; precedes M6.

### M6: Cancellation + Queued Steering  *(live-turn control)*
Make the live turn controllable on the M5 boundary. **Esc cancels a running turn
for real (option a):** the in-flight model stream is aborted via the
`ReqLLM.StreamResponse` `cancel` callback registered by the running turn task,
then the supervised turn task is shut down through `Coding.TurnSupervisor`'s
registry-level cancel path (grace window, then hard-kill fallback if the task is
still alive). Running actions remain bounded by their own Runner/action policy
timeouts and brutal-kill behavior; work done so far is kept, a `turn_cancelled`
stream event is emitted, and a **partial-turn trace** is written. A **queued
correction** redirects the next turn without killing the current one. (Where a
provider stream cannot be aborted, registry task shutdown still frees the turn and
the late result is discarded — but the default and tested path is a real stream
cancel.)
- **Implementation:** TUI coding-mode input-loop Esc handler -> turn registry
  cancel -> registered `StreamResponse.cancel.()` callback + bounded task exit;
  one-slot queued-correction buffer; partial-turn trace writer; normal non-coding
  TUI turns stay synchronous.
- **Settings:** `coding.steer.enabled`, `coding.cancel.grace_ms`.
- **Tests:** Esc/cancel mid-turn invokes the registered stream cancel callback,
  shuts down the supervised runtime task, emits `turn_cancelled`, writes the
  partial-turn trace, leaves no registered orphan, and starts the queued correction
  as the next coding turn without granting authority. M3 bash/action tests remain
  the policy proof for command timeout/output caps and brutal-kill behavior.
- Gate: M6 depends on M5 async boundary and M3 bash cancellation behavior.

### M7: Local-Coding Trust Tier + Approval Modes + Remembered Command Grants  *(serial authority gate)*
Implemented 2026-06-23. M7 lands the **local-coding operator** tier and
coder-facing approval modes.
Security Central resolves the tier only for the trusted operator in the main
session on `tui`; all other callers keep the full gate and argv-only shell path.
Modes and remembered grants change confirmation cost only, never authority.
- **Implementation:**
  - `Security.Policy.coding_tier/1` now resolves from Settings Central plus the
    normalized request context: `coding.pi_mode.enabled=true`, a configured
    `coding.trusted_operator_id`, actor match, main session, and `tui` channel are
    all required; channel-originated, scheduled, and generated-code contexts never
    resolve the tier. `Security.Context` preserves the normalized M7 fields
    Security Central needs (`session.main?`, coding mode, trust flags, and hash-only
    command-grant refs) and labels `tui` as local trust.
  - `Security.Decision.build/1` implements the **M0 confirmation-cost seam**:
    `accept-edits` suppresses only `write`/`edit` prompts, `tier` suppresses local
    coding file/shell prompts, remembered command grants suppress only their exact
    bash prompt, and `plan` denies `write`/`edit`/`bash`. The returned decision
    remains `:needs_confirmation` when the policy floor says so; policy, risk,
    trace, audit, and redaction still run. `Actions.Outbound.Gate` executes only
    when the decision is allowed, an approved confirmation resumes, or a
    `:needs_confirmation` decision has `requires_confirmation: false`.
  - **Reuse the existing remembered-grant mechanism**, do not build a parallel
    allowlist. "Always allow this command" extends
    `AllbertAssist.Resources.Grants` / `resource_grants.remembered` with
    `AllbertAssist.Coding.CommandGrants`: a **command-scoped** record keyed by repo
    fingerprint + permission + realpath-canonical cwd + canonical command hash.
    The stored record keeps hash-only command/cwd refs plus bounded metadata
    (`executable`, arg count, mode), so raw argv/raw shell text does not enter the
    Security decision context. Same lifecycle (TTL, revoke, list, audit); same
    Allbert Home-rooted storage convention as resource grants — not a new
    `coding/allowlists/*.json` store.
- **Settings:** `coding.default_approval_mode`, `coding.trusted_operator_id`,
  `coding.pi_mode.enabled`, `coding.bash.allow_raw_shell`,
  `coding.command_grants.default_ttl_ms`, `coding.command_grants.max_entries_per_repo`.
- **Tests:** trusted/main/`tui` resolves tier; channel-originated/scheduled/
  generated-code callers do not; the cheap gate **preserves the decision + trace +
  audit and only drops the prompt**; `plan` blocks `write`/`edit`/`bash`;
  `accept-edits` auto-approves only file edits; remembered command grants are scoped
  by repo fingerprint + canonical command + permission + cwd, expire, list, revoke,
  and audit through the existing grant lifecycle.
- Gate: M7 follows M1-M3 and rejoins before M8; focused M7 coverage is
  `m7_trust_approval_test.exs` plus the M2/M3/Security/Resources regressions.

## Non-Goals
*(from the Pi analysis §8 "what not to do")*
- **No YOLO-by-default.** The local-coding operator tier is a named, gated,
  non-default tier (sandbox Level 1) — never the default, never for
  channel-originated or generated-code sessions.
- **No weakening of the action boundary / Security Central / confirmations.** The
  gates are made *cheap at the tier*, not *absent*; every effect still routes
  through `Actions.Runner.run/3` at Level 1.
- **No "sandbox level 0" label / no isolation relaxation.** The tier changes the
  confirmation burden, not the sandbox level; host execution stays Level 1.
- **No model-decides-it's-done** for effectful or generated-code work;
  deterministic acceptance rules.
- **Keep MCP-first with lazy disclosure** (ADR 0038/0039) for capabilities
  **beyond the six default tools**; do not adopt Pi's no-MCP stance. The minimalism
  is in the *default* surface, not a ban — and code search (`grep`/`glob`) is part
  of that default surface, not an MCP round-trip.
- **No sub-agent-via-bash-spawn**; use restricted-registry delegate agents.
- **No sibling runtime**; Pi-mode is a channel/app on the one spine — same
  authority boundary, same trace, same memory.
- No new TUI renderer choice; reuse the v0.55 `owl` terminal surface (Pi's
  scrollback-native model stays reserved until evidence).
- **No long-term memory pollution.** Coding tool I/O (reads, searches, diffs, shell
  output) stays in the session trace; it is **not** auto-promoted to markdown-first
  long-term memory unless the operator explicitly saves it — coding context lives in
  the session, as in peer CLIs.
- **No parallel approval-memory store.** "Always allow this command" reuses the
  existing `Resources.Grants` remembered-grant lifecycle, not a second allowlist
  subsystem.

## Definition Of Done
v0.57 is done when:
- a Pi-mode coding surface runs as a channel/app on the `tui` channel, exposing
  exactly **six default tools** — read-only/sensitive `read`/`grep`/`glob` plus
  effectful `write`/`edit`/`bash` — each resolving through `Actions.Registry` and
  executing via `Actions.Runner.run/3` under Security Central;
- a named "local-coding operator" trust tier (ADR 0056 lineage, running at ADR
  0009 **Level 1**, ADR 0068) plus coder-facing approval modes
  (`default`/`accept-edits`/`plan`/`tier`) and `Resources.Grants`-backed per-repo
  "always allow" make the gate present-but-cheap **only** for a single trusted
  operator in the main session on `tui` (and enable raw-shell `bash` only there)
  via a confirmation-cost seam that **preserves the decision/trace/audit and drops
  only the prompt**, grant no authority, support list/revoke/audit/expiry, and
  provably cannot be reached by channel-originated, scheduled, or generated-code
  sessions, and are never the default;
- a coding turn runs under a **supervised Task** (the async turn-execution
  boundary), the live region streams **assistant text token-by-token** and renders
  diffs via the **net-new v0.57** stream-event contract and live-region substrate
  over the v0.55 static split payload (ADR 0067/0029/0030) with no `surface_payload`

  `turn_cancelled`, partial-turn trace) with a queued correction available;
- the coding prompt **+ six tool definitions combined** are under the token budget,
  context gathering uses chunked reads + artifacts (not whole-file ingestion), the
  default turn exposes only the six tools, the coding slash set
  (`/help`/`/mode`/`/model`/`/clear`/`/init`/`/diff`/`/compact`) is non-routable —
  `/init` action-backed, `/mode`/`/model`/`/clear`/`/compact` ungated session ops —
  entering Pi-mode pins the working repo to `coding.workspace.cwd_jail` (symlink + normalized-
  path checked) at session start, and a mid-session model switch preserves session

- acceptance for effectful/generated-code work is deterministic (evidence +
  confirmation), with no model-confidence completion and no bash-spawned sub-agent;
  the ordinary interactive turn ends when the model stops calling tools; coding tool
  I/O stays session-scoped and is not auto-promoted to long-term memory;
- `:v057` evals + `release.v057` + the full release gate pass across M0-M9;
- ADR 0068, the minimalism-budget coding policy, roadmap, vision, and
