## Purpose

Give Allbert a **Pi-mode coding surface**: a gated terminal coding loop that
feels as minimal as Pi and as **intuitive as Claude Code / Codex / Gemini CLI** (a
tight six-tool surface â€” read-only/sensitive `read`/`grep`/`glob` plus effectful
`write`/`edit`/`bash` â€” a sub-1000-token prompt+tool-defs budget, streamed diffs
**and** assistant-text token streaming, coder-facing approval modes,
Esc-to-cancel, a familiar slash set, deliberate chunked-read context discipline)
**while running on the one authority spine** â€” the same
`AllbertAssist.Actions.Runner.run/3`, Security Central, trace, and markdown-first
memory as every other surface.

Stated structurally (rationale Â§4): Allbert's cognitive runtime is the
seven-stage machine **Receive â†’ Interpret â†’ Frame/resume objective â†’
Propose/evaluate steps â†’ Authorize step â†’ Execute (`Actions.Runner.run/3`) â†’
Observe/advance.** Pi is `{Receive â†’ Interpret(implicit) â†’ Execute(YOLO) â†’
Observe}` â€” it deletes stages 3â€“5. v0.57 makes **stages 6â€“7 and the tool surface
Pi-minimal** (read/search/file/shell tools, streamed diffs + token streaming,
observable, sub-1000-token prompt+tool-defs budget)
**while leaving stages 3â€“5 fully intact**.
A Pi-mode session is one where the objective is "pair on this repo," step proposal
is trivial, and authorization runs under the **local-coding operator trust tier
at sandbox Level 1** for a single trusted local operator. The gates are *present
but cheap*, never *absent*.

This is a **channel/app under ADR 0016 on the existing spine** â€” NOT a sibling
runtime, NOT a new authority. What lands:

1. **Six default tools:** three **read-only but sensitive** tools
   (`read`/`grep`/`glob`) + three **effectful** tools (`write`/`edit`/`bash`),
   all routed through the action runner and Security Central. Read/search tools
   run without a confirmation prompt but still enforce cwd jail, ignore policy,
   output caps, redaction, trace, and audit; `bash` runs host processes at
   sandbox Level 1.
2. **Coder-facing approval modes** (`default` / `accept-edits` / `plan` / `tier`)
   plus persistent **"always allow this command"** per repo â€” a presentation +
   persistence layer over Security Central (Policy allowlist + confirmations),
   stored under Allbert Home by repo fingerprint, revocable and auditable,
   granting **no new authority**.
3. A named **"local-coding operator"** trust tier (ADR 0056 lineage, running at
   ADR 0009 **Level 1**) â€” single trusted operator, main session, terminal
   channel; never the default, never for channel-originated or generated-code
   sessions. Not a sandbox level; never "level 0".
4. **Net-new live rendering:** a coding-diff live-region renderer + progressive
   tool-argument streaming + **assistant-text token streaming** over the v0.55
   static split payload, plus **Esc-to-cancel** and a **queued-correction** steer.
5. A sub-1000-token prompt+tool-defs budget + chunked-read context discipline; a
   familiar coding **slash set** (`/help`, `/model`, `/clear`, `/init`, `/diff`,
   `/compact`) with an explicit effect matrix; a mid-session model switch
   (`req_llm` history handoff); and the minimalism budget as a standing coding
   policy.

## Readiness Sweep
- **Contract fit.** The Pi-mode surface is a delivery+driver experience over the
  v0.55 `tui` channel; it owns no intent/Security Central/Resource Access/
  confirmation storage/conversation history. Coding effects reach the host *only*
  through registered actions executed by `Actions.Runner.run/3`. The runner +
  Security Central decide; the surface renders and dispatches.
- **Reuse fit.** Reuse `plugins/allbert.tui` (normalize â†’ `Identity.resolve` â†’
  `Channels.derive_session_id` â†’ `Runtime.submit_user_input` â†’ `ChannelThread` â†’
  `channel_events`), the v0.55 **static** split-result contract
  (`AllbertAssist.Runtime.Response` `model_payload`/`surface_payload`, ADR 0067
  extending ADR 0029/0030) as the payload v0.57's **net-new** streaming renders
  over, the existing `actions/` boundary â€” specifically `run_shell_command`
  (argv-only Level 1) and `execution/skill_script_runner` (timeout + brutal-kill)
  â€” as the template for the `bash` policy, the ADR 0056 trust-tier convention +
  ADR 0009 Level 1 for the tier, the v0.03 skill loader
  (`AllbertAssist.Skills`) + `AGENTS.md` hierarchy for context, and req_llm
  message-history serialization/streaming for the model switch and live turn.
  Cross-provider conversion is best-effort; M0/M4 must verify the concrete
  `ReqLLM.Context`, `ReqLLM.StreamResponse`, and `ReqLLM.StreamChunk` APIs against
  vendored `deps/req_llm` source because Context7 currently resolves `Req`, not a
  first-class `req_llm` package. ADR 0047/0051 remain provider doctor/capability
  contracts, not this handoff.
- **Upstream check.** Confirm v0.55 delivered the `tui` channel AND the **static**
  split tool-result payload (`model_payload`/`surface_payload`) through the typed
  response contract + renderer. v0.55 did **not** ship a progressive coding-diff
  renderer or streaming substrate; diff rendering is v0.57 M4 and token
  streaming/cancel is v0.57 M5. If the `tui` channel or the static split is
  missing/regressed, stop and reconcile v0.55/v0.56 before implementing v0.57.
- **Authority check (AGENTS.md Non-Negotiables).** Advisory output is never
  authority: the coding model may propose `read`/`write`/`edit`/`bash` calls but
  may not bypass `Actions.Runner.run/3`, Security Central, or confirmations. The
  bash action is policy-bounded host execution behind a registered action â€” OTP
  supervision is **not** an OS security boundary. No model-decides-it's-done for
  effectful or generated-code work. No sub-agent-via-bash-spawn.
- **Library fit.** No new library is required. `owl` (v0.55, terminal render)
  supports live-updating blocks, but its own docs position it as CLI/script UI
  tooling rather than a full-screen terminal framework; Esc handling and turn
  cancellation are Allbert TUI/input-loop work. `req_llm` covers provider-neutral
  context, streaming chunks, and cancellation hooks in local source, but M0/M4/M5
  must verify provider-specific tool-call streaming and cancellation semantics.

## Coding-Surface Ergonomics & CLI Parity
The surface must feel intuitive to a developer arriving from Claude Code, Pi,
Codex CLI, or Gemini CLI. The minimalism is in the *authority spine and default
surface*, **not** in the interaction model â€” the everyday coding ergonomics those
tools share are first-class deliverables here, expressed as Allbert primitives.

| Convention | Claude Code / Codex / Gemini / Pi | Allbert Pi-mode (v0.57) |
|---|---|---|
| **Tool surface** | read/write/edit/bash + Grep/Glob (Claude, Gemini); Codex/Pi = shell does search | **Six**: read-only/sensitive `read`/`grep`/`glob` + effectful `write`/`edit`/`bash` (registered actions). Beyond the six â†’ MCP lazy disclosure. |
| **Code search** | first-class Grep/Glob, gitignore-aware, unprompted | `grep`/`glob` actions run without a confirmation prompt but still pass Runner/Security Central, cwd jail, ignore policy, output caps, trace, and redaction; `bash` `rg`/`find` also available at the tier. |
| **Approval UX** | named modes (acceptEdits/plan/yolo...) + persistent "always allow" | modes `default` / `accept-edits` / `plan` / `tier` + Allbert Home-rooted per-repo "always allow this command" â€” a view over Security Central, no new authority. |
| **Streaming** | assistant text streams token-by-token; diffs/output render live | **net-new v0.57**: explicit stream-event contract, assistant-text token streaming, progressive tool-argument diff streaming, and turn-complete fallback. |
| **Interrupt / steer** | Esc cancels; queued correction / Enter-to-steer | **Esc-to-cancel** is Allbert input-loop/runtime work (not Owl-provided): cancel model stream + running action best-effort, keep prior work, partial-turn trace, queued correction; `bash` timeout/brutal-kill underneath. |
| **Slash set** | /help /model /clear /init /diff /compact ... | `/help`, `/model`, `/clear`, `/init`, `/diff`, `/compact` â€” slash-allowlisted and non-routable, with a per-command effect/action matrix. |
| **Context file** | AGENTS.md / CLAUDE.md / GEMINI.md auto-load + @file + /init | `AGENTS.md` hierarchy (Allbert already uses it â€” Codex made AGENTS.md the native default, so this is **parity**) + `@file` mentions + `/init`. |
| **Loop / "done"** | loop until model stops emitting tool calls, then return to prompt | same interactive REPL: the turn ends when the model stops calling tools. "Deterministic acceptance / model never decides done" applies to **effectful/generated-code acceptance gates**, not to ending a conversational turn. |

Two of these choices carry **runtime dependencies beyond rendering** and are flagged
in the milestones: assistant-text **token streaming** needs the runtime to emit
model token deltas (today's v0.55 response is turn-complete), and **interrupt**
needs clean turn cancellation (abort the in-flight model call + any running action
with a partial-turn trace; `bash` brutal-kill exists, model-call abort is net-new).

## Locked Decisions
| # | Decision | Chosen value | Trade-off if changed |
|---|---|---|---|
| 1 | Surface authority | Pi-mode is a **channel/app on the one spine** (ADR 0016): every coding effect routes through `Actions.Runner.run/3` + Security Central. The surface has **no authority beyond any other channel**. | A sibling runtime or a direct-effect coding loop reintroduces Pi's YOLO threat model â€” the thing Allbert exists to reject. |
| 2 | Tool surface | **Six default tools**: read-only/sensitive `read`, `grep`, `glob` + effectful `write`, `edit`, `bash`. Each is a registered Jido action via `Actions.Registry`/`Runner`. `read`/`grep`/`glob` use the `:coding_file_read` permission and run without a confirmation prompt, but still enforce cwd jail, symlink policy, ignore policy, binary handling, output caps, redaction, trace, and audit. `write`/`edit` use `:coding_file_write`; `bash` uses `:coding_shell_execute`. Dotted strings such as `permissions.coding_file_write` are Settings Central keys, not permission atoms. Capabilities **beyond** the six are lazy-disclosed (MCP-first, ADR 0038/0039), not pre-loaded. | Omitting `grep`/`glob` reads as impoverished to Claude Code / Gemini users (2 of the 3 ship them first-class) and forces an MCP round-trip just to search a repo. A *wide* default surface beyond search would defeat the minimalism budget. |
| 3 | Trust tier | A named **"local-coding operator"** trust tier (ADR 0056 lineage) running at **sandbox Level 1** (ADR 0009): single trusted operator, main session, `tui` channel. **Never** the default; **never** for channel-originated inbound or generated-code sessions. The tier lowers only the *confirmation burden*; policy/redaction/trace/audit still run at Level 1. It is **not** a sandbox level and never "level 0". | An implicit or default tier turns a low-friction convenience into Pi-by-default and breaks the action boundary. Mislabeling it "level 0" collides with ADR 0009 (level 0 = inert/no-exec) and conflates caller-trust with isolation. |
| 4 | Acceptance | **Deterministic acceptance rules**, not model confidence. For effectful or generated-code work the model never "decides it's done"; evidence + confirmation rule (same posture Allbert took against Hermes). | Model-confidence completion contradicts "advisory output is never authority" and the codegen committee's deterministic gates. |
| 5 | Prompt + context | A **sub-1000-token prompt+tool-defs budget** (system prompt and all six tool definitions combined, against the configured tokenizer). Context discipline follows **Pi's documented practice**: chunked reads (offset/limit) + separate context-gathering sessions + file artifacts, **not** whole-file ingestion. Reuse the `AGENTS.md` hierarchy + v0.03 skill loader; `@file` mentions resolve through the same bounded read action, never raw path ingestion. | A large default budget or whole-file context bloats the surface. (Note: full-file ingestion is *not* a Pi finding â€” Pi reads in bounded chunks and notes models resist full reads; do not attribute it to Pi.) |
| 6 | Live rendering | **Net-new in v0.57:** a coding-diff live-region renderer + progressive tool-argument rendering **and assistant-text token streaming**, built on the v0.55 **static** split payload (`model_payload` vs. `surface_payload`, ADR 0067) through the unified renderer (ADR 0030). M0 defines the stream events (`assistant_token_delta`, `tool_call_argument_delta`, `tool_call_argument_complete`, `tool_result_delta`, `turn_cancelled`, `turn_complete`). The terminal renders ephemeral deltas and reconciles to the final model/surface payloads; the model sees only the clean model payload. | Treating streaming as a v0.55 inheritance understates the work; v0.55 ships only the static split. A turn-complete reply (no token streaming) feels dead next to every peer CLI. A single conflated payload re-bloats model context. |
| 7 | Model switch | Mid-session model switch is an **operator affordance** over req_llm's canonical `Context` handoff (verified through Context7 when available, otherwise through vendored `deps/req_llm` docs/source), surfaced in the `tui` session; it changes the model, never the authority path. ADR 0047/0051 remain provider doctor/capability contracts and do not own this handoff. | Hiding the switch wastes a structurally-cheap req_llm capability Pi proved is worth surfacing. |
| 8 | Sub-agents | Delegation uses **restricted-registry delegate agents**, never an opaque `bash`-spawned sub-agent. | Self-spawn via bash is an unobservable, unrestricted child â€” the Hermes "narrow the child's tools" lesson, violated. |
| 9 | `bash` semantics | **Raw shell strings, but only at the local-coding operator tier** (Pi-faithful). For every other caller `bash` is unavailable or falls back to argv-only `run_shell_command`. Tier-restricted raw shell is bounded by cwd-confinement to the repo root, a wall-clock timeout + brutal-kill (reuse `skill_script_runner`), full trace/audit, and the stance that the host/container â€” not OTP â€” is the security boundary. | A general raw-shell action would break `run_shell_command`'s anti-injection (argv-only) posture for untrusted callers. Restricting raw shell to one trusted local operator keeps the convenience without widening the channel-originated attack surface. |
| 10 | Approval modes | Coder-facing modes `default` / `accept-edits` / `plan` / `tier`, plus persistent **"always allow this command"** per repo. This is a **presentation + persistence layer over Security Central** (Policy allowlist + confirmations) â€” it changes *confirmation cost*, never the authority path. `plan` executes only read/search; it never executes `write`/`edit`/`bash`. The allowlist lives under Allbert Home (`<ALLBERT_HOME>/coding/allowlists/<repo_hash>.json` or equivalent SQLite store), stores canonical command signatures with repo fingerprint + permission + cwd + optional TTL, and supports list/revoke/audit. The `tier` mode is the local-coding operator tier (cheapest). | No coder-facing modes means a non-tier session prompts on every effect (permission fatigue â€” the exact thing Claude/Codex/Gemini solved with modes + remembered allowances). Modes that granted authority would violate "advisory output is never authority." |
| 11 | Interrupt / steer | **Esc cancels** a running turn (work done so far is kept + traced) and a **queued correction** lets the operator redirect without killing the turn. Cancellation cleanly aborts the in-flight model stream when the provider/ReqLLM path supports it and cancels or brutal-kills any running action by policy; failures degrade to bounded timeout + partial-turn trace, never silent orphaning. Owl provides live blocks, not the Esc semantics. | No interrupt is jarring â€” Esc-to-cancel is a universal reflex. Cancellation is a runtime concern (model-call abort + partial-turn trace), so it is scoped as net-new work, not assumed from v0.55. |
| 12 | Slash set | A familiar coding **slash allowlist**: `/help` (router-local read), `/diff` (read action), `/model` (session/model change), `/clear` (session-context mutation), `/compact` (context summarization/mutation), `/init` (file write through `:coding_file_write`), reusing the v0.55.1 slash-allowlist branch + `@file` mentions. Coding slash commands are non-routable (allowlist only), like the v0.55.1 operator set, but mutating slashes are action-backed or explicitly traced router-owned effects. | Only `/model` would feel bare; coders expect a baseline. Routable slash commands would re-introduce the operator/agent-surface boundary problem v0.56 M14a fixed. Treating slash as a bypass would create a hidden effect path. |

cancel/steer runtime -> **M6** trust tier + approval modes + command allowlist ->

channel + **static** split tool-result) exist or block. Define the six tools'
params/results/signals, permission atom vs. Settings key vocabulary, stream-event
contract, trust-tier resolver, and approval-mode vocabulary before effectful code.
- **Implementation:**
  - **Modules/signatures:** `Security.Policy`/`Security.PermissionGate` gain
    `coding_tier(context) :: :local_coding_operator | :none` and an
    `approval_mode/1` reader. Permission atoms are

    `permissions.coding_file_write` are Settings Central keys.
  - **Event contract:** define `assistant_token_delta`,
    `tool_call_argument_delta`, `tool_call_argument_complete`,

  - **Dependency/API check:** verify `Owl.LiveScreen`/live block usage and local

- Gate: ADR 0068 accepted; upstream/API check green.

### M1: Read/Search/Path Substrate  *(read-only but policy-bounded)*
Register `read`, `grep`, and `glob` as read-only/sensitive coding actions through
`Actions.Registry` and `Actions.Runner.run/3`. They run without a confirmation
prompt, but still enforce cwd jail, symlink policy, ignore policy, binary
handling, output caps, redaction, trace, and audit.
- **Implementation:** `Actions.Coding.{Read,Grep,Glob}` plus `Coding.PathPolicy`
  and optional `Coding.Search`. Decide whether search uses direct Elixir
  file-walk/globbing or a tightly bounded `rg` subprocess profile.
- **Settings:** `coding.workspace.cwd_jail`, `coding.read.default_limit`,
  `coding.search.max_results`, `coding.search.max_output_bytes`,
  `coding.search.respect_gitignore`, `coding.search.respect_allbertignore`,
  `permissions.coding_file_read`.
- **Tests:** all three actions resolve via Runner; `@file` uses the same bounded
  read path; grep/glob honor cwd jail and ignore policy; no search path mutates
  files; output is redacted/truncated.
- Gate: read/search substrate green before M2/M4 consume it.

### M2: Write/Edit Actions + Split-Payload Diffs  *(file effects)*
Register `write` and `edit` as effectful coding actions with permission
`:coding_file_write`. `write` creates files; `edit` performs exact-match
replacement and returns a clear miss so the model re-reads and retries. Results
produce clean `model_payload` plus renderable `surface_payload` diff for M4.
- **Implementation:** `Actions.Coding.{Write,Edit}` and shared diff builder.
- **Settings:** `coding.workspace.cwd_jail`, `coding.write.max_bytes`,
  `coding.edit.max_replacements`, `permissions.coding_file_write`.
- **Tests:** write/edit resolve only through Runner; default decision is
  confirmation-needed until M6 modes/tier make the prompt cheap; normalized path,
  symlink, overwrite, and outside-jail escapes are refused; split payload contains
  no UI/chrome in `model_payload`.
- Gate: M2 depends on M1 path policy; M4 consumes M2's diff payload contract.

### M3: Bash + Raw-Shell Runner Contract  *(host execution)*
Register Pi-mode `bash` as an effectful coding action with permission
`:coding_shell_execute`. Preserve existing `run_shell_command` argv-only semantics
for non-tier callers. Raw shell strings are accepted only when the local-coding
operator tier resolves later in M6.
- **Implementation:** `Actions.Coding.Bash`, `Coding.RawShellSpec` or equivalent,
  and the existing local-runner/skill-script timeout discipline.
- **Settings:** `coding.workspace.cwd_jail`, `coding.bash.timeout_ms`,
  `coding.bash.max_output_bytes`, `coding.bash.allow_raw_shell`,
  `permissions.coding_shell_execute`.
- **Tests:** bash resolves through Runner; cwd jail, timeout, output caps, env
  allowlist, and redaction are enforced; raw shell fails for non-tier callers;
  argv-only fallback remains intact; bash cannot spawn opaque sub-agents as an
  authority bypass.
- Gate: M3 depends on M1 path policy and precedes M6 raw-shell cheap gate.

### M4: Stream Event Pipeline + Live Renderer  *(rendering, no authority)*
Build the live-region render substrate the v0.55 static split lacks. It consumes
the M0 stream-event contract and M2 split diff payloads: assistant token deltas
render as live prose, tool-call argument deltas render as progressive diffs, and
final payloads reconcile to `model_payload`/`surface_payload`. Rendering grants no
authority and must degrade to the v0.55 turn-complete static render.
- **Implementation:** runtime stream-event emitter (req_llm stream -> Allbert
  runtime events), TUI live-region renderer over `owl`, and progressive
  tool-argument parser.
- **Settings:** `coding.streaming.enabled`, `coding.streaming.turn_complete_fallback`.
- **Tests:** assistant tokens render incrementally and reconcile; progressive-arg
  chunks render in order; `surface_payload` never leaks into `model_payload` or
  memory; turn-complete fallback works.
- Gate: M4 depends on M0 event contract and M2 split diff payloads.

### M5: Cancellation + Queued Steering  *(live-turn control)*
Make the live turn controllable: Esc cancels a running turn and queued correction
redirects the next turn. Cancellation is best-effort for provider model streams
through the `ReqLLM.StreamResponse` cancellation callback or equivalent; action
cancellation uses action-specific cancellation or bounded timeout/brutal-kill.
Every cancellation writes a partial-turn trace.
- **Implementation:** turn supervisor/cancellation registry, TUI input-loop Esc
  handler, queued-correction buffer, partial-turn trace writer.
- **Settings:** `coding.steer.enabled`, `coding.cancel.grace_ms`.
- **Tests:** Esc mid-turn cancels model stream + running action when supported;
  fallback timeout is traced; no orphaned action/model call remains; queued
  correction applies on the next turn and grants no authority.
- Gate: M5 depends on M3 bash cancellation behavior and M4 live events.

### M6: Local-Coding Trust Tier + Approval Modes + Allowlist  *(serial authority gate)*
Implement the **local-coding operator** tier and coder-facing approval modes.
Security Central resolves the tier only for the trusted operator in the main
session on `tui`; all other callers keep the full gate and argv-only shell path.
Modes and allowlist change confirmation cost only, never authority.
- **Implementation:** wire `coding_tier/1` and `approval_mode/1` into
  `Security.PermissionGate`; add Allbert Home-rooted per-repo command allowlist
  persistence (`<ALLBERT_HOME>/coding/allowlists/<repo_hash>.json` or SQLite).
- **Settings:** `coding.default_approval_mode`, `coding.trusted_operator_id`,
  `coding.pi_mode.enabled`, `coding.bash.allow_raw_shell`, `coding.allowlist.store`,
  `coding.allowlist.default_ttl_ms`, `coding.allowlist.max_entries_per_repo`.
- **Tests:** trusted/main/`tui` resolves tier; channel-originated/scheduled/
  generated-code callers do not; `plan` blocks `write`/`edit`/`bash`;
  `accept-edits` auto-approves only file edits; allowlist entries are scoped by
  repo fingerprint + canonical command + permission + cwd, expire, list, revoke,
  and audit.
- Gate: M6 follows M1-M3 and rejoins before M7.

### M7: Minimal Prompt + Context + Model-Switch + Slash Matrix  *(parallel lanes)*
- Lane A: sub-1000-token prompt+six-tool-defs budget, named tokenizer, chunked
  read context discipline, `AGENTS.md` hierarchy, and `@file` through bounded read.
- Lane B: mid-session model switch over req_llm message-history serialization,

## Non-Goals
*(from the Pi analysis Â§8 "what not to do")*
- **No YOLO-by-default.** The local-coding operator tier is a named, gated,
  non-default tier (sandbox Level 1) â€” never the default, never for
  channel-originated or generated-code sessions.
- **No weakening of the action boundary / Security Central / confirmations.** The
  gates are made *cheap at the tier*, not *absent*; every effect still routes
  through `Actions.Runner.run/3` at Level 1.
- **No "sandbox level 0" label / no isolation relaxation.** The tier changes the
  confirmation burden, not the sandbox level; host execution stays Level 1.
- **No model-decides-it's-done** for effectful or generated-code work;
  deterministic acceptance rules.
- **Keep MCP-first with lazy disclosure** (ADR 0038/0039) for capabilities
  **beyond the six default tools**; do not adopt Pi's no-MCP stance. The minimalism
  is in the *default* surface, not a ban â€” and code search (`grep`/`glob`) is part
  of that default surface, not an MCP round-trip.
- **No sub-agent-via-bash-spawn**; use restricted-registry delegate agents.
- **No sibling runtime**; Pi-mode is a channel/app on the one spine â€” same
  authority boundary, same trace, same memory.
- No new TUI renderer choice; reuse the v0.55 `owl` terminal surface (Pi's
  scrollback-native model stays reserved until evidence).
