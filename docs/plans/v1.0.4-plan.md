# Allbert v1.0.4 -> v1.0.5 - Packaged Browser Recovery

Status: **v1.0.4 published but macOS acceptance failed; signed v1.0.5 RC.1
published but WSL2 acceptance failed; v1.0.5 M8 remediation and RC.2 are now
required. Stable publication is blocked on the remediated WSL2 rerun and
real-host Linux.**
The operator authorized this immutable hotfix on 2026-07-20 after the published
v1.0.3 macOS rehearsal proved that browser runtime ownership was neither shipped
nor declared: the artifact carried the Allbert bridge but no resolvable host
Playwright/Chromium contract. The `v1.0.3` tag and GitHub Release remain
immutable. v1.0.4 established the external-runtime packaging boundary, but its
macOS packaged browser crashed because the BEAM port applied the Windows-only
`:hide` option on Darwin. The immutable v1.0.5 tag will carry the platform-port
correction and must complete the acceptance that v1.0.3 and v1.0.4 did not.

## Purpose

This recovery line is not a feature release. v1.0.5 must:

1. Keep Node, Playwright, Chromium, and their caches out of every Allbert
   artifact. The binary contains the reviewed Allbert bridge/manifests only;
   external runtimes come from operator-approved host package paths.
2. Make the packaged driver consume explicit host Node, Playwright module, and
   browser executable paths; retain `node_unavailable` and add an actionable
   `playwright_unavailable` diagnostic. Discovery, doctor, and session execution
   must never run npm, npx, an OS package manager, or a downloader.
3. Turn the v1.0.3 escape into a permanent release gate: each matrix artifact
   must prove the forbidden runtime trees are absent and then pass a live
   `browser_doctor` launch against separately provisioned host dependencies and
   a disposable Allbert Home.
4. Repair the operator-flow drift found during the real v1.0.3 rehearsal:
   existing-tap refresh, daemon/TUI single-writer sequencing, `localhost`
   browser origin, explicit browser/research enablement and doctor, and Linux
   container verification against the published cosign bundle.
5. Apply `:hide` only when opening the Node bridge on Windows; omit it on macOS
   and Linux, where it is not a background-process primitive. Prove the option
   selection in a permanent regression and prove a real packaged macOS doctor.
6. Build and cosign immutable `v1.0.5-rc.N` GitHub prereleases, validate the
   latest candidate across macOS, both Linux artifacts, real-host Linux, and
   WSL2, and keep every pre-stable platform correction inside the still-open
   v1.0.5 milestone. RC.1 is preserved as failed WSL2 evidence; RC.2 is the
   first candidate eligible to close the newly exposed Settings, service, and
   configured-local-endpoint defects.
   Publish stable `v1.0.5`, move the tap, and run final published-artifact
   checks only after both operator-host rows pass or receive an explicit
   operator disposition.

The hard acceptance bar is this Purpose section. Nothing here may be deferred
without an explicit operator decision recorded in Build Progress.

## Why v1.0.4 And v1.0.5 Exist

The source release was healthy and publication plumbing worked, but product
acceptance did not:

| Evidence | Recorded result |
| --- | --- |
| v1.0.3 tag commit | `329b9d28551315e0a82962ad306927ccd86ee62a` |
| v1.0.3 CI | run `29797899746`, gate/build/rehearsal/publish green |
| Published release | `https://github.com/lexlapax/allbert-assist/releases/tag/v1.0.3` |
| Tap fill | `lexlapax/homebrew-allbert` commit `28ef6c22251519578feaaa978dea1d38126820d8` |
| macOS packaged doctor | FAIL: `Cannot find module 'playwright/package.json'`; persisted category `bridge_exited` |
| Artifact inspection | `bridge.js`, `package.json`, and `package-lock.json` present; `node_modules` and a Chromium executable absent |
| Local evidence root | `/tmp/allbert-v103-evidence.81IzF5/`, including `macos-browser-doctor.json` |

The failure is release-specific: checkout external smoke could use an already
installed local package tree, while the packaged contract neither supplied nor
declared a resolvable external runtime. `scripts/smoke/artifact_smoke.sh`
checked plugin registration but never launched the packaged browser. The green
v1.0.3 matrix therefore could not detect the missing ownership contract.

Moving or replacing `v1.0.3` would make published checksums, signatures,
release assets, and the tap lie about immutability. The only valid repair is a
new version with a new tag and artifacts. This plan, the v1.0.3 transfer entry,
the roadmap, CHANGELOG, and ADR 0040 amendment are the durable decision trail.

v1.0.4 then published immutably at
`337e3ddbbc14a3832d700d2ac63655a2e068c721`. CI run `29841331741`, all three
artifact jobs, hosted Linux rehearsal, cosign publication, the GitHub Release,
tap commit `5a970b994dcfe03136f0ed106a0da40f95081835`, curl install, and both
published Linux container rehearsals passed. The artifacts correctly excluded
Node, Playwright, Chromium, `node_modules`, and browser caches.

The macOS Homebrew doctor was the blocking exception. With external Node,
Playwright 1.58.2, and OS Chrome 150.0.7871.129 configured, Chrome aborted in
macOS `TransformProcessType`. Isolation runs proved direct headless Chrome,
direct Playwright, the packaged bridge, and a BEAM `Port.open/2` bridge without
`:hide` all passed; the production driver with `:hide` reproduced SIGABRT.
Erlang documents `hide` as suppressing a new console window on Windows, so the
portable contract is Windows-only `:hide`. The operator approved immutable
v1.0.5 on 2026-07-21 rather than moving v1.0.4 or accepting a failed macOS row.

## Scope And Locked Decisions

- Product scope is only the unmet v1.0.3 binary acceptance and directly exposed
  release-runbook defects. No v1.1 feature or parked v1.0.3 optimization enters.
- `bridge.js`, `package.json`, and `package-lock.json` remain reviewed Allbert
  source. `node_modules`, `.local-browsers`, browser executables, Node, and
  package-manager caches are forbidden artifact content even when present in a
  developer checkout. Release staging must exclude them rather than relying on
  `.gitignore` or a clean builder.
- Node, the Playwright module, and Chromium/Chrome are explicit host
  prerequisites. The driver resolves Node from `browser.driver.node_path` or
  PATH, accepts a host module root through `browser.driver.node_module_path` /
  `NODE_PATH`, and uses `browser.driver.binary_path` for the OS browser.
- Port visibility is platform-specific: use Erlang `open_port/2` option
  `:hide` only for Windows and omit it for Darwin/Linux. This changes neither
  artifact contents nor host-runtime ownership.
- **Operator decision (2026-07-21):** Playwright is installed outside the
  artifact as a pinned host-managed npm package:
  `PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1 npm install --prefix <host-root>
  --ignore-scripts --no-audit --no-fund --no-save playwright@1.58.2`.
  Node (18+) and Chromium/Chrome come from OS packages. Each validation row
  records the exact Node, Playwright, and browser versions/paths. Playwright's
  upstream caveat that custom executables are not its bundled-browser support
  path is accepted only when the live doctor/product smoke passes on that host.
- Artifact assembly does not access npm or a browser CDN. Host provisioning is
  an operator/package-manager step before Allbert starts; installed discovery,
  doctor, research, and action execution remain unable to install or download
  dependencies. An empty package-manager/download audit is a gate.
- The 317,912,214-byte bundled macOS prototype is rejected implementation
  evidence, not a size exception. Final artifacts should remain near the direct
  Allbert dependency baseline; every target size is still measured and a
  target more than 5% larger than its v1.0.3 counterpart blocks release absent
  an explicit operator exception.
- v1.0.3 and v1.0.4 remain published but **not accepted/closed**. v1.0.5
  inherits the incomplete macOS browser, real-host Linux, WSL2, and any
  product-smoke rows. Previously green source gates may be cited as history but
  are rerun at the v1.0.5 candidate SHA.
- **Operator decision (2026-07-21):** real-host Linux and WSL2 are pre-stable-tag
  acceptance barriers. `v1.0.5-rc.1` is a signed GitHub prerelease built from
  the clean candidate SHA; it must not become Latest and must not move the tap.
  The binary reports product version `1.0.5`, while its candidate provenance is
  the exact RC tag, commit SHA, Release URL, checksums, and cosign identity. A
  host-discovered code change stays in v1.0.5 and produces `rc.2+`; a code
  change after stable `v1.0.5` would require v1.0.6.
- **RC.1 WSL2 disposition (2026-07-21):** RC.1 is superseded, not moved and not
  accepted. Its signed install, browser prerequisites, Windows-host Ollama
  provider proof, and safe uninstall are diagnostic evidence only; no RC.1 PASS
  is carried forward to RC.2. The WSL2 row exposed product defects in
  cross-process Settings persistence, confirmed service lifecycle execution,
  and first-run/TUI readiness against a configured local endpoint. Those defects
  are v1.0.5 M8 work because they were found before the stable tag.
- **Configured-local endpoint contract:** a `local_ollama` provider whose
  operator-configured `base_url` passes the existing focused model doctor and
  exposes the selected model is a local-ready model path even when the Ollama
  executable lives on the Windows side of WSL2 and WSL `localhost:11434` is
  empty. First-run detection must reuse the configured provider/model contract;
  it must not maintain a competing localhost/binary-only truth. This grants no
  install authority and does not bundle or start Ollama.
- **Settings and service consistency contract:** every Settings read observes a
  complete YAML generation, and concurrent release-CLI/daemon writers serialize
  the read-modify-write transaction across OS processes. Confirmation approval
  records the operator decision before a self-starting/self-stopping service
  effect; the target outcome is annotated separately. systemd start/stop work
  must not wait in a same-Home deadlock, and service unit removal must agree with
  the returned target outcome. `approved` means the operator authorized the
  attempt, not that a failed target is rewritten as `denied`.
- **Onboarding completion contract:** QuickStart cannot mark onboarding complete
  while its derived first chat is not ready. A healthy configured local endpoint
  produces `Ready`, enables the existing direct-answer/model-assist settings at
  the existing readiness gate, and admits TUI. A genuinely unavailable endpoint
  remains on the repair step and cannot produce a false `complete` marker.
- `future-features.md` is unchanged: this is correction of an active release
  obligation, not new operator demand.

  artifact verification, then M10 closeout.

  M6.1 decision trail. The `v1.0.5-rc.1` prerelease waits for M6.2 plus the

  artifacts publish. M7 is an operator-held pre-stable barrier; failures rejoin
  M6.2-M6.3 through M8. Stable M9 cannot start before that barrier resolves.

  caches across targets.
