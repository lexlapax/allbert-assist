# Allbert v1.0.4 - Packaged Browser Recovery

Status: **approved; M0 documentation and red-evidence freeze in progress.**
The operator authorized this immutable hotfix on 2026-07-20 after the published
v1.0.3 macOS rehearsal proved that the release artifact omitted the Playwright
package tree and Chromium payload. The `v1.0.3` tag and GitHub Release remain
immutable. v1.0.4 exists to satisfy the binary acceptance that v1.0.3 published
but did not complete.

## Purpose

v1.0.4 is a recovery release, not a new feature release. It must:

1. Ship the release-owned, lockfile-pinned Playwright package tree and the
   target-matching Chromium payload inside every artifact. Browser discovery,
   doctor, and session execution must never run npm, npx, or a downloader.
2. Make the packaged driver resolve that hermetic browser payload and retain
   the existing explicit `node_unavailable` diagnostic for hosts without the
   documented Node prerequisite. v1.0.4 does not silently fetch or install a
   host runtime.
3. Turn the v1.0.3 escape into a permanent release gate: each matrix artifact
   must prove the bridge module, bundled Chromium executable, and a live
   `browser_doctor` launch through the extracted binary and a disposable
   Allbert Home.
4. Repair the operator-flow drift found during the real v1.0.3 rehearsal:
   existing-tap refresh, daemon/TUI single-writer sequencing, `localhost`
   browser origin, explicit browser/research enablement and doctor, and Linux
   container verification against the published cosign bundle.
5. Publish, cosign, tap-fill, and validate v1.0.4 across macOS and both Linux
   artifacts. Stop after those agent-runnable rows and ask the operator to run
   real-host Linux and WSL2. Close only after those results and every remaining
   product row have an operator disposition.

The hard acceptance bar is this Purpose section. Nothing here may be deferred
without an explicit operator decision recorded in Build Progress.

## Why v1.0.4 Exists

The source release was healthy and publication plumbing worked, but product
acceptance did not:

| Evidence | Recorded result |
| --- | --- |
| v1.0.3 tag commit | `329b9d28551315e0a82962ad306927ccd86ee62a` |
| v1.0.3 CI | run `29797899746`, gate/build/rehearsal/publish green |
| Published release | `https://github.com/lexlapax/allbert-assist/releases/tag/v1.0.3` |
| Tap fill | `lexlapax/homebrew-allbert` commit `28ef6c22251519578feaaa978dea1d38126820d8` |
| macOS packaged doctor | FAIL: `Cannot find module 'playwright/package.json'`; persisted category `bridge_exited` |
| Artifact inspection | `bridge.js`, `package.json`, and `package-lock.json` present; `node_modules` and a Chromium executable absent |
| Local evidence root | `/tmp/allbert-v103-evidence.81IzF5/`, including `macos-browser-doctor.json` |

The failure is release-specific: checkout external smoke can use an already
installed local package tree, while `stage_plugins/1` copied the plugin `priv/`
directory from a tree where ignored build dependencies were not assembled by
CI. `scripts/smoke/artifact_smoke.sh` checked plugin registration but never
launched the packaged browser. The green v1.0.3 matrix therefore could not
detect the missing payload.

Moving or replacing `v1.0.3` would make published checksums, signatures,
release assets, and the tap lie about immutability. The only valid repair is a
new version with a new tag and artifacts. This plan, the v1.0.3 transfer entry,
the roadmap, CHANGELOG, and ADR 0040 amendment are the durable decision trail.

## Scope And Locked Decisions

- Product scope is only the unmet v1.0.3 binary acceptance and directly exposed
  release-runbook defects. No v1.1 feature or parked v1.0.3 optimization enters.
- Playwright stays exactly pinned by
  `plugins/allbert.browser/priv/playwright_bridge/package-lock.json`. Assembly
  runs `npm ci --ignore-scripts`, then the checked-in Playwright CLI installs
  Chromium with `PLAYWRIGHT_BROWSERS_PATH=0`. It runs once per native target;
  no cross-target browser cache is reused.
- The staged payload is
  `plugins/allbert.browser/priv/playwright_bridge/node_modules/`, including
  `playwright-core/.local-browsers/`. The runtime-spawned bridge receives
  `PLAYWRIGHT_BROWSERS_PATH=0`, so it cannot fall back to a user cache.
- Node remains an explicit host prerequisite in this bounded hotfix. Homebrew
  declares it; curl/standalone install documentation and doctor identify it.
  Bundling a Node distribution would be a separate supply-chain decision and
  is not needed to repair the v1.0.3 omission. Tests may not substitute a fake
  Node process for artifact acceptance.
- Artifact assembly may access the npm registry and Playwright CDN. Installed
  browser doctor/research may not execute a package manager or download a
  browser. An empty package-manager/download audit after extraction is a gate.
- The artifact-size increase is measured per target and recorded; a target may
  not exceed 350 MiB compressed without operator approval. Correctness and a
  live packaged doctor are hard gates; size is an explicit review gate.
- v1.0.3 remains published but **not accepted/closed**. v1.0.4 inherits its
  incomplete macOS browser, real-host Linux, WSL2, and any product-smoke rows.
  Previously green source gates may be cited as history but are rerun at the
  v1.0.4 candidate SHA.
- `future-features.md` is unchanged: this is correction of an active release
  obligation, not new operator demand.

## Workstreams And Serial Barriers

- **W1 — release payload:** M1 build-time assembly and M2 runtime resolution.
- **W2 — proof:** M0 red fixture, M3 artifact structure/live-doctor smoke, and
  `release.v104` wiring.
- **W3 — operations/docs:** M4 runbook, install prerequisite, version,
  dependency review, and evidence reconciliation.
- W1 and W3 may proceed in parallel after M0. M3 waits for M1/M2. M5 waits for
  W1-W3 to rejoin and for the complete gate cascade. M6 starts only after the
  immutable tag publishes all three artifacts. M7 is an operator-held barrier.
- Gates run serially; release/artifact builds never share mutable npm/browser
  caches across targets.

### M1 — Hermetic browser payload assembly

- Add one release step before `:assemble`/`stage_plugins/1` that runs, in the
  bridge directory:

  ```sh
  npm ci --ignore-scripts
  PLAYWRIGHT_BROWSERS_PATH=0 node node_modules/playwright/cli.js install chromium
  ```

- Fail assembly unless the lockfile, `playwright/package.json`, and at least one
  executable target Chromium are present beneath the hermetic tree.
- Stage the resulting ignored build outputs into the plugin payload. Do not
  add generated `node_modules` or Chromium bytes to Git.

Acceptance: a clean native release build contains the pinned package version
and target Chromium; deleting either makes assembly fail with a stable message.

### M2 — Runtime resolution and dependency diagnostics

- Set `PLAYWRIGHT_BROWSERS_PATH=0` only on the Playwright bridge port process;
  preserve existing environment keys and default driver behavior.
- Prefer `browser.driver.node_path` when configured, otherwise PATH, and retain
  `node_unavailable` as the redacted doctor category.
- Add focused tests for hermetic path environment, configured-node precedence,
  and unchanged default-context error behavior.

Acceptance: focused driver/doctor tests pass; checkout and extracted-release
doctors both launch the hermetic Chromium and report Playwright `1.58.2`.

### M3 — Artifact escape gate and `release.v104`

## Definition Of Done

- Every Purpose outcome is met or explicitly dispositioned by the operator.
- v1.0.3’s immutable publication and failed acceptance are traceable to this
