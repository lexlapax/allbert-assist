## Main User/Operator Flows

- **Faster, honest daily gates.** After M3/M4: fast-local high-coverage and
  prepush drop measurably vs the M0 baseline (recorded, not asserted); the five
  formerly flaky tests pass deterministically in any order; `inventory
  --check-tags` reconciles every lane change.
- **The web long pole is gone.** `WorkspaceLiveTest` (256.6s, ~83% of web
  wall-clock) becomes partitionable split files; the remaining
  `external_runtime_serial` web set is small and enumerated.
- **Packaged users catch up.** `brew upgrade allbert` / curl installer move
  1.0.0 → 1.0.2: the TUI launches, browser research completes end-to-end behind
  one consent, channel-send routes correctly, assets are content-hashed — all
  attested against the PUBLISHED artifact (§F–§I below).
- **Dependency hygiene.** swoosh CVE fix, jsv hardening, tailwind wrapper, and
  the coordinated ecto 3.14 move land with byte-diff/round-trip proofs.
- **Governance.** ADR 0081 defines how a Tier-2 contract is promoted to frozen
  Tier-1; ADR 0082 pins the isolation-seam rules so future tests don't regrow
  the serial surface.

## Data Flow

- Test-lane selection: `mix allbert.test` gate → lane inventory (one primary
  tag per file) → per-lane runners (async single-VM for `pure_async`; serial
  in-VM with OS partitions for partitionable lanes; forced `--partitions 1`
  for security/external). The M2 seam changes NO runtime data flow: production
  reads resolve the same global registries by default; only tests pass private
  servers.
- Release: tag (no marker) → CI build matrix → artifact smoke →
  linux-rehearsal → publish (cosign-signed SHA256SUMS, version-less `latest`
  aliases) → GitHub Latest moves 1.0.0 → 1.0.2 → manual tap fill → installers
  resolve 1.0.2.

## Authority Invariants

### §D — Binary release (release commit; §E of release-rehearsal applies in full)

```sh
mix precommit
mix allbert.test release.v1
mix allbert.test release.v102
git push origin main
HEAD_SHA="$(git rev-parse HEAD)"
REMOTE_SHA="$(git ls-remote origin refs/heads/main | awk '{print $1}')"
test "$HEAD_SHA" = "$REMOTE_SHA"
git tag -a v1.0.2 -m "Allbert v1.0.2 — test suite speed & isolation phase 1 + binary catch-up"
git push origin v1.0.2
TAG_SHA="$(git rev-parse 'v1.0.2^{}')"
REMOTE_TAG_SHA="$(git ls-remote origin 'refs/tags/v1.0.2^{}' | awk '{print $1}')"
test "$HEAD_SHA" = "$TAG_SHA"
test "$TAG_SHA" = "$REMOTE_TAG_SHA"
echo "PASS: HEAD, origin/main, and peeled v1.0.2 tag agree"
```

(NO `[skip-artifacts]` — this is the binary release.) After CI publishes
(cosign is a hard step; publish waits on the Linux rehearsal): fill the tap
(`gh release download v1.0.2 --pattern SHA256SUMS`; `homebrew/fill-sha256.sh
SHA256SUMS Formula/allbert.rb`; `brew audit --strict --online`; commit+push to
`lexlapax/homebrew-allbert`), then the §3 per-OS install rehearsal. PASS:
GitHub Release `v1.0.2` is Latest with 3 tarballs + `SHA256SUMS` +
`SHA256SUMS.cosign.bundle`; `brew info allbert` and the curl installer both
resolve 1.0.2; no `REPLACE_*` placeholder remains in the formula.
