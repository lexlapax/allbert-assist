# Allbert v1.0.3 Request Flow - Test Isolation Phase 2 & Catch-up Binary Release

Status: **implementation-ready — second-pass reconciled 2026-07-19.** This is a
developer-infrastructure release: the "request" here is a test run. This doc
fixes the end-state lane architecture, the conversion decision tree, and the
validation matrix; it is maintained as waves land.

## §A — Test-Run Flow (end state)

```
mix allbert.test <gate>
  │
  ├─ statics (format / credo / dialyzer / compile)
  ▼
pure_async group ── one VM, async: true, max concurrency
  │                 (phase-2 waves GROW this group: every converted file
  │                  leaves a serial bin and joins here)
  ▼
serial lanes ── per lane: PartitionPacker (store-cost bin-pack, M8.8)
  │             → N explicit per-partition file lists
  │             → N VMs, owned envs (per-partition HOME/DB, MIX_TEST_PARTITION
  │               naming), --only <lane>, --max-cases 1, --slowest 25
  │             → TestMetrics.record per partition (walls, totals, slowest)
  ▼
external_runtime_serial ── single VM today; M6 experiment: 2-VM packed
  │                        split with fully owned envs, go/no-go by
  │                        3x-green + no cross-VM interference (LD 6)
  ▼
security_eval_serial ── single VM (eval determinism)
  ▼
gate verdict + store records (every timing claim cites these)
```

## §B — Conversion Decision Tree (per file, ADR 0086)

```
file is serial because…
  ├─ shared DB writes            → contract 1: existing Sandbox owner
  │                                + allowances to spawned processes
  │                                → db_partition_safe; never pure_async
  ├─ Application.put_env globals → contract 2: explicit process-scoped
  │     │                          configuration context (distinct from
  │     │                          settings snapshots/RegistryContext)
  │     └─ PRODUCTION path still reads a global?
  │           → record seam gap, STAY serial with reason (feeds phase-3)
  ├─ named singleton process     → contract 3: injected name/registry,
  │                                pid-qualified per test → async
  ├─ shared ALLBERT_HOME fs      → contract 4: pid-qualified per-test home
  │                                (M8.3/M8.7 idiom) → async
  │     └─ ADR 0031 global read (browser_actions class)
  │           → decide + record (convert only if the boundary allows)
  └─ real external runtime       → contract 5: NOT converted; multi-VM
                                   partition experiment only (M6)
```

Every conversion: red-first serial-requirement proof → convert through the
class contract → solo green ×3 → both-orders composition green → reviewed
test-body diff → manifest identity/multiplicity proof → check-tags
classification proof.

## §D — Measurement Protocol

- Pre column: M0 baseline on build HEAD (post-M8.8 packing), identical
  commands, sequential runs, unpiped exits.
- Per wave: lane max-partition + totals before/after from the store;
  targets set at M0, never fitted after.
- Concurrency is only realised where BOTH hold (M4 verdict, 2026-07-20):
  the file is `async: true` via its ADR 0086 contract, AND its lane runs
  with `--max-cases` > 1 (`pure_async` already does; every other lane is
  pinned at 1 until its files are converted).
- M7 production fix: eprof attribution + `mix allbert.test bench-decide`
  (corpus decide-v1, provenance-recorded, dirty:false rows only) — the
  scratchpad protocol is retired (v1.0.2 M8.10).
- RC: 20-seed monolith campaign (seeds 1000–20000 step 1000, clean
  pid-qualified homes) — both retired classes 0/20, no new unexplained
  signatures.
- Never two gates concurrently (shared tmp roots); campaign owns `_build`.

Run and record the RC cascade sequentially:

```sh
MIX_ENV=test mix precommit

git status --porcelain=v1
```

PASS: all four commands exit 0, their store rows name the same full SHA with
`dirty: false`, and status prints nothing.

Push parity, tag, and publication verification:

```sh
git push origin main
HEAD_SHA="$(git rev-parse HEAD)"
REMOTE_SHA="$(git ls-remote origin refs/heads/main | awk '{print $1}')"
test "$HEAD_SHA" = "$REMOTE_SHA"
git tag -a "$VERSION" -m "Allbert $EXPECTED_VERSION"
git push origin "$VERSION"
TAG_SHA="$(git rev-parse "$VERSION^{}")"
test "$HEAD_SHA" = "$TAG_SHA"
RUN_ID="$(gh run list --repo "$REPO" --workflow release-artifacts.yml \
  --branch "$VERSION" --limit 1 --json databaseId --jq '.[0].databaseId')"
echo "$RUN_ID" >"$EVIDENCE_ROOT/ci-run-id.txt"
gh run watch "$RUN_ID" --repo "$REPO" --exit-status
gh release view "$VERSION" --repo "$REPO" --json tagName,publishedAt,url \
  >"$EVIDENCE_ROOT/github-release.json"
test "$(gh release view --repo "$REPO" --json tagName --jq .tagName)" = "$VERSION"
```

PASS: HEAD, origin/main, and peeled tag agree; workflow exits 0; v1.0.3 is
Latest. Record `$RUN_ID`, tag SHA, and release URL.

Verify exact assets and mandatory signature:

```sh
gh release view "$VERSION" --repo "$REPO" --json assets --jq \
  '.assets[].name' >"$EVIDENCE_ROOT/assets.txt"
for asset in \
  "allbert-$VERSION-macos-arm64.tar.gz" \
  "allbert-$VERSION-linux-x64.tar.gz" \
  "allbert-$VERSION-linux-arm64.tar.gz" \
  SHA256SUMS SHA256SUMS.cosign.bundle
do
  rg -Fx "$asset" "$EVIDENCE_ROOT/assets.txt"
done
gh release download "$VERSION" --repo "$REPO" --pattern 'SHA256SUMS*' \
  --dir "$EVIDENCE_ROOT"
cosign verify-blob --bundle "$EVIDENCE_ROOT/SHA256SUMS.cosign.bundle" \
  "$EVIDENCE_ROOT/SHA256SUMS" \
  --certificate-identity-regexp \
  'https://github.com/lexlapax/allbert-assist/.github/workflows/release-artifacts.yml@refs/tags/v1.0.3' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' \
  >"$EVIDENCE_ROOT/cosign.txt"
```

PASS: all five required assets exist and cosign verification exits 0.

Fill the tap only after those checksums exist:

```sh
export ALLBERT_ASSIST_CHECKOUT="$(pwd)"
export TAP_CHECKOUT="$(mktemp -d /tmp/homebrew-allbert-v103.XXXXXX)"
git clone https://github.com/lexlapax/homebrew-allbert "$TAP_CHECKOUT"
sh homebrew/fill-sha256.sh "$EVIDENCE_ROOT/SHA256SUMS" \
  "$TAP_CHECKOUT/Formula/allbert.rb"
cd "$TAP_CHECKOUT"

brew tap lexlapax/allbert "$TAP_CHECKOUT" --custom-remote
brew trust --formula lexlapax/allbert/allbert
brew audit --strict --online --formula allbert \
  >"$EVIDENCE_ROOT/tap-audit.txt"
git add Formula/allbert.rb
git commit -m "allbert 1.0.3"
git push origin main
git rev-parse HEAD >"$EVIDENCE_ROOT/tap-commit.txt"
```

PASS: no placeholder/1.0.0 URL remains, audit exits 0, and the tap commit is
pushed. The repository formula stays at packaged Latest 1.0.0 through RC;
only published checksums populate the tap.

### §G.1 — Per-platform validation (paste-executable, run per host)

Each block is runnable as printed with inline PASS criteria — no external
lookup required on the main path. `$VERSION`/`$EXPECTED_VERSION`/`$EVIDENCE_ROOT`
carry over from §G. A SKIP requires a named owner, a policy reason, and a
follow-up location recorded in the ledger; absence of a host is never a silent
PASS. Command bodies are the audited ones in
`docs/operator/release-rehearsal.md` §3–§5 (kept in sync there).

**macOS install + product smokes (`v103-macos-install`, `v103-macos-product`):**

```sh
# install via curl, then confirm the packaged version resolves
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v103-macos.XXXXXX)"
export ALLBERT_VERSION="$VERSION"
curl -fsSL https://raw.githubusercontent.com/lexlapax/allbert-assist/main/scripts/install/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"
test "$(allbert --version)" = "allbert $EXPECTED_VERSION"   # PASS: exact match
allbert admin status                                       # PASS: renders through the spine
# Homebrew resolves the same version (use the local formula until the tap is pushed)
brew install "$TAP_CHECKOUT/Formula/allbert.rb"
test "$(allbert --version)" = "allbert $EXPECTED_VERSION"
brew test allbert                                          # PASS: exit 0
# four product smokes — capture each to its own log
allbert tui        # warm one session, send one prompt, /quit   -> macos-tui.log
allbert admin status   # browser workspace: open, one chat turn  -> macos-browser.log
#   channel + acp smokes per release-rehearsal.md §4/§5          -> macos-channel.log, macos-acp.log
# uninstall with Home preserved
allbert admin uninstall --preserve-home                   # PASS: binary gone, $ALLBERT_HOME intact
```

PASS: `--version` exact on both curl and brew paths; `brew test` exit 0; all
four product smokes green; uninstall leaves `$ALLBERT_HOME` intact.
