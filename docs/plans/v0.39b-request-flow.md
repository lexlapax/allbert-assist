## Main User/Operator Flows

- Operator optionally writes an `identity` persona file via the v0.21 memory
  review surface.
- Operator submits a prompt; the runtime runs
  `retrieve_active_memory` before composing the model call.
- Runtime composes the model call with top-K reviewed memory chunks scoped
  to `{thread_id, active_app, identity}`.
- Operator inspects the `### Active Memory` trace subsection to see which
  chunks were retrieved, their scores, and which candidates were excluded.

## Data Flow

`Runtime.submit_user_input/1` → registered `retrieve_active_memory` action →
deterministic lexical scoring over reviewed-promoted memory entries scoped to
`{thread_id, active_app, identity_namespace}` → top-K chunks (bounded by
`active_memory.top_k`) → model context composition → reply → trace metadata.

## Authority Invariants

- The identity persona file is inert content; it never grants permission or
  executes.
- `retrieve_active_memory` is read-only; it cannot promote, mutate, or infer
  durable memory.
- Retrieval results are advisory context, not authority.
- Same query + same memory state + same settings → same chunks
  byte-for-byte (deterministic and replayable).
- No embeddings; no learned ranking; no LLM-driven scoring in v0.39b.

## Handles / CLI Surfaces

- Existing `mix allbert.memory review` paths (used to write/edit the identity
  namespace).
- `mix allbert.memory list --namespace identity`
- `identity` memory namespace under `<ALLBERT_HOME>/memory/identity/`
- Settings Central `active_memory.*` keys.

## Smoke Checklist

- Create or skip the identity persona file from the memory review surface.
- Submit a prompt and confirm `### Active Memory` trace metadata is present.
- Submit the same prompt twice with no state changes and confirm identical
  retrieved chunk ids (replayability).
- Promote a chunk via v0.21 review and confirm subsequent retrievals reflect
  the change.
- Confirm cross-thread chunks do not leak into the current thread retrieval
  unless tagged appropriately.
