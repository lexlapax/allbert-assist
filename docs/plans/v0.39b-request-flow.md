## Main User/Operator Flows

- Operator optionally creates or edits `identity` markdown files under
  `<ALLBERT_HOME>/memory/identity/`, then uses the v0.21 memory review/update
  surface to mark entries `:kept`. Each file is one `Memory.Entry`; one or
  many files both work.
- Operator submits a prompt with `intent.direct_answer_model_enabled` true; if
  the selected route is a direct-answer model call, `retrieve_active_memory`
  runs after intent ranking/classification and before composing the
  direct-answer model prompt.
- Runtime composes the direct-answer model call with top-K `:kept` memory
  chunks scoped to `{thread_id, active_app, identity}` (see
  "Neutral-Context Retrieval Behavior" in the plan for `active_app: nil` and
  `active_app: :allbert` semantics).
- Operator inspects the `## Active Memory` trace section (placed after
  `## Intent Candidates`, before `## Memory Review`) to see which
  chunks were retrieved, their score breakdown, and which candidates were
  excluded.

## Data Flow

`Runtime.submit_user_input/1` → normal intent ranking/classification without
raw Active Memory chunks → direct-answer route selected →
registered `retrieve_active_memory` action → deterministic lexical scoring
over `review_status: :kept` memory entries scoped to
`{thread_id, active_app, identity_namespace}` → snapshot the candidate set
once per direct-answer turn → chunk each `Memory.Entry` body into
`active_memory.chunk_max_bytes` windows → top-K chunks (bounded by
`active_memory.top_k`) → direct-answer model context composition → reply →
trace metadata.

System namespace declarations flow through
`AllbertAssist.Memory.SystemNamespaces` (new in v0.39b M2) → merged with app
namespace declarations by the combined memory namespace facade → recognized by
memory write/read authorization. `:_system` is not an app id and is never
passed through app registry normalization or app validation.

## Authority Invariants

- The identity persona files are inert content; they never grant permission
  or execute.
- `retrieve_active_memory` is read-only; it cannot promote, mutate, or
  infer durable memory.
- If `intent.direct_answer_model_enabled` is false, Active Memory retrieval is
  skipped for direct-answer turns in v0.39b.
- Retrieval results are advisory context, not authority.
- Retrieval results never feed the intent classifier and never affect action
  selection, permission floors, confirmations, or authorization.
- Same query + same memory state + same settings → same chunks
  byte-for-byte (deterministic and replayable).
- Snapshot rule: concurrent v0.21 writes during scoring are reflected on
  the next turn, not the current one.
- No embeddings; no learned ranking; no LLM-driven scoring in v0.39b.
- System-namespace writes use the existing `:memory_write` permission;
  no new permission class.
- Neutral/core-context retrieval (`active_app` is `nil` or `:allbert`)
  surfaces identity + general chunks only; app-tagged chunks for non-active
  apps are excluded.

## Handles / CLI Surfaces

- Existing `mix allbert.memory review` and update paths (used to review/edit
  identity namespace entries after the operator creates local markdown files).
- `mix allbert.memory list --namespace identity` (new flag added in M4;
  does not exist today).
- `mix allbert.memory list --category identity` (new flag using the
  new `:identity` category from M2).
- `mix allbert.memory retrieve --query "..."` (new developer/operator
  helper added in M4 that runs `retrieve_active_memory` and prints the
  deterministic top-K result with score breakdown).
- `identity` memory namespace under `<ALLBERT_HOME>/memory/identity/`.
- Settings Central `active_memory.*` keys.

## Smoke Checklist

- Create or skip the identity persona file(s) under
  `<ALLBERT_HOME>/memory/identity/`; mark one entry `:kept` through the memory
  review/update path and verify non-`:kept` identity entries are not retrieved.
- Submit a direct-answer prompt and confirm `## Active Memory` trace metadata
  is present at the pinned placement (after `## Intent Candidates`, before
  `## Memory Review`).
- Disable `intent.direct_answer_model_enabled` and confirm the direct-answer
  fallback does not run Active Memory retrieval.
- Submit the same prompt twice with no state changes and confirm
  identical retrieved chunk ids in the same order (replayability).
- Mark a chunk `:kept` via v0.21 review and confirm subsequent retrievals
  reflect the change on the next turn (snapshot rule).
- Confirm cross-thread chunks do not leak into the current thread
  retrieval unless tagged appropriately.
- Confirm neutral/core-context retrieval (`active_app` nil or `:allbert`)
  surfaces identity + general chunks and excludes app-tagged chunks for
  non-active apps.
- Confirm the intent classifier prompt/input never includes raw Active Memory
  chunk text.
- Verify `mix allbert.memory list --namespace identity` returns identity
  entries only.
- Verify `mix allbert.memory list --category identity` returns the same
  set (deterministic alternative).
- Verify `mix allbert.memory retrieve --query "..."` prints the same
  chunks as the trace metadata for the same query and state.
