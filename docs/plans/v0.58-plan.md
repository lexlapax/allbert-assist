## Purpose

Make the platform **internally consistent and freezable** before the v1.0 contract
freeze. Four pillars, all on the existing authority spine:

1. **Surface consistency (ADR 0073).** Every surface — web, TUI, channels, Mix,
   public protocol, Pi-mode — becomes a thin, uniform view over one runtime/action/
   settings spine: one renderer, one event/audit path, one identity/session seam,
   operator reads through the registered-action layer, and shared invocation
   helpers. Today the eight channel surfaces conform; web, Mix, and public-protocol
   diverge (five bespoke renderers, no event/audit on three surfaces, web reads
   confirmations/settings directly and hardcodes identity).
2. **Settings enforcement (ADR 0004/0031 note).** All operator-tunable config goes
   through Settings Central; the known bypasses are fixed and a CI guard prevents
   new ones. (Distinct from v0.59's settings *migration* tooling, ADR 0046.)
3. **Web design system & UX language (ADR 0074).** A real token system, a
   prop-driven component-variant registry, a shared pattern library, one app shell,
   and the catalog as the rendering boundary for *every* page (Jobs/Objectives are
   folded in). The chat-primary re-layout and the operator panels are executed on
   this system.
4. **Redundancy consolidation.** Comb the codebase and consolidate the duplicated
   helper families (~8 patterns, 100+ sites) into shared modules, so v0.59 inherits
   a clean base for its final cleanup.

The authority boundary is unchanged: this release changes *how surfaces reach the
spine* and *how the web is built*, never what any surface may do. Security Central
remains the only authority; descriptors remain routing vocabulary; surface policy
remains presentation governance.

## Cross-Cutting Invariants (apply to every milestone)
- **One spine.** User turns through `Runtime.submit_user_input/1`; operations
  through `Actions.Runner.run/3`. No surface calls business logic, `Confirmations`
  storage, or `Settings.Store` directly (ADR 0073).
- **Settings Central is the only config authority.** Every new key (surface
  policy, the fixed bypasses, web prefs) is registered with specs + defaults +
  `safe_write_keys`; the CI guard fails on ad-hoc operator-tunable config.
- **Security Central is the only authority boundary.** Surface policy, panels, and
  the design system grant no authority, make no `:internal` action routable, lower
  no confirmation floor, and grant no egress.
- **Catalog is the rendering boundary.** No model- or data-generated UI; every page
  renders through the unified catalog inside one shell (ADR 0030/0074).
- **Redaction everywhere.** No panel or renderer prints secret refs, endpoint URLs,
  raw prompts, raw descriptor/evidence payloads, or provider response bodies.

## Locked Decisions
| # | Decision | Chosen value | Trade-off if changed |
|---|---|---|---|
| 1 | Scope | Full consolidation: full surface unification + full web design system + all redundancy, anchored by ADR 0073/0074. | Deferring leaves divergence/debt to carry through the 1.0 freeze. |
| 2 | One renderer | The five bespoke response renderers collapse into one `Surface.Renderer` over the typed `Runtime.Response` + a surface descriptor. | N renderers re-diverge and re-bloat model context. |
| 3 | Uniform events | Web/Mix/public-protocol record inbound/rejection/error to the shared event spine by `surface_id`. | An audit gap on three surfaces blocks a freezable, auditable platform. |

| 5 | Settings access | No operator-tunable config outside Settings Central; CI guard enforces it; the 6 known bypasses fixed. | Ad-hoc config re-accumulates and is unmanageable at 1.0. |
| 6 | Design system | Tokens + prop-driven variant registry + pattern library + one app shell + catalog-for-all-pages; UX language doc. | Class-driven variants + hand-rolled pages keep the web inconsistent. |
| 7 | No rebuild / no rename / no new routes | ADR 0023 substrate, `Conversations.Thread` internals, `Session.Scratchpad`, `/workspace`, and the catalog boundary are unchanged. "Conversations" is UI-string-only. | A rebuild/rename collides with the 1.0 freeze and the existing `Session` concept. |
| 8 | Surface policy | Operator-managed presentation/report-shape policy is Settings-Central-backed and action-mediated, distinct from descriptor YAML and from Security Central authority. | Mixing report shape into descriptors or permissions violates ADR 0060/0062/0006. |

| 10 | v0.59 boundary | v0.58 = consistency + consolidation + web system. v0.59 keeps migration (0046), param-contract (0065), onboarding (0069), security sweep, RC. | Pulling 0065 forward over-couples v0.58; leaving consolidation to v0.59 bloats the RC. |

## v0.56 DTO Handoff For Web Panels
The web panels render the v0.56 read/action contracts through `Actions.Runner.run/3`
(ADR 0073) — no Mix scraping, no `DescriptorStore`/`Settings.Store` reads, no new
authority. Reads: `intent_coverage`, `intent_list_descriptors`,
`intent_show_descriptor`, `intent_eval_run`, `intent_list_review`, `model_doctor`,
`list_model_profiles`, `list_provider_profiles`. Mutations (explicit operator
actions, gated): `edit_intent_descriptor`, `disable_intent_descriptor`,
`enable_intent_descriptor`, `promote_intent_descriptor` (ADR 0071 gate),
`optimize_intent_descriptors`, `reindex_intent_descriptors`. CLI/TUI parity oracles:
`mix allbert.intent …`, `mix allbert.settings model-doctor`, `mix allbert.model
list`, TUI `/intents`, `/models`. Redaction + advisory-only authority contract per
the cross-cutting invariants.

### M0: Contracts (serial barrier)
Author ADR 0073 + 0074; amend ADR 0016/0030/0070; add the ADR 0004/0031 settings
note; revise ADR 0024 (chat-primary). Define the conformance matrix, the
design-token/variant/pattern spec, and the `surface_id` vocabulary. No code.
- **Verify:** ADRs Proposed/revised; conformance matrix + token spec reviewed.
- **Rejoin:** barrier for M1–M3.

### M1: Unified Surface Renderer
Collapse the five bespoke response renderers (web, TUI, channels, Mix, MCP) into one
`Surface.Renderer.render_response(response, surface_descriptor)` over the typed
`Runtime.Response`. Surfaces pass a descriptor (render primitives — the
`[:typed_command, :list, :button, :link]` set defined by ADR 0016); `model_payload`
never carries `surface_payload` chrome; only `model_payload` threads to memory.
- **Files:** a shared renderer module; thin per-surface descriptor adapters; remove
  `telegram/renderer.ex`, `tui/renderer.ex`, web/mix/MCP response formatting in favor
  of the shared path.
- **Tests:** byte-for-byte render parity per surface against the prior output; no UI
  leak into `model_payload`. Eval `surface-renderer-unified-parity-001`.

### M2: Uniform Surface Event Recording + Audit
Web, Mix, and public-protocol record inbound / rejection / error events to the shared
channel-event spine keyed by `surface_id`
(`live_view`/`cli`/`mcp_stdio`/`mcp_http`/`acp`/`openai_api`), with dedupe + trace,
matching the channel pattern. Operator event reports must render `surface_id` so
manual validation can prove cross-surface parity without database/file greps.
- **Files:** event recording in `workspace_live.ex`, `mix/tasks/allbert.ask.ex`,
  `public_protocol/*`; a shared recorder (decorator) so all surfaces call one path.
- **Tests:** a turn on each surface records an event + trace; a rejection/error is
  persisted, not discarded. Eval `surface-event-audit-parity-001`.

### M3: Web Via Actions + Identity
Route web confirmation reads and settings reads through the ADR 0070 read-only action
layer; replace hardcoded `user_id:"local"`/`session_id:"web-local"` with
`Identity.resolve` + the shared session seam; apply operator-read discipline to web
panels (only `:internal`/`:read_only` reads; mutations stay explicit gated actions).
- **Files:** `workspace_live.ex` (remove direct `Confirmations`/`Settings.Store`
  reads), identity wiring.
- **Tests:** web reads resolve through registered actions; identity resolves through
  the map; no direct store read remains. Evals `web-reads-action-backed-001`,
  `web-identity-resolved-001`.

### M4: Shared Surface-Invocation Helpers
Introduce `Surfaces.ContextBuilder` (`cli_context`/`live_view_context`/…),
`Actions.Helper.completed_action/2`, and `ErrorExtraction.from_response/1`; route all
surfaces + Mix tasks + web components through them, deleting the ~20× duplicated
`context/0`/`completed_action/2`/`response_error/1`.
- **Tests:** each surface builds context via the shared builder; error extraction is
  identical everywhere. Eval `surface-invocation-shared-001`.

### M5: Settings Central Enforcement
Fix the known bypasses (verified present 2026-06-24): the trace key
`runtime.trace_default` **already exists** — M5 removes the `ALLBERT_TRACE_ENABLED`
env-var precedence in `trace.ex` so the Settings key is authoritative (the env var
is checked first today). The remaining bypasses become **new** keys: browser
host-resolver (`plugins/allbert.browser/.../playwright.ex` `System.get_env`) →
`browser.driver.host_resolver_rules`; intent `@decisive_confidence 0.8`
(`intent/router/disambiguator.ex`) → `intent.router_decisive_confidence`;
active-memory `@candidate_limit`/`@excluded_sample_limit` (`memory/active_memory.ex`)
→ keys; trace `@workspace_recent_limit 5` (`trace.ex`) →
`runtime.trace_recent_entries_limit`; artifact `@default_timeout 15_000`
(`artifacts/ingestion_consumer.ex`) → `artifacts.ingestion_timeout_ms`. Add a **CI
guard** (Credo check + test) failing on `Application.get_env`/`System.get_env` for
operator-tunable config, with an allowlist for genuine infra/test config.
- **Settings:** the new keys above (specs + defaults + `safe_write_keys`); defaults
  in the table below mirror the current hardcoded values.
- **Tests:** each former bypass now reads through Settings Central and is operator-
  settable; the guard fails on a seeded ad-hoc read. Eval `settings-no-bypass-001`.

### M6: Design Tokens + UX Language
Add documented token scales (color, **type**, **spacing**, radius, **motion**) as
global CSS custom properties (not scoped to `#workspace-shell`); apply high-contrast +
reduced-motion globally. Author the UX-language doc (terminology, scales, patterns,
a11y, copy conventions).
- **Files:** `assets/css/app.css` token layer; `docs/developer/web-design-system.md`.
- **Tests:** components consume tokens (no hardcoded sizes in changed components);
  reduced-motion/high-contrast apply on a non-workspace page. Eval `design-tokens-global-001`.

### M7: Component-Variant Registry + Pattern Library
Make variants prop-driven (`<.workspace_button variant=...>`, status tones, card
kinds) through the catalog; add shared accessible patterns (modal/popover,
loading/streaming, empty state, error callout) with built-in focus + ARIA. Replace
ad-hoc variant classes + hand-built modals.
- **Tests:** variant selection is prop-driven; the shared modal carries
  dialog/aria-modal/FocusTrap. Eval `component-variant-registry-001`,
  `pattern-library-a11y-001`.

### M8: Shared App Shell + Catalog For All Pages
One shared app shell (nav, header/switchers, mobile shellbar) wrapping every operator
page; fold **Jobs** and **Objectives** into the catalog/shell (no hand-rolled HEEx);
define `grid-template` explicitly in `app.css`.
- **Tests:** Jobs/Objectives render through the catalog inside the shell; no
  undeclared atoms; a11y affordances present on those pages. Eval
  `all-pages-catalog-shell-001`.

### M9: Chat-Primary Re-Layout (on the design system)
Chat is the primary surface; ephemerals are modals/popovers; canvas demoted to a
launcher/drawer; **"Conversations"** relabel (UI strings only). Executed via the
CoreApp shell tree + tokens + pattern library — not a one-off styling pass.
- **Tests:** chat-primary default; ephemeral renders as modal dialog + dismiss;
  canvas reachable as drawer with tiles intact; relabel UI-only (zero internal
  identifier diffs). Evals `chat-primary-default-001`,
  `ephemeral-renders-as-modal-dialog-001`, `conversations-relabel-ui-only-001`.

### M10: Operator Panels (Intents / Settings-Models / Surface-Policy)
The three operator panels as catalog components on the design system, rendering the
v0.56 DTOs through `Actions.Runner.run/3`. Intents: coverage, source badges, slot
counts, eval/gate status, review queue, edit/disable/promote (ADR 0071 gate).
Settings/Models: recommendation matrix, status, diagnostics, bounded inventories
(raw rows require an explicit affordance + surface policy). Surface-Policy: the M11
policy editor.
- **Tests:** panels render from action DTOs (no store reads / no CLI scraping); no
  secret/endpoint leak; gated promotion mutates via action; regressing promotion
  rejected. Evals `intents-panel-v056-dto-parity-001`,
  `intents-panel-gated-promotion-001`, `models-panel-v056-dto-redaction-001`.

### M11: Operator-Managed Surface Policy Layer
A `surface_policy.*` Settings Central namespace + a read/update action pair + redacted
DTOs + integration points used by agent-facing read actions before rendering. Governs,
per surface/action: which may request `operator_report` vs `assistant_summary`, the
redaction/display profile, row/count bounds, and the explicit affordance required.
Now applies across the *uniform* surfaces (M1–M4). Security Central stays final
authority; policy cannot make `:internal` actions routable, lower floors, bypass
confirmation, or grant egress.
- **Tests:** raw reports require explicit affordance; NL turns stay bounded; policy
  edits audited + fail-closed; policy grants no authority; web/TUI/CLI parity proves
  one policy DTO. Evals `surface-policy-raw-report-affordance-001`,
  `surface-policy-settings-central-001`, `surface-policy-no-authority-grant-001`.

### M12: Redundancy Consolidation
Consolidate the 8 helper families into shared modules — `AllbertAssist.Maps`
(`field/3`, `get_any/2`), `AllbertAssist.Serialization` (`stringify_keys/1`),
`AllbertAssist.Validation` (`clamp_limit/4`), `AllbertAssist.Settings.Helpers`

`redact_release_output/1`, stale `context/0` post-ADR-0070).
- **Tests:** consolidated helpers pass each call site's prior behavior; no public-API
  change. Eval `redundancy-consolidation-no-regression-001`.

## Non-Goals
- No rebuild of the canvas/ephemeral substrate; no model-generated UI.
- No internal `Thread`→`Session` rename; no DB/schema rename; no new routes.
- No descriptor-YAML control of report shape or authority.
- No settings *migration* tooling (v0.59, ADR 0046); v0.58 governs the access path,
  not schema versioning.
- No central action param-contract enforcement (v0.59 M7, ADR 0065); v0.58 makes the
  invocation path uniform so v0.59 can enforce on it.
- No new user-facing capability or authority.

- every surface (web, TUI, channels, Mix, public protocol, Pi-mode) conforms to ADR
