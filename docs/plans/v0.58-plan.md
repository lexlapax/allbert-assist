## Purpose

Re-layout `/workspace` so **chat is the primary surface**, modeled on the
ChatGPT and Claude chat-UI grammar (Hermes — https://hermes-agent.nousresearch.com/
— informs the conversation-first framing; its public site documents a multi-channel
conversational agent, not a specific web component grammar). This is a
**re-skin / re-layout of the existing ADR 0023/0024 Surface substrate, not a
rebuild**:
- Chat becomes the primary surface (today it is the left pane of a 2-pane
  chat+canvas grid — the split is CSS, `grid-template-areas` in `app.css`).
- Ephemeral surfaces become **modals/popovers** (they already carry
  `role="dialog"`/`aria-modal`/FocusTrap/escape-dismiss; only the CSS places them
  inline in a bottom row).
- The canvas is **demoted** to a launcher destination / side-drawer (reusing the
  existing `canvas_destination` routing), no longer a co-equal pane.
- UI labels cleaned up — **"Conversations"** replaces the "threads" label
  (UI strings only; the internal `Conversations.Thread` module/schema/DB, catalog
  atoms, events, signal topics, and settings keys are unchanged, and the distinct
  volatile `Session.Scratchpad` concept is untouched — chosen to avoid the
  thread/session terminology collision).

v0.58 also formalizes the **operator-managed surface policy** exposed by the web
UX: which surfaces may request raw operator reports, which render bounded assistant
summaries, which redaction/display mode applies, and which affordances are explicit
operator controls. This is presentation policy, not intent vocabulary and not
permission authority. It lives in Settings Central / registered actions, is visible
and editable through the web operator UX, and never makes an internal action routable
or lowers confirmation/security requirements.

## Readiness Sweep
- **Re-skin fit.** The shell IS a Surface tree (ADR 0023 §2); the 2-pane split,
  maximize/focus, and ephemeral placement are CSS + presentation, so chat-primary
  is largely a CoreApp shell-tree + `app.css` + presentation change. ADR 0024's
  v0.34 revision already made nav a view-only launcher and canvas a single
  destination — the substrate is already chat-and-launcher oriented.
- **Authority fit.** No new authority/route/permission; launcher selection stays
  view-only; `active_app` only via v0.33 handoff; fragment emission stays
  HMAC-validated; rendering executes no model-generated UI (catalog is the
  boundary).
- **Terminology fit.** "Conversations" matches the internal `Conversations`
  module while `Thread` stays the row type — no internal rename, no collision.

## v0.56 DTO Handoff For Web Panels

v0.56 M12 ships the read/action contract v0.58 must render. The web panels must
call registered actions through `Actions.Runner.run/3`; they must not scrape Mix
output, bypass into `DescriptorStore`, or derive new authority from descriptor
or model data.

- **Intents panel read DTOs:** `intent_coverage`, `intent_list_descriptors`,
  `intent_show_descriptor`, `intent_eval_run`, and `intent_list_review`.
  CLI/TUI parity oracles: `mix allbert.intent coverage|list|show|eval run|review`
  and TUI `/intents`. Render coverage, descriptor source badges, slot counts,
  eval/gate status, and review-queue metadata from these DTOs.
- **Intents panel mutations:** `edit_intent_descriptor`,
  `disable_intent_descriptor`, `enable_intent_descriptor`,
  `promote_intent_descriptor`, `optimize_intent_descriptors`, and
  `reindex_intent_descriptors`. They remain explicit operator actions.
  `promote_intent_descriptor` and optimizer-generated promotions run the
  ADR 0071 gate; a failure displays diagnostics and mutates nothing.
- **Settings/Models panel read DTO:** `model_doctor`. CLI/TUI parity oracles:
  `mix allbert.settings model-doctor` and TUI `/models`. Render each row's
  purpose, recommended profile/model, configured profile/model, status,
  diagnostics, privacy note, and fallback note.
- **Redaction contract:** no panel may print secret refs, provider endpoint URLs,
  raw prompts, raw descriptor examples/synonyms beyond the v0.56 DTO fields, raw
  evidence payloads, or private provider response bodies. The DTOs are the
  source of truth; add a v0.58 action contract change if a new field is needed.
- **Authority contract:** model recommendations, eval results, descriptors, and
  learned proposals are advisory only. The panels do not make internal actions
  routable, do not lower confirmation requirements, and do not grant egress.
- **Surface-policy handoff:** v0.56 M14a leaves raw-vs-summary report shape as
  action presentation policy. v0.58 owns the operator-managed policy layer and UI:
  Settings Central keys/actions define which surfaces can request `operator_report`
  versus `assistant_summary`, the redaction/display profile, row/count bounds, and
  the explicit affordance required. Descriptor YAML remains routing vocabulary.

## Milestones

### M1: Chat-Primary Layout
Re-declare the CoreApp `:workspace` shell so chat is the main column with a
collapsible left **Conversations** rail and a pinned composer; rewrite the
`app.css` grid (`nav | chat | resizer | canvas` → chat-dominant). Header gets the
existing conversation switcher (+ optional model switcher via existing
`:button`/`:status_badge`). Seams: `core_app.ex` `workspace_nodes/0`, `app.css`
grid + maximize/focus rules, `renderer.ex` split-resizer.

### M2: Ephemeral → Modal/Popover
Move the `ephemeral` grid-area to a fixed overlay + backdrop; promote the active
ephemeral to a centered modal, stack the rest. Reuse the shipped dialog/focus-trap/
escape-dismiss semantics and `dismiss_workspace_ephemeral`. Seams: `app.css`
`.workspace-ephemeral-node*`, `renderer.ex` ephemeral branch.

### M3: Canvas Demotion
Canvas becomes a launcher destination / slide-over drawer rather than a permanent
pane; default view is chat. Reuse `canvas_destination` routing + focus/maximize
toggles; update mobile tabs + canvas empty-state copy. Seams: `workspace/catalog.ex`,
`workspace_live.ex`.

### M4: "Conversations" Relabel (UI strings only)
Replace the user-facing literals (`core_app.ex` thread-list title;
`structural.ex` "New thread"/"Recent workspace threads"/"Threads"/"No threads
yet"/switcher copy; `jobs_live.ex` table header; `workspace_live.ex` notice).
Leave canvas-internal "this thread" wording or relabel to "this conversation" per
copy review. Guard: zero diffs to `thread_id`, atoms, events, topics, settings
keys.

### M5: Polish / Accessibility / Evals / Closeout
WCAG re-check (real modals → focus traps, keyboard nav, reduced-motion),
dark-mode parity, mobile single-column; `:v058` eval rows; `release.v058`;
closeout discipline; ADR 0024 v0.58 revision accepted.

### M6: Browser Manual Validation
Real browser pass on `/workspace`: chat-primary renders; new/switch conversation;
ephemeral approval card appears as a modal and escapes/dismisses; canvas reachable
as launcher/drawer with tiles intact; relabels visible; streaming + message
affordances work; offline shell still registers. Capture screenshots.

### M7: Intents Operator Panel (deferred from v0.54 M9.4)
The operator-facing **Intents** web panel for the intent-descriptor lifecycle
(ADR 0062) — deferred from v0.54 M9.4 because it is LiveView/workspace-surface work
that overlaps this UX redo (v0.54 shipped the full **CLI** curation:
`mix allbert.intent list|show|disable|promote|review|optimize|reindex` + `doctor`
coverage). Scope: an `intents` entry in `WorkspaceLive` `@workspace_tools` + a
LiveView listing resolved descriptors with **source-layer badges**
(code/action/generated/override), the **coverage report**, per-descriptor
**edit/disable**, a **review queue** for generated-but-unpromoted descriptors
(promote/discard), and a **trigger-optimize/reindex** button. Edits write the
operator override (`<ALLBERT_HOME>/intents/overrides/*.yaml`) and are audited + gated
exactly like other operator writes (no new authority; routing remains advisory —
ADR 0060/0062). Reuses the settings/marketplace panel pattern. `:v058` eval row:
edit/disable/promote round-trips through the panel and the resolver reflects it;
a seeded regressing promotion is rejected by the ADR 0071 gate and leaves review
storage unchanged.

### M8: Settings/Models Operator Panel
Add a **Settings/Models** workspace panel that renders the v0.56 `model_doctor`
DTO without introducing a new settings authority path. The first v0.58 version is
read-only: it displays the per-purpose model recommendation matrix, configured
profile/model, status, diagnostics, privacy/fallback notes, and local-pull or
remote-egress warnings. It may deep-link or point to existing Settings Central
write surfaces, but any write remains an existing registered settings action and
is outside this panel unless v0.58 explicitly adds a gated mutation row.
`:v058` eval rows: recommendations render with no secret/endpoint leakage; the
panel grants no egress and does not make `model_doctor` routable.

### M9: Operator-Managed Surface Policy
Introduce a first-class surface policy layer and web operator panel section for
presentation/report-shape controls. The policy is Settings Central-backed and
action-mediated, not descriptor YAML. It answers questions such as:

- Which surfaces (`tui`, `web`, `cli`, public protocol clients, future panels) may request
  `operator_report` for a given read action.
- Which surfaces default to `assistant_summary`.
- Which redaction/display profile and row/count bounds apply.
- Which UI affordance proves explicit operator intent (slash command, Mix task,
  authenticated web operator button/panel).

Implementation shape: add a `surface_policy.*` Settings Central namespace, a small
read/update action pair through `Actions.Runner.run/3`, redacted DTOs for the web panel,
and integration points used by agent-facing read actions before rendering their message.
Security Central remains final authority: surface policy cannot lower permission floors,
make `:internal` actions routable, grant egress, bypass confirmation, or override action
schema validation. Descriptor YAML can still tune `list_settings`/`list_channels` routing
vocabulary, but cannot select raw report mode.

`:v058` eval rows: raw reports require explicit operator affordance; natural-language
turns render bounded summaries; policy edits are audited and fail closed on invalid keys;
policy never grants new action authority; web/TUI/CLI parity proves the same policy DTO.

## Non-Goals
- No removal of the canvas/ephemeral substrate (re-skin, not rebuild).
- No internal `Thread`→`Session` rename; no DB/schema rename.
- No model-generated UI code; the catalog stays the rendering boundary.
- No new routes; `/workspace` stays canonical.
- No descriptor-YAML control of report shape or authority; descriptor files remain
  routing vocabulary only.

## Definition Of Done
v0.58 is done when:
- `/workspace` renders chat-primary (single main chat column + collapsible
  Conversations rail + pinned composer);
- ephemeral surfaces render as modals/popovers (dialog semantics + dismiss),
  not an inline bottom row;
- the canvas is reachable as a launcher destination / drawer, default view is
  chat, and durable tiles + offline edit + multi-tab sync are preserved;
- the UI says "Conversations" with **zero** internal identifier churn (guard
  eval passes);
- the Intents panel renders the v0.56 coverage, descriptor, eval/gate, and
  review DTOs and uses only registered v0.56 actions for mutations;
- the Settings/Models panel renders the v0.56 model recommendation DTO with no
  secret/endpoint leakage and no new authority;
- the operator-managed surface policy layer ships with a web panel, Settings Central
  keys, registered read/update actions, redacted DTOs, and eval proof that raw
  reports require explicit operator affordances while natural-language turns stay
  bounded;
- authority is unchanged (fragment HMAC + view-only launcher evals pass);
- `:v058` evals + `release.v058` + the full release gate pass, plus a real
  browser manual-validation pass;
- ADR 0024 v0.58 revision, roadmap, vision, and agent-context-map agree.
