# Allbert v0.41 Everyday Integration Pack 1

Request flow: `docs/plans/v0.41-request-flow.md`

ADR: `docs/adr/0039-mcp-first-native-plugin-second-integrations.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Make Allbert useful for ordinary personal work by adding the first native or
MCP-backed integration pack: calendar, mail, GitHub, and notes/files.

## Expected Inputs

- v0.40 MCP client.
- Plugin/app contract and Surface DSL.
- Settings fragments and encrypted secrets.
- Existing email channel credentials, where reused by mail integration.
- Proposed ADR 0039 accepted before implementation closeout.

## Operator-Visible Win

An operator can ask for agenda, mail, GitHub, and notes/files help from the
workspace or a supported channel, with each effect routed through visible
actions and confirmations.

## Planned Interfaces

- Calendar plugin/app surface.
- Mail plugin/app surface next to the email channel adapter.
- GitHub workspace surface, using MCP for heavy operations where sufficient.
- Notes/files plugin/app surface with memory promotion candidates.
- App intent descriptors and workspace panels for each integration.

## Security Posture

Integrations are plugin-shaped, not core dependencies. MCP is preferred when it
is sufficient; native plugins exist only for richer surfaces or memory
namespace ownership. Effects use Resource Access, Security Central,
confirmations, trace, and audit.

## Non-Goals

- No broad provider dependency in Allbert core.
- No silent credential reuse across unrelated integrations.
- No automatic memory promotion from notes or mail.
- No hosted multi-user authorization model.

## Milestones

- M1: ADR 0039 and integration capability inventory.
- M2: Calendar and mail surfaces/actions.
- M3: GitHub and notes/files surfaces/actions.
- M4: Workspace panels, intent descriptors, and channel rendering.
- M5: Security evals, docs, and closeout.

## Documentation Deliverables

- Operator integration setup guide.
- Security-hardening notes for credentials, grants, and memory sync.
- Agent context map entries for each integration class.

## Definition Of Done

v0.41 is done when each first-wave integration has a usable workspace surface,
at least one confirmed effect or read flow, clear credential/grant posture, and
focused tests/evals proving no integration bypasses Allbert's action boundary.
