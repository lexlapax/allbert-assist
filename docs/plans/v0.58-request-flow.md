## Cross-Surface Data Flow (the one path, ADR 0073)
```text
inbound (web keypress/submit | TUI line | channel message | mix arg | MCP/ACP call)
  -> normalize -> Channels.Identity.resolve (list-shaped identity map; no hardcoded "local")
  -> session derive (provider channel id OR surface_id: live_view|cli|mcp_stdio|mcp_http|acp|openai_api)
  -> record inbound event (dedupe + trace) on the shared event spine
  -> user turn:  Runtime.submit_user_input/1
     operation:  Actions.Runner.run/3   (operator reads: :internal/:read_only actions)
  -> Security Central gate (unchanged authority) + Settings Central (only config authority)
  -> typed Runtime.Response (model_payload | surface_payload)
  -> ONE Surface.Renderer.render_response(response, surface_descriptor)
       descriptor = render primitives (ADR 0016); model sees model_payload only; memory persists model_payload only
  -> surface presents (web catalog component | TUI scrollback | channel message | mix stdout | MCP payload)
  -> record rejection/error event uniformly (not discarded)
  -> [web] panels render v0.56 DTOs via registered actions; surface policy governs report shape
```

## Web Render Flow (ADR 0074)
```text
every operator page -> ONE shared app shell (nav + header/switchers + mobile shellbar)
  -> WorkspaceRenderer over the unified catalog (ADR 0030) — workspace, Jobs, Objectives,
     Intents, Settings/Models, Surface-Policy all render through catalog atoms
  -> components consume global design tokens (color/type/spacing/motion scales)
  -> variants are prop-driven (variant registry); modals/loading/empty/error via the pattern library
  -> chat primary (timeline+composer) | collapsible Conversations rail | canvas launcher/drawer
  -> ephemeral surfaces -> pattern-library modal (role=dialog, FocusTrap, escape/click-away)
```

## Authority Invariants
- One spine: surfaces call only `Runtime.submit_user_input/1` + `Actions.Runner.run/3`;
  no direct `Confirmations`/`Settings.Store`/business-logic reads (ADR 0073).
- Security Central is the only authority; surface policy, panels, and the design
  system grant no authority, make no `:internal` action routable, lower no
  confirmation floor, and grant no egress.
- Catalog is the rendering boundary on every page; no model-/data-generated UI.
- Settings Central is the only config authority; no operator-tunable config is read
  ad-hoc (CI guard).
- Identity via `Identity.resolve` on every surface; no hardcoded `"local"`.
- "Conversations" is a UI-string change only; internal identifiers unchanged.
- Descriptor YAML remains routing vocabulary; it does not own report shape.

### S5 - Public-Protocol Smoke And Internal-Read Denial
Public protocol validation must use public-safe tools. Internal operator reads such
as `model_doctor`, `intent_coverage`, `intent_list_descriptors`, and
`promote_intent_descriptor` must not appear in MCP/OpenAI tool exposure.

```sh

rg 'direct_answer|get_public_call_result' "$V058_EVIDENCE_DIR/s5-mcp-tools.txt"
! rg 'model_doctor|intent_coverage|intent_list_descriptors|promote_intent_descriptor' "$V058_EVIDENCE_DIR/s5-mcp-tools.txt"

export V058_MCP_TOKEN="$(mix allbert.public_protocol token create --surface mcp_http --client v058-mcp | awk -F= '/^token=/{print $2; exit}')"
export V058_OPENAI_TOKEN="$(mix allbert.public_protocol token create --surface openai_api --client v058-openai | awk -F= '/^token=/{print $2; exit}')"
test -n "$V058_MCP_TOKEN"

mix allbert.public_protocol token list --surface openai_api > "$V058_EVIDENCE_DIR/s5-openai-token-list.txt"
```
