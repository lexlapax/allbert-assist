## Render Flow
```text
WorkspaceLive.render -> single WorkspaceRenderer over the CoreApp :workspace tree
  shell tree (re-laid out): header (switchers) | chat (primary: timeline+composer)
    | collapsible Conversations rail | canvas (launcher destination/drawer)
  ephemeral surfaces -> fixed-overlay modal/popover (role=dialog, FocusTrap,
    escape/click-away dismiss) instead of inline bottom row
  canvas tiles injected via inject_runtime_node (unchanged); reached via launcher
```

## Authority Invariants
- Catalog is the rendering boundary; no model-generated UI (ADR 0023 搂9).
- Fragment emission stays HMAC-signed + catalog-validated; launcher selection is
  view-only (sets Canvas destination, never `active_app`/permission).
- `active_app` only via the v0.33 handoff; Runner鈫扴ecurity Central unchanged.
- "Conversations" is a UI-string change only; internal identifiers unchanged.

## Handles / Surfaces
- `/workspace` (canonical, unchanged route). No new routes.
- Conversation rail/switcher; canvas launcher/drawer; ephemeral modals.
- Intents operator panel and Settings/Models panel render the v0.56 read-model DTOs:
  coverage, descriptor source badges, eval/gate status, review queue, and
  recommended-vs-configured model status. The panels must not print secrets,
  provider endpoints, raw prompts, or private payloads, and any promote/edit action
  still runs through the v0.56 gate/action boundary.
