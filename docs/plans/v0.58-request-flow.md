## Locked Decisions
Recorded in `v0.58-plan.md`. Re-skin not rebuild; Surface substrate + catalog
kept; no internal `Thread`→`Session` rename (UI label "Conversations" only);
`Session.Scratchpad` untouched; `/workspace` canonical; no model-generated UI;
authority unchanged.
Surface policy is added as presentation/report-shape governance: Settings
Central-backed, action-mediated, and separate from descriptor YAML and Security
Central authority.

## Render Flow
```text
WorkspaceLive.render -> single WorkspaceRenderer over the CoreApp :workspace tree
  shell tree (re-laid out): header (switchers) | chat (primary: timeline+composer)
    | collapsible Conversations rail | canvas (launcher destination/drawer)
  ephemeral surfaces -> fixed-overlay modal/popover (role=dialog, FocusTrap,
    escape/click-away dismiss) instead of inline bottom row
  canvas tiles injected via inject_runtime_node (unchanged); reached via launcher
```

## Authority Invariants
- Catalog is the rendering boundary; no model-generated UI (ADR 0023 §9).
- Fragment emission stays HMAC-signed + catalog-validated; launcher selection is
  view-only (sets Canvas destination, never `active_app`/permission).
- `active_app` only via the v0.33 handoff; Runner→Security Central unchanged.
- "Conversations" is a UI-string change only; internal identifiers unchanged.
- Surface policy can select display/report modes for explicit surfaces, but cannot
  make internal actions routable, lower permission floors, bypass confirmations,
  grant egress, or override redaction requirements.
- Descriptor YAML remains operator-managed routing vocabulary only; it does not
  own raw-vs-summary report shape.

## Handles / Surfaces
- `/workspace` (canonical, unchanged route). No new routes.
- Conversation rail/switcher; canvas launcher/drawer; ephemeral modals.
- Intents operator panel and Settings/Models panel render v0.56 action DTOs.
  They call `Actions.Runner.run/3`; they do not parse CLI output, read
  descriptor/model stores directly, or create new authority.
- Surface Policy panel/section renders policy DTOs from registered actions and
  lets the operator configure per-surface display modes, redaction/display
  profile, bounds, and explicit-affordance requirements.

## v0.56 DTO Handoff

The v0.58 UI implementation consumes these v0.56 handles:

- **Intents reads:** `intent_coverage`, `intent_list_descriptors`,
  `intent_show_descriptor`, `intent_eval_run`, `intent_list_review`.
  Render coverage, source badges, slot counts, eval/gate status, and review queue.
  CLI/TUI parity references: `mix allbert.intent coverage|list|show|eval run|review`
  and `/intents`.
- **Intents mutations:** `edit_intent_descriptor`, `disable_intent_descriptor`,
  `enable_intent_descriptor`, `promote_intent_descriptor`,
  `optimize_intent_descriptors`, `reindex_intent_descriptors`. Promotion and
  optimize/promotion paths run the ADR 0071 gate; failure displays diagnostics
  and mutates nothing.
- **Models reads:** `model_doctor`, `list_model_profiles`, and
  `list_provider_profiles`. Render the recommendation matrix, configured
  profile/model, status, diagnostics, privacy note, fallback note, and bounded
  model/provider profile inventory summaries. Raw profile rows require an
  explicit web-operator affordance plus surface-policy permission for
  `operator_report`. CLI/TUI parity references:
  `mix allbert.settings model-doctor`, `mix allbert.model list`,
  `mix allbert.settings providers list`, and `/models`.
- **Redaction/authority:** panels must not print secret refs, endpoint URLs, raw
  prompts, raw descriptor examples/synonyms outside the DTO fields, raw evidence,
  or provider response bodies. Descriptors, learned proposals, evals, and model
  recommendations remain advisory and never grant routing authority, confirmation
  bypass, or egress.
- **Surface policy:** v0.58 adds Settings Central policy keys plus read/update
  actions for presentation/report-shape controls. The policy describes when a
  surface may request `operator_report` versus `assistant_summary`, not whether an
  action is authorized. Natural-language turns remain bounded unless an explicit
  operator affordance is present.

## Smoke Checklist
- Chat-primary renders; new/switch conversation works.
- Ephemeral approval appears as a modal; escape/click-away dismiss.
- Canvas reachable via launcher/drawer; tiles + offline edit + multi-tab intact.
- Intents panel renders v0.56 descriptor coverage, source badges, eval/gate status,
  and review queue from action DTOs; a seeded promote regression is rejected by
  the ADR 0071 gate and mutates nothing.
- Settings/Models panel renders the v0.56 recommendation matrix and model/provider
  profile inventories with no secret or endpoint leakage, uses `model_doctor`,
  `list_model_profiles`, and `list_provider_profiles`, and grants no egress.
- Surface Policy panel shows per-surface/action report mode, redaction/display
  profile, row/count bounds, and explicit-affordance requirements; edits are
  audited and invalid policy fails closed.
- Natural-language settings/channel/report phrasing renders bounded summaries;
  explicit slash/Mix/web-operator affordances render raw operator reports only
  when policy allows them.
- Relabels visible; grep guard shows no internal identifier diffs.
- Streaming + message affordances; mobile single-column.
