## Pre-Flight

Use a disposable Allbert Home and open `/agent`:

```sh
export SMOKE_HOME=$(mktemp -d /tmp/allbert-v030-smoke.XXXXXX)
ALLBERT_HOME="$SMOKE_HOME" mix ecto.migrate
ALLBERT_HOME="$SMOKE_HOME" mix phx.server
```

v0.30 assumes v0.27 real StockSage renderers exist and v0.28 has audited the
workspace Fragment/canvas path.

## Flow 1: Emit Durable Canvas Tiles From Analysis

Operator opens `/agent?app_id=stocksage` and asks to analyze a ticker.

Expected behavior:

1. StockSage runs through the existing `RunAnalysis` and confirmation flow.
2. When analysis completes, `canvas_ops` create durable workspace tiles using
   registered StockSage component ids.
3. Tiles render in the `/agent` canvas with the same component semantics proven
   in v0.27.
4. The chat response links to the StockSage detail surface and references the
   created tile ids.

Failure behavior:

- If tile creation fails validation, the analysis result remains persisted and
  the operator sees a bounded canvas-emission failure.
- No analysis is rerun merely because tile emission failed.

## Flow 2: Reload And Resume

Operator refreshes `/agent` or restarts the app.

Expected behavior:

1. Persistent StockSage tiles reappear for the same user/thread.
2. Tile provenance points back to analysis id, objective id, engine, and trace.
3. Deleted or evicted tiles follow existing soft-delete/restore behavior.
4. No tiles leak across users or threads.

## Flow 3: Dashboard Uses Persistent Tiles

Operator visits StockSage workspace/dashboard after recent analyses exist.

Expected behavior:

1. Recent analysis summaries can be pinned or surfaced as persistent canvas
   tiles.
2. Dashboard links and `/agent` tiles point to the same underlying analysis
   records.
3. The StockSage app surface remains usable without opening `/agent`.

## Flow 4: Invalid Canvas Output Is Rejected

Fixture attempts to emit unknown component ids, cross-app component ids, raw
HTML, secret-like props, or unauthorized action bindings.

Expected behavior:

1. Catalog validation rejects the tile.
2. The rejection is visible in trace/workspace diagnostics.
3. No arbitrary model-generated UI renders.

## Smoke Checklist

- Analysis completion can create registered StockSage canvas tiles.
- Tiles survive reload/restart and preserve provenance.
- Unknown StockSage component ids are rejected.
- v0.27 StockSage LiveViews still render the same analysis without depending on
  `/agent` canvas.
- Milestone closeout runs focused workspace/canvas tests and `mix precommit`.
