## Current Code State

**R15 — asset pipeline.** The digest infrastructure already exists end to end; only the
CSS link bypasses it:

- `assets.deploy` = tailwind + esbuild + **`phx.digest`**
  (`apps/allbert_assist_web/mix.exs:86-90`); the release build runs it via
  `build_web_assets/1` (root `mix.exs:159-176`, release step at `mix.exs:41`) with
  `MIX_ENV=prod` in CI (`release-artifacts.yml:17`), and
  `config/prod.exs:12-14` sets `cache_static_manifest: "priv/static/cache_manifest.json"`.
  Digested files + manifest are present in `priv/static/`.
- The JS link is already correct: `src={~p"/assets/js/app.js"}` (`root.html.heex:40`)
  digests to `app-<hash>.js` in prod.
- The defect: `root.html.heex:33-37` renders
  `href={~p"/assets/css/app.css?#{[v: static_asset_version()]}"}` where
  `static_asset_version/0` (`components/layouts.ex:603-607`) returns the OTP app
  version. Phoenix's `~p` marks the route static from the *path* but passes
  `path <> "?" <> query` to `static_path/1`, which only splits on `#`, never `?`
  (`phoenix/lib/phoenix/endpoint.ex:602-609`) — so the manifest lookup key
  `"/assets/css/app.css?v=1.0.0"` misses and the URL is served undigested. Same-version
  rebuilds therefore serve browser-cached CSS (the original R15 incident,
  `docs/plans/archives/v1.0-plan.md:507-517`).
- Theme CSS links (`root.html.heex:38-39`) are **dynamic router routes**
  (`router.ex:38-40`, `ThemeController`) with a content-derived version query
  (`theme_asset_version/0` → `Theme.Version.stylesheet_version()`) — correct, out of
  scope.
- One existing test pins the old behavior:
  `theme_controller_test.exs:184-192` asserts `html =~ ~s(/assets/css/app.css?)`.
- Service worker: `priv/static/workspace-sw.js` is cache-first keyed by request URL;
  its runtime precache list reads the **rendered DOM URLs**
  (`assets/js/app.js:983-993`, posted via `ALLBERT_WORKSPACE_CACHE_ASSETS` at
  `app.js:1005-1042`), so it adapts to digested URLs with no change. The static
  `DEFAULT_SHELL_ASSETS` plain entries stay harmless because `phx.digest` keeps the
  undigested files on disk.

## Locked Decisions

1. **Binary point release, Latest moves.** v1.0.1 tags without `[skip-artifacts]`; the
   full CI build → cosign → publish pipeline runs; the Homebrew tap is filled manually
   per `release-rehearsal.md` §2; v1.0.1 becomes GitHub Latest. `release.v1` must be
   green at the release commit.
2. **Content-hash cache-busting uses the existing digest manifest — no new mechanism.**
   The fix is subtractive: drop the `?v=` query from the app.css link so `~p` flows
   through `static_lookup`. Theme CSS keeps its content-derived version query (dynamic
   routes, not static files). `config/prod.exs`, `endpoint.ex`, the `assets.deploy`
   alias, and the release steps are already correct and stay untouched.
3. **No new `Patterns` variant.** Daisy `btn-ghost` and bare `btn` map to
   `"secondary"` (the lowest-emphasis existing variant). If a distinct low-emphasis
   look is ever wanted, that is a future web-design-system decision, not a 1.0.1 drift
   fix.
4. **The drift test is re-tightened, not baselined.** The `btn` refute regex is
   extended to also catch the list form `class={["btn ..."]}`. No allowlist, no ratchet
   file — zero raw `btn` in production web source is the contract
   (`web-design-system.md:404-406`).
5. **The offline guard stays `:pure_async` and gets a gate-level backstop.** Fix option
   (a): derive the expected version at compile time (`Mix.Project.config()[:version]`
   module attribute) instead of runtime `Application.spec`, keeping the lane contract.
   Additionally, the SW cache-name lockstep assertion is added to
   `version_consistency_test.exs` so a missed bump fails the `v1_version_consistency`
   step of every future release gate — the guard no longer lives only in one
   sweepable web test.
6. **SW cache name bumps with the release version, in the same commit as the
   `mix.exs` bumps.** M3 first syncs it to the current `1.0.0` (making the guard
   green); the M6 version-bump commit moves all three `mix.exs` files **and**
   `workspace-sw.js` `CACHE_NAME` to `1.0.1` together.
7. **No new EvalInventory rows, no `:v101` milestone atom.** Nothing here adds
   authority, signals, settings, or contract surface. The `release.v101` gate re-runs
   the existing sweeps (`v1_freeze_sweep`, `v1_product_rc_sweep`) plus focused v1.0.1
   test steps. DIT-5 is operator-attested evidence, not an `AssertBinding` row.
8. **`release.v101` = `release.v1` steps ++ focused v1.0.1 steps.** The steps attribute
   is defined as `@release_v1_steps ++ @v101_focused_steps` so freeze coverage cannot
   drift from the point gate. Evidence JSON goes to
   `<ALLBERT_HOME>/release_evidence/v101/`.
9. **DIT-5 transcript is operator-supplied and redacted before commit.** Secrets,
   tokens, and real paths outside the disposable Home are scrubbed per the
   evidence-redaction rules; the agent does not fabricate or reconstruct the
   transcript.
10. **The standing dependency refresh (M5) is review-always, apply-bounded.** Every
    outdated dependency gets a recorded disposition (applied / scoped-out with
    reason / blocked); major or breaking upgrades ride the next release rather than
    bloating a remediation point release; the refresh lands as its own commit; the
    ADR 0050 vendored-`:memento` exit is re-verified and its verdict recorded
    (2026-07-15 working rule — `roadmap.md` Working Rules, AGENTS.md).
