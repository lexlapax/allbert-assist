# Allbert v0.44 Channel Pack 1: Discord And Slack

Request flow: `docs/plans/v0.44-request-flow.md`
ADR: `docs/adr/0016-channel-adapter-boundary-and-identity-mapping.md` (v0.44
amendment for approval primitives)

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`, post-v0.37 planning pass.

This milestone broadened in the post-v0.37 planning pass to lock the
channel-approval primitive contract (ADR 0016 amendment) so v0.50 mobile
channels inherit it rather than re-invent rendering.

## Purpose

Expand Allbert reach into team and community chat by adding Discord and Slack
channel plugins that reuse the existing channel substrate. Formalize the
channel approval-primitive contract through an ADR 0016 amendment so all
later channels honor one rendering rule.

## Expected Inputs

- v0.16 channel adapter boundary.
- v0.17 plugin contract and shipped channel plugins.
- Existing Approval Handoff, confirmations, identity mapping, dedupe, and
  redaction posture.

## Operator-Visible Win

Allbert can be reached from Discord and Slack with safe mention handling,
threaded replies, and confirmation callbacks.

## Planned Interfaces

- `./plugins/allbert.discord/`
- `./plugins/allbert.slack/`
- Discord bot settings and identity map.
- Slack app settings and identity map.
- Channel status surfaced through existing channel list/show paths.

## Security Posture

Channels do not invent permission classes or confirmation shapes. Group/server
authorization, workspace mapping, and callback ownership must be explicit.
Approval Handoff remains provider-neutral and action-owned.

## Non-Goals

- No SMS.
- No WhatsApp, Signal, iMessage, or Matrix in this milestone.
- No channel-specific action authority.
- No private durable goal loop inside a channel adapter.

## Milestones

- M0: ADR 0016 amendment — declare the four standardized approval primitives
  (`:list`, `:button`, `:typed_command`, `:link`), selection rule, adapter
  declaration requirement, and the per-adapter declaration table covering
  CLI / LiveView / Telegram / Email / Discord / Slack / WhatsApp / Signal /
  Matrix. Implementation lands as part of M3-M4 below; existing Telegram and
  email adapters retroactively declare their primitive subsets.
- M1: Discord plugin skeleton and simulated provider tests.
- M2: Slack plugin skeleton and simulated provider tests.
- M3: Identity mapping, group authorization, and threaded replies.
- M4: Confirmation callbacks rendered through each adapter's declared
  primitives (Discord `:button`; Slack `:button`; both fall back to
  `:typed_command` and `:list`). Existing Telegram + email adapters updated
  to declare their primitive subsets per the amendment.
- M5: Security evals (including `approval-primitive-honor` per adapter)
  and closeout docs.

## Documentation Deliverables

- Operator channel setup docs for Discord and Slack.
- Security-hardening notes for group/team channels.
- ADR 0016 amendment text.
- Developer notes for the channel approval-primitive contract.
- Agent context map update.

## Definition Of Done

v0.44 is done when:

- Discord and Slack adapters simulate inbound messages, render replies and
  approvals, preserve identity/thread mapping;
- ADR 0016 amendment is accepted with the four primitives and adapter
  declaration table;
- existing Telegram + email adapters declare their primitive subsets and
  pass the `approval-primitive-honor` eval;
- cross-channel spoofing, replay, group leakage, and callback-scope evals
  pass.
