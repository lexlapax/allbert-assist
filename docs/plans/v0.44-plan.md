# Allbert v0.44 Plan/Build Mode And Operator Workflow YAML

Request flow: `docs/plans/v0.44-request-flow.md`

ADR: `docs/adr/0041-plan-build-and-operator-workflow-yaml.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Expose Objective Runtime as a user-facing Plan/Build mode and allow operators
to author declarative workflow YAML that expands into objective steps.

## Expected Inputs

- Objective Runtime and `:delegate_agent` step kind.
- v0.40 MCP, v0.42 integrations, and v0.43 browser.
- Existing workspace Canvas and Approval Handoff.
- Proposed ADR 0041 accepted before implementation closeout.

## Operator-Visible Win

Operators can preview multi-step work, inspect required resources and
confirmation points, edit or cancel the plan, then run it step by step.

## Planned Interfaces

- Workspace Plan/Build destination.
- Declarative workflow YAML schema.
- Workflow files live under `<ALLBERT_HOME>/workflows/<workflow-id>.yaml`
  per the ADR 0041 post-v0.37 amendment. `<workflow-id>` is a deterministic
  operator-chosen slug (`^[a-z0-9][a-z0-9_-]*$`).
- Discovery is on-demand: the runtime reads the file only when an operator
  references the workflow id. No autoload, no scanning, no compilation.
- Channel-visible plan summaries for CLI, Telegram, email, and LiveView using
  the renderers available at v0.44.
- Background objective progress rendering.

## Forward Pin

Discord and Slack are intentionally later in the arc. When Channel Pack 1 lands
in v0.49, those adapters inherit Plan/Build plan summaries and progress
rendering through the channel approval-primitive contract. v0.44 does not need
the v0.49 ADR 0016 amendment to ship Plan/Build for existing surfaces.

## Security Posture

Workflow YAML produces objective steps only. It is not an execution engine and
does not grant action authority. Every step still resolves through
`Actions.Runner.run/3`, Security Central, confirmations, traces, and audits.

Workflow files are inert data. The runtime never executes them. Schema
validation rejects unknown keys closed. Workflows under
`<ALLBERT_HOME>/workflows/` are operator-readable Markdown-adjacent artifacts;
v0.51 export/import preserves them.

## Non-Goals

- No autonomous approval of plans.
- No workflow YAML script execution.
- No private objective store outside `AllbertAssist.Objectives`.
- No lowering of action-level confirmations.

## Milestones

- M1: ADR 0041 and YAML schema.
- M2: Plan/Build workspace destination.
- M3: Objective step generation, preview, and edit/cancel UX.
- M4: Channel plan summaries and background progress.
- M5: Security evals, docs, and closeout.

## Documentation Deliverables

- Operator Plan/Build guide.
- Developer workflow YAML schema notes.
- Runtime-boundary and agent-context updates.

## Definition Of Done

v0.44 is done when an operator can author or request a plan, preview/edit it,
approve execution, see progress from workspace and existing supported surfaces,
and prove every step still obeys normal action/security boundaries.
