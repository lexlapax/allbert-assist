## Purpose

Expose Allbert as a public **agent surface** across three protocols so external
agent runtimes can call Allbert's registered actions as tools and read enabled
memory namespaces as resources:

1. **MCP server** — Claude Desktop, Cursor, ChatGPT MCP, JetBrains MCP, other
   MCP agents. Symmetric to the v0.40 MCP client work. (The original v0.52b scope.)
2. **OpenAI-compatible HTTP API** — `POST /v1/chat/completions` (+ optional
   `/v1/models`, SSE streaming) so the large OpenAI-API client ecosystem can
   reach Allbert.
3. **ACP server** — Agent Client Protocol (editor agents such as Zed) over stdio
   JSON-RPC.

The request path is a thin adapter over Allbert's existing boundaries:
conversational requests enter through `Runtime.submit_user_input/1`; exposed
tool/effectful calls execute through `Actions.Runner.run/3`; none gains more
authority than a local workspace user; none can self-approve confirmations. The
**return path for confirmation-gated calls is genuinely new substrate** — a
poll-by-id result readback (§"Result Readback"), since nothing today returns a
post-approval result to a stateless external client. The public **AG-UI/A2UI
bridge and MCP Apps iframe UI stay parked** post-1.0 (ADR 0044).

## Readiness Sweep

Status: complete; implementation-ready for M1.

- **Substrate fit.** Public surfaces are adapters over the runtime, not a new
  runtime. Every effectful call still resolves through `Actions.Runner.run/3` +
  Security Central + Resource Access + confirmations + traces + audits. The one
  new primitive set (inbound auth, rate-limit, result readback) is recorded in
  ADR 0055.
- **Roadmap fit.** v0.51 follows v0.50b and precedes v0.52 Channel Pack 1. It is
  the first public-interop surface; channels (v0.52/v0.53) and the v0.57 RC
  sweep build on it.
- **Past-plan fit.** Reuses, in place: the v0.40 MCP **client** modules
  (`AllbertAssist.Mcp.Codec` Hermes message wrapper, `Mcp.Transport` stdio
  `Port`/HTTP-SSE, `Mcp.Client` lifecycle/`@protocol_version`, trust/audit);
  `hermes_mcp` 0.14.1 **server** support (`use Hermes.Server`,
  `Frame.register_tool/3`, the `handle_tool_call/3` callback,
  `Hermes.Server.Registry`, `Hermes.Server.Transport.StreamableHTTP.Plug`);
  `Actions.Registry` + `Capability` (`exposure: :agent`) as the tool-schema
  source; `Memory.Namespaces` (app vs system) for resources; the confirmation
  store + resumable-action resume path; `Runtime.Redactor`; the
  `Voice.LocalRuntime.Auth` token pattern (`Plug.Crypto.secure_compare`).
- **New-build fit (honest).** There is **no authenticated HTTP API today** (the
  web `:api` pipeline is unused / `/api` scope commented out), **no inbound
  rate-limiter** anywhere, and **no post-approval result-readback for a stateless

- **Dependency fit.** **No v0.52 dependency.** v0.51 renders confirmations via
  the existing `Intent.ApprovalHandoff` (a text/format + routing helper that
  exists today) and the workspace approval path — NOT the net-new v0.52
  `Approval.Handoff.render/2` channel-primitive module. Ordering is safe.
- **ADR fit.** ADR 0044 amended (3 surfaces, exposure). ADR 0055 new (inbound
  trust tier + permission class + auth + readback). ADR 0038 reused inbound.
  ADR 0025 CSP reconciled for APIs in ADR 0055.

|---|---|---|---|
| 1 | Surface scope | All three surfaces ship in v0.51 (MCP server + OpenAI-compatible API + ACP server). AG-UI/A2UI + MCP Apps iframe parked post-1.0. | A larger single release, but the full public-interop story ships together; splitting would defer the OpenAI ecosystem reach. |
| 2 | Adapter posture | Conversational requests → `Runtime.submit_user_input/1`; exposed tool/effectful calls → `Actions.Runner.run/3`. No surface gets more authority than a local workspace user. | Bypassing the runner/runtime boundary would create a parallel authority path. |
| 3 | Inbound trust tier | New `:public_surface_call_inbound` permission, floor `:needs_confirmation`; per-client Settings-Central tokens (HTTP surfaces); inbound rate-limit; API secure-headers — all decided in **ADR 0055**. | Plan-only prose would break the "every permission class is ADR-introduced" convention and leave auth under-specified. |
| 4 | Result readback | **Poll-by-id**: a new read-only `:agent`-exposable action returns a confirmation/call result by id (`pending`/`approved+result`/`denied`), **client-scoped**, **never before operator approval**. | Without it, an external client can never retrieve the result of a confirmation-gated call; re-using `approve_confirmation` would allow self-approval. |
| 5 | Confirmation authority | External clients **cannot self-approve**; confirmation-decision actions stay `:internal`/non-exposable; ApprovalHandoff stays operator-owned on the workspace/origin channel. | Exposing approve/deny would hand external clients self-approval. |
| 6 | OpenAI mapping | `/v1/chat/completions`: **flatten** the `messages` array into one `Runtime.submit_user_input/1` turn; **reject** client-supplied `tools`/`tool_calls`/`tool_choice` (tool selection stays operator-allowlisted + internal). | A 1:1 `tools` passthrough would let clients drive tool selection, conflicting with the exposure model. |
| 7 | Exposure | Default-off surfaces; default-empty tool/namespace allowlists; only `exposure: :agent` actions + app namespaces are eligible; non-exposable internals are config errors. | Default-on or auto-enabled exposure widens the attack surface. |
| 8 | hermes_mcp boundary | `hermes_mcp` provides MCP protocol/lifecycle/framing only; Allbert owns ingress auth + authority. Native `Req`/`Plug` fallback decided at M1 (mirrors ADR 0038 §6). | Letting Hermes own ingress would bypass Allbert auth/CSP/rate-limit. |

## Exposure Rules (all three surfaces)

- A registered action becomes an exposable tool only when `exposure == :agent`
  AND explicitly listed in the surface's `tools_enabled` (default empty).
- An app memory namespace becomes a resource only when explicitly listed in
  `memory_namespaces_enabled` (default empty).
- Tool descriptions, parameter schemas, and resource summaries are bounded and
  redacted before exposure.
- **Not exposable** (config error if attempted, across all surfaces): settings,
  secrets, raw runtime signals, raw trace bodies, confirmation storage,
  plugin/registry internals, `:internal`-exposure actions, system memory
  namespaces. **Confirmation-decision actions (`approve_confirmation`/
  `deny_confirmation`) are never exposable** — the self-approval-denial point.
  The read-only result-readback action (Locked Decision 4) is the *only* new
  `:agent`-exposable confirmation-related surface, and it is client-scoped.

## Authority Rules (all three surfaces)

- Inbound requests **never** receive more authority than a local workspace user.
- External clients **cannot approve their own confirmations**. A confirmation-
  needing call returns `:confirmation_pending` (MCP/ACP) or an OpenAI-shaped
  equivalent carrying the confirmation id; the operator approves through their
  usual surface; the client retrieves the result via the poll-by-id readback.
- ACP `session/request_permission` maps to a confirmation but is **advisory only
  and never authorizes execution** (ADR 0038 / ADR 0055).
- All effectful calls route through `Actions.Runner.run/3`, Security Central,
  Resource Access, confirmations, traces, and audits.
- HTTP-bearing surfaces (MCP streamable-HTTP + OpenAI API) require a per-client
  token (ADR 0055), behind an API secure-header policy + inbound rate-limit +
  redaction. stdio surfaces (MCP stdio + ACP) run under ADR 0009 bounds.

## Result Readback

Confirmation-gated effectful calls need a return path for stateless clients.
Today: an action returns `:needs_confirmation`, the operator approves out-of-band
(`approve_confirmation` re-runs a resumable action and returns the result to the
*operator*), and the result persists only in the confirmation record — the read-
by-id actions are `:internal`, so an external client can never retrieve it. v0.51
adds:

- A new **read-only, `:agent`-exposable** action (e.g. `get_public_call_result`)
  keyed by the confirmation/call id, returning `pending | approved-with-result |
  denied`. The result is the redacted action result (via `Runtime.Redactor`), not
  confirmation-store internals.
- **Client-scoped**: a client may read only confirmations/calls it originated; a
  per-client call→confirmation ownership record backs this (no cross-client
  leak).
- **Never before approval**: returns `pending` until the operator resolves the
  confirmation.
- Each surface shapes this idiomatically: MCP as a tool / resource read; OpenAI
  as a follow-up request carrying the id; ACP as a `session/*` read.

ADR 0055 records the trust decision; this section is the design seam.

## Non-Goals
- No public AG-UI/A2UI bridge or MCP Apps iframe UI (parked post-1.0).
- No hosted multi-user authorization.
- No external-client self-approval of confirmations.
- No client-supplied tool selection (OpenAI `tools` rejected).
- No exposure of settings, secrets, raw signals, raw trace bodies, or
  confirmation storage; no auto-enablement from request payloads.

## Definition Of Done

v0.51 is done when:
- an MCP client, an OpenAI-API client, and an ACP editor can each reach Allbert
  in a disposable Allbert Home, seeing only operator-enabled tools and memory
  namespaces;
- a confirmation-gated call on each surface returns `:confirmation_pending` (or
  its OpenAI-shaped equivalent), the operator approves through their workspace,
  and the client retrieves the result via the **client-scoped poll-by-id
  readback** (never before approval, never another client's);
- conversational prompts enter through `Runtime.submit_user_input/1`; every
  effectful call routes through `Actions.Runner.run/3` + Security Central +
  confirmations; non-exposable internals are config errors; no surface can
  self-approve;
- HTTP surfaces require a per-client token (rotatable/revocable) behind the API
  secure-header policy + inbound rate-limit; the `:public_surface_call_inbound`
  permission + floor are in `Security.Policy`;
- the `:v051` eval set + `release.v051` + the full release gate pass; AG-UI/A2UI
  + MCP Apps remain internal/parked;
- roadmap/vision/agent-context-map/ADR 0044/0055/CHANGELOG reflect the shipped
  state; the v0.57 export/import sweep round-trips the three surfaces' settings
  without exposing secrets.
