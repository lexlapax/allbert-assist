## Main User/Operator Flows

- A technical prosumer installs Allbert from a packaged binary — `brew install
  allbert` or a one-line curl — on a machine with no Elixir/OTP toolchain.
- Operator runs `allbert serve` to start the runtime + web workspace + channels and
  installs the launchd/systemd/Scheduled-Task service.
- Operator watches the three-leg (runtime/web/channels) health check go green.
- Operator drives day-to-day operation through `allbert <group> <command>`
  (`ask|chat|tui|serve|admin <area>|gen`) and the mix-free TUI console, never
  touching raw `mix`.
- Operator stores provider credentials in the OS vault and migrates existing
  plaintext credentials with an explicit redacted step.
- Developer/CI continues to use `mix` tasks from a checkout.
- Maintainer runs `mix allbert.test release.v062` for the deterministic gate.

## Data Flow

The packaged binary boots the same supervision tree the checkout boots; `allbert
<group> <command>` resolves each operator command to the same registered action or
read-report interface the equivalent `mix` task used, routing through
`Actions.Runner.run/3` and the runtime/action/settings spine (ADR 0073). `allbert
serve` supervises the runtime, Phoenix web workspace, and channel adapters under one
managed process and reports a health snapshot. The completed TUI console resolves the
remaining admin-inspection reads as `exposure: :internal` read-only actions through
the shared read-report interface (ADR 0070). Credential reads resolve a settings-held
reference to an OS-vault value injected at launch; Settings Central holds references,
the OS vault holds secret values, and no raw secret crosses into logs or inspection
output.

## Authority Invariants

No new capability is introduced. Packaging changes how Allbert is installed and
invoked, not what any surface may do — the binary, dispatcher, daemon, TUI console,
and vault all route through the ADR 0073 spine under Security Central, confirmations,
and surface policy (ADR 0006). The CLI dispatcher exposes no command the equivalent
`mix` task did not. The TUI convergence adds read-only internal actions only — no
mutation, no `:operator` exposure, excluded from the intent-router candidate set
(ADR 0060 / ADR 0070). The OS vault never logs or renders raw secret values. Allbert
Home layout settles here for the v1.0 freeze.

  confirm it boots the same supervision tree.
- Run `allbert <group> <command>` across `ask|chat|tui|serve|admin <area>|gen`;
  confirm grouped help and that each operator command routes through the spine and
  exposes no command the equivalent `mix` task did not.
- Confirm developer/CI commands stay `mix`-only and are absent from the binary.
- Run `allbert serve`, install the launchd/systemd/Scheduled-Task service, and watch
  the three-leg (runtime/web/channels) health check succeed; uninstall cleanly.
- Drive the remaining admin-inspection reads through the mix-free TUI console;
  confirm they are read-only internal actions, mutate nothing, grant nothing, are
  redacted, and are not intent-router candidates.
- Store a credential in the OS vault, migrate an existing plaintext credential, and
  scan logs/inspection output for any raw secret.
- Confirm the Allbert Home layout is settled and the v0.61 → v0.62 migration is
  documented.
- Confirm docs and gates are clean.

## Operator Validation

Status: planned. Commands are illustrative until the milestone lands; verify and
replace with observed output at closeout.

```sh
# Install path (no Elixir/OTP toolchain on the host).
brew install allbert            # or: curl -fsSL https://.../install.sh | sh
allbert --help                  # grouped dispatcher help
allbert serve --health          # start runtime+web+channels; watch health go green
allbert admin status            # operator read via the spine, not raw mix
rg -i 'token|secret|api_key' "$ALLBERT_HOME" "$HOME/Library/Logs/allbert" || true
```

Expected behavior:

- the packaged binary installs and runs with no Elixir/OTP toolchain and boots the
  same supervision tree.
- `allbert <group> <command>` exposes the grouped operator surface, routes through
  the spine, and adds no command/authority over the equivalent `mix` tasks; dev/CI
  commands remain `mix`-only.
- `allbert serve` runs the managed daemon and the three-leg health check succeeds.
- the mix-free TUI console covers day-to-day operation with read-only internal reads
  only; credentials live in the OS vault with no raw secret in logs or output.
