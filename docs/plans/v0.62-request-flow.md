## Main User/Operator Flows

- New operator starts Allbert with the packaged entry points; the web wizard
  auto-opens on first run.
- Operator picks QuickStart or Advanced.
- Operator enters a masked provider key, stores it through the OS vault, sees
  inline doctor verification, chooses local or hosted provider/model, and can
  switch providers without editing config.
- Operator selects a user-category profile, reviews exactly what it seeds, and
  confirms.
- QuickStart reaches a working first chat; Advanced may continue into optional
  channel/integration setup.
- Operator sees health/trust-spine confirmation.
- Headless operator runs the same flow through the interactive terminal wizard.

## Data Flow

The shared wizard state machine owns step sequence, validation, and transitions.
The web and terminal surfaces render it without forking logic. Provider/model
setup writes credential values to the OS vault through Settings Central
references and verifies them through existing doctor flows. Profile selection
reads repo-maintained declarative presets and, after explicit confirmation,
seeds Settings Central defaults plus suggested apps, channels, and intents.
Every effectful step remains behind the existing action/settings/security gates.

## Authority Invariants

No new capability is introduced. Profiles seed defaults only; they lower no
confirmation floor, grant no egress, and enable nothing effectful without normal
gates. Settings/profile writes flow only through Settings Central. Masked-key
entry never echoes or logs raw secrets; wizard diagnostics are redacted.

## Handles / CLI Surfaces

- Web auto-first-run wizard.
- Interactive terminal wizard through the packaged `allbert` CLI.
- Provider/model setup with masked OS-vault entry, inline doctor, and provider
  switch.
- Registered settings action to apply profile presets with review/confirm.
- Health/trust-spine confirmation surface.
- `mix allbert.test release.v062` deterministic gate.

## Smoke Checklist

- Launch a disposable home through the packaged entry points; confirm the web
  wizard auto-opens.
- Run QuickStart to a working first chat.
- Enter a masked provider key; confirm OS-vault storage, inline doctor
  verification, and no raw secret echo/log.
- Switch provider without editing config.
- Select each shipped profile and confirm the review step shows exactly what is
  seeded and writes nothing before confirm.
- Run the same flow through the terminal wizard.
- Confirm the trust spine is surfaced.
- Confirm docs and gates are clean.
