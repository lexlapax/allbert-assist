## Pre-Flight

Use a disposable Allbert Home and open `/agent`:

```sh
export SMOKE_HOME=$(mktemp -d /tmp/allbert-v031-smoke.XXXXXX)
ALLBERT_HOME="$SMOKE_HOME" mix ecto.migrate
ALLBERT_HOME="$SMOKE_HOME" mix phx.server
```

v0.31 assumes the v0.26 workspace shell, v0.27 StockSage renderers, v0.28
security evals, and v0.30 canvas emission exist.

## Flow 1: Workspace Shell Renders Three Zones

Operator opens `/agent`.

Expected behavior:

1. The shell renders a left rail (new-chat, recent threads, app launcher,
   context panels), a center chat thread, and a right canvas.
2. The thread list is the operator's recent threads (existing
   `recent_threads/1`); selecting one switches the active thread.
3. The app launcher lists registered apps (Allbert, StockSage); selecting one
   sets `active_app` and reveals that app's panels.
4. On a narrow viewport the rail collapses to a drawer; chat/canvas tab
   behavior from v0.26 is preserved.

Failure behavior:

- An app with no panels still appears in the launcher; its zone renders an
  accessible empty placeholder.

## Flow 2: App Contributes Panels Into The Workspace

Operator selects StockSage in the launcher.

Expected behavior:

1. StockSage dashboard, recent analyses, queue, and trends render as panels in
   the canvas/context zones — no navigation to `/stocksage/*`.
2. Panels are catalog-validated Surface nodes; no arbitrary HTML renders.
3. Panel data reflects the same StockSage records as durable canvas tiles and
   the analysis detail page.

Failure behavior:

- A panel that fails catalog/zone validation is dropped with a bounded
  diagnostic; the rest of the workspace renders.

## Flow 3: Dedicated Page Survives Only For Detail

Operator opens an analysis from a panel.

Expected behavior:

1. The analysis detail opens at `/apps/stocksage/analyses/:id` (deep-linkable,
   shareable, print-friendly).
2. No other StockSage route exists; `/stocksage/*` 404s or redirects to the new
   locations.
3. The detail page renders the same v0.27 cards as the panels and tiles.

## Flow 4: Operator Retints The UI With Tokens

Operator writes `<ALLBERT_HOME>/themes/midnight.yaml` and sets
`workspace.theme.active=midnight`.

Expected behavior:

1. `/theme/user.css` serves a `:root` token block reflecting the file.
2. The UI recolors on reload with no asset rebuild.
3. An invalid or missing token file falls back to defaults; the operator sees a
   bounded warning.

## Flow 5: Operator Adds A CSS Snippet

Operator drops `<ALLBERT_HOME>/themes/snippets/compact.css`, sets
`workspace.theme.snippets_enabled=true`, and enables the file.

Expected behavior:

1. The snippet is sanitized: `@import` is rejected, `url()`/`image-set()` are
   stripped, and the operator sees a warning naming what was removed.
2. The sanitized snippet serves last (outside Tailwind layers) and overrides
   base styling.
3. With snippets disabled (default), no snippet CSS is served.

Failure behavior:

- A snippet that is entirely unsafe serves as empty with a warning; the UI is
  never broken by snippet content.

## Flow 6: Operator Overrides Layout

Operator writes `<ALLBERT_HOME>/workspace/layout.yaml` to hide a zone and
reorder panels, and sets `workspace.layout.override_enabled=true`.

Expected behavior:

1. The workspace tree applies the override: zones reorder/hide, pinned panels
   move.
2. Unknown zones/atoms in the config are ignored per-key with a warning, not a
   crash.
3. With override disabled, the default layout renders.

## Flow 7: Injection Is Rejected

A fixture attempts to contribute a panel with unknown component ids, raw HTML,
secret-like props, a foreign app's action binding, or an unknown zone; a layout
config names a non-catalog atom; a snippet carries an exfiltration `url()`.

Expected behavior:

1. Catalog/zone validation rejects the panel; the sanitizer strips the snippet
   construct; layout validation drops the bad key.
2. Rejections are visible in trace/workspace diagnostics.
3. No arbitrary model- or operator-supplied UI executes; CSP blocks remote
   style/image fetches.

## DOM Ids And Events

Stable handles for automated tests and Chrome manual verification:

| Surface | Handle | Expected use |
|---|---|---|
| Left nav rail | `#workspace-nav-rail` | Three-zone shell rail |
| Thread list | `#workspace-thread-list` | Recent threads from `recent_threads/1` |
| App launcher | `#workspace-app-launcher` | Registered app entries (`:nav_apps`) |
| Zone | `#workspace-zone-<zone>` | Host-owned named zone wrapper |
| Panel | `#workspace-panel-<surface-id>` | Catalog-validated panel surface |
| User theme link | `link[href^="/theme/user.css"]` | Token stylesheet loaded after app.css |
| Snippet link | `link[href^="/theme/snippets/"]` | Sanitized snippet stylesheet |

## Smoke Checklist

- `/agent` renders the three-zone shell; rail lists threads + apps.
- StockSage dashboard/list/queue/trends render as panels; only detail has a
  route under `/apps/stocksage/...`.
- CoreApp domain cards render through zones.
- Token retint works with no rebuild; snippet sanitizer strips/rejects unsafe
  CSS with warnings; layout override applies and validates.
- CSP headers present; remote style/image fetches blocked.
- Injection attempts (panel/zone/layout/snippet) are rejected with diagnostics.
- Milestone closeout runs focused workspace/surface/theme tests, the v0.28 eval
  additions, and `mix precommit`.

## Milestone User Testing

### M1-M2 Zones And Panels

1. Confirm `/agent` zones render (empty placeholders before panels exist).
2. Add a fixture panel; confirm it renders in its zone and a bad panel is
   rejected.

### M3-M4 Shell And Domain Cards

1. Confirm the left rail lists recent threads and apps.
2. Confirm objective/job/confirmation cards render in zones.
3. Confirm a11y and mobile drawer behavior.

### M5-M6 Routes And StockSage Panels

1. Confirm `/apps/stocksage/analyses/:id` deep-links; old routes redirect/404.
2. Confirm StockSage dashboard/queue/trends render as workspace panels with no
   StockSage nav chrome.

### M7-M9 Theming

1. Token file retints with no rebuild.
2. Snippet sanitizer strips/rejects unsafe constructs with warnings.
3. Layout override reorders/hides zones and pins panels; invalid keys are
   ignored safely.
