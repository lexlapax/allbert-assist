## Pre-Flight

Use a disposable Allbert Home:

```sh
export SMOKE_HOME=$(mktemp -d /tmp/allbert-v026c-smoke.XXXXXX)
ALLBERT_HOME="$SMOKE_HOME" mix ecto.migrate
ALLBERT_HOME="$SMOKE_HOME" mix phx.server
```

Open `/agent` in one or two browser tabs. v0.26c assumes the `0.26.1`
workspace shell already loads, submits turns, renders approval ephemerals, and
uses the existing workspace canvas tables.

## Flow 1: Inspect A Tile

Operator opens `/agent`, submits a prompt that creates at least one tile, then
opens the tile inspector from the tile body or kebab menu.

Expected behavior:

1. The inspector opens without navigating away from `/agent`.
2. Focus moves into the inspector; Escape or close returns focus to the tile.
3. The inspector shows tile id, kind, status, user/thread context, provenance,
   trace link when present, and readable body content.
4. Copy affordances copy ids or payload snippets without exposing secrets.
5. No tile mutation happens unless the operator invokes an existing registered
   action such as remove/restore/pin.

Failure behavior:

- If body YAML is missing or corrupt, the inspector shows a bounded error and
  still displays SQLite metadata.
- If a trace link points to a missing trace, the trace affordance is disabled
  with explanatory text.

## Flow 2: Switch Threads

Operator clicks the AppBar thread chip.

Expected behavior:

1. A thread switcher dropdown opens with recent threads for the current user.
2. Selecting a thread navigates/reassigns the workspace to that thread.
3. "New thread" creates or selects a fresh thread through existing runtime
   boundaries and updates the URL/session state.
4. The canvas and ephemerals switch to the selected thread; no cross-thread
   tiles leak.

Failure behavior:

- If there are no previous threads, the menu still offers "New thread".
- If a selected thread is stale or belongs to another user, the workspace shows
  the existing not-found/fallback behavior and creates or returns to a valid
  thread.

## Flow 3: Verify Multi-Tab Sync

Operator opens the same `/agent` thread in two tabs.

Expected behavior:

1. A tile created or updated in tab 1 appears in tab 2.
2. An ephemeral opened in tab 1 appears in tab 2.
3. Dismissing an ephemeral in either tab dismisses it in the other.
4. The verification record includes screenshots or browser-smoke notes for the
   before/after states.

Failure behavior:

- If a tab disconnects, it catches up on reconnect using persisted canvas and
  ephemeral state.
- If PubSub delivery is delayed, the UI remains usable and converges after
  reload.

## Non-Flows

- No StockSage progress streaming.
- No new StockSage card rendering.
- No schema, catalog, security, settings, or signal changes.

## Smoke Checklist

- `/agent` loads with a disposable home.
- Tile inspector opens, closes, and preserves focus.
- Thread switcher lists recent threads and starts a new thread.
- Two tabs stay synchronized for tile and ephemeral state.
- `git diff --check` passes for docs-only work; implementation work runs the
  focused LiveView tests plus the normal warning gate.

## DOM Ids And Events (operator-facing handles)

- Inspect (tile kebab item): `#workspace-tile-inspect-<tile-id>`, event
  `open_tile_inspector`. Modal: `#workspace-tile-inspector` (role=dialog,
  FocusTrap); close: `close_tile_inspector` (button + Escape).
- Thread chip toggle: `#workspace-thread-switcher-toggle`, event
  `toggle_thread_switcher`. Menu: `#workspace-thread-switcher-menu` (role=menu).
  Recent item: `#workspace-thread-item-<thread-id>` (navigate). New thread:
  `#workspace-thread-new`, event `new_thread` (calls
  `Conversations.resolve_thread/1` with `new_thread: true`, then navigates to the
  returned `/agent?thread_id=<id>`). Copy id: `#workspace-thread-copy-id`.
