## Main User/Operator Flows

- Operator configures an MCP server with secret refs.
- Operator lists MCP tools and resources.
- Operator reads an MCP resource.
- Operator approves an MCP tool call.
- Operator inspects MCP trace/audit evidence.

## Data Flow

Settings Central resolves enabled MCP servers and secret refs. MCP discovery
returns descriptive tool/resource metadata. MCP calls become registered actions
with `mcp://` resources, Security Central decisions, confirmations, trace, and
audit records.

## Authority Invariants

MCP server schemas are never authority. MCP resource URIs are inert until a
registered action consumes them. Per-tool filters and confirmation policy can
tighten authority only. MCP calls cannot bypass Resource Access.

## Smoke Checklist

- Configure one fake/test MCP server.
- List tools/resources.
- Deny and approve one tool call.
- Confirm traces redact secrets and include MCP server/tool metadata.
- Confirm disabled or filtered tools cannot run.
