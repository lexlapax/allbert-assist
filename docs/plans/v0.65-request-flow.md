## Main User/Operator Flows

- Operator connects or confirms a local notes/files root.
- Operator asks Allbert to find or summarize a local note.
- Operator reads a note through the notes/files path.
- Operator confirms a proposed note write.
- Operator reviews a memory candidate before it becomes durable memory.
- Operator asks a later question that uses reviewed memory.
- Maintainer runs docs, warning, drift, leak-scan, and `release.v065` gates.

## Data Flow

No new authority data flow is introduced. Notes/files reads and writes continue
to use Resource Access file refs under configured roots. Note writes remain
confirmation-gated. Memory writes remain reviewable and never derive authority
from notes/files output, plugin metadata, or model output.

## Authority Invariants

Local file roots are explicit. Read/write operations stay scoped. `write_note`
requires confirmation. The notes/files memory namespace stays non-writable and
does not auto-promote. Reviewed agent memory remains user-controlled,
inspectable, and removable.

## Handles / CLI Surfaces

- Notes/files workspace destination and settings.
- `search_notes`, `read_note`, and `write_note`.
- Memory review and active-memory operator surfaces.
- Confirmation, trace, and audit views for note writes and memory decisions.
- `mix allbert.test release.v065` deterministic gate.

## Smoke Checklist

- Configure or confirm a notes root in a disposable Home.
- Search/read a sample note and verify the Resource Access refs are scoped.
- Confirm a note write and verify denial/approval behavior.
- Create or surface a memory candidate and keep/reject it through review.
- Ask a later question that can use the kept memory.
- Verify no notes content or secrets leak into evidence beyond explicit test
  fixtures.
- Run docs gate, warning gate, drift checks, and `release.v065`.
