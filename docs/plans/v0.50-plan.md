# Allbert v0.50 Hardening, Export/Import, And Final RC

Request flow: `docs/plans/v0.50-request-flow.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Prepare 1.0 after the final feature milestone. The milestone makes the
v0.39-v0.49 surface portable, auditable, documented, and boring to operate,
then gathers the final release-candidate evidence before the v1.0 contract
freeze.

## Expected Inputs

- v0.39 through v0.49 implemented and accepted.
- Current Allbert Home layout.
- Existing settings, memory, conversation, trace, audit, objective, plugin,
  marketplace, protocol, and self-improvement storage.

## Operator-Visible Win

An operator can export an Allbert Home, dry-run import on a second machine,
confirm behavior, inspect the final cross-surface security posture, and enter
the 1.0 contract freeze with evidence.

## Planned Interfaces

- Profile export/import dry run.
- Schema/version metadata summary.
- Rollback documentation.
- Final security review/status surfaces.
- Final RC acceptance checklist.

## Security Posture

No new authority. The milestone audits all newly added capability classes:
MCP, browser, channels, voice, vision, marketplace, API, ACP, MCP server,
AG-UI/A2UI bridge, and operator-supervised self-improvement. Secret migration
is explicit and redacted.

## Non-Goals

- No broad remote sync service.
- No hosted multi-user auth.
- No new capability surface.
- No expansion of v0.49 self-improvement behavior.
- No contract freeze until v1.0.

## Milestones

- M1: Export/import inventory and dry-run design.
- M2: Secret migration, rollback, and schema/version metadata docs.
- M3: Cross-surface security eval sweep, including v0.49 self-improvement.
- M4: Operator docs, performance checks, and CSP final reconciliation.
- M5: Release-candidate closeout gates for v1.0.

## Documentation Deliverables

- Operator export/import guide.
- 1.0 release-candidate checklist.
- Security-hardening, onboarding, self-improvement, and protocol-interop
  reconciliation.

## Definition Of Done

v0.50 is done when a disposable profile can be exported and re-imported with
documented behavior, all v1.0 acceptance-matrix gates have evidence, and no
known warning/security/doc drift remains.
