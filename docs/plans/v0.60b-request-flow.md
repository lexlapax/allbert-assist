## Operator / Reviewer Validation

Run only after `release.v060b` is green. Record observed output at closeout. This is
a **visual-design-review + rendering** validation, not a runtime capability
validation: there is no new authority to exercise.

**Two-phase timing of the choice.** The design choice is MADE and formalized **during
M5** — an initial design review scores the rendered directions, picks one, and writes
it into `visual-language-selected.md` + ADR 0079 so the gate can go green (the gate
requires a recorded choice and an Accepted-with-choice ADR). The post-gate **S4.5**
pass then RE-CONFIRMS that already-formalized choice against the same rendered
directions; it does not invent a new one. S5's `$CHOSEN` binding (edit below) asserts
the S4.5-recorded direction equals the one named in the selected doc + ADR 0079, so
the two phases cannot drift. The heart of the validation is **S4.5**, where the
operator views the ≥3 candidate hero-screen renderings side-by-side and **re-confirms
the chosen direction**. Stop at the first non-zero status, missing artifact,
un-accepted ADR, unresolved route, a11y failure, or an absent recorded choice.
