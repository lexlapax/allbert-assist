# Allbert v0.40 MCP Client Integration

Request flow: `docs/plans/v0.40-request-flow.md`

ADR: `docs/adr/0038-mcp-client-trust-tier.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Add MCP client support as the first ecosystem-scale capability expansion after
onboarding. MCP servers become configured resources and tools that Allbert can
inspect and call only through registered actions, Resource Access, Security
Central, confirmations, traces, and audits.

## Expected Inputs

- v0.39 provider/model control and onboarding.
- ADR 0047 provider/model doctor contract for reusable doctor return shape,
  redaction, and read-only diagnostic posture.
- ADR 0013 URI-first resource identity with reserved `mcp://`.
- Existing Resource Access posture and Approval Handoff.
- Existing Settings Central secrets.
- Proposed ADR 0038 accepted before implementation closeout.

## Operator-Visible Win

An operator can configure an MCP server, list its tools/resources, approve a
tool call, and inspect trace/audit evidence without treating MCP metadata as
authority.

## Planned Interfaces

- Settings namespace `mcp.servers.*`.
- Registered actions: `mcp_doctor_server`, `mcp_list_tools`,
  `mcp_call_tool`, `mcp_list_resources`, `mcp_read_resource`.
- Resource identity: `mcp://<server-id>/<encoded-uri>`.
- First tested server shapes: GitHub, Notion, Google Drive, Postgres.

## Security Posture

MCP schemas are descriptive only. MCP calls get new permission classes defined
by ADR 0038, with confirmation floors that settings cannot loosen. Server env
and headers use secret refs. `mcp_doctor_server` reuses ADR 0047's redacted
doctor envelope and fixed-catalog diagnostics for reachability/discovery, but
does not grant tool/resource authority. Tool/resource confusion, prompt
injection, server impersonation, and stdio process startup all require security
eval coverage.

## Non-Goals

- No MCP Apps iframe or third-party UI model.
- No `agent://` endpoint execution.
- No arbitrary auto-enablement of MCP tools.
- No MCP server authority over Allbert permissions.

## Milestones

- M1: ADR 0038 and Settings Central server schema.
- M2: MCP client transport, `mcp_doctor_server`, and read-only discovery
  actions.
- M3: Tool/resource call actions, confirmations, and traces.
- M4: First server-shape validation and operator docs.
- M5: Security evals and closeout.

## Documentation Deliverables

- Operator guide for MCP server configuration.
- Developer notes for MCP action/resource boundaries.
- Roadmap/future-features reconciliation if the supported server set changes.

## Definition Of Done

v0.40 is done when at least one MCP server can be configured in a disposable
Allbert Home, tools/resources can be listed, a tool can be called only through
Allbert policy, and MCP-specific security evals pass.
