## Main User/Operator Flows

- Operator configures WhatsApp, Signal, iMessage, or Matrix.
- External channel event maps to a local user/session identity.
- Allbert replies through the same channel adapter boundary as Telegram/email.
- Operator configures a local protocol client.
- External API/editor/agent client submits work through Allbert Runtime.
- Confirmation ownership remains local/operator-controlled.

## Data Flow

Channel events are deduped, mapped to local identity, normalized into runtime
input, and rendered back through provider-specific delivery rules. Protocol
requests authenticate, rate-limit, redact, trace, and enter the same runtime
and action boundaries used by workspace and channels.

## Authority Invariants

- Phone number, device session, Matrix id, or iMessage account identity is not
  permission authority.
- External protocol clients never receive more authority than workspace users.
- External clients cannot approve their own confirmations.
- Channel metadata and protocol metadata never grant action permissions.
- All effectful work still routes through registered actions, Security Central,
  confirmations, traces, and audits.
