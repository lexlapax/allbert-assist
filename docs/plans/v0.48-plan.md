# Allbert v0.48 Marketplace Lite And API/ACP/Protocol Interop

Request flow: `docs/plans/v0.48-request-flow.md`

ADRs: `docs/adr/0043-marketplace-lite-trust-tier.md`,
`docs/adr/0044-public-protocol-exposure.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Add reviewed discovery and external protocol access without adding arbitrary
remote code authority before 1.0.

## Expected Inputs

- v0.38 templated creation.
- v0.40 MCP client lessons.
- v0.43/v0.47 channel identity lessons.
- Proposed ADRs 0043 and 0044 accepted before implementation closeout.

## Operator-Visible Win

Operators can discover reviewed skills/templates and drive Allbert from local
API/editor/agent clients under the same policy boundary as the workspace.

## Planned Interfaces

- Marketplace-lite skill discovery/install.
- Reviewed-source plugin index metadata.
- Template catalog metadata in `workspace:create`.
- OpenAI-compatible local HTTP endpoint.
- ACP server mode.
- MCP server mode.
- Public AG-UI/A2UI bridge.

## Security Posture

Marketplace installs are disabled/untrusted by default and code-bearing remote
plugin install is excluded. Public protocol clients never receive more
authority than local workspace users. One auth/rate-limit/CSP/redaction policy
covers API, ACP, MCP server, and AG-UI/A2UI bridge.

## Non-Goals

- No arbitrary remote code-bearing plugin install.
- No marketplace theme/snippet distribution.
- No MCP Apps iframe execution.
- No external client self-approval of confirmations.

## Milestones

- M1: ADR 0043 marketplace trust tier and skill discovery path.
- M2: Reviewed plugin/template index metadata.
- M3: ADR 0044 public protocol policy.
- M4: API, ACP, MCP server, and AG-UI/A2UI bridge surfaces.
- M5: Security evals, CSP reconciliation, docs, and closeout.

## Documentation Deliverables

- Operator marketplace-lite guide.
- Developer protocol interop guide.
- Security-hardening updates for external clients and CSP.

## Definition Of Done

v0.48 is done when reviewed skills/templates can be discovered without granting
authority, public protocol clients can submit work only through Allbert policy,
and security evals cover marketplace and protocol exposure threats.
