# Allbert v0.64 Product RC & Integrated Validation

Request flow: `docs/plans/v0.64-request-flow.md`
ADR: no new authority ADR. This release consumes ADR 0046, ADR 0065, ADR 0069,
ADR 0074, ADR 0075, ADR 0076, ADR 0077, and ADR 0078 as accepted inputs and
proves their combined operator path before the v1.0 freeze.

Status: planned (pre-1.0 integrated product release candidate).

Source: post-v0.58 product-readiness review. v0.59 is engineering substrate;
v0.60 is the product-experience design; v0.61-v0.63 add product capability on
that design. v0.64 validates the v0.61-v0.63 product capability releases
(presentation overhaul, packaging, onboarding) integrated, on the v0.60
product-experience design. It exists because the first complete install -> serve
-> onboard -> first-use -> inspect -> export/import path should not be validated
for the first time during v1.0.

## Purpose

Run the integrated product release candidate after the presentation overhaul,
packaging, and onboarding/profile work has landed, on the v0.60
product-experience design. v0.64 adds no new *capability* — it carries the two
named v0.62 packaging-trust deferrals (M0.a) and otherwise proves that a
technical prosumer can install Allbert, start it, complete guided onboarding,
reach first value, inspect safety/trust state, use at least one real integration
path, export/import or upgrade a Home, and uninstall/rollback without hidden
state, warning, security, or docs drift.

## Expected Inputs

- v0.59 hardening substrate accepted: export/import dry-run portability,
  settings version contract/enforcement, central action param contracts, and
  security sweep.
- v0.60 product-experience design accepted: the entry-point / CLI UX and
  First-Model-Path decisions (ADR 0077, ADR 0078) the later releases implement.
- v0.61 presentation layer overhaul accepted.
- v0.62 packaged binary, unified CLI, daemon, mix-free TUI, and OS vault
  accepted.
- v0.63 guided onboarding and user-category profiles accepted.
- Version-specific gates `release.v059` through `release.v063` passing — this
  range now includes `release.v060`.

## Operator-Visible Win

An operator can follow one precise validation script from a clean machine/home:
install Allbert without Elixir/OTP, start `allbert serve`, complete QuickStart,
ask a first useful question, inspect confirmations/security/events, exercise one
tool/channel/browser path under policy, export/import or upgrade the Home, and
uninstall cleanly. Evidence is redacted, durable, and ready for the v1.0 freeze.

## Planned Interfaces

- Product RC operator validation checklist.
- Cross-platform install/onboard/serve evidence template.
- Browser-controlled web smoke for `/`, `/workspace`, `/jobs`, `/objectives`,
  onboarding, settings/model, and operator panels.
- CLI/TUI smoke for grouped command help, `allbert serve`, `allbert onboard`,
  `allbert tui`, and admin reads.
- Upgrade/export/import smoke from a v0.58/v0.59-style Home into the packaged
  v0.64 shape.
- Secret and evidence leak scan.
- `mix allbert.test release.v064` deterministic gate.

## Security Posture

No new capability and no new authority. This is validation only. Failures found
in this release must be fixed in the owning capability area rather than deferred
past 1.0. Any exception must be documented as non-blocking with an explicit
rationale and user-visible workaround.

## Non-Goals

- No feature work beyond the named M0.a packaging-trust intake (no new
  capability, permission class, or product surface).
- No native desktop GUI client.
- No new public protocol surface.
- No new profile category unless required to fix a v0.63 correctness issue.
- No contract freeze. v1.0 owns the freeze after this RC is green.

## Milestones

- M0.a: Packaging-trust intake (deferred by v0.62, recorded on both sides —
  v0.62 plan Locked Decisions 14/15 and ADR 0076 Distribution Trust):
  Developer-ID signing + notarization decision and implementation (Apple
  Developer account, hardened-runtime `codesign` + `notarytool` in the release
  CI, the self-extraction × hardened-runtime interaction test the v0.62 spike
  recorded), and upgrade rollback/restore automation (restore from the v0.62
  backup-before-migrate artifact; down-migration or restore-based rollback;
  delta-upgrade yes/no decision). If either is consciously re-parked post-1.0,
  that decision is recorded in ADR 0076 and the v1.0 plan — it does not drop
  silently.
- M1: Product RC evidence harness and disposable-home validation script.
- M2: Cross-platform install/serve/onboard smoke: macOS, Linux, Windows/WSL2
  where supported by the packaged binary.
- M3: Browser-controlled web smoke across landing, workspace, jobs, objectives,
  onboarding, settings/model, and operator panels.
- M4: CLI/TUI smoke across grouped help, serve health, terminal onboarding, TUI
  admin reads, and operator/dev command split.
- M5: Integration smoke for one real policy-bounded path: MCP, browser research,
  or a configured channel, with explicit secret redaction.
- M5.1: Cross-surface security eval delta-sweep over the surfaces added since the
  v0.59 M4 sweep — landing/`/` (v0.61), the packaged binary / grouped CLI /
  daemon / OS secret-vault (v0.62), and the onboarding wizard / masked-credential
  entry (v0.63) — reusing the v0.59 M4 sweep methodology, not only the leak scan.
  The secret-vault and credential-entry surfaces are authority-adjacent and must
  be swept, not just leak-scanned, before the v1.0 freeze.
- M6: Upgrade/export/import/uninstall validation and Allbert Home layout
  reconciliation.
- M7: Drift sweep, docs closeout, `release.v064`, docs gate, warning gate, and
  v1.0 handoff.

## Definition Of Done

v0.64 is done when:

- the complete clean-machine/home install -> serve -> onboard -> first chat ->
  inspect -> integration smoke -> export/import or upgrade -> uninstall path is
  validated with redacted evidence;
- browser-controlled web smoke covers `/`, `/workspace`, `/jobs`,
  `/objectives`, onboarding, settings/model, and operator panels with no console
  errors or layout regressions;
- grouped CLI and TUI operator flows work without raw `mix`;
- no raw secret appears in logs, output, evidence, docs, screenshots, or release
  artifacts;
- `mix allbert.test release.v064`, docs, warning, and drift checks pass;
- all v1.0 acceptance-matrix inputs are current and linked;
- no known warning/security/doc/product-readiness drift remains.
