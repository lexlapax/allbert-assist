## M1 Locked Decisions

Each decision below is recorded in `v0.43-plan.md` §"M1 Locked Decisions"
with a one-paragraph rationale. These are chosen values, not open defaults.

1. Browser driver — Playwright Chromium through a reviewed local bridge.
2. OS support — macOS + Linux only at v0.43; Windows/WSL2 parked.
3. Headless vs headed — headless only.
4. Profile persistence — ephemeral browser contexts only; cookies,
   localStorage, and IndexedDB are discarded on session close.
5. Routing browser vs HTTP — keep `external_network_request` (v0.10) and
   v0.11 inert `summarize_url`/`inspect_document`; browser is the graduated
   path when extraction needs DOM/JS, screenshots/rendering, PDF extraction,
   an allowlisted browser domain, or the operator asks (predicate in plan
   §"Browser-Fetch Vs HTTP-Fetch Routing").
6. JavaScript default — JS enabled.

## Main User/Operator Flows

- Operator enables browser (`browser.enabled true`) and runs
  `mix allbert.browser doctor` to confirm the driver is verified.
- Operator asks Allbert to research a URL or topic; the intent agent picks
  the browser path per the routing predicate.
- Allbert requests confirmation for a session start (one-time per session)
  and for a navigation scope (per-domain, with a "remember for <host>"
  affordance).
- Browser plugin navigates, extracts (HTML/markdown/text/PDF), screenshots,
  or reads bounded documents through registered actions; subresource
  requests pass through the plugin's `NetworkPolicy`.
- Operator reviews extracted evidence (text + screenshot link) in the
  workspace browser results panel and inspects redacted trace/audit
  metadata.
- Subsequent navigation within the same host skips confirmation
  (remembered grant applies); a new host triggers a fresh confirmation;
  a cross-domain redirect pauses for re-confirmation rather than silently
  following.
- Operator can close a session through `browser_close_session`; cache
  retention is enforced by a paused-by-default sweep job.

## Intent Phrase Corpus

Source of truth in `v0.43-plan.md` §"Planned Interfaces" Intent phrase
corpus. M4 implements one routing test per pattern; M5's
`release.v043` gate covers each pattern as a step.

| Pattern | Routes to |
|---|---|
| `research <URL>` | `browser_navigate` + `browser_extract` (text) |
| `research <URL> and summarize` | `browser_navigate` + `browser_extract` + handoff to summarizer |
| `summarize the page at <URL>` | `browser_navigate` + `browser_extract` |
| `screenshot <URL>` | `browser_navigate` + `browser_screenshot` |
| `what does <URL> look like` | `browser_navigate` + `browser_screenshot` |
| `render <URL>` | `browser_navigate` + `browser_screenshot` |
| `extract <text\|html\|markdown\|pdf> from <URL>` | `browser_navigate` + `browser_extract` (specified format) |

Patterns match case-insensitively; `<URL>` is the rightmost recognizable
URL token; `<format>` accepts the four extractor formats. The corpus is
M4-locked — additions or edits require an explicit re-decision in this
flow and in the plan.

## Data Flow

Browser requests flow through `Actions.Runner.run/3`:

1. Intent layer (v0.11 + v0.33 descriptors) proposes one of the browser
   actions; the M1-locked predicate in `v0.43-plan.md` §"Browser-Fetch Vs
   HTTP-Fetch Routing" picks browser vs the v0.10 HTTP path. M4
   implements one routing test per predicate row; drift between table and
   implementation fails the test.
2. `browser_navigate` looks up a remembered per-domain grant in
   `AllbertAssist.Resources.Grants` (operation class
   `:browser_navigate`, `:url_prefix` scope on `https://<host>/`,
   permission `:browser_navigate`).
3. Without a grant, Security Central evaluates `:browser_navigate`, the
   action creates an `AllbertAssist.Confirmations` pending with
   `ResourceMetadata` showing the redacted target URL, per-domain scope,
   and "remember for <host>" affordance; the v0.07 confirmation flow
   pauses the action.
4. On approval (and a `remember=true` choice), a remembered grant is
   written and the action resumes.
5. The action invokes `AllbertBrowser.Session.Port`. The driver pre-flights
   the URL through the v0.43 browser navigation preflight helper, which reuses
   `External.HttpPolicy`'s existing SSRF/private-network posture; the driver's
   request-interception hook enforces `AllbertBrowser.NetworkPolicy` on
   subresources and redirects.
6. The page loads; `browser_extract` reads bounded text per
   `browser.extraction.*` caps; `browser_screenshot` writes a redacted
   bitmap to `<ALLBERT_HOME>/cache/browser/<session_id>/`.
7. The workspace browser results panel reads the cache reference and
   renders text + screenshot link via the v0.27 SurfaceProvider contract;
   the panel calls only registered actions, never the driver.
8. Trace/audit records carry the action name, redacted URL, operation
   class, byte/page counts, success/failure, and confirmation id;
   `AllbertAssist.Security.Redactor` scrubs cookies, Authorization, full
   URLs with userinfo, and sensitive query parameter values; raw page
   content never lands in `memory/traces/`.

A redirect chain that leaves the granted domain pauses (denied at the
driver-interception layer) and surfaces a re-confirmation against the new
host.

## Authority Invariants

- **Page content is not authority.** Extracted HTML/text/PDF/comments
  never alter agent behavior; they are evidence only (ADR 0038 §2
  precedent applied here).
- **Browser plugin process ownership is not core authority.** The session
  GenServer lives under `AllbertBrowser.Supervisor`; core spawns no
  browser. A driver crash takes the session, not core.
- **Each operation has a separate grant.** A `:browser_navigate` grant
  does not authorize `:browser_form_fill`, `:browser_download`, or
  `:browser_interact`. A `:summarize_url` or `:inspect_document` grant
  does not authorize `:browser_navigate`. A `:browser_extract` grant
  authorizes only further read of the already-loaded page within the
  same session and same domain.
- **`browser://session/<id>` is identity, not grant authority.**
  Remembered grants are stored against the navigated target URL
  (`https://<host>/` `:url_prefix` scope), not the session URI.
- **Default-deny on credential-bearing operations.** Form fill, download,
  and any future authenticated operation are denied by default; opt-in
  raises the floor only to `:needs_confirmation`.
- **Browser content does not auto-promote to memory.** Manual promotion
  uses existing memory actions with extracted text as input; no new
  memory namespace at v0.43.
- **Doctor gates session start.** `browser_session_start` refuses to
  allocate when `browser_doctor` reports an unverified driver or missing
  dependency.
- **Network policy applies at two layers.** Top-level navigation passes
  through the browser navigation preflight helper that reuses
  `External.HttpPolicy` checks; subresources and redirects pass through
  `AllbertBrowser.NetworkPolicy`. Both deny private networks, loopback,
  link-local, and cloud metadata endpoints.

## Channel Approval-Primitive Forward Pin

## Smoke Checklist

- Confirm `browser.enabled` defaults to `false`; `headless` is read-only;
  `screenshot.redact_credential_inputs` is read-only.
- Run `mix allbert.browser doctor`; verify the redacted ADR 0047 envelope.
- Approve a session start and a research navigation; deny once first to
  confirm fail-closed.
- Extract text from HTML / markdown / plain text / PDF fixtures and
  confirm byte/page caps are enforced.
- Capture and render one screenshot; confirm credential-input nodes are
  redacted before encoding.
- Trigger a cross-domain redirect; confirm the operation pauses for
  re-confirmation rather than silently following.
- Point a navigation at a private IP / loopback / metadata host; confirm
  SSRF denial.
- Open the workspace browser results panel; confirm the panel renders
  text + screenshot link without calling the driver directly.
- Confirm form-fill and download deny by default and require explicit
  opt-in plus confirmation when tightened.
- Confirm `BrowserExtract` is bounded; oversized HTML truncates with a
  redacted diagnostic; malformed PDF fails closed.
- Confirm cross-operation grant reuse is denied (a `:browser_extract`
  grant does not authorize `:browser_navigate` to a new path/domain).
- Confirm session isolation: two sessions do not share cookies or local
  storage; closing a session discards both.
- Confirm `cache/browser/<session_id>/` files are swept after retention
  when the sweep job is resumed.
- Confirm traces redact cookies, Authorization, credential query values,
  and full URLs with userinfo; raw page content is in cache, not in
  traces.

## Operator Validation

Status: planned. The per-milestone blocks below are the source list for
`mix allbert.test release.v043` (M5 / v0.42 R5 lesson): each block
becomes a deterministic step in that gate at M5. The plan also requires
each block to run end-to-end on a disposable `ALLBERT_HOME` without edits
as a per-milestone acceptance criterion (v0.42 R4 lesson). Manual
reproduction remains as debugging when an operator needs to step through
a single phase; it is no longer the release-validation primitive.

### M1 — substrate

```sh
export ALLBERT_HOME=/tmp/allbert-v043-m1-home
export DATABASE_PATH=/tmp/allbert-v043-m1.db
rm -rf "$ALLBERT_HOME"
rm -f "$DATABASE_PATH"
mix do ecto.create --quiet + ecto.migrate --quiet + run -e '
  IO.inspect(AllbertAssist.Settings.get("browser.enabled"));
  IO.inspect(AllbertAssist.Settings.put("browser.session.headless", false, %{actor: "local", channel: :test}));
  IO.inspect(Enum.member?(AllbertAssist.Security.Policy.permission_classes(), :browser_form_fill));
  IO.inspect(AllbertAssist.Security.Policy.safety_floor(:browser_form_fill));
  IO.inspect(AllbertAssist.Resources.OperationClass.operation_classes() |> Enum.filter(&(&1 |> Atom.to_string() |> String.starts_with?("browser_"))))
'
```

### M2 — navigation, extraction, screenshot

```sh
export ALLBERT_HOME=/tmp/allbert-v043-m2-home
export DATABASE_PATH=/tmp/allbert-v043-m2.db
rm -rf "$ALLBERT_HOME"
rm -f "$DATABASE_PATH"
mix do ecto.create --quiet + ecto.migrate --quiet + allbert.settings set browser.enabled true
mix allbert.browser doctor
mix run -e '
  context = %{actor: "local", channel: :cli, confirmation: %{approved?: true}}
  {:ok, started} = AllbertAssist.Actions.Runner.run("browser_start_session", %{}, context)
  IO.inspect(started.session_id, label: "session_id")
  {:ok, nav} = AllbertAssist.Actions.Runner.run("browser_navigate", %{session_id: started.session_id, url: "https://example.com"}, context)
  IO.inspect(nav.status, label: "navigate")
  {:ok, extracted} = AllbertAssist.Actions.Runner.run("browser_extract", %{session_id: started.session_id, format: "text"}, %{actor: "local", channel: :cli})
  IO.inspect(extracted.status, label: "extract")
  {:ok, shot} = AllbertAssist.Actions.Runner.run("browser_screenshot", %{session_id: started.session_id}, %{actor: "local", channel: :cli})
  IO.inspect(shot.status, label: "screenshot")
'
rg -i 'cookie|authorization|session=' "$ALLBERT_HOME/memory/traces" || true
```

### M5 — deterministic release gate (primary)

```sh
mix allbert.test release.v043
# evidence: <ALLBERT_HOME>/release_evidence/v043/release-v043-<ts>.json
```

The gate runs against the stub driver, with no external network. It
exercises the full v0.43 surface (doctor live-verify, denied + approved
session start, denied + approved + remembered navigation, per-domain
grant lookup, cross-domain redirect re-confirmation, credential-URL
preflight rejection, subresource policy denial, HTML/markdown/text/PDF
extraction with byte + page caps, malformed-PDF fail-closed, screenshot
redaction, cache sweep lifecycle, intent phrase routing, secret grep
on settings/traces) and writes a reproducible JSON evidence artifact.

### M5 — manual closeout (debugging fallback)

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v043.XXXXXX)"
export ALLBERT_TRACE_ENABLED=true
mix allbert.settings set browser.enabled true
mix allbert.browser doctor
mix allbert.ask --trace "research https://example.com and summarize it"
mix allbert.ask --trace "screenshot https://example.com/login"
mix allbert.browser sessions list
mix allbert.browser sessions close <id>
rg -i 'cookie|authorization|session=' "$ALLBERT_HOME/memory/traces" || true
PORT=4041 mix phx.server   # /workspace: browser results panel
```

Expected behavior across milestones:

- the navigation scope must be approved before any fetch; a denied scope
  fetches nothing.
- extracted evidence is bounded (HTML/markdown/plain text/PDF) and
  recorded with a trace/audit entry; raw bodies live in
  `<ALLBERT_HOME>/cache/browser/<session_id>/`, not in
  `memory/traces/`.
- screenshots render without leaking sensitive page state; known
  credential-input nodes are redacted before encoding.
- form-fill and authenticated operations are denied by default; opt-in
  raises the floor only to confirmation.
- traces/audits redact cookies, credentials, and full URLs with userinfo.
- per-domain remembered grants apply to subsequent navigations within the
  same host; new hosts trigger fresh confirmations; cross-domain
  redirects pause for re-confirmation.
- doctor refuses to start sessions when the driver is unverified.
