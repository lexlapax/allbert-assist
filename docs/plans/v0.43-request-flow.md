## Main User/Operator Flows

- Operator asks Allbert to research a URL or topic.
- Allbert requests confirmation for a new navigation scope.
- Browser plugin navigates, extracts, screenshots, or reads bounded documents.
- Operator reviews extracted evidence and trace/audit metadata.

## Data Flow

Runtime frames a browser/research action, Resource Access records
`browser://session/<id>` and target URL scopes, Security Central confirms the
operation, the browser plugin performs bounded work, and results render in the
workspace with redacted traces.

## Authority Invariants

Browser plugin process ownership is not core authority. Each operation has a
separate grant. A navigate grant does not authorize fill/submit/download.
Browser content does not auto-promote to memory.

## Operator Validation

Status: planned. Commands are illustrative until the milestone lands; verify and
replace with observed output at closeout.

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v043.XXXXXX)"
export ALLBERT_TRACE_ENABLED=true
mix allbert.ask --trace "research https://example.com and summarize it"
# approve the navigation scope when prompted; deny once to confirm fail-closed
rg -i 'cookie|authorization|session' "$ALLBERT_HOME/memory/traces" || true
```

Expected behavior:

- the navigation scope must be approved before any fetch; a denied scope fetches
  nothing.
- extracted evidence is bounded (HTML/markdown/plain text/PDF) and recorded with
  a trace/audit entry.
- screenshots render without leaking sensitive page state; form-fill and
  authenticated operations are denied or require stricter confirmation.
- traces/audits redact cookies, credentials, and full URLs.
