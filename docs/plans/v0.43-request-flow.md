# Allbert v0.43 Request Flow - Discord And Slack Channels

Status: planned.

Implementation details live in `docs/plans/v0.43-plan.md`.

## Source Documents

- `docs/plans/v0.43-plan.md`
- ADR 0016, ADR 0017

## Main User/Operator Flows

- Operator configures Discord or Slack credentials and identity maps.
- External user mentions Allbert in an authorized channel.
- Allbert replies in thread or channel.
- External user approves or denies a confirmation callback.

## Data Flow

Provider events become durable channel events, resolve external identity to
local `user_id`, submit input through `Runtime.submit_user_input/1`, render
responses through provider-specific renderers, and update channel event status.

## Authority Invariants

Channels are delivery adapters only. They do not own runtime logic,
confirmation authority, resource grants, app context, or durable objectives.
Approval Handoff remains action-owned.
