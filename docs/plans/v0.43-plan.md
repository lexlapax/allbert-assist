# Allbert v0.43 Browser And Web Research

Request flow: `docs/plans/v0.43-request-flow.md`

ADR: `docs/adr/0040-browser-session-and-web-research-policy.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Let Allbert research web targets through a policy-bounded browser path. The
first release focuses on navigation, extraction, screenshots, and bounded
document extraction rather than arbitrary authenticated browser operation.

## Expected Inputs

- v0.40 MCP and v0.42 integration policy lessons.
- Existing Resource Access and external-network posture.
- Existing workspace surface and trace infrastructure.
- Proposed ADR 0040 accepted before implementation closeout.

## Operator-Visible Win

An operator can ask Allbert to research a web target, approve the navigation
scope, see extracted evidence, and inspect trace/audit records.

## Planned Interfaces

- Plugin package `./plugins/allbert.browser/`.
- Browser session resource `browser://session/<id>`.
- Registered actions: `BrowserStartSession`, `BrowserNavigate`,
  `BrowserExtract`, `BrowserScreenshot`, `BrowserClick`, `BrowserFill`,
  `BrowserDownload`, `BrowserCloseSession`.
- Bounded extraction for HTML, markdown, plain text, and PDF.

## Security Posture

Start with research, extraction, and screenshots. Form fill, submit, download,
and authenticated account work are denied or highly constrained until policy UI
is proven. Grants are per-domain and per-operation. Browser process ownership
stays in the plugin supervisor; core never spawns an unbounded browser.

## Non-Goals

- No arbitrary crawling.
- No automatic memory promotion from browser content.
- No broad Office/archive extraction.
- No account-changing browser actions by default.

## Milestones

- M1: ADR 0040, resource model, and browser plugin skeleton.
- M2: Research, navigate, extract, and screenshot actions.
- M3: Bounded document extraction and workspace rendering.
- M4: Confirmation UX, trace/audit metadata, and operator docs.
- M5: Browser-specific security evals and closeout.

## Documentation Deliverables

- Operator browser/research guide.
- Security-hardening updates for browser sessions and web prompt injection.
- Future-features split notes for broader document extraction.

## Definition Of Done

v0.43 is done when a disposable-home operator can approve a web research task,
receive bounded extracted evidence, and see eval coverage for prompt injection,
data exfiltration, cookie/session leakage, screenshot redaction, and
cross-domain grant escape.
