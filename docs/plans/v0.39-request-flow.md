## Main User/Operator Flows

- Operator runs `mix allbert.onboard` from a fresh checkout.
- Operator opens `/workspace?destination=workspace:onboard` and resumes the
  same onboarding objective.
- Operator lists, selects, and doctors a model profile (credentialed-remote or
  local-endpoint branch).
- Operator optionally toggles `intent.model_assist_enabled` during step 6.
- Operator optionally registers a channel (telegram or email).
- Operator sees an identity-slot preview pointing at v0.39b
  (`docs/operator/active-memory.md`); no write happens in v0.39.

## Data Flow

Input enters Runtime or an onboarding action, frames or resumes a single
durable onboarding `Objective` identified by
`{user_id, source_intent: "first_run_onboarding", non-terminal status}`
through the public `AllbertAssist.Objectives` facade, writes Settings Central
values only through registered actions (`set_active_model_profile`,
`set_provider_credential`, and the channel credential paths reused by
`mix allbert.channels`), stores secrets in Settings Central secrets, and emits
redacted doctor summaries per ADR 0047.

`mix allbert.model doctor PROFILE` calls `doctor_model_profile`, where
`PROFILE` is a model profile name. The action resolves the referenced provider
profile, chooses the branch from `providers.<name>.endpoint_kind`, reads only
resolved provider config and secret refs, and never returns raw secret values.
Its return shape is the canonical doctor summary from ADR 0047.

## Authority Invariants

- Onboarding metadata grants nothing.
- Provider doctor checks never expose secrets, raw error bodies, full URLs,
  backend stack traces, or credential fragments (per ADR 0047 §3).
- Provider doctor checks accept configured model profile names only; they do
  not accept arbitrary URLs, model-output targets, or remembered resource
  grants.
- Every effectful setup step runs through `Actions.Runner.run/3`.
- Onboarding writes only safe-keys (see v0.39 plan "Settings Central
  Writes During Onboarding").
- Onboarding never grants new permission classes and never lowers a
  safety floor.
- v0.39 does not write identity-slot content; the identity-slot step is
  preview-only and points at v0.39b.

## Smoke Checklist

- Complete onboarding in a disposable Allbert Home (`ALLBERT_HOME=$(mktemp -d
  /tmp/allbert-onboard-smoke.XXXXXX)`).
- Re-run onboarding mid-flight (close CLI, re-run, confirm resumes at first
  incomplete step). Same behavior from `/workspace?destination=workspace:onboard`.
- Select a remote model profile (e.g., `anthropic_fast`) and verify
  `mix allbert.model doctor anthropic_fast` returns the ADR 0047 shape with
  `endpoint_kind: :credentialed_remote`, `credential_ok: true`,
  `model_available: true`, redacted host only, no raw error body.
- Select a local model profile (e.g., `local` against
  `http://localhost:11434` Ollama with the shipped default model not yet
  pulled) and verify `mix allbert.model doctor local` returns
  `endpoint_kind: :local_endpoint`, `endpoint_ok: true`,
  `model_available: false`, `credential_ok: nil`, redacted host only, and a
  fixed remediation message.
- Pull the shipped local default model explicitly, rerun
  `mix allbert.model doctor local`, and verify it returns
  `endpoint_kind: :local_endpoint`, `endpoint_ok: true`,
  `model_available: true`, `credential_ok: nil`, redacted host only.
- Verify the shipped `model_profiles.local.model` default is a real Ollama
  model name (`llama3.2:3b`) and that the doctor does not auto-run
  `ollama pull`.
- Override `providers.local_ollama.endpoint_kind` to `credentialed_remote`
  via `mix allbert.settings set` and confirm the doctor branches on the
  override, not on heuristics.
- Verify `intent.model_assist_enabled` toggle requires explicit operator
  confirmation during step 6; default opt-in renders as the recommended
  choice, declining leaves the setting unchanged.
- Confirm onboarding objective resumes from `/workspace?destination=workspace:onboard`
  after CLI start, and vice versa.
- Confirm channel registration step is optional and skippable; `:skipped`
  outcome recorded on the step.
- Confirm identity-slot preview step renders the v0.39b pointer and writes
  nothing to `<ALLBERT_HOME>/memory/identity/`.
- Cross-OS smoke matrix:
  - macOS: completes from CLI and workspace.
  - Linux: completes from CLI and workspace.
  - Windows/WSL2: completes from CLI and workspace; localhost reachability
    against WSL2-side Ollama works; Windows-host Ollama bridging via
    `host.docker.internal` documented but not required for acceptance.
