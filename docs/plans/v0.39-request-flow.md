## Main User/Operator Flows

- Operator runs `mix allbert.onboard` from a fresh checkout.
- Operator opens `/workspace` onboarding destination and resumes the same setup
  objective.
- Operator lists, selects, and doctors a model profile.
- Operator creates or skips the optional identity memory.
- Runtime retrieves reviewed memory before a reply.

## Data Flow

Input enters Runtime or an onboarding action, frames/resumes an onboarding
objective, writes Settings Central values only through registered actions,
stores secrets in Settings Central secrets, optionally writes inert identity
memory, and adds read-only Active Memory retrieval metadata to traces.

## Authority Invariants

Onboarding metadata grants nothing. Provider doctor checks never expose secrets.
Identity memory is inert content. Active Memory does not promote or mutate
memory. Every effectful setup step runs through `Actions.Runner.run/3`.
