## Main User/Operator Flows

- Operator runs `mix allbert.onboard` from a fresh checkout.
- Operator opens `/workspace` onboarding destination and resumes the same
  setup objective.
- Operator lists, selects, and doctors a model profile (credentialed-remote or
  local-endpoint branch).
- Operator optionally registers a channel.

## Data Flow

Input enters Runtime or an onboarding action, frames/resumes an onboarding
objective, writes Settings Central values only through registered actions,
stores secrets in Settings Central secrets, and emits redacted doctor
summaries.

## Authority Invariants

Onboarding metadata grants nothing. Provider doctor checks never expose
secrets or raw error bodies. Every effectful setup step runs through
`Actions.Runner.run/3`.

## Handles / CLI Surfaces

- `mix allbert.onboard`
- `mix allbert.model list`
- `mix allbert.model use PROFILE`
- `mix allbert.model doctor PROFILE`

## Smoke Checklist

- Complete onboarding in a disposable Allbert Home.
- Select a remote provider profile (e.g., Anthropic) and verify doctor returns
  redacted summary.
- Select a local provider profile (e.g., Ollama) and verify doctor returns
  endpoint + model availability without credential probe.
- Confirm onboarding objective resumes from `/workspace` after CLI start.
- Confirm channel registration step is optional and skippable.
