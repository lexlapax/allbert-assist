# Allbert v0.30 StockSage Canvas Integration

Formerly tracked as M-Canvas, previously planned as v0.28 before the
project-direction rethink inserted v0.23 Jido State-Machine Convergence and
v0.24 Objective Runtime Foundation.

## Purpose

v0.30 registers StockSage components with the v0.26 Allbert canvas substrate.
This is wiring existing StockSage surfaces into the canvas catalog, not new
analysis/domain work.

## Expected Inputs

- v0.26 workspace surface and canvas substrate exist.
- v0.27 StockSage LiveViews and v0.29 polished domain behavior exist.
- v0.18 app/surface contract includes canvas/catalog extension points.

## Expected Direction

- Add `StockSageWeb.Canvas.StockChart`.
- Add `StockSageWeb.Canvas.AnalysisCard`.
- Implement StockSage canvas component registration through the v0.18 Surface
  DSL, v0.26 workspace/canvas substrate, and v0.27 StockSage component catalog.
- Add `canvas_ops` for useful analysis-result responses.
- Let StockSage workspace/dashboard use persistent tiles for recent analysis
  summaries.

## Non-Goals

- No new trading agents or domain records.
- No arbitrary model-generated components.
- No canvas contract changes outside v0.26 extension points.

## Test Plan

- `mix allbert.ask --user alice "analyze AAPL"` can push a registered
  `stock_chart` tile when an analysis completes.
- Persistent tiles survive reload when marked persistent.
- Unknown StockSage component ids are rejected by catalog validation.

## Definition Of Done

v0.30 is done when StockSage analysis can produce validated canvas tiles using
the Allbert surface catalog.
