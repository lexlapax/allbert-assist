v0.36 is the self-extending-runtime milestone. Allbert detects a capability
gap, generates **code and config** to the proven v0.27â€“v0.35 contract shapes,
trials it in an OS-level sandbox, and â€” only after it passes the full warning

template-driven developer/operator creation experience (Mix tasks, operator

This is the highest-capability and highest-risk milestone in the roadmap. Its
safety rests entirely on a strict separation between an untrusted trial phase
and a gated, operator-confirmed integration phase; the gates are
non-bypassable and integration is always operator-confirmed.

## Purpose

When a request needs a capability no existing plugin/app/action provides,
Allbert should be able to build that capability and make it live â€” not just
propose an inert draft. v0.36 delivers the full loop: generate â†’ sandbox trial â†’

## Safety / Trust Model

Two strictly separated phases (ADR 0032):

1. **Untrusted phase (may be proactive/auto).** Generated code/config is
   compiled and trialed only in an OS-level sandbox. Default backend is a local
   container (Docker/Podman); gVisor/Firecracker microVMs are the stronger
   future tier. BEAM processes and distributed-Erlang nodes are not a valid
   boundary (ADR 0009). If no OS backend is configured, the workflow stays
   disabled. Tiers: `:draft` â†’ `:sandbox_compiled` â†’ `:sandbox_trialed`.
2. **Integration gate (the trust grant).** An artifact becomes eligible only
   when it (a) passed the sandbox trial, (b) passed the **same warning gate as
   human code inside the sandbox** (`compile --warnings-as-errors`,
   `credo --strict`, `dialyzer`, focused tests, v0.36 security evals â†’ tier
   `:gate_passed`), and (c) gets **explicit operator confirmation**. Advisory
   /agent output and auto-trials never authorize integration.
3. **Trusted phase.** A gate-passing, operator-confirmed artifact is hot-loaded
   and registered live (tier `:integrated`) via the ADR 0035 loader, which
   recompiles the operator-reviewed source in core (with an integrity hash) and
   registers actions/app/children/destinations at runtime â€” no restart.
   Reversible live to `:rolled_back`. Page-route surfaces still need a restart;
   panel/destination apps integrate fully live.

## Design Decisions

### 1. Generation targets code and config, to proven shapes
Generated artifacts cover the full contract surface: plugin/app manifest and
modules, `AllbertAssist.Action` actions, `SurfaceProvider` panels and
`intent_descriptors`, settings fragments, memory namespace, objective wiring,
and theming/layout stubs. Generation is language-pluggable behind a
`CodeGenTarget` behaviour; v0.36 ships the Elixir/OTP target only. Other
languages stay parked.

### 2. LLM code-gen agent committee (the engine)
A committee of advisory `Jido.Agent` agents (Planner, Author, TrialAuthor,
Critic, Repair) authors drafts using Jido.AI (ADR 0035), reusing the v0.25
pattern. Agent output is advisory only (ADR 0021); it never enables, trusts, or
integrates. The deterministic, template-driven creation experience (Mix tasks,
operator flows, Canvas) is v0.37, built on this engine's sandbox/gate/loader.

### 3. Untrusted trial is OS-sandboxed and may be proactive
Generation and sandbox compile/trial may run proactively on a detected gap;
they are confined to the sandbox and grant no authority. The trial runs bounded
scenarios with limited time, memory, processes, filesystem, and no network,
secrets, Repo credentials, dependency changes, migrations, NIFs, or
package-manager steps.

### 4. Integration is gated, operator-confirmed, audited, and reversible
Integration runs only when `:gate_passed` and the operator confirms. The ADR
0035 loader recompiles the reviewed source in core (integrity-hashed), registers
the capability live (actions overlay, `App.Registry`, `DynamicSupervisor`,
catalog/destinations), and emits audit events. `rollback/1` purges and
unregisters live. Route-page apps require a restart (documented limit).

### 5. Default off; objective-runtime owned
The whole workflow is gated by a Settings Central master switch, default off,
and only enables when an OS sandbox backend is configured. Capability gaps,
generation, trial, and integration are objective steps; no app/plugin/LiveView
runs a private generation loop.

### 6. No new supply-chain or schema authority
Generated artifacts use only existing project dependencies and approved
post-v0.31 facades. No Hex deps, package-manager steps, NIFs, ports, migrations,
Repo credentials, unrestricted network, secrets, or external installers.

## Planned Interfaces

- `AllbertAssist.DynamicPlugins`: lifecycle context (generate, sandbox compile,
  trial, gate, integrate, rollback, discard).
- `AllbertAssist.DynamicPlugins.Draft`: normalized metadata, slug, trust tier,
  path, objective id, capability gap, gate report, integrity hash, status.
- `AllbertAssist.DynamicPlugins.Sandbox`: OS-isolation backend contract
  (container first) with no access to secrets, Repo, or core state.
- `AllbertAssist.DynamicPlugins.Loader`: gated in-core hot-load + live
  registration + rollback (ADR 0035).
- `AllbertAssist.Codegen.{Planner,Author,TrialAuthor,Critic,Repair}`: advisory
  Jido code-gen agents (the v0.37 templated path reuses this engine).
- `AllbertAssist.CodeGenTarget`: language-target behaviour (Elixir shipped).
- Registered actions: `GenerateDraft`, `CompileDraft`, `RunDraftTrial`,
  `RunWarningGate`, `IntegrateDraft`, `RollbackIntegration`, `DiscardDraft`
  (integration/rollback/discard require confirmation).
- `AllbertAssist.Objectives.CapabilityGap`: objective gap/acquisition vocabulary.
- Actions registry runtime overlay + `DynamicSupervisor` for live registration.

Names may change during implementation if docs/request-flow record the final
public surface.

## Milestones

Each milestone closes with focused tests, docs/request-flow notes, and
`mix precommit` unless a narrower gate is recorded with why.

### M1: ADRs, tiers, capability-gap vocabulary, settings
Accept ADR 0032/0033 amendments and new ADR 0035. Add trust tiers incl.
`:gate_passed`/`:integrated`/`:rolled_back`. Add capability-gap + proactive-trial
vocabulary. Add the default-off Settings master switch and draft roots under
Allbert Home. Tests: temporary-home paths, gap serialization,
disabled-workflow + confirmation-required security tests.

### M2: Container sandbox backend
Implement the sandbox adapter and the local container backend (compile + trial),
with policy denials (no secrets/Repo/network/deps/migrations/NIFs/package
managers/core load) and audit. Tests: compile/trial success/failure, forbidden-
construct denials, timeout/cleanup, and "core node never loads trial modules."

### M3: Generation output contract + validation
Pin the code/config output contract (the proven v0.27â€“v0.35 shapes) the engine
must emit under `<ALLBERT_HOME>/plugins/<slug>/`, with slug/path/allow-list/size
/no-traversal validation, forbidden-construct static checks, and redacted
provenance. This is the shared contract the v0.37 templated path also targets.
Tests: draft-tree, traversal/size denial, forbidden-construct static checks.

### M4: Code-gen agent committee + repair loop
Add the advisory Jido agents and the sandbox-grounded repair loop. Tests:
agents propose only; agent output cannot enable/trust/integrate; repair re-authors
on sandbox failure; raw prompts/completions are not traced.

### M5: In-sandbox warning gate
Run the full warning gate (compile-WAE, Credo, Dialyzer, focused tests, security
evals) inside the sandbox as the integration precondition â†’ `:gate_passed`.
Tests: gate pass/fail; a failing artifact can never reach integration.

### M6: Live integration loader + rollback (no restart)
Add the runtime-mutable actions overlay, `DynamicSupervisor`, and the ADR 0035
loader: operator-confirmed hot-load + live registration of a `:gate_passed`
artifact without a restart, plus `rollback`. Tests: live capability becomes
usable without restart; rollback reverts live; route-page restart limit
documented; unconfirmed/ungated integration denied.

### M7: Security evals and closeout
Add evals: untrusted core-load attempt, gate-skip, unapproved/auto integration,
loader provenance/integrity tamper, rollback correctness, exfiltration,
supply-chain, sandbox escape, advisory-approval bypass. Update README, roadmap,
agent map, future-features, AGENTS, DEVELOPMENT, ADR pointers. Bump version
metadata to `0.36.0`.

## Non-Goals

- No untrusted in-core loading; no integration without the gate + operator
  confirmation; no integration authorized by advisory/agent output or an
  auto-trial result alone.
- No remote marketplace, remote plugin install, or remote binary loading.
- No package-manager execution, dependency additions, migrations, NIFs, ports,
  or unrestricted network adapters in generated artifacts.
- No multi-language target beyond Elixir/OTP (others parked).
- No autonomous trace-to-skill creation (parked).
- No distributed-Erlang/BEAM node as the untrusted-trial boundary.

to the proven shapes (via the code-gen agent committee), trial it in an
