# v1.0 Handoff — Acceptance Matrix

Status: current (authored at v0.66 M11; reconciled after v0.66 release). v0.66 is
the released integrated product RC; v1.0 (roadmap item 67) is the stability
release + tiered public-contract freeze + non-developer product launch. This note
is the handoff: the acceptance-matrix inputs v1.0 inherits from v0.66, what each
input already proves, and which operator-attested DIT rows still block the freeze.

v0.66 uses two-layer verification (`docs/plans/v0.66-plan.md` Locked Decision 1): a
deterministic `mix allbert.test release.v066` gate for contract/routing/boundary
invariants, plus operator-attested evidence (`docs/validation/v0.66/`) for the
install/browser/model/cross-platform claims the gate cannot run in a checkout. New
v1.0 DIT evidence is recorded under `docs/validation/v1.0/` and links back to the
inherited v0.66 evidence rather than rewriting it.

## Contracts v1.0 freezes (roadmap item 67)

- Tier 1: Runtime/turn signals, Actions/permissions, Plugin, App, Settings Central schema
  shape, Allbert Home layout, Channel adapter boundary, Resource Access URI/grants,
  model/provider doctor shape, and installer-side cosign fail-closed verification.
- Tier 2: SurfaceProvider, Surface DSL (additive-only carve-out), workspace
  canvas/ephemeral substrate (minus single-consumer components), templated creation,
  public-protocol surface policy, packaged-entry contracts, and v0.65 local-knowledge
  action/settings/memory-review contracts.

## Open items carried to v1.0

The `PENDING-OPERATOR` rows above (WSL2 install, one-click model download from empty,
fresh no-docs first-run walkthrough, model-backed memory recall in chat, real
cross-version upgrade, host uninstall, and live advanced-surface exercise) are the
operator-attested evidence v1.0 must complete on real hosts before the freeze. The
recorded usability caveats (transient reconnect toast and ensuring QuickStart always
reaches the direct-answer enable step) are tracked in
`docs/validation/v0.66/item11-usability-audit.md` and rechecked in `docs/validation/v1.0/`.

Decision lock:

- DIT-2 uses a true non-developer operator, not a maintainer role-playing with
  project docs or hidden setup knowledge.
- One-click in-app model download from an empty model cache is freeze-blocking.
  A preloaded/local-ready model can prove row 8 first chat, but cannot satisfy
  the consumer-default setup bar.
- DIT-5's required upgrade source is a real `v0.66.0` packaged Home upgraded or
  imported into v1.0 on a second machine/disposable Home. Older pre-v0.66 Homes
  are compatibility notes unless a schema change or release note explicitly
  expands support.

## DIT Milestones Before Freeze

DIT here means direct integration test: evidence that cannot be proven by the
checkout-bound `release.v066` gate and must run against real hosts, real packaged
artifacts, or real configured providers before the v1.0 freeze.

- **DIT-1: Cross-host install coverage.** Run the WSL2 install/serve walkthrough and
  attach the redacted transcript to `docs/validation/v1.0/`; link the macOS and Linux
  `0.66.0` evidence inherited from `docs/validation/v0.66/`.
- **DIT-2: No-docs first-run completion.** Complete the fresh true non-developer
  path from install -> QuickStart -> local model setup/download from an empty model
  cache -> first useful chat without reading developer docs, and confirm QuickStart
  enables direct answers before the user's first real question.
- **DIT-3: Model-backed local knowledge.** Keep a memory candidate and verify a later
  model-backed chat uses only `:kept` memory; connect notes and verify grounded
  note-backed answer/write flows with confirmations.
- **DIT-4: Live advanced-surface regression.** Exercise browser research,
  browser-research delegate, configured remote-channel outbound/inbound, MCP/OpenAI/ACP,
  and Plan/Build approval smokes; record unconfigured classes as scoped out or blocking.
- **DIT-5: Home portability and teardown.** Run a real `v0.66.0` packaged Home ->
  v1.0 upgrade or import round-trip on a second machine/disposable Home, then
  uninstall the service/binary and verify Allbert Home preservation.
- **DIT-6: Source/release-line parity.** Treat `v0.66.0` as the packaged artifact line and
  `origin/main` after v0.66 M12/M13 as the freeze-prep source/evidence line; if a public
  source tag is required before v1.0, use a docs/source point tag with `[skip-artifacts]`.
