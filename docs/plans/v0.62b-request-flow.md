version **0.62.1** if a point release is needed. Mirrors
`docs/plans/v0.62b-plan.md`. v0.62.0 is already shipped and tagged; this flow
owns the remaining Homebrew, packaged TUI, both-Linux Docker, and reusable

## Authority Invariants

- No new authority, permission atom, Settings Central key, confirmation-floor
  change, provider egress, runtime action, or trust semantic.
- Tap fill and installer rehearsals verify distribution plumbing only. They do
  not grant a plugin, package, formula, checksum file, or container extra
  authority inside Allbert.
- Docker rehearsals are package smokes. They are not proof of a real desktop
  Secret Service session or real user systemd environment unless those host
  services are explicitly present and exercised inside the container.
- Packaged TUI validation must use the packaged `allbert` entry point and a
  disposable `ALLBERT_HOME`; it must not silently fall back to `mix allbert.tui`.

## M1 - Homebrew Tap Fill

Use a scratch clone unless the operator explicitly wants to work in an existing
tap checkout:

```sh
ALLBERT_ASSIST_CHECKOUT="${ALLBERT_ASSIST_CHECKOUT:-$(pwd)}"  # run from allbert-assist checkout
mkdir -p /tmp/allbert-v062b
gh release download v0.62.0 --repo lexlapax/allbert-assist \
  --pattern SHA256SUMS --dir /tmp/allbert-v062b
git clone https://github.com/lexlapax/homebrew-allbert /tmp/homebrew-allbert
sh "$ALLBERT_ASSIST_CHECKOUT/homebrew/fill-sha256.sh" \
  /tmp/allbert-v062b/SHA256SUMS \
  /tmp/homebrew-allbert/Formula/allbert.rb
```

Review and validate:

```sh
cd /tmp/homebrew-allbert
git diff -- Formula/allbert.rb
rg -n 'PLACEHOLDER|TODO|sha256' Formula/allbert.rb
brew audit --strict --online --formula Formula/allbert.rb
```

Commit and push in the tap only after audit output is acceptable:

```sh
git add Formula/allbert.rb
git commit -m "allbert v0.62.0"
git push origin main
```

## M4 - Both-Linux Docker Rehearsals

Docker Desktop may need to be started before `docker info` succeeds:

```sh
docker desktop status || docker desktop start
docker info
```

The Docker command shape uses `--platform`, bind mounts, environment variables,
`--rm`, and a fixed workdir. Keep the container command checked into the evidence
record before running so the exact platform is visible.

Prepare artifacts:

```sh
ALLBERT_ASSIST_CHECKOUT="${ALLBERT_ASSIST_CHECKOUT:-$(pwd)}"  # run from allbert-assist checkout
mkdir -p /tmp/allbert-v062b/artifacts
gh release download v0.62.0 --repo lexlapax/allbert-assist \
  --pattern 'allbert-v0.62.0-linux-*.tar.gz' \
  --pattern SHA256SUMS \
  --dir /tmp/allbert-v062b/artifacts
```

Run native Linux arm64 on Apple Silicon Docker:

```sh
docker run --rm --platform linux/arm64 \
  --mount type=bind,source=/tmp/allbert-v062b,target=/work,readonly \
  --mount type=bind,source="$ALLBERT_ASSIST_CHECKOUT",target=/repo,readonly \
  --workdir /tmp \
  ubuntu:22.04 \
  bash -lc 'set -euo pipefail
    apt-get update
    apt-get install -y ca-certificates curl tar gzip libstdc++6 openssl
    useradd -m -u 1000 allbert
    mkdir -p /tmp/rehearsal
    cp -R /work/artifacts /tmp/rehearsal/artifacts
    chown -R allbert:allbert /tmp/rehearsal
    su -s /bin/bash allbert -c "set -euo pipefail
    cd /tmp/rehearsal
    export LANG=C.UTF-8 LC_ALL=C.UTF-8
    (cd artifacts && sha256sum -c SHA256SUMS --ignore-missing)
    mkdir -p extract-arm64
    tar -xzf artifacts/allbert-v0.62.0-linux-arm64.tar.gz -C extract-arm64
    /repo/scripts/smoke/linux_rehearsal.sh /tmp/rehearsal/extract-arm64/allbert"'
```

Run Linux x64 under emulation:

```sh
docker run --rm --platform linux/amd64 \
  --mount type=bind,source=/tmp/allbert-v062b,target=/work,readonly \
  --mount type=bind,source="$ALLBERT_ASSIST_CHECKOUT",target=/repo,readonly \
  --workdir /tmp \
  ubuntu:22.04 \
  bash -lc 'set -euo pipefail
    apt-get update
    apt-get install -y ca-certificates curl tar gzip libstdc++6 openssl
    useradd -m -u 1000 allbert
    mkdir -p /tmp/rehearsal
    cp -R /work/artifacts /tmp/rehearsal/artifacts
    chown -R allbert:allbert /tmp/rehearsal
    su -s /bin/bash allbert -c "set -euo pipefail
    cd /tmp/rehearsal
    export LANG=C.UTF-8 LC_ALL=C.UTF-8 ERL_AFLAGS=\"+JMsingle true\"
    (cd artifacts && sha256sum -c SHA256SUMS --ignore-missing)
    mkdir -p extract-x64
    tar -xzf artifacts/allbert-v0.62.0-linux-x64.tar.gz -C extract-x64
    /repo/scripts/smoke/linux_rehearsal.sh /tmp/rehearsal/extract-x64/allbert"'
```

Container runtime notes:

- Run package smoke commands as a non-root user inside Docker. The packaged
  runtime uses `erlexec`, which refuses root startup without an explicit
  effective user setting; the first root-form command reached install/version
  and then failed `admin status` before application code could run.
- On Apple Silicon Docker Desktop, `linux/amd64` emulation needs
  `ERL_AFLAGS="+JMsingle true"` for the x64 BEAM JIT memory mapping. Without it,
  the x64 VM fails in `prim_tty/user_drv` before Allbert application boot.
- If Docker Hub credential helpers hang or fail for public Ubuntu pulls, isolate
  this rehearsal with `DOCKER_CONFIG=/tmp/allbert-v062b/docker-config`.

If `scripts/smoke/linux_rehearsal.sh` needs to be bypassed for diagnosis, run the
same checks manually inside the container and record the commands:

```sh
apt-get update
apt-get install -y ca-certificates curl tar gzip libstdc++6 openssl
sha256sum -c artifacts/SHA256SUMS --ignore-missing
tar -xzf artifacts/allbert-v0.62.0-linux-<target>.tar.gz -C /work
ln -sf /work/allbert/bin/allbert /usr/local/bin/allbert
allbert --version
allbert admin status
allbert serve &
DAEMON_PID=$!
curl -fsS http://localhost:4000/health
allbert admin status
kill "$DAEMON_PID"
test -d "$ALLBERT_HOME"
```

Vault/service rules:

- Mark vault PASS only if Secret Service is actually available and exercised
  with the packaged command path.
- Mark service PASS only if user systemd is actually available and the dry-run or
  install/uninstall path is exercised as specified.
- Otherwise mark SKIP with the missing host service. Do not convert those skips
  into release failures unless the active closeout explicitly requires real-host
  Linux validation.
