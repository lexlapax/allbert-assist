## Locked Decisions
Recorded in `v0.59-plan.md`: hardening-only scope; ship the settings version
*contract* + enforcement, defer the runtime runner; M3 is reference round-trip not
secret migration; reuse `validate_params/1` at the Runner with key-normalize-first;
the context/param split mostly already exists; explicit `schema: []` disposition;
the param seam is delivered freeze-ready; dry-run-only import with documented manual
rollback; an explicit `envelope_version` distinct from per-fragment `schema_version`.

## Cross-Surface Data Flow (the one path)
```text
EXPORT (operator)
  mix allbert.home.export -> Portability.Export
    -> read settings (+ per-fragment schema_version), memory, skills, threads,
       traces, audits, jobs, objectives, confirmations, plugin/marketplace meta,
       self-improvement state, media retention meta, channel pairing meta,
       public-protocol settings  (through documented facades, never raw stores)
    -> redact secret VALUES; keep secret REFERENCES only
    -> envelope { envelope_version, exporting_version, manifest, fragments[...] }

IMPORT (dry-run only, second Home)
  mix allbert.home.import --dry-run -> Portability.Import
    -> validate envelope_version
    -> per-fragment: Settings.VersionContract boot check
         stored == known  -> current
         stored <  known  -> pending (runner DEFERRED; manual step documented)
         stored >  known  -> FAIL CLOSED (operator diagnostic via doctor + security status)
    -> write redacted diagnostic; APPLY NOTHING
    -> rollback = documented manual restore of the prior Home dir

PARAM CONTRACT (every operation)
  Actions.Runner.run/3
    -> key-normalize params (string->atom)
    -> action.validate_params/1   (reused; was bypassed at runner.ex:85)
       fail -> redacted :invalid_params  (ADR 0064 shape, before any body)
    -> safe_run -> action.run(params, context)   (context is a SEPARATE arg)
       system context is never read from params (3 sites fixed)
```

## Authority Invariants
- No new capability, no new authority surface, no confirmation-floor change, no
  egress grant. Security Central remains the only authority boundary.
- Export envelopes, dry-run diagnostics, the boot-check diagnostic, and sweep
  evidence never print raw secret values/refs, endpoint URLs, raw prompts, or
  provider bodies.
- Secret VALUES stay in the encrypted Settings Central secret store
  (`<ALLBERT_HOME>/settings/secrets.yml.enc` today); only REFERENCES round-trip.
- Self-improvement (v0.47) and voice/vision (v0.48-v0.49) stay inert in the import
  plan; dry-run applies nothing.
- Import is dry-run only; it applies nothing and the runtime migration runner is
  deferred (ADR 0046 §5).
- The model proposes params only; the Runner enforces the registered schema.

### S4 - Fail-Closed Boot Check (forward-version fixture)
```sh
# Craft a fixture envelope that bumps one fragment's schema_version beyond known max.
cp "$V059_EVIDENCE_DIR/home-a.envelope.json" "$V059_EVIDENCE_DIR/home-a-forward.envelope.json"
# Bump the first schema_version found to a forward value (e.g. 2 where known max is 1).
mix run -e '
path = System.fetch_env!("V059_EVIDENCE_DIR") <> "/home-a-forward.envelope.json"
data = path |> File.read!() |> Jason.decode!()
bump = fn bump, term ->
  cond do
    is_map(term) and Map.has_key?(term, "schema_version") ->
      {Map.put(term, "schema_version", 2), true}
    is_map(term) ->
      Enum.reduce(term, {%{}, false}, fn {key, value}, {acc, done?} ->
        if done? do
          {Map.put(acc, key, value), true}
        else
          {new_value, hit?} = bump.(bump, value)
          {Map.put(acc, key, new_value), hit?}
        end
      end)
    is_list(term) ->
      Enum.reduce(term, {[], false}, fn value, {acc, done?} ->
        if done? do
          {[value | acc], true}
        else
          {new_value, hit?} = bump.(bump, value)
          {[new_value | acc], hit?}
        end
      end)
      |> then(fn {values, hit?} -> {Enum.reverse(values), hit?} end)
    true ->
      {term, false}
  end
end
{updated, hit?} = bump.(bump, data)
unless hit?, do: raise("no schema_version found in envelope fixture")
File.write!(path, Jason.encode!(updated, pretty: true))
IO.puts("bumped schema_version")
'
ALLBERT_HOME="$V059_HOME_B" ALLBERT_HOME_DIR="$V059_HOME_B" \
  mix allbert.home.import --dry-run --in "$V059_EVIDENCE_DIR/home-a-forward.envelope.json" \
  > "$V059_EVIDENCE_DIR/s4-forward-import.txt" 2>&1 || true
rg -i 'unknown|forward|fail.closed|refus' "$V059_EVIDENCE_DIR/s4-forward-import.txt"
```
Expected: the forward `schema_version` trips the fail-closed diagnostic; the
import refuses to load the forward fragment and surfaces an operator-visible
message (mirrored in `mix allbert.security status`). The boot check is exercised by
the fixture alone — no migration runner is required.

### Pass Criteria
- Portability: Home A exports a versioned, redacted envelope; dry-run import into
  Home B produces a diagnostic and applies nothing; the documented manual rollback
  path is recorded.
- Version contract: every registered fragment carries a first-class
  `schema_version` at `1`; additive-only is enforced; the boot check fails closed
  on a forward fixture via doctor + `security status`; the runtime runner is
  explicitly deferred.
- Secrets: references round-trip; values never leave the encrypted store.
- Param contract: malformed params yield `:invalid_params` before any body; the
  three context sites are fixed; every `schema: []` action has a disposition; the
  ~190-action sweep shows no regression.
- Security sweep: every named v0.40-v0.51 surface passes with redacted evidence;
  inert invariants hold.
- The gate `mix allbert.test release.v059` is green and deterministic; durable
  redacted evidence is retained outside the repo.
