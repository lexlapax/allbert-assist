## Cross-Surface Data Flow (the one path)
```text
EXPORT (operator)
  mix allbert.home.export -> Portability.Export
    -> read settings (+ per-fragment schema_version), memory, skills, threads,
       traces, audits, jobs, objectives, confirmations, plugin/marketplace meta,
       self-improvement state, media retention meta, channel pairing meta,
       public-protocol settings  (through documented facades, never raw stores)
    -> redact secret VALUES; keep secret REFERENCES only
    -> envelope { envelope_version, exporting_version, manifest, fragments[...] }

IMPORT (dry-run only, second Home)
  mix allbert.home.import --dry-run -> Portability.Import
    -> validate envelope_version
    -> per-fragment: Settings.VersionContract boot check
         stored == known  -> current
         stored <  known  -> pending (runner DEFERRED; manual step documented)
         stored >  known  -> FAIL CLOSED (operator diagnostic via doctor + security status)
    -> emit redacted diagnostic to stdout or operator evidence path outside Home B; APPLY NOTHING
    -> rollback = documented manual restore of the prior Home dir

PARAM CONTRACT (every operation)
  Actions.Runner.run/3
    -> normalize only schema-known string keys to existing atoms; never create atoms
    -> reject unknown keys unless explicit open-key/compat disposition allows them
    -> action.validate_params/1   (reused; was bypassed at runner.ex:85)
       fail -> redacted :invalid_params  (ADR 0064 shape, before any body)
    -> safe_run -> action.run(validated_params, context)   (context is a SEPARATE arg)
       system context is never read from params (3 sites fixed)
```

## Authority Invariants
- No new capability, no new authority surface, no confirmation-floor change, no
  egress grant. Security Central remains the only authority boundary.
- Export envelopes, dry-run diagnostics, the boot-check diagnostic, and sweep
  evidence never print raw secret values/refs, endpoint URLs, raw prompts, or
  provider bodies.
- Secret VALUES stay in the encrypted Settings Central secret store
  (`<ALLBERT_HOME>/settings/secrets.yml.enc` today); only REFERENCES round-trip.
- Self-improvement (v0.47) and voice/vision (v0.48-v0.49) stay inert in the import
  plan; dry-run applies nothing.
- Import is dry-run only; it applies nothing and the runtime migration runner is
  deferred (ADR 0046 §5).
- The model proposes params only; the Runner enforces the registered schema.
