## Purpose

Three goals:
1. **Channel parity** — make the cross-channel capability matrix explicit and
   machine-checkable (which approval primitives `{:list,:button,:typed_command,
   :link}`, attachments, streaming, identity mapping, and approval rendering each
   channel supports), and close the gaps, including the v0.54-deferred Matrix
   generic outbound implementation behind `Channels.Outbound`.
2. **A proper TUI/terminal channel** — a persistent supervised terminal channel
   that is a real ADR 0016 channel (descriptor + `channel_events` dedupe +
   identity mapping + Approval Handoff + redaction), distinct from the one-shot
   `mix allbert.ask` label.
3. **Split tool-result foundation** — harvest Pi's split tool-result pattern (a
   model-facing payload separate from the surface render payload; ADR 0067
   extending ADR 0029/0030) into the typed response contract so the TUI can
   stream diffs and rich output. This is the substrate the v0.57 Pi-mode coding
   surface builds on.

## Readiness Sweep
- **Contract fit.** Channels are delivery adapters around
  `Runtime.submit_user_input/1` + registered actions (ADR 0016 §boundary); they
  do NOT own intent/actions/Security Central/Resource Access/confirmation storage/
  conversation history. The TUI is just another adapter under
  `Channels.Supervisor` (`:one_for_one`) and must use the ADR 0057
  `ChannelThread` substrate for conversation continuity.
- **Reuse fit.** `plugins/allbert.telegram` is the adapter template
  (`channels/0` descriptor → supervised GenServer → normalize → `Identity.resolve`
  → `Channels.derive_session_id` → `Runtime.submit_user_input` → `channel_events`).
  Email's `ALLBERT:APPROVE:<id>` renderer is the `:typed_command` template.
- **Upstream check.** Confirm v0.52 delivered `primitives:` +
  `Approval.Handoff` + registry validation and ADR 0057 delivered `threading:` +
  `Conversations.ChannelThread` + owner/account/key mapping. If missing, stop
  and reconcile the v0.52/v0.53 dependency chain before implementing v0.55.
- **Library fit.** `owl` is present but **transitive** (via `igniter`, `:dev,:test`
  only), so M0 must promote it to a direct runtime dep. It is pure-Elixir and
  explicitly top-to-bottom (not full-screen), matching the scrollback-native
  rendering model (Locked Decision #7): `Owl.IO`/`Owl.Data`/`Owl.LiveScreen` cover
  input, styled lines, and the single live block. `ratatouille`/`ex_termbox` are
  rejected (dormant, native NIF, full-screen takeover that loses scrollback).

## Locked Decisions
| # | Decision | Chosen value | Trade-off if changed |
|---|---|---|---|
| 1 | TUI authority | The TUI is a delivery adapter with **no authority beyond any other channel** (ADR 0016). | A privileged channel breaks the boundary. |
| 2 | TUI library | **`owl`** (maintained, pure-Elixir) over `ratatouille` (dormant, `ex_termbox` NIF). Promote owl from transitive (`igniter`, `:dev,:test`) to a direct, top-level **runtime** dep. | ratatouille adds a native-build + unmaintained dependency and is full-screen (loses scrollback); owl is top-to-bottom and matches the rendering model. |
| 3 | CLI promotion | Do **not** promote `mix allbert.ask` (stays a non-channel developer label, `channel: :cli`). The TUI is the first-class persistent terminal channel (`channel_id: "tui"`, provider `"terminal"`). | Promoting the one-shot task duplicates the TUI and muddles the label. |
| 4 | TUI primitives | Declare `primitives: [:typed_command, :list]` (numbered options + `ALLBERT:APPROVE:<id>`); **not** `:button`/`:link` (no GUI affordance / no in-terminal workspace URL). `:list` is the mandatory universal fallback. | Declaring `:button`/`:link` would advertise affordances the terminal can't honor. |
| 5 | TUI session/identity/threading | Maps to local `user_id` via `channels.tui.identity_map` (default `"local"`); new `session_strategy` (`{:tui_session, prefix: "ch_tui_"}`) via `Channels.derive_session_id`; declares `threading: :rich` and writes through `ChannelThread` with `owner_scope: "local"`, `receiver_account_ref: "tui:<profile>"`, and a deterministic `provider_thread_key` for the terminal conversation. | Implicit `"local"` claim violates ADR 0016 identity mapping. Bypassing ADR 0057 would fork terminal conversation history from web/Slack/Discord/mobile continuity. |
| 6 | Parity matrix home | Canonical prose table in the ADR 0016 amendment + machine-derived from descriptors (`ChannelParity` / `mix allbert.channels --parity`) + `docs/developer/channel-parity.md`. | A prose-only matrix drifts from the registry truth. |
| 7 | TUI rendering model | **Scrollback-native, line-oriented (Pi model).** Completed turns are static scrollback lines (`Owl.Data`/`IO.ANSI`); a single `Owl.LiveScreen` block renders the active prompt + streaming line (differential bottom-region, synchronized-output to avoid flicker). No alternate-screen, no full-viewport redraw. | A full-screen/alt-screen TUI (`ratatouille`/`ex_termbox`) loses native scrollback + search and boxes out the v0.57 Pi-mode streaming substrate. |

## Foundation For v0.57 Pi-mode

v0.55 is the foundation v0.57 Pi-mode (ADR 0068) extends **additively** — Pi-mode
must need no rework of this channel. v0.55 therefore lands these seams:

- **Same channel + adapter loop.** Pi-mode runs *in* the `tui` channel and its
  persistent session (M4 console); it adds `read`/`write`/`edit`/`bash` actions +
  a trust tier, it does **not** fork a new channel or runtime.
- **Split tool-result payload seam.** v0.55 establishes the model-facing vs
  surface-render split (ADR 0067 extending ADR 0029/0030) so Pi-mode's streamed
  diffs and tool-result rendering slot in without touching the response contract.
- **Live-region streaming substrate.** The single `Owl.LiveScreen` block (Locked
  Decision #7) is the surface Pi-mode streams diffs into.
- **Level-0-compatible boundary.** v0.55 maps to `"local"` identity and runs every
  action through `Actions.Runner.run/3` + Security Central, with no assumption
  (per-action confirmation, identity model) that blocks v0.57 adding the named
  "local coding / sandbox level 0" trust tier on the same adapter.
- **Session/threading shaped for a coding mode.** `:tui_session` + `ChannelThread`
  are structured so a Pi-mode coding session is a *mode* on the same channel.

## Milestones

Lane summary: M0 serial barrier (deps + introspection) → (M1 parity ∥ Matrix
outbound) → M2 TUI adapter → M3 approval render → M4 operator/validation console →
M5 evals + docs → **M6 operator validation (manual gate)** → **M7 release
closeout**. M6 blocks M7.

### M0: Locked Decisions + Dependency + Parity Substrate
Promote `owl` from a transitive (`igniter`, `:dev,:test`) dep to a **direct,
top-level runtime dep** in the umbrella `mix.exs` (it is otherwise unavailable at
runtime, where the TUI runs); verify the v0.52 `primitives:`/`Approval.Handoff`/
registry-validator and ADR 0057 `threading:`/`ChannelThread` substrates exist
(audit 2026-06-21: **confirmed present** — `Approval.Handoff.render/2`,
`Conversations.ChannelThread`, and the `primitives:`/`threading:` validators are
shipped); define the parity-matrix data model + `ChannelParity` introspection over
`PluginRegistry.registered_channels()`.
- **Lane:** serial barrier (dep/build change + shared introspection). Parallel:
  parity-matrix data-model design (pure) ∥ owl promotion.
- **Focused:** `ChannelParity` introspection over registered channels; owl
  resolvable at runtime.
- **Rejoin:** owl is a direct runtime dep; `ChannelParity` module exists.

### M1: Parity Matrix + Matrix Generic Outbound
Build the explicit matrix (primitives, attachments, streaming, identity mapping,
approval rendering per channel). Ensure every shipped descriptor declares
`primitives:`; document CLI/LiveView as local surfaces (declared-but-not-plugin)
consistently; record a canonical "streaming: not supported (turn-complete)" cell
and per-channel attachment posture. Close the **Matrix generic outbound** gap from
v0.54 by implementing `AllbertAssist.Channels.Matrix.Adapter.deliver_outbound/3`
through `Channels.Outbound` (today `Channels.Outbound.send/4` returns
`:outbound_not_implemented` for Matrix via the `function_exported?/3` guard). Add
the `mix allbert.channels --parity` report.
- **Lane:** parallel — `mix allbert.channels --parity` (pure + CLI) ∥ Matrix
  `deliver_outbound/3` (plugin code; distinct modules, no shared barrier).
- **Focused:** `send_channel_message` no longer degrades for mapped Matrix
  targets; parity report == registry truth.
- **Rejoin:** every descriptor declares `primitives:`; Matrix outbound is live.

### M2: TUI Channel Adapter + Rendering + Split-Payload Seam
New shipped plugin `plugins/allbert.tui`: `channels/0` descriptor (`channel_id:
"tui"`, adapter module, `child_spec`, `settings_prefix: "channels.tui"`,
`identity_map_key`, `session_strategy: {:tui_session, prefix: "ch_tui_"}` — **add
the `:tui_session` case to `Channels.derive_session_id`** — `primitives:
[:typed_command, :list]`, `threading: :rich`). Supervised GenServer under
`Channels.Supervisor`: `owl` input loop → normalize → `Identity.resolve` →
`derive_session_id` → `Runtime.submit_user_input` → `ChannelThread`
lookup/link/message-ref → `channel_events` create/update → **render per the
Rendering Model** (Locked Decision #7): completed turns as static scrollback lines
(`Owl.Data`/`IO.ANSI`) + a single `Owl.LiveScreen` block for the active
prompt/streaming line. Land the **split tool-result payload seam** (model-facing
vs surface-render; ADR 0067 extending ADR 0029/0030) — the Pi-mode foundation.
Settings schema + enabled flag.
- **Lane:** serial barriers — `Channels.Supervisor` child (named process),
  `channel_events` (db_serial), the `derive_session_id` `:tui_session` case, and
  the split-payload seam (touches the ADR 0029/0030 response contract — shared
  seam). Terminal IO is external_runtime.
- **Focused:** identity-map rejection, session/turn dedupe, crash isolation
  (`:one_for_one`), scrollback + live-region render, split-payload seam (surface
  framing/ANSI never leaks into the model-facing payload).
- **Rejoin:** TUI loads, simulated inbound green, split-payload seam in place.

### M3: Approval-Primitive Rendering In TUI
TUI `render_approval_handoff/1` consumes `{primitive_kind, payload}` from
`Approval.Handoff.render/2`; renders `:list` (numbered approve/deny/show) and
`:typed_command` (`ALLBERT:APPROVE:<id>`) within the scrollback/live-region model;
resolves via `Actions.Runner.run("approve_confirmation"|"deny_confirmation"|"show_confirmation")`
like Telegram. No primitive embeds targets/commands/URLs (ADR 0016).
- **Lane:** depends on M2; focused single-channel.
- **Focused:** primitives never embed targets/commands/URLs; confirmation resolve
  isolated from other channels.
- **Rejoin:** approve/deny/show work in the TUI like Telegram.

### M4: TUI Persistent Operator/Validation Console
Make the TUI the persistent, mix-free operator console so interactive operator
work runs in **one warm BEAM (the DB stays open)** instead of cold per-turn `mix
allbert.ask` invocations. Add a `mix allbert.tui` launcher that boots the app once
and holds the session open; add the in-session slash-commands operators need
during validation — `/status`, `/confirmations`, `/events`, `/channels`,
`/settings get` (each resolves through the same `Actions.Runner`/context boundary,
**no new authority**). This persistent session is also the session v0.57 Pi-mode
runs in. `mix allbert.test` (deterministic CI gates) is unchanged — this targets
interactive operator/validation flows only.
- **Lane:** depends on M2/M3. Serial: slash-commands touch `Actions.Runner` + read
  paths (db_serial for `/events`/`/confirmations`); the launcher manages app
  lifecycle (named processes). Parallel: launcher ∥ slash-commands.
- **Focused:** a full validation turn (send → approve → inspect) completes in one
  session with no BEAM restart; each slash-command resolves against the warm node.
- **Rejoin:** interactive operator validation runs warm, persistent, mix-free.

### M5: Evals + Docs

- **Lane:** parallel doc-writing lanes ∥ eval-row authoring.
- **Focused:** `:v055` evals green on stubs; `release.v055` deterministic +

- **Rejoin:** punchlist + parity + TUI guide written; deterministic gate green.

### M6: Operator Validation (manual gate)
A real operator runs the request-flow A1–A6 punchlist **through the persistent TUI
console** (M4) — one warm BEAM, no DB reopen — and records evidence. This is the
manual real-operator gate; **it blocks release (M7)**.
- **Lane:** serial, manual, single-operator, external-terminal — not
  parallelizable.
- **Focused / gate:** A1 (turn + dedupe), A2 (identity rejection), A3 (approval
  render+resolve), A4 (crash isolation), A5 (Matrix outbound), A6 (split-payload
  seam) all pass with recorded evidence.
- **Rejoin:** operator-validation evidence recorded.

- **Lane:** serial closeout.
- **Focused:** full release gate passes; docs agree.
- **Rejoin:** `v0.55.0` tagged with operator-validation + eval evidence.

## Non-Goals
- No CLI (`mix allbert.ask`) promotion to a channel.
- No `:button`/`:link` primitives in the terminal.
- No new authority for any channel; channels stay delivery adapters.
- No streaming contract change (records the current turn-complete reality).

## Definition Of Done
v0.55 is done when:
- an explicit, machine-derived channel capability/parity matrix exists and matches
  the registry (`mix allbert.channels --parity` + `docs/developer/channel-parity.md`),
  with every shipped descriptor declaring `primitives:`;
- a persistent TUI/terminal channel (`plugins/allbert.tui`) runs under
  `Channels.Supervisor`, normalizes terminal input through the channel substrate
  (identity map, session derivation, `channel_events` dedupe), and renders
  responses + `:list`/`:typed_command` approvals, resolving confirmations via the
  Runner;
- the TUI carries no authority beyond any other channel (eval rows pass), and a
  TUI crash is isolated;
- the TUI renders scrollback-native (static lines + one live block), and the split
  tool-result payload seam + live region exist as the v0.57 Pi-mode foundation;
- the persistent `mix allbert.tui` operator console runs interactive validation in
  one warm BEAM (no cold per-turn `mix` calls);
- the **M6 operator-validation punchlist (A1–A6) passed through the TUI console
  with recorded evidence before tag**;
- `:v055` evals + `release.v055` + the full release gate pass;
- ADR 0016 v0.55 amendment + ADR 0067 are Accepted; version is `0.55.0` and tagged;
- ADR 0016 v0.55 amendment, roadmap, vision, and agent-context-map agree.
