## Purpose

Re-layout `/workspace` so **chat is the primary surface**, modeled on the
ChatGPT and Claude chat-UI grammar (Hermes — https://hermes-agent.nousresearch.com/
— informs the conversation-first framing; its public site documents a multi-channel
conversational agent, not a specific web component grammar). This is a
**re-skin / re-layout of the existing ADR 0023/0024 Surface substrate, not a
rebuild**:
- Chat becomes the primary surface (today it is the left pane of a 2-pane
  chat+canvas grid — the split is CSS, `grid-template-areas` in `app.css`).
- Ephemeral surfaces become **modals/popovers** (they already carry
  `role="dialog"`/`aria-modal`/FocusTrap/escape-dismiss; only the CSS places them
  inline in a bottom row).
- The canvas is **demoted** to a launcher destination / side-drawer (reusing the
  existing `canvas_destination` routing), no longer a co-equal pane.
- UI labels cleaned up — **"Conversations"** replaces the "threads" label
  (UI strings only; the internal `Conversations.Thread` module/schema/DB, catalog
  atoms, events, signal topics, and settings keys are unchanged, and the distinct
  volatile `Session.Scratchpad` concept is untouched — chosen to avoid the
  thread/session terminology collision).

## Readiness Sweep
- **Re-skin fit.** The shell IS a Surface tree (ADR 0023 §2); the 2-pane split,
  maximize/focus, and ephemeral placement are CSS + presentation, so chat-primary
  is largely a CoreApp shell-tree + `app.css` + presentation change. ADR 0024's
  v0.34 revision already made nav a view-only launcher and canvas a single
  destination — the substrate is already chat-and-launcher oriented.
- **Authority fit.** No new authority/route/permission; launcher selection stays
  view-only; `active_app` only via v0.33 handoff; fragment emission stays
  HMAC-validated; rendering executes no model-generated UI (catalog is the
  boundary).
- **Terminology fit.** "Conversations" matches the internal `Conversations`
  module while `Thread` stays the row type — no internal rename, no collision.

## Milestones

### M1: Chat-Primary Layout
Re-declare the CoreApp `:workspace` shell so chat is the main column with a
collapsible left **Conversations** rail and a pinned composer; rewrite the
`app.css` grid (`nav | chat | resizer | canvas` → chat-dominant). Header gets the
existing conversation switcher (+ optional model switcher via existing
`:button`/`:status_badge`). Seams: `core_app.ex` `workspace_nodes/0`, `app.css`
grid + maximize/focus rules, `renderer.ex` split-resizer.

### M2: Ephemeral → Modal/Popover
Move the `ephemeral` grid-area to a fixed overlay + backdrop; promote the active
ephemeral to a centered modal, stack the rest. Reuse the shipped dialog/focus-trap/
escape-dismiss semantics and `dismiss_workspace_ephemeral`. Seams: `app.css`
`.workspace-ephemeral-node*`, `renderer.ex` ephemeral branch.

### M3: Canvas Demotion
Canvas becomes a launcher destination / slide-over drawer rather than a permanent
pane; default view is chat. Reuse `canvas_destination` routing + focus/maximize
toggles; update mobile tabs + canvas empty-state copy. Seams: `workspace/catalog.ex`,
`workspace_live.ex`.

### M4: "Conversations" Relabel (UI strings only)
Replace the user-facing literals (`core_app.ex` thread-list title;
`structural.ex` "New thread"/"Recent workspace threads"/"Threads"/"No threads
yet"/switcher copy; `jobs_live.ex` table header; `workspace_live.ex` notice).
Leave canvas-internal "this thread" wording or relabel to "this conversation" per
copy review. Guard: zero diffs to `thread_id`, atoms, events, topics, settings
keys.

### M5: Polish / Accessibility / Evals / Closeout
WCAG re-check (real modals → focus traps, keyboard nav, reduced-motion),
dark-mode parity, mobile single-column; `:v055` eval rows; `release.v055`;
closeout discipline; ADR 0024 v0.55 revision accepted.

### M6: Browser Manual Validation
Real browser pass on `/workspace`: chat-primary renders; new/switch conversation;
ephemeral approval card appears as a modal and escapes/dismisses; canvas reachable
as launcher/drawer with tiles intact; relabels visible; streaming + message
affordances work; offline shell still registers. Capture screenshots.

## Non-Goals
- No removal of the canvas/ephemeral substrate (re-skin, not rebuild).
- No internal `Thread`→`Session` rename; no DB/schema rename.
- No model-generated UI code; the catalog stays the rendering boundary.
- No new routes; `/workspace` stays canonical.

## Definition Of Done
v0.55 is done when:
- `/workspace` renders chat-primary (single main chat column + collapsible
  Conversations rail + pinned composer);
- ephemeral surfaces render as modals/popovers (dialog semantics + dismiss),
  not an inline bottom row;
- the canvas is reachable as a launcher destination / drawer, default view is
  chat, and durable tiles + offline edit + multi-tab sync are preserved;
- the UI says "Conversations" with **zero** internal identifier churn (guard
  eval passes);
- authority is unchanged (fragment HMAC + view-only launcher evals pass);
- `:v055` evals + `release.v055` + the full release gate pass, plus a real
  browser manual-validation pass;
- ADR 0024 v0.55 revision, roadmap, vision, and agent-context-map agree.
