## Purpose

Three goals:
1. **Channel parity** — make the cross-channel capability matrix explicit and
   machine-checkable (which approval primitives `{:list,:button,:typed_command,
   :link}`, attachments, streaming, identity mapping, and approval rendering each
   channel supports), and close the gaps, including the v0.54-deferred Matrix
   generic outbound implementation behind `Channels.Outbound`.
2. **A proper TUI/terminal channel** — a persistent supervised terminal channel
   that is a real ADR 0016 channel (descriptor + `channel_events` dedupe +
   identity mapping + Approval Handoff + redaction), distinct from the one-shot
   `mix allbert.ask` label.
3. **Split tool-result foundation** — harvest Pi's split tool-result pattern
   (`model_payload` separate from `surface_payload`; ADR 0067 extending ADR
   0029/0030) into the typed response contract so the TUI can
   render terminal-specific framing without leaking it into conversation history.
   This is the substrate the v0.57 Pi-mode coding surface builds on; true
   streamed diff/token semantics remain v0.57 scope.

## Readiness Sweep
- **Contract fit.** Channels are delivery adapters around
  `Runtime.submit_user_input/1` + registered actions (ADR 0016 §boundary); they
  do NOT own intent/actions/Security Central/Resource Access/confirmation storage/
  conversation history. The TUI is just another adapter under
  `Channels.Supervisor` (`:one_for_one`) and must use the ADR 0057
  `ChannelThread` substrate for conversation continuity.
- **Reuse fit.** `plugins/allbert.telegram` is the adapter template
  (`channels/0` descriptor → supervised GenServer → normalize → `Identity.resolve`
  → `Channels.derive_session_id` → `Runtime.submit_user_input` → `channel_events`).
  Email's `ALLBERT:APPROVE:<id>` renderer is the `:typed_command` template.
- **Upstream check.** Audit 2026-06-21 confirms v0.52 delivered `primitives:` +
  `Approval.Handoff` + registry validation and ADR 0057 delivered `threading:` +
  `Conversations.ChannelThread` + owner/account/key mapping — all shipped. The M0
  block-if-not check passes.
- **Library fit.** `owl` is present but **transitive** (via `igniter`, `:dev,:test`
  only), so M0 must promote it to a direct runtime dep. It is pure-Elixir and
  explicitly top-to-bottom (not full-screen), matching the scrollback-native
  rendering model (Locked Decision #7): `Owl.IO`/`Owl.Data`/`Owl.LiveScreen` cover
  input, styled lines, and the single live block. `ratatouille`/`ex_termbox` are
  rejected (dormant, native NIF, full-screen takeover that loses scrollback).

## Locked Decisions
| # | Decision | Chosen value | Trade-off if changed |
|---|---|---|---|
| 1 | TUI authority | The TUI is a delivery adapter with **no authority beyond any other channel** (ADR 0016). | A privileged channel breaks the boundary. |
| 2 | TUI library | **`owl`** (maintained, pure-Elixir) over `ratatouille` (dormant, `ex_termbox` NIF). Promote owl from transitive (`igniter`, `:dev,:test`) to a direct, top-level **runtime** dep. | ratatouille adds a native-build + unmaintained dependency and is full-screen (loses scrollback); owl is top-to-bottom and matches the rendering model. |
| 3 | CLI promotion | Do **not** promote `mix allbert.ask` (stays a non-channel developer label, `channel: :cli`). The TUI is the first-class persistent terminal channel (`channel_id: "tui"`, provider `"terminal"`). | Promoting the one-shot task duplicates the TUI and muddles the label. |
| 4 | TUI primitives | Declare `primitives: [:typed_command, :list]` (numbered options + `ALLBERT:APPROVE:<id>`); **not** `:button`/`:link` (no GUI affordance / no in-terminal workspace URL). `:list` is the mandatory universal fallback. | Declaring `:button`/`:link` would advertise affordances the terminal can't honor. |
| 5 | TUI session/identity/threading | Maps a terminal profile to local `user_id` via the existing list-shaped channel identity-map contract: `channels.tui.identity_map = [%{"external_user_id" => "default", "user_id" => "local", "enabled" => true}]`. No implicit `"local"` claim. The descriptor declares `session_strategy: {:tui_session, prefix: "ch_tui_"}` and uses the existing descriptor-driven `Channels.derive_session_id/3` strategy branch unless implementation proves a new special case is necessary. It declares `threading: :rich` and writes through `ChannelThread` with `owner_scope: "local"`, `receiver_account_ref: "tui:<profile>"`, and a deterministic `provider_thread_key` for the terminal conversation. | A TUI-only identity shape or implicit local claim violates ADR 0016 and forks Settings/identity behavior. Bypassing ADR 0057 would fork terminal conversation history from web/Slack/Discord/mobile continuity. |
| 6 | Parity matrix home | Canonical prose table in the ADR 0016 amendment + machine-derived from descriptors (`ChannelParity` / `mix allbert.channels --parity`) + `docs/developer/channel-parity.md`. `mix allbert.channels status` is **not** v0.55 scope; it lands in v0.55.1 with the operator console. v0.55 crash/isolation evidence comes from the deterministic gate plus the live TUI session evidence in the request-flow punchlist. | A prose-only matrix drifts from the registry truth; pulling status back into v0.55 would blur the split with v0.55.1. |
| 7 | TUI rendering model | **Scrollback-native, line-oriented (Pi model).** Completed turns are static scrollback lines (`Owl.Data`/`IO.ANSI`); a single `Owl.LiveScreen` block renders the active prompt + in-progress status/live line (differential bottom-region, synchronized-output where supported). No alternate-screen, no full-viewport redraw. No v0.55 token/diff streaming contract. | A full-screen/alt-screen TUI (`ratatouille`/`ex_termbox`) loses native scrollback + search and boxes out the v0.57 Pi-mode streaming substrate. |

## Foundation For v0.57 Pi-mode

v0.55 is the foundation v0.57 Pi-mode (ADR 0068) extends **additively** — Pi-mode
must need no rework of this channel. v0.55 therefore lands these seams:

- **Same channel + adapter loop.** Pi-mode runs *in* the `tui` channel and its
  persistent session (the basic v0.55 launcher, hardened into the v0.55.1
  console); it adds `read`/`write`/`edit`/`bash` actions +
  a trust tier, it does **not** fork a new channel or runtime.
- **Split tool-result payload seam.** v0.55 establishes the model-facing vs
  surface-render split (ADR 0067 extending ADR 0029/0030) so Pi-mode's streamed
  diffs and tool-result rendering slot in without touching the response contract.
- **Live-region render substrate.** The single `Owl.LiveScreen` block (Locked
  Decision #7) is the surface Pi-mode later streams diffs into.
- **Level-0-compatible boundary.** v0.55 resolves the terminal profile through
  the same list-shaped channel identity map to a configured local `user_id`
  (the documented default maps `"default"` to `"local"`; there is no implicit
  local claim) and runs every action through `Actions.Runner.run/3` + Security
  Central, with no assumption (per-action confirmation, identity model) that
  blocks v0.57 adding the named "local coding / sandbox level 0" trust tier on
  the same adapter.
- **Session/threading shaped for a coding mode.** The descriptor-owned
  `:tui_session` strategy + `ChannelThread` are structured so a Pi-mode coding
  session is a *mode* on the same channel.

### M3: Approval-Primitive Rendering In TUI
TUI `render_approval_handoff/1` consumes `{primitive_kind, payload}` from

like Telegram. No primitive embeds targets/commands/URLs (ADR 0016).
- **Lane:** depends on M2; focused single-channel.
- **Focused:** primitives never embed targets/commands/URLs; confirmation resolve
  isolated from other channels.
- **Rejoin:** approve/deny/show work in the TUI like Telegram.
- **Implementation status (2026-06-21):** landed. The TUI renderer consumes

  adapter parses typed commands before runtime submission and resolves them via
  `AllbertAssist.Channels.ConfirmationCallback`, including identity proof and
  same-channel checks; wrong-channel confirmations are rejected without runtime
  submission.

## Non-Goals
- No CLI (`mix allbert.ask`) promotion to a channel.
- No `:button`/`:link` primitives in the terminal.
- No new authority for any channel; channels stay delivery adapters.
- No token/diff streaming contract change in v0.55 (records the current
  turn-complete runtime reality). v0.55 lands the split payload and live-region
  render substrate that v0.57 Pi-mode can stream over.

## Definition Of Done
v0.55 is done when:
- an explicit, machine-derived channel capability/parity matrix exists and matches
  the registry (`mix allbert.channels --parity` + `docs/developer/channel-parity.md`),
  with every shipped descriptor declaring `primitives:`;
- a persistent TUI/terminal channel (`plugins/allbert.tui`) runs under
  `Channels.Supervisor`, normalizes terminal input through the channel substrate
  (identity map, session derivation, `channel_events` dedupe), and renders
  responses + `:list`/`:typed_command` approvals, resolving confirmations via the
  Runner;
- the TUI carries no authority beyond any other channel (eval rows pass), and a
  TUI crash is isolated;
- the TUI renders scrollback-native (static lines + one live block), and the split
  tool-result payload seam + live region exist as the v0.57 Pi-mode foundation;
- the basic `mix allbert.tui` launcher runs interactive validation in one warm
  BEAM;
- the **M5 operator-validation punchlist (A1–A5) passed through the `mix allbert.tui`
  session with recorded evidence before tag**, and the separate Matrix outbound
  real-provider smoke passed;
- `:v055` evals + `release.v055` + the full release gate pass;
- ADR 0016 v0.55 amendment + ADR 0067 are Accepted; version is `0.55.0` and tagged;
- ADR 0016 v0.55 amendment, roadmap, vision, and agent-context-map agree.
