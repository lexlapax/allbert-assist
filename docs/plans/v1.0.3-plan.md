## Purpose

Slowness and flakiness share one root cause — global-state tests forced into
serial lanes (the v0.53 thesis, proven by phase 1). Phase 1 (v1.0.2) made
the lanes HONEST (reconciliation 173→0, residue de-flaked, web split,
cost-packed partitions, decide-turn remediation) but converted only a
handful of tests. Phase 2 does the actual migration: convert the measured
serial floor to async where an ownership seam exists, retire the two
monolith-only failure classes at their global-state root, take the last
attributed decide-turn hot path, and then ship the catch-up binary so the
packaged line finally carries the v1.0.1 + v1.0.2 fixes.

The acceptance bar (AGENTS.md: Purpose is the bar): measured lane-floor
reduction with ZERO test-feature loss — every conversion proves identical
test totals and unchanged assertions; the 20-seed monolith campaign rerun
shows both residual classes at 0; `release.v1` + `release.v103` green; the
packaged catch-up validated on a real host.

**Why each lane is serial (the conversion target classes).**
- `db_serial`: shared global SQLite DB writes; `DataCase` non-async;
  `pool: Ecto.Adapters.SQL.Sandbox` is ALREADY configured (config/test.exs)
  but ownership is not exercised per-test — conversion = real sandbox
  checkout ownership per test (ADR 0086 contract 1).
- `app_env_serial`: tests mutate `Application.put_env` globals (Paths,
  Settings config); conversion = process-scoped context via the existing
  seams — ADR 0082 registry-context opts + `Store.with_resolved_settings/1`
  pin (contract 2).
- `global_process_serial`: named singleton processes (registries, agents)
  collide across tests; conversion = per-test Registry-addressed instances
  through the ADR 0082 injection seam (contract 3).
- `home_fs_serial`: shared ALLBERT_HOME filesystem; the pid-qualified
  per-test home idiom (M8.3/M8.7) makes most of these convertible
  (contract 4).
- `external_runtime_serial`: real external runtime state (browser, OS
  processes, ports). NOT converted — evaluated for multi-VM partitioning
  with owned envs (contract 5, measured go/no-go).

## Locked Decisions

1. **Phase 2 + catch-up binary are ONE version (operator, 2026-07-19).**
   Rationale recorded above. M9 is the transferred binary scope.
2. **Conversion order is store-ranked (derived).** Waves work the measured
   floor top-down (lane max-partition × file cost), never alphabetically or
   by convenience; every wave's acceptance cites pre/post from the store on
   identical commands (the pre/post metrics rule).
3. **No test-feature loss, proven per wave (derived from the Purpose bar).**
   A conversion changes HOW a test isolates (ownership), never WHAT it
   asserts. Per-wave proof: lane totals identical (moved tests counted in
   their new lane), assertions untouched (identity review), full accuracy
   suites unchanged, and the wave's files green solo AND in composition
   both orders (the residue-matrix pattern).
4. **ADR 0086 binds the conversion contracts (derived).** One contract per
   serial class (sandbox ownership, app-env context, named-process
   injection, home-fs pid idiom, external-runtime partitioning go/no-go);
   a test converts ONLY through its class contract — no bespoke isolation
   inventions per file.
5. **Monolith classes are fixed at the OWNERSHIP root, not the symptom
   (derived).** No retries, no test deletions, no monolith-specific skips:
   Sidebar's fix is sandbox allowance threading; ListChannels' fix is
   registry convergence + context adoption. Acceptance is the rerun
   campaign at 0/10 for both.
6. **external_runtime stays honest (derived).** If M6's partitioning
   experiment cannot prove owned-env isolation across VMs (browser/port
   state), the lane STAYS single-VM and the measured no-go is recorded —
   an honest floor beats a flaky split. (The v0.66 lesson: never gate the
   whole workspace_live battery on optimism.)
7. **Binary scope transfers verbatim (operator).** M9 = the v1.0.2 plan's
   M9 text with 1.0.2→1.0.3 substitutions: operator-held annotated tag
   WITHOUT `[skip-artifacts]`, CI tarballs + SHA256SUMS cosign bundle,
   Latest 1.0.0→1.0.3, tap fill (separate repo, manual sha256), install
   rehearsal, published-artifact §F–§I + spot smokes. Version bumps (mix
   trio, SW CACHE_NAME, gz, formula template) land at M8 RC.

## Code Seams

- **S1 — sandbox ownership (M1/M2, ADR 0086 c1).** `DataCase` gains an
  async-capable ownership mode: explicit `Ecto.Adapters.SQL.Sandbox`
  checkout per test + allowance helpers for spawned processes (LiveView
  mounts, Tasks, agents). Pilot on the Sidebar neighborhood (the
  DBConnection class), then the db_serial wave (M4 wave A).
- **S2 — app-env context adoption (M1/M5, c2).** Tests reading
  Paths/Settings via `Application.get_env` move to the pinned/context read
  path; writes become per-test pins, not global put_env. Files whose
  PRODUCTION code path still reads globals get the seam gap recorded (the
  engine `channel_candidates` list seeds this) and stay serial with reason.
- **S3 — named-process injection (M1/M6, c3).** ADR 0082 pattern: the
  test-facing constructor takes a registry/name override; per-test unique
  names via the pid-qualified idiom.
- **S4 — monolith-class retirement (M2/M3).** Sidebar: thread sandbox
  allowances through the LiveView/live-Runtime boundary
  (`workspace_live_case.ex` + the 5-test remainder). ListChannels:
  ShippedRegistries convergence for its monolith neighbors + registry
  context in the SurfacePolicy read it exercises. Red-first: reproduce each
  class OUTSIDE the monolith with a minimal composition before fixing.
- **S5 — external-runtime partition experiment (M6, c5).** Attempt 2-VM
  packed partitioning with fully owned envs (per-VM homes/DBs/ports);
  accept only on 3× consecutive green + no cross-VM interference evidence;
  otherwise record the measured no-go.
- **S6 — prefilter tokenization (M7, production).** Profile-attributed:
  cache/precompile the prefilter's per-turn regex work (compile-constant
  patterns to persistent_term, single-pass tokenization where semantics
  are provably identical — the M8.8 red-first/identity discipline).
- **S7 — release plumbing (M8/M9).** `release.v103` gate =
  `@release_v1_steps ++` focused v1.0.3 steps (per-wave conversion proofs,
  campaign-class regression solos, packer/no-loss checks); version bumps;
  CHANGELOG; the transferred binary publication.

  before all waves (contracts before conversions). M2/M3 before M8's
  campaign (class fixes need seeds to prove). M7 independent of waves

  M4/M5/M6 waves can interleave once M1's contracts exist — but never two

  strictly sequential.

  OWNER APP directory (root `mix test` path rule, v1.1 plan matrix
  precedent); `inventory --check-tags` after every lane move (conversions
  RE-TAG files — the check proves the move); no `mix | tail` without
  unpiped exit capture; metrics store records cited for every claim.
- **Full-precommit timing:** M8 RC assembly and once at M0 baseline.
