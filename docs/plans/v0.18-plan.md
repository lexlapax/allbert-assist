# Allbert v0.18 StockSage Plugin, Umbrella App, And Domain

Formerly tracked as M-D2a.

## Purpose

v0.18 introduces StockSage as the first plugin-contributed domain app inside
the Allbert workspace. It adds a source-tree plugin package under
`./plugins/stocksage/`, the `stocksage` umbrella app, SQLite-first domain
storage, local skill pack, and import path from the existing Python
`stocksage.db`.

## Expected Inputs

- v0.12 local string `user_id` exists.
- v0.15 minimal app registration contract exists.
- v0.17 plugin contract exists, including source-tree plugin packages,
  plugin-contributed apps, skill roots, actions, settings schema entries, and
  diagnostics.
- `aiworkspace-plan.md` has been folded into `allbert-jido-vision.md` as
  durable product direction.

## Expected Direction

- Add `./plugins/stocksage/` with `allbert_plugin.json`, README,
  `StockSage.Plugin`, and StockSage skill roots.
- Add `apps/stocksage/` as the OTP core app for domain storage, migrations,
  contexts, and supervision.
- Add `apps/stocksage_web/` as a stub Phoenix surface app for later v0.23
  LiveViews.
- Implement `StockSage.Plugin` as the plugin entrypoint.
- Implement `StockSage.App` using the v0.15 lite app contract and contribute it
  through `StockSage.Plugin`.
- Add SQLite-backed domain records for `Analysis`, `AnalysisDetail`,
  `Outcome`, `AnalysisQueue`, `QueueRun`, and `MemoryEntry`.
- Every user-facing/request/queue row carries string `user_id`.
- Add StockSage local skill pack: `run-analysis`, `get-trends`, and
  `queue-analysis`, contributed from `./plugins/stocksage`.
- Add `mix stocksage.import_sqlite /path/to/stocksage.db`.
- Preserve the no-server local posture: no PostgreSQL, no Oban hard
  dependency.

## Non-Goals

- No Python bridge execution; that is v0.20.
- No native Jido trading agents; that is v0.22.
- No StockSage LiveViews; that is v0.23.
- No account foreign keys.
- No plugin-system changes beyond consuming the v0.17 source-tree plugin
  contract.

## Test Plan

- StockSage domain records round-trip through SQLite.
- Python `stocksage.db` import is idempotent for representative fixtures.
- `StockSage.Plugin` appears in `mix allbert.plugins list` as a source-tree
  plugin from `./plugins/stocksage`.
- `StockSage.App` is registered through the plugin/app bootstrap path.
- StockSage skills appear in `mix allbert.skills list` from the
  plugin-contributed skill root.
- No local PostgreSQL server is required.

## Definition Of Done

v0.18 is done when StockSage exists as a plugin-contributed registered local
Allbert app with SQLite-backed domain records, importable historical data, and
discoverable skills.
