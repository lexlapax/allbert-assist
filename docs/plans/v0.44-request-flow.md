## Main User/Operator Flows

- Operator asks Allbert to plan a multi-step task.
- Operator reviews objective steps, resources, confirmations, and failure
  preview.
- Operator edits, approves, or cancels the plan.
- Operator submits workflow YAML that expands to objective steps.
- Background objective progress renders in workspace and origin channel.

## Data Flow

Plan input becomes an objective draft, candidate steps are validated, workflow
YAML is parsed as declarative data, approved steps execute through
`Actions.Runner.run/3`, and objective events drive status rendering.

## Authority Invariants

Plans and YAML do not execute. Objective ids are not authority. Every step keeps
normal action permission and confirmation behavior. Unknown YAML keys fail
closed.
