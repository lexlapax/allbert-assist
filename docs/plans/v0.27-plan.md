# Allbert v0.27 App Surface Contract - StockSage LiveViews

Request flow: `docs/plans/v0.27-request-flow.md`

Formerly tracked as M-D3a, previously planned as v0.23, then v0.24, then
v0.25 before the v0.23 Jido State-Machine Convergence and v0.24 Objective
Runtime Foundation milestones were inserted by the project-direction rethink.
Renamed after the post-v0.26 roadmap reconciliation to make the contract
visible: StockSage is the reference implementation for the app surface
contract, not a special-case app.

## Purpose

v0.27 proves that a plugin-contributed app can expose real Phoenix LiveView
surfaces through `AllbertAssist.App.SurfaceProvider` from day one. StockSage
is the first non-core app to carry that proof.

The milestone ships real StockSage renderers in StockSage-owned
`/stocksage/...` LiveViews. These renderers replace the v0.26 reserved-card
stubs for `:analysis_card`, `:agent_report_card`, `:parity_card`, and
`:debate_round_card` when those cards are displayed in the StockSage app
surface. v0.30 later wires the same proven component types into durable
`/agent` workspace canvas tiles through `canvas_ops` and the audited v0.26
canvas mechanism. v0.27 is renderer and app-surface proof; v0.30 is durable
canvas-emission proof.

v0.27 also adds app memory namespace declaration/registration as an ownership
claim, but does not sync lessons, reflections, or analysis results into
Allbert memory. That separation lets v0.28 audit namespace claim abuse and
namespace isolation before v0.29 adds privileged memory writes.

## Implementation Research Notes

Current Phoenix docs consulted during planning:

- Phoenix LiveView streams support incremental collection rendering with
  `stream/3`, `stream_insert/4`, and `phx-update="stream"`. v0.27 should use
  this shape for analysis progress lists rather than replacing the full page.
- Phoenix.PubSub provides `subscribe/3` and `broadcast/4` over named topics.
  v0.27 progress events should subscribe per analysis/objective/user context
  and catch up from persisted StockSage/objective state on reconnect.
- LiveView JS commands and hooks are appropriate only for client-owned behavior
  such as focus/keyboard helpers or chart integration; StockSage domain state
  remains server-owned behind contexts/actions.

## Expected Direction

- Add plugin-owned `StockSageLive.WorkspaceLive`, `AnalysisLive`,
  `QueueLive`, and `TrendsLive` modules under `plugins/stocksage`.
- Extend `StockSage.App` with concrete `/stocksage/...` surfaces declared via
  `AllbertAssist.App.SurfaceProvider` and `AllbertAssist.Surface`; route
  mounts live in the host router, while navigation metadata comes from the
  provider.
- Implement real StockSage card renderers for `:analysis_card`,
  `:agent_report_card`, `:parity_card`, and `:debate_round_card`. These
  renderers must display persisted native and bridge analysis output, objective
  state, specialist reports, debate rounds, parity comparisons, warnings, and
  trace/provenance links without arbitrary model-generated UI.
- Declare StockSage component/catalog entries such as `:analysis_card`,
  `:queue_entry`, and `:trend_summary` through the Surface DSL. The component
  types are declared and rendered in StockSage app surfaces here; v0.30 owns
  durable `/agent` canvas emission through `canvas_ops`.
- Update `StockSage.Actions.RunAnalysis` results to include validated
  `AllbertAssist.Surface.Node` output for analysis results. Nodes are validated
  at action completion and render inside StockSage LiveViews in this milestone.
- Add app memory namespace declaration/registration to
  `AllbertAssist.App.Registry` and have `StockSage.App` declare its namespace.
  Namespace declaration is ownership metadata only in v0.27; no memory sync,
  reflections, lesson promotion, or markdown-memory writes are added here.
- Stream StockSage analysis progress through Phoenix.PubSub and LiveView
  `stream/3` without page reloads.
- Set `active_app: :stocksage` in scratchpad when navigating under
  `/stocksage/`.
- Add app-flow UX for StockSage surfaces: loading, empty, error, and failure
  states; keyboard/focus behavior; mobile fit; and clear bridge/native/parity
  labeling.
- Keep LiveViews thin. Domain behavior stays behind StockSage contexts,
  registered actions, objectives, and runtime boundaries.

## Milestones

- M1: SurfaceProvider/routes - declare StockSage surfaces, mount routes, and
  validate app registration.
- M2: App memory namespace declaration - add registry support and StockSage
  namespace declaration without memory writes.
- M3: Real card renderers - replace v0.26 reserved-card stubs in StockSage
  app surfaces.
- M4: RunAnalysis Surface nodes - return and validate analysis result nodes.
- M5: Objective and confirmation rendering - show objective state, steps,
  confirmation links, and cancellation affordances.
- M6: Progress streaming - stream analysis progress through PubSub/LiveView.
- M7: App-flow UX - finish empty/loading/error/mobile/focus behavior.
- M8: Closeout - focused tests pass and `mix precommit` passes.

## Non-Goals

- No durable `/agent` canvas tile emission and no `canvas_ops`; that is v0.30.
- No memory sync, reflections, lesson promotion, or markdown-memory writes;
  that is v0.29.
- No AG-UI protocol emission. Surface nodes are Allbert-native validated
  structures.
- No arbitrary model-generated component shapes.
- No PostgreSQL or Oban dependency.

## Test Plan

- `StockSage.App` implements `AllbertAssist.App.SurfaceProvider`;
  `mix allbert.validate_app StockSage.App` passes.
- `AllbertAssist.App.Registry.registered_apps/0` includes StockSage surfaces,
  route metadata, catalog entries, and memory namespace declaration.
- StockSage LiveViews load from `/stocksage/...` routes and set
  `active_app: :stocksage`.
- Real card renderers display native, bridge, and parity result fixtures
  without falling back to v0.26 placeholder stubs.
- `RunAnalysis` action results carry validated `AllbertAssist.Surface.Node`
  output; undeclared component types are rejected.
- Namespace declaration is visible to v0.28 evals, while memory sync/write
  actions remain absent.
- Live progress updates stream without polling or page reload.
- Loading, empty, error, mobile, and keyboard/focus behavior pass focused
  LiveView tests and manual review.
- Milestone closeout runs the full gate: compile warnings-as-errors, format
  check, Credo strict, Dialyzer, and `mix precommit`.

## Definition Of Done

v0.27 is done when StockSage is a usable plugin-contributed web app through
declared `SurfaceProvider` surfaces; real StockSage renderers replace the
reserved v0.26 card stubs inside `/stocksage/...`; `RunAnalysis` emits
validated Surface nodes; app memory namespace ownership is declared but not
consumed; progress streams live; and the v0.30 durable-canvas boundary is
documented and unimplemented.
