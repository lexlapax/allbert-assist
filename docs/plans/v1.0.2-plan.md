## Purpose

Retire the test suite's structural debt in one deliberate phase, and put the
1.0.1 fixes in packaged users' hands:

1. **De-flake the documented residue.** Five order/selection-dependent failures
   are individually root-caused (see Current Code State); the v1.0.1 build tripped
   over four of them repeatedly. Each becomes deterministically green solo AND in
   batch, red-first where the fix has a testable seam.
2. **Shrink the serial surface.** Virtually the whole core suite (~291
   explicit-serial files + 127 DataCase + 28 SecurityEvalCase vs 21 `pure_async`)
   is serial-within-a-VM — the dominant wall-clock cost and the flake source. Add
   the missing registry injection seam (ADR 0082), then convert the
   seam-unblocked and trivially-isolatable files to `pure_async`/partitionable
   lanes, lane by lane.
3. **Split the web long pole.** `WorkspaceLiveTest` alone is ~256.6s — ~83% of web
   wall-clock — and the only web `external_runtime_serial` file. Split it into
   partitionable `liveview_serial` files per the v0.41 M9 deferral's "own safety
   story" requirement.
4. **Clear the queued tails**: v0.58 cleanup tails A+B, the Tier-2→Tier-1
   promotion ADR, and the four scoped-out dependency updates.
5. **Binary ship**: `release.v102` gate, 1.0.2 bump, tag WITHOUT
   `[skip-artifacts]`, CI publish + cosign, tap fill (1.0.0 → 1.0.2), install
   rehearsal, and the packaged §F–§I catch-up attestations.

## Locked Decisions

1. **Binary release; Latest moves to 1.0.2.** Tag WITHOUT `[skip-artifacts]`;
   full CI build → cosign → publish; manual tap fill (1.0.0 → 1.0.2); the
   packaged §F–§I re-attestations against the PUBLISHED artifact are the v1.0.1
   artifact-line catch-up proof and part of this release's Definition of Done.
2. **Per-test isolation, never invariant weakening.** The ADR 0015/0031
   registration↔Settings-schema coupling stands; every de-flake seeds or owns
   state per test. Production code gains injection SEAMS (default = the global
   singleton) — behavior in production is unchanged, proven by the freeze sweep,
   Dialyzer, and zero-diff default paths (ADR 0082 binds this).
3. **Seam before conversion.** M2's registry thread-through lands and is proven
   (default-path zero-diff) before any M3 lane demotion that depends on it.
   Files that cannot isolate cleanly stay in their serial lane — no tag changes
   without the isolation actually done (the reconciliation rule stays honest).
4. **Security-eval and external-runtime lanes stay serial by policy.** No
   promotion of `security_eval_serial`/`external_runtime_serial` files; their
   fix (where needed) is solo-determinism via seeding, not lane demotion.
5. **The web split has a safety story** (the v0.41 M9 condition): a split
   `WorkspaceLiveTest` file may carry `liveview_serial` only if it runs with
   `server: false` semantics, owns its home/settings state, and touches no
   runtime singleton that the partition runner doesn't own — the checklist is
   codified in test-strategy.md as part of M4, and every split file passes
   `inventory --check-tags` + the solo/batch determinism proof.
6. **De-flake is red-first where a seam exists**: each residue fix demonstrates
   the failure solo on the pre-fix code, then green solo AND green in the batch
   combination that previously flipped it (both orders recorded in Build
   Progress).
7. **Measurable acceptance, honest scope.** Acceptance = the five named residues
   deterministic; the seam landed; wave-1 conversions + the web split measured
   against the M0 re-baseline. The "~20" full-`mix test` estimate is re-measured
   and recorded at M9 — reducing it is the goal; eliminating every unknown
   order-dependence is NOT promised by phase 1 (remainder stays on the track's
   future-features entry with the new count).
8. **Dependency refresh per the standing rule**: review-always (three-root
   `hex.outdated`, dedup, disposition table), apply-bounded (the four verdicted
   items in two waves: swoosh+tailwind+jsv, then the ecto pair with the exqlite
   lock-diff decision recorded), own commit(s).
9. **ADR 0081 (Tier-2→Tier-1 promotion) is a docs deliverable** — it changes no
   code and no current tier assignment; it defines the process (criteria,
   freeze-sweep row addition, versioning interplay) for future promotions.
10. **No new EvalInventory rows / no `:v102` milestone atom.** Nothing here adds
    authority. `@release_v102_steps = @release_v1_steps ++ @v102_focused_steps`
    (precedent: v066/v101 — no predecessor bespoke steps).
