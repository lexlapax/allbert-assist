# v0.10 — Provider expansion and local-first model default

Status: Proposed

## Context

v0.1 intentionally shipped two providers, Anthropic and OpenRouter, to force the provider seam to exist early. By v0.9 that seam had proven useful, but it was too narrow for the next phase of local-first operation:

- fresh profiles still assumed a hosted API key;
- OpenAI and Gemini, two common direct-provider choices, were missing;
- local Ollama models could not be used through the normal provider path;
- job, agent, daemon protocol, setup, and REPL model selection all assumed every provider had an API-key env var.

v0.10 widens the provider layer while keeping Allbert's owned provider seam. Fresh profiles default to local Ollama with `gemma4`. Remote providers remain available for operators who prefer Anthropic, OpenRouter, OpenAI, or Gemini.

## Research checkpoint

| Topic | Decision |
| --- | --- |
| Rig crate | Do not replace Allbert's provider seam with Rig in v0.10. Rig is ergonomic and modular for LLM apps, but Allbert's seam is tightly coupled to kernel-owned tools, cost logs, daemon protocol, setup, channel image gating, and local-first policy. Revisit Rig later if Allbert needs embeddings/RAG/provider-side agent abstractions. |
| OpenAI | Implement direct OpenAI through the Responses API. OpenAI positions Responses as the primary API for new AI applications and documents `store: false` for applications that do not want response storage. Default OpenAI model: `gpt-5.4-mini`. |
| Gemini | Implement direct Gemini REST `generateContent` using `GEMINI_API_KEY`, `system_instruction`, `contents`, `generationConfig.maxOutputTokens`, and `usageMetadata`. Default Gemini model: `gemini-2.5-flash`. |
| Ollama | Implement direct local Ollama `/api/chat` with `stream = false`, no API key, default base URL `http://127.0.0.1:11434`, and token usage from `prompt_eval_count`/`eval_count`. |
| Gemma 4 | Use `gemma4` exactly as the fresh-profile default. Allbert does not install Ollama or pull the model; operators run `ollama run gemma4` themselves. |

## Goal

Ship a provider layer that supports:

1. Anthropic, OpenRouter, OpenAI, Gemini, and Ollama behind the same `LlmProvider` trait.
2. Fresh local-first profiles that can start without a hosted API key.
3. Explicit remote-provider switching via setup and `/model`.
4. Provider configs where `api_key_env` is optional and `base_url` is available for local/custom endpoints.
5. Provider-free default validation plus ignored live smokes for opt-in real-provider checks.

## Milestones

### M1 — Config, proto, and metadata shape

- Extend kernel and protocol provider enums with `openai`, `gemini`, and `ollama`.
- Bump daemon protocol version because model payload JSON compatibility changes.
- Change model config from mandatory `api_key_env: String` to `api_key_env: Option<String>`.
- Add `base_url: Option<String>` for local/custom endpoints.
- Change `ModelConfigPayload` to carry `api_key_env: Option<String>` and `base_url: Option<String>` so session-local `/model`, job overrides, and daemon status roundtrip the same shape.
- Update daemon and session status payloads so keyless providers do not require fake env vars. Status renderers show Ollama's API-key state as `not required`; API-key-present checks must not warn, fail, or block startup when `api_key_env` is `None`.
- Preserve old TOML configs with `api_key_env = "..."`.
- Add one canonical kernel-owned provider metadata surface for parse, label, default model, default API key env, default base URL, API-key requirement, and known image-input posture.
- Route setup, REPL parsing/rendering, daemon protocol conversion, jobs, skill-contributed agent model overrides, status text, and docs examples through the shared provider metadata surface rather than duplicating provider `match` logic.

### M2 — Provider clients

- Keep Anthropic and OpenRouter behavior intact.
- Keep OpenRouter metadata bootstrap best-effort: metadata fetch failures fall back to the static compatibility list and must not block daemon startup or model construction.
- Add `openai.rs`:
  - POST `/v1/responses`;
  - `Authorization: Bearer`;
  - `store: false`;
  - system prompt via `instructions`;
  - image inputs as data URLs;
  - output text and usage parsing;
  - pricing table for the GPT-5.4 family.
- Add `gemini.rs`:
  - POST `/models/{model}:generateContent`;
  - `x-goog-api-key`;
  - `system_instruction`, `contents`, `generationConfig.maxOutputTokens`;
  - inline image data;
  - `usageMetadata` parsing;
  - pricing table for Gemini 2.5 Flash/Flash-Lite/Pro.
- Add `ollama.rs`:
  - POST `/api/chat`;
  - `stream = false`;
  - system/user/assistant messages;
  - image attachments as base64 arrays;
  - `options.num_predict`;
  - `prompt_eval_count`/`eval_count` usage;
  - zero pricing.

### M3 — Setup and REPL model UX

- Fresh `Config::default_template()` uses:
  - `provider = "ollama"`;
  - `model_id = "gemma4"`;
  - no API key env;
  - `base_url = "http://127.0.0.1:11434"`.
- Guided setup asks for the default provider and model.
- Hosted providers prompt for an API-key env var.
- Ollama prompts for the local base URL.
- Existing profiles preserve their configured provider/model unless setup or `/model` changes them.
- `/model` supports `anthropic`, `openrouter`, `openai`, `gemini`, and `ollama`.
- Status renders Ollama API-key state as `not required`.

### M4 — Daemon, jobs, skills, and protocol roundtrip

- Daemon model payloads include optional API-key env and base URL.
- Daemon status and session status preserve base URL and optional API-key state for every provider; hosted providers report missing API keys, while Ollama reports `not required`.
- Session-local model changes support every provider and remain scoped to the session.
- Job frontmatter and AgentSkills-contributed agent model overrides accept optional `api_key_env` and `base_url`.
- Job definition writers omit `api_key_env` for providers that do not need it.

### M5 — Docs and operator guidance

- README and onboarding docs explain Ollama/Gemma4 as the fresh-profile default.
- `.env.example` lists optional Anthropic, OpenRouter, OpenAI, Gemini, and Ollama variables without implying secrets are required for Tier A validation.
- Operator docs explain that Allbert does not install Ollama or pull `gemma4`.
- Roadmap and vision move self-improvement to v0.11.
- ADR 0066 records why v0.10 keeps an owned provider seam instead of adopting Rig.

### M6 — Test coverage and validation

- Provider unit tests use local mock HTTP servers for OpenAI, Gemini, and Ollama request/response mapping, image serialization, usage parsing, and error handling.
- Config tests cover fresh Ollama defaults, optional `api_key_env`, provider base URL defaults, and legacy `api_key_env = "ANTHROPIC_API_KEY"` configs.
- Daemon integration tests cover `set_model` roundtrip for OpenRouter, OpenAI, Gemini, and Ollama, including `base_url` preservation and optional API-key status rendering.
- Setup tests cover fresh-profile model defaults and remote-provider API-key warnings.
- Setup/status tests assert fresh Ollama profiles do not warn for missing API keys, while hosted providers still surface missing-key diagnostics.
- REPL tests cover new provider parsing, provider labels, default API-key env behavior, no-key Ollama switching, and base URL defaults.
- Ignored live smokes exist for `OPENAI_API_KEY`, `GEMINI_API_KEY`, and local Ollama/Gemma4.
- Required validation remains:
  - `cargo fmt --check`;
  - `env -u RUSTC_WRAPPER cargo clippy --workspace --all-targets -- -D warnings`;
  - `env -u RUSTC_WRAPPER cargo test -q`;
  - `env -u RUSTC_WRAPPER cargo run -q -p allbert-cli -- --help`;
  - temp-home `daemon status` smoke.

## Non-goals

- No automatic Ollama installation.
- No automatic `gemma4` model pull.
- No runtime dependency on Rig.
- No embeddings/RAG/provider-side agent work.
- No hosted sync or hosted model gateway.
- No version/tag release action in this implementation pass.

## Exit criteria

- Fresh profiles default to Ollama/Gemma4 and do not warn for a missing API key.
- Existing Anthropic/OpenRouter configs still load.
- OpenAI, Gemini, and Ollama providers can complete provider-free mocked tests.
- Daemon protocol and session model switching roundtrip all five providers.
- Docs explain local default, optional hosted-provider keys, and the no-auto-install Ollama posture.
- Tier A validation is green.
