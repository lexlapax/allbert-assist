# Allbert v0.56 Intent Descriptor Learning + Registration Lifecycle Completion

Request flow: `docs/plans/v0.56-request-flow.md`

ADRs:
- `docs/adr/0062-intent-descriptor-lifecycle-generation-and-operator-curation.md`
  (v0.56 completion amendment).
- `docs/adr/0060-two-stage-intent-router-and-approval-gate-separation.md`
  (routing grants no authority).
- `docs/adr/0061-local-embedding-capability-and-router-model-tiers.md`
  (local model tiers, no default egress).

Status: planned. Inserted after the v0.54 post-implementation audit. The prior
hardening/final-RC milestone moved to v0.59.

## Purpose

Complete the self-optimizing parts of ADR 0062 that v0.54 deliberately narrowed
after implementation: v0.54 shipped the safe foundation (dual-source
`DescriptorResolver`, data-only YAML descriptor/vocabulary files, heuristic
generation, CLI curation, review/promote/disable/edit, and dynamic-codegen-only
reindex hooks). v0.56 adds the planned local-model generation, learned-review
mining, operator-exposed optimize action, and full registration-signal lifecycle.

The milestone keeps the same authority boundary: descriptors are routing hints
only. They never grant permission, lower confirmation floors, or execute an action.

## Locked Decisions

| # | Decision | Chosen value |
|---|---|---|
| 1 | Generation model | Use `router_local` through the existing local model profile stack; fall back to the v0.54 heuristic generator when unavailable. |
| 2 | Learning output | Learned memory/trace/confirmation proposals land only in `<ALLBERT_HOME>/intents/learned/review/`; never active routing. |
| 3 | Promotion | Operator promotion writes to `generated/` or `overrides/` with audit. No silent route mutation from memory analysis. |
| 4 | Optimize action | Add `optimize_intent_descriptors` as an operator-exposed registered action in addition to `mix allbert.intent optimize`. |
| 5 | Registration hooks | Emit and consume `allbert.app.registered`, `allbert.plugin.registered`, and `allbert.action.registry_changed` in addition to existing `allbert.dynamic_codegen.**` signals. |

## Milestones

### M1: ADR 0062 Completion Reconciliation
Amend ADR 0062 to split shipped v0.54 foundation from v0.56 completion work.
Update developer docs so agents do not assume model generation or learned mining
exists before this milestone lands.

Focused gates: docs check; v0.54 release gate remains unchanged.

### M2: Local-Model Descriptor Generator
Introduce a bounded/redacted descriptor generator over action `name/0`,
`description/0`, `capability/0`, and schema metadata. It uses `router_local`
through the existing local-first model profile stack, writes YAML-compatible attrs,
and falls back to the v0.54 heuristic when the model is missing, disabled, timed
out, or returns an invalid object.

Focused gates: generator unit tests with fake local model; no raw prompts/secrets
in traces/audits; heuristic fallback tests.

### M3: Learned-Review Proposal Mining
Mine reviewed/kept memory, resolved clarification turns, approved confirmations,
redacted intent traces, and explicit operator corrections into learned-review YAML
proposal files with evidence refs, support counts, timestamps, and confidence.
The resolver must continue to ignore learned-review proposals until promoted.

Focused gates: proposal writer path safety; redaction; support-count merge;
learned-review inert-until-promoted.

### M4: Optimize Action + CLI Parity
Add `AllbertAssist.Actions.Intent.OptimizeIntentDescriptors` with operator
exposure, confirmation/permission posture appropriate for local advisory
maintenance, and output matching the doctor coverage envelope. Keep
`mix allbert.intent optimize|reindex|doctor|review|promote` as the operator CLI.

Focused gates: action route creates/reviews descriptors without granting
authority; CLI/action coverage reports agree; invalid model output fails closed.

### M5: Full Registration Signal Lifecycle
Emit app/plugin/action registration signals from the canonical registration paths
and subscribe the intent index to those signals plus the existing
`allbert.dynamic_codegen.**` pattern. Debounce rebuilds and ensure new actions do
not become routable unless a code, generated, or promoted descriptor exists.

Focused gates: app/plugin/action signal emission; debounced rebuild; rollback /
unregister removes routability; setting `intent.reindex_on_registration_signal`
remains the operator escape hatch.

### M6: Evals, Docs, And Closeout
Add `:v056` security/eval rows and a release lane. Reconcile roadmap, ADR 0062,
agent-context-map, operator/developer intent docs, and v0.59 inputs.

Focused gates: `release.v056`; docs gate; full warning gate before tag.

## Non-Goals

- No web Intents panel work; v0.58 owns that surface.
- No central action param-contract enforcement; v0.59 M7 / ADR 0065 owns that.
- No hosted model generation by default. Hosted generation requires explicit
  operator opt-in and must preserve the no-secrets/no-authority posture.
- No autonomous action/plugin creation or permission changes.

## Definition Of Done

v0.56 is done when local-model descriptor generation, learned-review mining, the
operator-exposed optimize action, and full registration reindex hooks are
implemented; learned proposals remain inert until operator promotion; all
authority invariants from ADR 0060/0062 are covered by eval rows; and roadmap,
ADR, request-flow, CHANGELOG, and agent-context-map agree.
