## Purpose

Three intertwined goals, all on the existing authority spine:

1. **Complete ADR 0062's self-optimizing lifecycle** that v0.54 deliberately
   narrowed: local-model descriptor generation, learned-review proposal-mining infrastructure, the
   operator-callable optimize action, and the full app/plugin/action
   registration-signal lifecycle. v0.54 shipped the safe foundation (dual-source
   `DescriptorResolver`, data-only YAML store, heuristic generation, CLI curation,
   review/promote/disable/edit, dynamic-codegen-only reindex).
2. **Make routing correct, not just covered.** The v0.54 audit found only **12
   descriptor-covered actions** in the routable inventory. v0.56 curates
   descriptors for the current `Actions.Registry.agent_modules/0` inventory
   (`57/57` at closeout) **and** introduces a deterministic routing-accuracy
   evaluation harness (ADR 0071) wired as a **blocking promotion and release
   gate** so the right agent/action fires and stays fired as descriptors change.
3. **Tell operators which model to use for what** (ADR 0072): a consolidated
   recommended-model-profile-per-purpose guide, recommended Settings Central
   defaults, and per-purpose doctor coverage — because this release leans on local
   model profiles (generation, the eval live lane, escalation).

The authority boundary is unchanged: descriptors are routing hints only; they never
grant permission, lower a confirmation floor, or execute an action.

## Cross-Cutting Invariants (apply to every milestone)

- **Operator Action Layer (systemic).** Every operator operation (doctor, list,
  show, coverage, eval run/baseline/capture/add, review, promote, optimize, reindex,
  enable/disable/edit, model-doctor) is a **registered Jido action** resolved through
  `Actions.Runner.run/3`. Reads are `exposure: :internal`, `permission: :read_only`,
  `confirmation: :not_required`; mutations are callable only from explicit
  operator surfaces/tasks, gated, and audited. Mix tasks, TUI slash commands, and the
  v0.58 web operator panels are **thin views** over the same actions — single source
  of truth (extends ADR 0070; satisfies the AGENTS.md "runtime-facing invocation
  resolves through Registry + Runner" rule). Natural-language channel routing never
  reaches these internal operator operations.
- **Settings Central is the only config authority.** Every new setting
  (`intent.eval.*` thresholds, recommended model-profile keys/defaults, reuse of
  `intent.descriptor_autoaccept` / `intent.reindex_on_registration_signal`) is
  registered in the Settings Central schema (specs + defaults + `safe_write_keys`);
  no module reads ad-hoc config or hard-codes a threshold/model tag.
- **Security Central is the only authority boundary.** Generation, learned-mining infrastructure,
  the optimize/promote/capture actions, and the eval gate are advisory/fail-closed;
  all security/permission decisions resolve through `Actions.Runner.run/3` + Security
  Central. Egress (any hosted model) stays an explicit, audited operator opt-in.
- **UI/UX is a first-class deliverable.** Every operator-visible read-model is a
  redacted DTO produced by a read action and surfaced in CLI + minimal TUI reads now;
  web (v0.58 Intents/Models panels) is flagged with the action+DTO contracts it
  consumes (M12).
- **Redaction everywhere.** No raw prompts, private payloads, or secrets in traces,
  audits, eval capture, or doctor output.

## Operator Action Layer (single source of truth)

The action set every surface reuses. Read actions resolve through the redacted DTOs
in `## Read-Model DTOs`; mutation actions run the ADR 0071 gate where they change
routing.

**Read actions** — `exposure: :internal`, `permission: :read_only`,
`confirmation: :not_required`, absent from `Actions.Registry.agent_modules/0`, never
intent candidates:

| Action | `name` | Returns |
|---|---|---|
| Router/coverage/gate status | `intent_doctor` | doctor DTO (router strategy, profiles, index state, coverage summary, eval baseline + gate status) |
| Resolved descriptors | `intent_list_descriptors` | `[descriptor_dto]` with source-layer badges |
| One descriptor | `intent_show_descriptor` | `descriptor_dto` |
| Coverage report | `intent_coverage` | coverage DTO (routable / missing / generated / review / override / disabled) |
| Run the harness | `intent_eval_run` | eval-result DTO (accuracy/per-domain/per-surface/confusion/slots/negatives/gate) |
| Learned proposals | `intent_list_review` | `[proposal_dto]` |
| Per-purpose models | `model_doctor` | recommendation DTO (purpose → recommended/configured/status) |

**Mutation actions** — callable only from explicit operator surfaces/tasks, audited,
gate-checked where they touch routing:

| Action | `name` | Posture |
|---|---|---|
| Generate + reindex | `optimize_intent_descriptors` | gate-checked; advisory; no authority |
| Promote review/generated → active | `promote_intent_descriptor` | **gate-checked** (rejects on regression) |
| Override edit / disable / enable | `edit_intent_descriptor` / `disable_intent_descriptor` / `enable_intent_descriptor` | audited override write; gate-checked before active routing changes |
| Rebuild index | `reindex_intent_descriptors` | no authority |
| Record baseline | `intent_eval_baseline` | writes committed baseline artifact |
| Capture a case | `intent_eval_capture` | redacted candidate to `<ALLBERT_HOME>/intents/eval/captured/` |
| Promote a case to the fixture | `intent_eval_add` | dev/repo action; reviewed candidate → committed fixture |

## Read-Model DTOs (redacted; produced by read actions)

- `descriptor_dto`: `{app_id, action_name, label, source (:app|:plugin|:action|:generated|:overrides), examples_count, synonyms_count, required_slots, disabled?}`.
- `coverage_dto`: `{routable: int, covered: int, missing: [action], generated: int, review: int, overridden: int, disabled: int, per_domain: %{domain => %{covered, total}}}`.
- `eval_result_dto`: `{overall_accuracy, per_domain: %{}, per_surface: %{}, confusion: [...], slot_accuracy, clarify_vs_execute, negative_violations: [...], gate: %{pass?, regressions: [...], floor, baseline}}`.
- `recommendation_dto`: `{purpose, recommended, configured, status (:ok|:missing|:under_capable|:not_pulled|:remote_egress_warning)}` (no endpoints/keys).

#### M14a: Natural-Language Operator Summary Boundary

Manual B3 found an implementation drift risk: natural-language inspection phrasing could
route to agent-exposed settings/channel/model/provider list actions and render raw
operator-shaped reports.
The corrective rule is:

- Raw operator reports are returned only from explicit operator affordances: TUI slash
  commands, Mix tasks, and future v0.58 operator panels.
- Natural-language turns may still route to agent-exposed read actions such as
  `list_settings`, `list_channels`, `list_model_profiles`, or
  `list_provider_profiles`, but those actions render bounded assistant summaries
  by default and include slash/operator guidance instead of dumping report rows.
- Operator/Mix callers that need full inventory output request `render_mode=operator_report`
  or use the existing internal `operator_*` actions.
- Descriptor YAML and operator overrides remain routing-vocabulary controls only. They may
  influence whether an action is shortlisted, but they do not own report shape, redaction,
  or operator authority. `render_mode` is action presentation policy, not descriptor
  vocabulary.
- Tests cover the action default, explicit operator-report mode, profile read summaries,
  TUI slash preservation, and the B3 natural-language phrasing including the observed
  typo variant.

M14a stays inside M14 and requires a fresh B1-B5 warm-TUI revalidation before closeout.
The formal operator-managed surface policy UI/Settings layer is handed to v0.58
alongside the Web UX redo; v0.56 keeps only the safe action default plus explicit
operator-report mode needed for validation.
