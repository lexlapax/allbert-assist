## Purpose

Three intertwined goals, all on the existing authority spine:

1. **Complete ADR 0062's self-optimizing lifecycle** that v0.54 deliberately
   narrowed: local-model descriptor generation, learned-review mining, the
   operator-exposed optimize action, and the full app/plugin/action
   registration-signal lifecycle. v0.54 shipped the safe foundation (dual-source
   `DescriptorResolver`, data-only YAML store, heuristic generation, CLI curation,
   review/promote/disable/edit, dynamic-codegen-only reindex).
2. **Make routing correct, not just covered.** Today only **12 of ~47 routable
   actions** have descriptors. v0.56 curates descriptors for the full routable
   inventory **and** introduces a deterministic routing-accuracy evaluation harness
   (ADR 0071) wired as a **blocking promotion and release gate** so the right
   agent/action fires and stays fired as descriptors change.
3. **Tell operators which model to use for what** (ADR 0072): a consolidated
   recommended-model-profile-per-purpose guide, recommended Settings Central
   defaults, and per-purpose doctor coverage — because this release leans on local
   model profiles (generation, the eval live lane, escalation).

The authority boundary is unchanged: descriptors are routing hints only; they never
grant permission, lower a confirmation floor, or execute an action. All four pillars
respect it.

## Foundations This Builds On

- **v0.54** two-stage router (ADR 0060), local model tiers (ADR 0061), descriptor
  lifecycle foundation (ADR 0062): `Router`/`Prefilter`/`Disambiguator`/`Index`/
  `DescriptorResolver`/`DescriptorStore`/`Optimizer`/`Bench` are shipped.
- **v0.55 / v0.55.1** split tool-result payload, the TUI/terminal channel, and the
  operator console whose inspection actions are `exposure: :internal` and **must
  never become routable** — a standing negative-route guarantee this release proves
  (ADR 0070).
- **v0.24** objective runtime (multi-step intents), **channels** (cross-surface
  routing must stay correct), Settings Central, Security Central, the ADR 0047
  doctor envelope, and Allbert Home `intents/` tree.

## Cross-Cutting Invariants (apply to every milestone)

- **Settings Central is the only config authority.** Every new setting
  (`intent.eval.*` thresholds, recommended model-profile keys/defaults,
  reuse of `intent.descriptor_autoaccept` / `intent.reindex_on_registration_signal`)
  is registered in the Settings Central schema (specs + defaults +
  `safe_write_keys`); no module reads ad-hoc config or hard-codes a threshold/model.
- **Security Central is the only authority boundary.** Generation, learned mining,
  the optimize action, promotion, and capture are advisory; all security/permission
  decisions resolve through `Actions.Runner.run/3` + Security Central. The eval gate
  is fail-closed and grants nothing. Egress (any hosted model) stays an explicit,
  audited operator opt-in.
- **UI/UX is a first-class deliverable.** Every operator-visible read-model
  (coverage report, descriptor list with source badges, eval results, model
  recommendations) is exposed as a redacted DTO and surfaced in CLI now; web (v0.58
  Intents/Models panels) and any TUI affordance are flagged with the contracts they
  consume (M11).
- **Redaction everywhere.** No raw prompts, private message dumps, or secrets in
  traces, audits, eval capture, or doctor output.

## Locked Decisions

| # | Decision | Chosen value |
|---|---|---|
| 1 | Generation model | Use `router_local` through the existing local model profile stack; fall back to the v0.54 heuristic generator when unavailable, disabled, timed out, or invalid. |
| 2 | Learning output | Learned memory/trace/confirmation proposals land only in `<ALLBERT_HOME>/intents/learned/review/`; never active routing. |
| 3 | Promotion | Operator promotion writes to `generated/` or `overrides/` with audit, **and must pass the ADR 0071 routing-accuracy gate**. No silent route mutation from memory analysis. |
| 4 | Optimize action | Add `optimize_intent_descriptors` as an operator-exposed registered action in addition to `mix allbert.intent optimize`; it runs through the gate. |
| 5 | Registration hooks | Emit and consume `allbert.app.registered`, `allbert.plugin.registered`, and `allbert.action.registry_changed` in addition to existing `allbert.dynamic_codegen.**` signals. |
| 6 | Routing-accuracy gate | A deterministic harness (ADR 0071) is a **blocking** promotion + release gate: **no-regression vs a committed baseline AND a ratcheting absolute floor AND zero negative-route violations.** |
| 7 | Eval corpus | Data-only YAML under `test/fixtures/intent/eval/`; hand-seeded across every domain, grown by `mix allbert.intent eval capture` from redacted real mis-routes; the `.exs` golden set is migrated to it. |
| 8 | Coverage target | Curate descriptors to the **full ~47-action routable inventory** to a passing per-domain accuracy bar this release — not just enable the machinery. |
| 9 | Model recommendations | Ship a consolidated per-purpose recommendation (ADR 0072) + Settings Central recommended defaults + per-purpose reporting folded into `mix allbert.intent doctor` and `mix allbert.settings model-doctor` (no new `mix allbert.models` task). |

## Milestones

Lane summary: M0 ADRs (serial barrier) → M1 eval harness (serial barrier) → (M2
corpus ∥ M3 model recs) → M4 generator → M5 full coverage+slots → M6 cross-surface
→ M7 learned mining → M8 promotion gate (serial barrier) → M9 optimize action + eval
CLI → M10 registration signals (serial barrier) → M11 UI/UX contracts → M12 evals +
release lane → **M13 operator validation (manual gate)** → **M14 closeout**. M13
blocks M14.

### M0: ADR Reconciliation + New ADRs
Draft ADR 0071 (eval harness + blocking gate) and ADR 0072 (model recommendations).
Amend ADR 0062 to split shipped v0.54 foundation from v0.56 completion and to state
that promotion/optimize now pass through the ADR 0071 gate. Update developer docs so
agents do not assume model generation, learned mining, or the gate exist before they
land.
- **Lane:** serial barrier — shared decisions other milestones cite; pure docs.
- **Settings/Security:** none (docs). **UI/UX:** none.
- **Focused:** docs check; v0.54 release gate unchanged.
- **Rejoin:** ADR 0071/0072 Proposed; ADR 0062 reconciled.

### M1: Intent Evaluation Harness Foundation
Build `Intent.Eval.{Corpus,Runner,Scorer,Gate}` (ADR 0071). Migrate the shipped
`.exs` golden anchors into the data-only YAML corpus format. The Runner replays the
corpus through the real Stage-1 ranking against a frozen embedding fixture + a
fake/seeded disambiguator (deterministic). The Scorer computes top-1 accuracy
(overall/per-domain/per-surface), confusion matrix, slot accuracy, clarify-vs-execute,
and negative-route violations.
- **Lane:** serial barrier — the measurement substrate everything downstream is
  gated by; touches the shared test-fixture tree.
- **Settings:** `intent.eval.min_accuracy`, `intent.eval.min_per_domain_accuracy`,
  `intent.eval.block_on_regression` registered in Settings Central (specs +
  defaults + `safe_write_keys`). **Security:** harness grants no authority; gate is
  fail-closed. **UI/UX:** Scorer emits a redacted result DTO for later surfaces.
- **Focused:** scorer math; determinism (same corpus + fake model → identical run);
  confusion/slot/negative metrics; corpus YAML validation + path safety.
- **Rejoin:** harness runs deterministically; golden set is YAML.

### M2: Baseline Corpus Authoring + BEFORE Baseline
Hand-seed the YAML corpus across **every** routable domain (notes, stocks, research,
browser, panels, email, calendar, memory, image-gen, channels, settings/model,
skills/plugin authoring, MCP, objectives, confirmations) plus negative-route cases
(v0.55.1 operator-inspection / `exposure: :internal` / doctor actions and TUI slash
lines must never route; ambiguous sibling-domain pairs). Record the BEFORE
routing-accuracy baseline (expected low: ~12/47 covered).
- **Lane:** parallel per-domain authoring lanes ∥ M1 runner.
- **Settings/Security:** none new. **UI/UX:** none.
- **Focused:** corpus loads; every `execute` case targets a real (or
  intentionally-uncovered) action; negative cases enumerated; BEFORE baseline
  committed via `mix allbert.intent eval baseline`.
- **Rejoin:** seed corpus + BEFORE baseline committed.

### M3: Model Recommendations + Recommended Profiles + Doctor Coverage
Implement ADR 0072. Ship `docs/operator/model-recommendations.md`; register
recommended model-profile defaults per purpose in Settings Central; fold per-purpose
reporting (recommended vs configured vs status: `ok|missing|under-capable|
not-pulled|remote-egress-warning`) into `mix allbert.intent doctor` (intent rows) and
`mix allbert.settings model-doctor` (all purposes), using the ADR 0047 redacted
envelope. Expose a recommendation read-model DTO for operator surfaces.
- **Lane:** parallel — guide (pure doc) ∥ recommended-default schema work ∥ doctor
  extension.
- **Settings:** recommended profile keys/defaults via Settings Central. **Security:**
  recommendations advisory; hosted profile flagged `remote-egress-warning`; no egress
  forced; doctor read-only/redacted. **UI/UX:** recommendation DTO consumed by CLI now,
  web/TUI flagged in M11.
- **Focused:** doctor reports per-purpose status truthfully; no secrets/endpoints in
  output; recommendation grants no capability.
- **Rejoin:** model guide + recommended defaults + doctor coverage shipped.

### M4: Local-Model Descriptor Generator
Add the `router_local` generation path to `Optimizer.generate/2` (ReqLLM,
json_schema, bounded/redacted; ADR 0061), with heuristic fallback when the model is
missing, disabled, timed out, or returns an invalid object.
- **Lane:** serial — touches the shared `Optimizer`.
- **Settings:** reuses `intent.router_model_profile` + `intent.descriptor_autoaccept`
  via Settings Central. **Security:** local-only, redacted; no raw prompts/secrets in
  traces; generated descriptors for dynamic/write-code actions stay inert until
  promotion. **UI/UX:** generation results in the coverage DTO.
- **Focused:** generator unit tests with a fake local model; redaction; heuristic
  fallback on every failure mode; invalid model output fails closed.
- **Rejoin:** model-backed generation works with deterministic fallback.

### M5: Full Descriptor Coverage + Slot Extraction To ~47 Actions
Generate + hand-curate descriptors for the full routable inventory in domain
batches, iterating each batch against the M1 harness until it meets the per-domain
bar. Add whitelisted slot extractors for new domains; missing required slot →
clarify, not mis-execute. Exclude internal-capability actions (`signal_doctor`,
`whatsapp_doctor`) and `exposure: :internal` actions from the routable set.
- **Lane:** parallel domain-batch lanes (comms / knowledge / platform / domain),
  each gated by the harness; slot work depends on each batch's descriptors.
- **Settings/Security:** slot extractors stay code-whitelisted (no operator-defined
  regex/code); routability never implies executability. **UI/UX:** coverage DTO
  reflects per-domain completion.
- **Focused:** each domain batch passes its accuracy + slot-accuracy bar; clarify on
  missing required slot; internal actions absent from the routable set.
- **Rejoin:** ~47-action coverage at/above the bar.

### M6: Cross-Surface Routing Correctness
Ensure routing is correct and consistent across surfaces (web, TUI, Telegram,
Discord, Slack, Matrix, WhatsApp, Signal, email) and that v0.55.1 TUI slash-commands
and operator-inspection actions stay non-routable on every surface.
- **Lane:** depends on M1/M5; parallel per-surface corpus slices.
- **Settings/Security:** per-surface negative-route guarantee (ties ADR 0070
  `tui-inspection-not-agent-candidate`). **UI/UX:** none new.
- **Focused:** same utterance routes consistently across channels; slash line never a
  model turn; inspection actions never route on any surface.
- **Rejoin:** cross-surface routing + negative-route guarantees pass.

### M7: Learned-Review Proposal Mining
Mine reviewed/kept memory, resolved clarification turns, approved confirmations,
redacted intent traces, and explicit operator corrections into `learned/review/`
YAML proposals (evidence refs, support counts, timestamps, confidence). The resolver
keeps ignoring this tier until promotion.
- **Lane:** parallel; touches the descriptor store (db/file serial for writes).
- **Settings/Security:** redaction enforced; proposals inert until operator
  promotion; no egress; no raw private payloads. **UI/UX:** proposals listed in
  `review` + the v0.58 panel contract.
- **Focused:** proposal writer path safety; redaction; support-count merge;
  inert-until-promoted.
- **Rejoin:** learned mining lands inert proposals with evidence.

### M8: Promotion Gate Wiring (Blocking)
Wire the ADR 0071 `Gate` into promotion and release: `mix allbert.intent promote` and
`optimize_intent_descriptors` run `Gate.check` against the candidate resolved set
before writing active; a failing gate rejects the change with a diagnostic and
mutates nothing. The release lane fails on a gate failure.
- **Lane:** serial barrier — the integration point for routing safety.
- **Settings:** gate thresholds read from Settings Central. **Security:** fail-closed;
  the gate only ever prevents a routing change, never enables one. **UI/UX:** gate
  result in the eval/coverage DTO.
- **Focused:** a regressing/negative-route-breaking promotion is rejected; a passing
  one proceeds through normal ADR 0062 promotion + audit.
- **Rejoin:** promotion + release are gated.

### M9: Optimize Action + Eval CLI
Add `AllbertAssist.Actions.Intent.OptimizeIntentDescriptors` (operator exposure,
read/advisory permission posture, doctor coverage envelope) that runs optimize +
generation **through the gate**. Add `mix allbert.intent eval run|baseline|capture|add`
(the operator corpus-growth flow). Keep `optimize|reindex|doctor|review|promote|
edit|disable|list|show|bench` as the operator CLI.
- **Lane:** depends on M1/M4/M8.
- **Settings:** thresholds/profiles via Settings Central. **Security:** action through
  `Actions.Runner.run/3` + Security Central; grants no authority; capture is redacted
  and path-safe. **UI/UX:** action result DTO; CLI renders coverage + gate status.
- **Focused:** action creates/reviews descriptors without granting authority;
  CLI/action coverage reports agree; capture redacts; invalid model output fails closed.
- **Rejoin:** optimize action + eval CLI shipped, gate-enforced.

### M10: Full Registration Signal Lifecycle
Emit `allbert.app.registered`, `allbert.plugin.registered`, and
`allbert.action.registry_changed` from the canonical registration paths; subscribe
the intent `Index` to those plus the existing `allbert.dynamic_codegen.**`. Debounce
rebuilds + enqueue a generation pass for newly-seen actions; new actions are not
routable unless a code/generated/promoted descriptor exists.
- **Lane:** serial barrier — signal emission across registration paths + the named
  Index process.
- **Settings:** `intent.reindex_on_registration_signal` (existing) is the operator
  escape hatch, via Settings Central. **Security:** registration signals update
  routing coverage only; never register actions or grant capability. **UI/UX:** none new.
- **Focused:** signal emission; debounced rebuild; rollback/unregister removes
  routability; escape hatch disables signal-driven rebuild.
- **Rejoin:** routing coverage tracks the live action set on all changes.

### M11: UI/UX Surface Contracts + Flags (Web / TUI / CLI)
Define and document the read-model DTOs every operator surface needs — coverage
report, descriptor list with source-layer badges, eval results + gate status, and the
model-recommendation matrix — and implement the CLI rendering now. **Flag** the web
deltas for v0.58 (the Intents panel + a Settings/Models panel rendering the
recommendation DTO) with the exact contracts they consume, and update the v0.58 plan
inputs; flag any TUI affordance (a future `/intents` read using the v0.55.1 console
pattern). No new web LiveView ships in v0.56; the boundary stays with v0.58's Web UX
redo.
- **Lane:** parallel — DTO/contract docs ∥ CLI rendering.
- **Settings/Security:** DTOs are redacted, read-only, authority-free. **UI/UX:** the
  whole milestone — CLI implemented; web/TUI contracted + flagged to v0.58.
- **Focused:** DTOs render in CLI; v0.58 inputs list the panels + contracts; no
  authority in any read surface.
- **Rejoin:** surface contracts documented; CLI surfaces shipped; v0.58 handoff recorded.

### M12: Evals + `release.v056` Lane
Add the `:v056` security/eval rows (authority rows **and** routing-accuracy rows) to
`EvalInventory` + a `release.v056` lane bundling: disposable migration; intent-eval
deterministic gate; descriptor-lifecycle units; model-doctor unit; the `:v056` eval
lane; and a channel-pack/secret scan. (Note: the disposable-home secret scan may
report `scanned_file_count: 0`; treat it as a tripwire, not positive proof.)
- **Lane:** parallel eval-row authoring ∥ lane wiring.
- **Settings/Security:** authority invariants asserted by eval rows. **UI/UX:** none.
- **Focused:** `:v056` evals green; `release.v056` deterministic + reproducible;
  routing-accuracy gate runs in the lane.
- **Rejoin:** deterministic gate + eval rows green.

### M13: Operator Validation (Manual Gate)
A real operator runs the live lane and curation flow: `mix allbert.settings
model-doctor` + `mix allbert.intent doctor` (recommended-vs-configured), `mix
allbert.intent optimize` with the local model, `mix allbert.intent bench` (live),
`eval capture` of a real mis-route, and a `promote` that the gate accepts/rejects.
Record **durable, redacted** evidence under `docs/release-evidence/v056/` (the v0.55
durability lesson), not an ephemeral `/tmp` home. **Blocks M14.**
- **Lane:** serial, manual, single-operator, local-model.
- **Settings/Security:** operator confirms egress posture; no secrets in evidence.
  **UI/UX:** operator walks the CLI surfaces.
- **Focused / gate:** doctors truthful; generation local + redacted; gate blocks a
  seeded regression; live bench recorded; evidence retained.
- **Rejoin:** operator-validation evidence committed + accepted.

### M14: Release Closeout
Flip ADR 0071 + ADR 0072 → Accepted; reconcile ADR 0062; version → `0.56.0`;
`CHANGELOG.md`; reconcile roadmap, vision, agent-context-map, and v0.59 inputs.
Depends on M13. Tag is the final operator-approved action and is held out of the
closeout commit per project convention.
- **Lane:** serial closeout.
- **Focused:** full release gate + docs gate + warning gate green; docs agree.
- **Rejoin:** docs/version metadata release-prepared; `v0.56.0` tag pending operator

## Non-Goals

- No web Intents/Models panel **implementation** in v0.56; v0.58 owns those surfaces.
  v0.56 ships the read-model DTOs/contracts and flags them (M11).
- No central action param-contract enforcement; v0.59 M7 / ADR 0065 owns that.
- No hosted model generation by default. Hosted generation/escalation requires
  explicit operator opt-in and preserves the no-secrets/no-authority posture.
- No autonomous action/plugin creation or permission changes.
- The eval gate never enables a routing change; it only blocks unsafe ones.

## Definition Of Done

v0.56 is done when:
- local-model descriptor generation, learned-review mining, the operator-exposed
  optimize action, and full registration reindex hooks are implemented; learned
  proposals remain inert until operator promotion;
- descriptors cover the full ~47-action routable inventory at/above the per-domain
  accuracy bar;
- the ADR 0071 routing-accuracy harness is a **blocking** promotion + release gate
  (no-regression vs baseline + ratcheting floor + zero negative-route violations),
  and the data-only YAML corpus + `eval` CLI ship;
- ADR 0072 model recommendations, recommended Settings Central defaults, and
  per-purpose `mix allbert.intent doctor` + `mix allbert.settings model-doctor`
  coverage ship, with the operator guide;
- all new settings go through Settings Central and all security decisions through
  Security Central; operator read-models are redacted, authority-free DTOs, with
  web/TUI rendering contracted + flagged to v0.58;
- all authority invariants from ADR 0060/0062 and the routing-accuracy + model-doctor
  invariants are covered by `:v056` eval rows;
- M13 operator validation passed with durable recorded evidence;
- ADR 0071 + ADR 0072 are Accepted, ADR 0062 reconciled; version is `0.56.0`; the tag
  is pending operator approval; and roadmap, vision, ADRs, request-flow, CHANGELOG,
  and agent-context-map agree.
