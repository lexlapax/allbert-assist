## Main User/Operator Flows

- Operator connects calendar, mail, GitHub, or notes/files integration.
- Operator asks for an agenda, inbox summary, GitHub summary, or notes search.
- Operator approves an effectful integration action.
- Operator opens integration panels in `/workspace`.

## Data Flow

Integration requests route through intent descriptors, app/plugin surfaces, MCP
servers where sufficient, native plugin actions where richer workspace or
memory ownership is required, and the standard action/security boundary.

## Authority Invariants

Integrations do not live in core. MCP is preferred for generic tool calls.
Native plugins own only their declared surfaces/actions/settings fragments and
never grant permission by registration.

## Handles / CLI Surfaces

- Calendar, mail, GitHub, and notes/files workspace panels.
- Integration-specific registered actions.
- Integration setup settings and secret refs.
- Existing confirmation and resource-grant commands.
