## M1 Locked Decisions

Each decision below is recorded in `v0.45-plan.md` §"M1 Locked
Decisions" with a one-paragraph rationale. These are chosen values,
not open defaults.

1. Hash algorithm — **SHA-256** via `:crypto.hash(:sha256, basis)`;
   matches v0.42 R9 precedent.
2. Permission strategy — **new `:marketplace_install`** permission
   class (default `:allowed`, floor `:allowed`).
3. URI scheme — **new `marketplace://entry/<author>/<name>`** top-level scheme;
   add to ADR 0013 as v0.45 amendment.
4. Install path layout — `<ALLBERT_HOME>/marketplace/<kind>/
   <entry-id>/` for installable skill/template bundles; catalog
   mirror at `<ALLBERT_HOME>/marketplace/cache/index.json`.
5. Installed-bundle state — **file-based** in
   `<ALLBERT_HOME>/marketplace/installed.json`; no new SQLite table
   at v0.45.
6. Workspace panel — **ship minimal**
   `Marketplace.SurfaceProvider` + `Panels.Catalog` under
   `:canvas_panels` zone.
7. Marketplace doctor — **ship `marketplace_doctor`** following
   ADR 0047 redacted shape.

## Main User/Operator Flows

- Operator browses the reviewed marketplace catalog (via workspace
  panel or `mix allbert.marketplace list`) and sees skills,
  templates, and reviewed-source plugin index entries with version,
  provenance, hash, and install state for installable entries.
  Plugin index entries render as browse-only metadata.
- Operator inspects an entry (panel affordance or `mix
  allbert.marketplace show <entry-id>`) and sees the full bundle
  manifest, file list, install target when applicable, and
  provenance metadata.
- Operator installs a reviewed skill/template (`mix
  allbert.marketplace install <entry-id>` or panel "Install"
  affordance). The action verifies the bundle hash, writes the
  bundle to `<ALLBERT_HOME>/marketplace/<kind>/<entry-id>/` in
  disabled/untrusted state, and updates
  `<ALLBERT_HOME>/marketplace/installed.json`. No skill becomes
  executable; no permission grant fires. Under the default
  `:marketplace_install` policy no confirmation prompt is required;
  operators may tighten the permission to confirmation or denial.
- Operator views installed bundles (`mix allbert.marketplace
  installed` or panel affordance); sees entry-id + version +
  install_state per row.
- Operator rolls back an install (`mix allbert.marketplace
  rollback <entry-id>` or panel affordance). The action removes
  `<ALLBERT_HOME>/marketplace/<kind>/<entry-id>/` and updates
  `installed.json`.
- Operator verifies a bundle hash on demand (`mix
  allbert.marketplace verify <entry-id>`).
- Operator runs the marketplace doctor (`mix allbert.marketplace
  doctor`) to check index parses, all bundle hashes match, no
  orphan installs, schema_version matches.
- Operator never fetches catalog data remotely — catalog is shipped
  data, updated by upgrading Allbert.

## Data Flow

Marketplace requests flow through `Actions.Runner.run/3`:

1. **Catalog browse:** `list_marketplace_entries` reads
   `priv/marketplace/index.json` via `Marketplace.Catalog`, which
   parses + validates per the v1 schema (rejects unknown
   schema_version, unknown keys, missing required fields). If the
   `<ALLBERT_HOME>/marketplace/cache/index.json` mirror is stale or
   missing, lazy mirror writes the read-only copy.
2. **Entry inspect:** `inspect_marketplace_entry` reads the bundle
   manifest from `priv/marketplace/bundles/<bundle_path>/bundle.json`,
   resolves the install target template when present, and returns the
   full descriptor.
3. **Bundle install pipeline** (M2):
   - `install_marketplace_bundle` looks up entry by id; validates
     the entry exists in the shipped index.
   - `Marketplace.Bundle.compute_hash/1` walks the bundle directory
     and computes the recursive SHA-256.
   - Hash mismatch with manifest's `bundle_hash` fails closed with a
     structured `error_category` diagnostic; install does not
     proceed.
   - `Marketplace.Install.write/2` copies bundle files to
     `<ALLBERT_HOME>/marketplace/<kind>/<entry-id>/`; updates
     `installed.json` with entry-id + version + installed_at +
     install_state (`"disabled_untrusted"`) + install_target +
     bundle_hash for installable `skill` and `template` kinds.
   - For skill kind: `Skills.Registry` receives the install with
     `source_scope: :marketplace_install, status: :disabled, trust: :untrusted`.
     For template kind: workspace:create template registry receives
     the descriptor. For plugin_index kind: install rejects; the
     entry is descriptive browse/inspect metadata only.
4. **Rollback pipeline** (M2):
   - `rollback_marketplace_install` removes
     `<ALLBERT_HOME>/marketplace/<kind>/<entry-id>/` recursively for
     installed skill/template bundles.
   - Updates `installed.json` to remove the entry.
   - `Skills.Registry` (for skill kind) deregisters the entry.
   - `plugin_index` rollback rejects because plugin index entries are
     never installed.
5. **Workspace panel render** (M4):
   - `Marketplace.SurfaceProvider` registers
     `Marketplace.Panels.Catalog` under `:canvas_panels` zone.
   - Panel reads catalog via `list_marketplace_entries` action.
   - Inspect / Verify Hash affordances route to
     `inspect_marketplace_entry` and
     `verify_marketplace_bundle_hash` for every entry.
   - Install / Rollback affordances route to
     `install_marketplace_bundle` and
     `rollback_marketplace_install` only for installable
     skill/template entries.
   - Panel never reads `priv/marketplace/` or
     `<ALLBERT_HOME>/marketplace/` directly — all data flows through
     registered actions.
6. **Doctor flow** (M5):
   - `marketplace_doctor` parses `priv/marketplace/index.json`,
     walks all bundle hashes, walks `installed.json` and verifies
     each installed bundle's hash still matches, checks for orphan
     installs, verifies `marketplace.schema_version` matches
     expected.
   - Records `last_verified_at` + `live_check_status` in
     `<ALLBERT_HOME>/marketplace/doctor/state.json`.
   - Failure modes produce documented `error_category` atoms
     per v0.43 R6 pattern.

## Authority Invariants

- **Marketplace metadata is descriptive only.** Catalog entries
  grant no permission, no skill enablement, no resource grants, no
  confirmation bypass.
- **Install writes disabled/untrusted state only.** Installed
  bundles register with `status: :disabled, trust: :untrusted` in
  the relevant registry (Skills or template); enablement and trust
  are separate operator actions through existing v0.03 + v0.06
  paths.
- **No marketplace-provided permission grants.** The
  `:marketplace_install` permission authorizes the install action
  (which writes inert state); it does not authorize execution of
  installed bundles.
- **Hash mismatch fails closed.** No partial install.
- **`bundle_path` sandboxed.** `..` traversal rejected; paths
  outside `priv/marketplace/bundles/` rejected.
- **`install_target` sandboxed.** For installable kinds, resolves
  under `<ALLBERT_HOME>/marketplace/`; any other prefix rejected.
  `plugin_index` entries must not declare an install target.
- **Unknown schema_version rejects.** Catalog with `schema_version
  != 1` rejects at v0.45 with JSON-Pointer diagnostic.
- **No remote network.** Catalog source is `"shipped"`; mirror is
  file-IO only.
- **Operator-modified mirror is advisory.** Shipped
  `priv/marketplace/index.json` remains the authority.
- **Workflow YAML files are never written.** Marketplace bundle
  kinds are `skill | template | plugin_index` only; no `workflow`
  kind at v0.45 (parked per v0.44 forward-pin).
- **Code-bearing remote plugin install denied.** `plugin_index`
  entries are descriptive metadata only; install rejects, and v0.45
  never fetches or compiles plugin code from the marketplace.
- **No theme/snippet distribution at v0.45** (parked).
- **`marketplace://entry/<author>/<name>` is identity, not authority.**
  Install authority comes from `:marketplace_install` permission,
  not the URI itself.

## Smoke Checklist

- Confirm `marketplace.enabled` defaults to `true`;
  `marketplace.schema_version` is read-only at `1`;
  `catalog.source` is read-only at `"shipped"`;
  `install.default_state` is read-only at `"disabled_untrusted"`;
  `provenance.hash_algorithm` is read-only at `"sha256"`.
- Run `mix allbert.marketplace doctor`; verify ADR 0047 redacted
  envelope; no orphan installs on a fresh `<ALLBERT_HOME>`.
- Run `mix allbert.marketplace list`; see the shipped seed
  entries with provenance + hash.
- Run `mix allbert.marketplace install <entry-id>`; verify hash
  match, install writes to
  `<ALLBERT_HOME>/marketplace/<kind>/<entry-id>/` in
  disabled/untrusted state, `installed.json` updated.
- For skill kind: confirm `mix allbert.skills list` shows the
  install with `source_scope: :marketplace_install, status: :disabled, trust:
  :untrusted`.
- For template kind: confirm `workspace:create` template registry
  shows the entry.
- For plugin_index kind: confirm install rejects (descriptive
  metadata only).
- Tamper with an installed bundle file; re-run `marketplace
  doctor`; confirm `error_category:
  :installed_bundle_hash_mismatch`.
- Run `mix allbert.marketplace rollback <entry-id>`; verify
  install directory removed, `installed.json` updated.
- Confirm cross-kind install_target sandbox: an entry with
  `install_target` outside `<ALLBERT_HOME>/marketplace/` rejects.
- Confirm bundle_path traversal: an entry with `..` in
  bundle_path rejects.
- Confirm workflow files: marketplace bundle with `kind:
  "workflow"` rejects (closed kind enum).
- Confirm operator-modified mirror: edit
  `<ALLBERT_HOME>/marketplace/cache/index.json`; re-run a
  marketplace action; confirm warning logged, shipped index still
  authority.
- Confirm trace redaction: bundle hashes show as `sha256:abcdef…`
  prefix-only in trace metadata.

## Operator Validation

Status: planned. The per-milestone blocks below are the source list
for `mix allbert.test release.v045` (M6 / v0.42 R5 lesson): each
block becomes a deterministic step in that gate at M6. The plan also
requires each block to run end-to-end on a disposable
`<ALLBERT_HOME>` without edits as a per-milestone acceptance
criterion (v0.42 R4 lesson). Manual reproduction remains as
debugging when an operator needs to step through a single phase; it
is no longer the release-validation primitive.

### M1 — substrate

```sh
export ALLBERT_HOME=/tmp/allbert-v045-m1-home
rm -rf "$ALLBERT_HOME"
mix run --no-start -e '
  IO.inspect(AllbertAssist.Settings.get("marketplace.enabled"));
  IO.inspect(AllbertAssist.Settings.get("marketplace.schema_version"));
  IO.inspect(AllbertAssist.Settings.put("marketplace.schema_version", 2, %{actor: "local", channel: :test}));
  IO.inspect(Enum.member?(AllbertAssist.Security.Policy.permission_classes(), :marketplace_install));
  IO.inspect(AllbertAssist.Security.Policy.safety_floor(:marketplace_install));
  IO.inspect(AllbertAssist.Resources.OperationClass.operation_classes() |> Enum.filter(&(&1 |> Atom.to_string() |> String.starts_with?("marketplace_"))))
'
```

### M2 — catalog parser + install + rollback

M2 ships the parser/install/rollback action substrate plus the smallest
operator-runnable CLI and seed path needed for this block:
`list`, `install`, `installed`, `rollback`, and
`allbert/research-helpers`. The richer seed set and remaining CLI
commands stay in M3.

```sh
export ALLBERT_HOME=/tmp/allbert-v045-m2-home
rm -rf "$ALLBERT_HOME"
# Assume priv/marketplace/ ships with allbert/research-helpers seed
mix allbert.marketplace list
mix allbert.marketplace install allbert/research-helpers
mix allbert.marketplace installed
ls "$ALLBERT_HOME/marketplace/skills/"
mix allbert.marketplace rollback allbert/research-helpers
mix allbert.marketplace installed
```

### M4 — workspace panel + intent routing

Implemented state:

- `workspace:marketplace` opens the Marketplace Catalog canvas panel.
- Skill and template entries expose Inspect, Verify Hash, and
  Install/Rollback depending on installed state.
- Plugin-index entries expose Inspect and Verify Hash only.
- The phrase corpus routes marketplace browse/install/rollback/verify
  prompts through registered marketplace actions.

```sh
export ALLBERT_HOME=/tmp/allbert-v045-m4-home
rm -rf "$ALLBERT_HOME"
mix allbert.ask --trace "show me the reviewed skill catalog"
PORT=4041 mix phx.server   # open /workspace?destination=workspace:marketplace
mix allbert.ask --trace "install the allbert/research-helpers skill"
```

Expected M4 outcomes:

- The first prompt routes to `list_marketplace_entries` with
  `kind=skill`.
- The workspace panel renders shipped catalog entries and dispatches
  affordances through registered actions.
- The install prompt routes to `install_marketplace_bundle` and writes
  disabled/untrusted marketplace state under the default allowed policy.

### M6 — deterministic release gate (primary)

```sh
mix allbert.test release.v045
# evidence: <ALLBERT_HOME>/release_evidence/v045/release-v045-<ts>.json
```

The gate runs against the shipped seed catalog with no external
network. It exercises the full v0.45 surface (substrate
permissions/operations/URI/settings, catalog parse, bundle hash
verification, install + rollback for installable skill/template
kinds, plugin_index install rejection, doctor success path + each
failure mode, intent routing for each phrase in the corpus,
workspace panel render, secret scan) and writes a reproducible JSON
evidence artifact.

### M6 — manual closeout (debugging fallback)

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v045.XXXXXX)"
export ALLBERT_TRACE_ENABLED=true
mix allbert.marketplace doctor
mix allbert.marketplace list
mix allbert.marketplace install allbert/research-helpers
mix allbert.marketplace installed
mix allbert.marketplace rollback allbert/research-helpers
rg -i 'token|secret' "$ALLBERT_HOME/settings" "$ALLBERT_HOME/memory/traces" || true
PORT=4041 mix phx.server   # /workspace: Marketplace Catalog panel
```

Expected behavior across milestones:

- catalog entries show provenance, version, hash, and trust
  metadata.
- an installed bundle lands in disabled/untrusted state; install
  grants no execution authority; hash verification runs before
  write.
- reviewed-source plugin_index metadata is viewable without
  installing code; no code-bearing remote plugin is installed.
- template metadata appears in `workspace:create` without
  executing templates.
- rollback removes the install directory and updates
  `installed.json`; the shipped bundle remains in `priv/marketplace/`
  for re-install.
- doctor returns ADR 0047 redacted shape; detects index parse
  errors, hash mismatches, orphan installs, and schema_version
  drift with documented `error_category` atoms.
- workflow YAML files are never installed (forward-pin invariant
  holds).
- `mix allbert.test release.v045` produces reproducible JSON
  evidence; secret scan finds nothing.
