## Main User/Operator Flows

- Operator browses reviewed skills/templates.
- Operator inspects provenance, version, hash, and trust metadata.
- Operator installs a reviewed skill into Allbert Home as disabled/untrusted.
- Operator views reviewed-source plugin index metadata without installing code.
- Operator sees marketplace templates as `workspace:create` catalog metadata.

## Data Flow

Marketplace metadata is fetched or read as reviewed index data, normalized into
local metadata, and written only to Allbert Home marketplace/cache locations.
Installed skills remain disabled/untrusted until a separate operator action
changes their state.

## Authority Invariants

- Marketplace metadata is descriptive only.
- Marketplace install does not enable skills, actions, plugins, templates, or
  permissions.
- Code-bearing remote plugins are not installed.
- Template catalog metadata does not execute templates.
- Any enablement still uses registered actions, Security Central,
  confirmations, traces, and audits.

## Smoke Checklist

- Discover reviewed skill metadata.
- Install a skill and confirm it is disabled/untrusted.
- Inspect reviewed-source plugin metadata without code install.
- Confirm template metadata appears without template execution.
- Confirm invalid provenance/hash metadata is rejected or marked unusable.
