## Purpose

Mark the first stable Allbert platform contract. v1.0 adds no features; it
freezes the public shapes operators, plugins, apps, channels, and external
clients can depend on. The freeze is **tiered** so contracts that have proven
themselves through ≥ 2 consumers and ≥ 6 months of operator-tested
implementation lock fully, while contracts that are still stabilizing freeze
with explicit additive-only carve-outs.

The 2026-06-25 1.0 planning pass also makes v1.0 a **product launch**. After the
v0.63 rescope, that launch is meaningful only if a true non-developer can install
Allbert, start the browser-first product surface through the persistent service, complete
the consumer-default guided local model path before opting into BYOK/custom setup,
connect local files/notes, review agent memory, and reach first value without developer
docs. v1.0 must pass the extended
acceptance matrix (toolchain-free trusted install, repairable onboarding, local
files/notes/memory launch path, professional web UX, existing implemented-surface
regression checks, v0.66 integrated RC evidence, and v1.0 no-docs DIT closeout),
not only the contract checks.

## Operator-Visible Win

Operators get a stable local-first assistant platform whose core contracts can
be trusted across upgrades. Non-developer operators can start with the packaged
binary plus browser UI; a native desktop client remains post-1.0. Plugin and app
authors can ship integrations against the frozen contracts without churn.

Three 1.0 product decisions (post-v0.63 review, operator-confirmed 2026-07-09; see
`roadmap.md` §v1.0 Strategic Frame and ADR 0077 / ADR 0078):

- **Web-first; native desktop post-1.0.** The web workspace plus packaged binary are the
  1.0 product surface. What makes "web via a server" acceptable for a non-developer is
  that Allbert runs as a **persistent background service** started once (`brew services` /
  `systemctl --user`) — the user opens the workspace, never re-runs `serve`.
- **Two-tier user target.** A friction-light **consumer default** and a **prosumer
  advanced** path (BYOK, custom endpoints, CLI). The consumer default is the primary
  first-run story validated by the v0.66 RC evidence plus the v1.0 no-docs DIT pass.
- **Zero-setup consumer model path.** The consumer default reaches first chat through
  guided local-runtime setup if needed, then a one-click in-app curated-local-model
  download (no manual model CLI, no API key), matching the LM Studio / Jan / Ollama / Msty
  one-click-model bar. Delivered in v0.64 through managed Ollama plus pull-API progress,
  as recorded in ADR 0078.

## Tiered Public Contract Freeze

### Tier 1 — Frozen Public Contracts

Contracts with ≥ 6 months of operator-tested implementation and ≥ 2 consumers.
Rename, remove, and shape-change are forbidden after 1.0:

- `AllbertAssist.Runtime.submit_user_input/1` plus canonical runtime turn
  signals (`allbert.input.received`, `allbert.agent.responded`,
  `allbert.runtime.turn.started`, `allbert.runtime.turn.completed`).
- `AllbertAssist.Actions.Registry` and `AllbertAssist.Actions.Runner.run/3`,
  including the ADR 0065 central param-contract seam (the Registry-exposed action
  schema and the Runner `:invalid_params` response shape) delivered in v0.59.
- Permission classes and safety floors as of v0.59.
- Plugin contract (`AllbertAssist.Plugin` behaviour + Registry shape).
- App contract (`AllbertAssist.App` behaviour).
- Settings Central schema **shape** — the fragment substrate, the per-fragment
  `schema_version` version contract (ADR 0046, delivered in v0.59), and the
  additive-only + deprecation-window policy. Individual keys evolve additively.
- Allbert Home layout (`<ALLBERT_HOME>` directory roots).
- Channel adapter boundary + identity-mapping shape (ADR 0016 plus the v0.52
  approval-primitive amendment and ADR 0057 threading substrate). The frozen
  shape includes canonical `conversation_threads.id`, `thread_channel_refs`,
  `conversation_message_refs`, `cross_channel_identity_links`, `owner_scope`,
  `receiver_account_ref`, and deterministic `provider_thread_key`. v1.0 remains
  local-first; post-1.0 multi-user/multi-tenant work may assign non-`"local"`
  owner scopes but must not introduce a second canonical conversation id.
- Resource Access `ResourceURI` shape, operation classes, and grant shape.
- Model provider/doctor return shape (ADR 0047) — reconciled with the roadmap
  acceptance-matrix item 2, which promotes the doctor shape to a Tier-1 freeze
  contract at v1.0. `AllbertAssist` model/provider readiness reads (`admin models
  doctor`, `first_model_detect`) keep their canonical result shape.
- Installer-side cosign **fail-closed verification** (v0.64) — a trust/safety floor
  frozen against weakening: the curl installer verifies `SHA256SUMS.cosign.bundle`
  and refuses to proceed on a missing/invalid signature.

### Tier 2 — Stabilizing Contracts (frozen with carve-outs)

Contracts with newer surface area or single-consumer atoms. Frozen against
rename/remove; additive-only changes permitted post-1.0:

- `AllbertAssist.App.SurfaceProvider`.
- Surface DSL catalog: **adding components allowed** post-1.0 under the
  registered fragment path; renaming/removing forbidden; signed Fragment
  envelope shape frozen.
- Workspace canvas and ephemeral surface contracts: **single-emitter component
  atoms** (those that ship with only one consumer at v1.0) are NOT frozen by
  name; only multi-consumer components and the underlying substrate are
  frozen. The substrate includes signed Fragment emission, per-thread Canvas
  + Ephemeral persistence shape, and SignalBridge validation.
- The presentation contracts frozen here (Surface DSL catalog, workspace canvas,
  and ephemeral substrate) freeze the released **post-v0.61b consolidated-shell
  baseline**. The freeze locks the overhauled presentation, not a pre-redesign one.
- v0.38 templated creation contracts: `AllbertAssist.Templates`,
  `AllbertAssist.Templates.Pattern`, reviewed pattern registry/listing,
  registered template actions (`render_template`, `validate_template`,
  `scaffold_template`, `create_from_template`), and the `workspace:create`
  destination are frozen against rename/remove. Adding new reviewed patterns,
  parameters, target shapes, and validation profiles is permitted.
- Template Settings Central keys are additive under ADR 0046:
  `templates.create.enabled` and `templates.allowed_patterns` retain their
  meaning, while future template settings must use safe defaults and migration
  metadata.
- v0.51 public protocol surface policy: `mcp_server.*`, `openai_api.*`, and
  `acp_server.*` Settings shape; default-off/default-empty exposure; per-client
  token auth for HTTP-bearing surfaces; self-approval denial; redacted
  confirmation-pending responses; and the rule that all effectful work reaches
  `Actions.Runner.run/3` + Security Central are frozen against removal or
  weakening. Protocol-specific MCP/OpenAI-compatible/ACP wire details track
  upstream specs and may evolve additively. The v0.51 text-first subset and
  denial of OpenAI/ACP image, audio, resource, filesystem-root, and
  client-supplied MCP-server authority are also frozen against weakening unless
  a later ADR adds an explicit capability-specific route.
- v0.62 packaged-entry contracts (ADR 0076): the unified `allbert` CLI
  **operator/dev command taxonomy** (`AllbertAssist.CLI.Commands.operator_table/0`
  dispositions — operator verbs and `admin <area>` reads are frozen against
  rename/removal; adding operator or dev commands is additive), the three-tier
  **secret vault** tier-resolution contract + `token_ref` semantics
  (`AllbertAssist.Settings.Vault` backends — OS vault → encrypted file → env; adding
  backends is additive), and the `allbert serve` daemon **`/health` JSON shape** +
  the **attach-over-UDS handshake** (`AllbertAssist.Runtime.Attach` token/version/
  Home/uid binding) are frozen against rename/removal/weakening; additive fields
  are permitted.
- v0.65 local-knowledge contracts: the `notes_files` reference-plugin launch action
  names (`search_notes`, `read_note`, `write_note`) and the `set_notes_root` settings
  action + its `apps.notes_files.notes_root` key are frozen against rename/removal
  (adding notes actions/keys is additive under ADR 0046). The **memory review-status
  vocabulary** (`:unreviewed`, `:kept`, `:flagged`, `:prune_nominated`) and the
  **`:kept`-only recall rule** are a behavioural safety contract frozen against
  weakening — the set of recall-eligible statuses may not expand beyond `:kept`
  without an explicit ADR; adding non-recall statuses is additive.

### Explicitly Not Frozen At 1.0

The 1.0 freeze does not lock the following shapes; they remain free to evolve
post-1.0 without a contract-break:

- ADR 0021 reserved advisory-provider vocabulary (`WorldModelProvider`,
  `DiffusionProposalProvider`, `MarketAllocatorProvider`,
  `ProbabilisticInferenceProvider`, `CriticEvaluatorProvider`,
  `ResourceDecisionProvider`, `CapabilityProvider`, `RouteProvider`). Only the
  `IntentProvider` **role** is implemented in v1.0, and it is implemented by the
  concrete `AllbertAssist.Intent.Classifier` module (there is no module literally
  named `IntentProvider` — it is a reserved ADR 0021 vocabulary token). The first
  concrete implementation of any other reserved role may amend ADR 0021 and is
  allowed to alter the role's behaviour shape post-1.0.
- Workspace zone/destination names beyond those with ≥ 2 consumers — including the
  single-surface v0.65 `workspace:notes` and `workspace:memory` destinations, whose
  names are not frozen (the notes/memory *actions* and review contract they dispatch
  are frozen in Tier 2 above).
- Workflow YAML schema (still evolving through v0.47 self-improvement
  suggestions).
- MCP/OpenAI-compatible/ACP wire and tool shapes (upstream protocol specs
  evolve; Allbert tracks upstream rather than freezing its own shape).
- Internal AG-UI bridge semantic mappings (the bridge stays internal-only at
  v1.0; public exposure is parked).

## Freeze Enforcement (release.v1 gate + `:v1` exact-name sweep)

The freeze is only meaningful if it is **enforced deterministically**: a future
rename or removal of a frozen contract must fail a gate, not slip through review.
v1.0 uses the proven v0.66 gate machinery with a freeze-specific mechanism.

**Mechanism — exact-name assertions (Locked Decision).** A `:v1` security sweep
(`v1_sweep_eval_test.exs`, cloning `v066_sweep_eval_test.exs`) asserts that every
frozen Tier 1/Tier 2 contract still exists by its **exact name**, using the v0.66
`AssertBinding.check!/2` pattern. Renaming or removing a frozen symbol fails its row;
Tier 2 additive changes stay green because the rows assert *presence*, not exhaustive
equality. Assertion primitives per contract kind:

- **Functions/behaviours** — `function_exported?/3` (e.g. `Runtime.submit_user_input/1`,
  `Actions.Runner.run/3`) and behaviour `@callback` presence.
- **Signals** — the four canonical turn-signal literals are present as emitted constants.
- **DB shapes** — `Schema.__schema__(:fields)` includes the frozen columns
  (`conversation_threads.id`, `owner_scope`, `receiver_account_ref`, `provider_thread_key`, …).
- **Settings keys** — the frozen key set (`mcp_server.*`, `openai_api.*`, `acp_server.*`,
  `templates.create.enabled`, `templates.allowed_patterns`, `apps.notes_files.notes_root`)
  resolves in the Settings schema; the per-fragment `schema_version` contract holds.
- **Home layout** — `AllbertAssist.Paths` exposes the frozen roots by name.
- **Permission classes / no-new-authority** — every registered action's permission is in
  the frozen `Policy.permission_classes()` set (reuses the v0.66 packaging check).
- **Enum vocabularies** — the memory review-status set and the `:kept`-only recall rule hold.

**`release.v1` gate.** Clone the v0.65/v0.66 quintet in
`apps/allbert_assist/lib/mix/tasks/allbert.test.ex`: usage line,
`def run(["release.v1"]), do: release_v1()`, `@release_v1_steps`, `release_v1/0` →
`release_evidence/v1/`, `run_release_v1_step/2`. Steps: static gates (migrate, format,
warnings-as-errors, credo --strict, dialyzer), version consistency, the `:v1` freeze
sweep, the reused v0.66 product-RC proof buckets (the v1.0 acceptance matrix's gate half),
and the extended docs gate. Add `| :v1` to `EvalInventory.milestone()`.

**The 7 freeze eval rows** (`:v1` milestone, each bound in `v1_sweep_eval_test.exs` with
`≥3 assert:` atoms):

- `v1-contract-freeze-no-new-features-001` — no new permission class beyond the frozen
  `Policy.permission_classes()` set; no new product surface vs v0.66.
- `v1-public-contract-docs-001` — the freeze notes doc exists and classifies each contract.
- `v1-action-boundary-regression-001` — `Actions.Registry` + `Runner.run/3` +
  `:invalid_params` shape exist; effectful actions route through the Runner.
- `v1-settings-home-layout-freeze-001` — the frozen Home roots (`Paths`) and the frozen
  Settings key set + `schema_version` contract exist by exact name.
- `v1-public-surface-policy-freeze-001` — `mcp_server.*`/`openai_api.*`/`acp_server.*`

- `v1-tier1-tier2-classification-001` — every frozen contract in the freeze notes has a
  tier + consumer-count entry; the sweep's contract set matches the freeze notes.
- `v1-adr0021-vocabulary-not-frozen-001` — ADR 0021 A20 records the

Attested-only acceptance-matrix rows (install/browser/model/cross-host) reuse the v0.66
evidence and the DIT milestones below; they are not `:v1` gate rows.
