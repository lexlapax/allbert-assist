# Allbert v1.0 Stability Release And Public Contract Freeze

Request flow: `docs/plans/v1.0-request-flow.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Mark the first stable Allbert platform contract. v1.0 adds no features; it
freezes the public shapes operators, plugins, apps, channels, and external
clients can depend on.

## Expected Inputs

- v0.50 final release-candidate hardening finished.
- All v1.0 acceptance matrix checks passing.
- No known warning, security, docs, or contract drift.

## Operator-Visible Win

Operators get a stable local-first assistant platform whose core contracts can
be trusted across upgrades.

## Planned Interfaces

Freeze the public contracts for:

- `AllbertAssist.Runtime.submit_user_input/1`
- `AllbertAssist.Actions.Registry` and `AllbertAssist.Actions.Runner.run/3`
- Plugin and app contracts.
- Surface DSL and signed Fragment envelope.
- Resource Access URI/grant shape.
- Workspace canvas and ephemeral surfaces.
- Channel adapter boundary.
- Settings Central schema.
- Allbert Home layout.

## Security Posture

No new authority. The release validates that every effectful path still routes
through registered actions, Security Central, confirmations, Resource Access,
traces, and audits.

## Non-Goals

- No new feature work.
- No new self-improvement features beyond the v0.45 operator-supervised
  precursor.
- No autonomous skill creation, distillation, or self-recompilation.
- No hosted multi-user model.
- No remote sync service.
- No compatibility-breaking API cleanup.

## Milestones

- M1: Contract inventory and freeze checklist.
- M2: Final docs reconciliation.
- M3: Final security/warning/precommit gates.
- M4: Operator acceptance and tag preparation.

## Documentation Deliverables

- 1.0 contract freeze notes.
- Operator upgrade and rollback notes.
- Developer compatibility guidance.

## Definition Of Done

v1.0 is done when all public contracts are documented, release gates pass,
operator acceptance is recorded, and no new features are introduced beyond the
v0.50 release candidate.
