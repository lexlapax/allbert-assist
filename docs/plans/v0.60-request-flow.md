## Cross-Surface Data / Decision Flow (the one path)
```text
DESIGN (no runtime behavior)
  M1 product-experience spec  (install -> first-run -> onboard -> first-value -> daily-use)
    -> first-value moment = "first useful chat"
    -> per-stage owning release map  (v0.61 / v0.62 / v0.63)

  M2 information architecture  (consumes M1)
    -> sitemap + screen inventory + navigation model + workspace/screen composition
    -> the layout redesign ADR 0074 deferred (chat-primary, ephemerals-as-modals)
    -> ADR 0077 Accepted (v0.60)
    -> feeds M6 (routes to render) and M7 (composition rules to gap-analyze)

  M3 first-model path  (consumes M1 first-value)
    -> decide assisted-local | managed-hosted | honest-BYOK
    -> QuickStart "first useful chat" contract
    -> packaging implications (handed to v0.62)
    -> ADR 0078 Accepted (v0.60)
    -> feeds M4 (QuickStart track) and M5 (first-run model-state check)

  M4 onboarding + persona design  (consumes M1, M3)  -> v0.63 input (ADR 0069 + ADR 0075 content)
  M5 entry-point + CLI UX design   (consumes M1, M3, M4)  -> v0.62 input (ADR 0076)
  M7 design-system gap analysis    (consumes M2, M6)  -> v0.61 input (ADR 0074 amendment)

WALKING SKELETON (the only running code v0.60 adds)
  M6 nav shell + placeholder screen per IA route
    -> render through the unified catalog inside one app shell (ADR 0030/0073/0074)
    -> routes resolve; nav model exposed; FocusTrap/high-contrast/reduced-motion hold
    -> NO live data, NO business-state read, NO effectful affordance, NO authority
    -> the v0.61 build scaffold + the M8 gate artifact

GATE
  M8 mix allbert.test release.v060
    -> format + compile-warnings-as-errors + Credo strict
    -> design artifacts present + docs-gate clean
    -> ADR 0077 + ADR 0078 Accepted (v0.60)
    -> walking-skeleton routes resolve + nav/a11y smoke
    -> :v060 evals + handoff drift-check (v0.61/v0.62/v0.63 consumers)
    -> redacted evidence under release_evidence/v060/
```

## Authority Invariants
- No new capability, no new authority surface, no confirmation-floor change, no
  egress grant, no Settings key, no contract freeze. Security Central remains the
  only authority boundary.
- The walking skeleton renders through the unified catalog inside one app shell
  (ADR 0030/0073/0074). No model-generated or data-generated markup is introduced.
- Placeholder screens read no business state, compose no effectful affordance, and
  grant no authority or egress. They prove the IA is navigable and scaffold v0.61.
- The design milestones (M1-M5, M7) change documents only; they add no code path,
  no setting, and no seeded value. The persona / First-Model-Path / onboarding
  designs are inputs to v0.62-v0.63, not v0.60 implementations.
- Exactly two ADRs are accepted (0077, 0078); ADR 0074/0075/0069/0076 receive
  design inputs, not implementation.
