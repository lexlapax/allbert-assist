# v0.12 — Self-improvement

Status: Proposed

## Context

After v0.10, self-improvement was resequenced out of v0.11 so that v0.11 can focus on TUI and adaptive memory. By v0.12, Allbert should have a stronger terminal operator surface, kernel-owned telemetry, and adaptive memory routing in addition to the v0.10 provider expansion. That makes v0.12 the safer point to ship self-improvement: the operator can review patches and skill drafts from a richer UI, with clear cost/token/context visibility and better memory recall.

v0.12 carries forward the previous self-improvement design: a Rust rebuild skill, a user-facing skill-authoring skill, and an embedded scripting seam. The sequencing changes; the trust posture does not. All artifacts still route through explicit review. The old v0.11 self-improvement stub has been removed; this file is the canonical plan for ADRs 0067-0073.

## Goal

Ship three capabilities, all framed as "Allbert does this for the user under review":

1. A `rust-rebuild` skill that reads, modifies, builds, and tests the Allbert codebase inside a sibling worktree, then produces diffs for operator review.
2. A `skill-author` skill that scaffolds AgentSkills-format skills through natural-language conversation and submits them through the standard install quarantine.
3. A `ScriptingEngine` trait with a sandboxed Lua implementation as the first embedded runtime.

## Non-goals

- No autonomous self-modification.
- No automatic binary swap of the running daemon.
- No bypass of skill install preview.
- No Lua filesystem/network access by default.
- No source-checkout requirement for ordinary Allbert usage.
- No local model retraining or distillation pipeline.

## Key decisions

- Self-modification happens in a sibling worktree, not the active checkout.
- Tier A validation is the rebuild safe-to-merge bar.
- Rebuild diffs route through a `patch-approval` inbox kind.
- Accepting a patch approval does not install or swap a binary.
- Self-authored skills carry `provenance: self-authored` and still use the install quarantine.
- Lua scripts receive serialized JSON inputs and return serialized JSON outputs; they do not share host memory.
- Lua is disabled unless both config and exec policy allow it.

## Exit criteria

- `rust-rebuild` can produce a patch against a fixture checkout, run Tier A, and route the result through patch approval.
- `skill-author` can draft and submit a self-authored skill through quarantine.
- Lua scripts run only through the sandboxed engine and cap enforcement.
- All operator docs and upgrade notes are complete.
- Tier A validation is green.
