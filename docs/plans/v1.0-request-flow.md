## Main User/Operator Flows

- Operator reviews the public contract inventory.
- Operator runs final acceptance on a disposable Allbert Home.
- Operator confirms inherited v0.66 evidence and completes the v1.0 DIT evidence for
  the primary local files/notes/memory launch path with a true non-developer operator.
- Maintainer confirms no new feature work is included.
- Maintainer prepares the stability release and tag.

## Data Flow

No new runtime data flow is introduced. The release validates documented public
contracts, release gates, security posture, and operator acceptance evidence.

## Authority Invariants

v1.0 adds no authority. All effectful work remains behind registered actions,
Security Central, confirmations, Resource Access, traces, and audits.

## Handles / CLI Surfaces

- Public contract inventory with per-tier classification.
- Release acceptance checklist.
- Existing release gates.

## Security Eval Rows

The 7 freeze rows are registered in `EvalInventory` (`:v1` milestone) and bound in
`v1_sweep_eval_test.exs` by **exact-name assertions** (plan Freeze Enforcement section):
each row proves a frozen contract still exists by exact name (`function_exported?/3`,
signal literals, `__schema__(:fields)`, Settings-key resolution, `Paths` roots,
`Policy.permission_classes()`). `mix allbert.test release.v1` fails if any row is
prose-only or unbound, or if a frozen symbol was renamed/removed. Additive Tier 2 changes
stay green (rows assert presence, not exhaustive equality).

- `v1-contract-freeze-no-new-features-001`
- `v1-public-contract-docs-001`
- `v1-action-boundary-regression-001`
- `v1-settings-home-layout-freeze-001`
- `v1-public-surface-policy-freeze-001`
- `v1-tier1-tier2-classification-001`
- `v1-adr0021-vocabulary-not-frozen-001`

The install/browser/model/cross-host acceptance-matrix rows are **operator-attested**
(the DIT milestones in `v1.0-handoff.md`), reusing v0.66 evidence; they are not `:v1`
gate rows.

## Smoke Checklist

- Run v1.0 acceptance matrix (17 freeze-blocking criteria).
- Confirm v0.59 substrate evidence and v0.66 product RC evidence remain current.
- Confirm v0.64 trusted install and v0.65 local knowledge evidence remain
  current.
- Confirm no new feature docs or code landed in v1.0.
- Confirm v0.47 self-improvement behavior is frozen rather than expanded.
- Confirm Tier 1 contracts have ≥ 2 consumers documented.
- Confirm Tier 2 carve-outs are explicit (additive-only, single-consumer
  components exempt).
- Confirm v0.51 public-surface policy is frozen for Allbert-owned
  auth/exposure/authority semantics, while MCP/OpenAI-compatible/ACP wire
  details remain upstream-tracking.
- Confirm ADR 0021 A20 records reserved-vocabulary not in the freeze.
- Run `mix allbert.test release.v1` — the `:v1` freeze sweep proves every frozen
  contract exists by exact name (rename/remove fails); confirm the prior
  `release.vNN` gates and docs-only checks pass.
- Complete the DIT operator-attested prerequisites (`v1.0-handoff.md`): WSL2 install,
  true non-developer no-docs first-run, hard one-click model download from empty,
  QuickStart direct-answer enablement, real `v0.66.0` -> v1.0 Home upgrade +
  uninstall, live advanced-surface exercise, and model-backed recall.

## Acceptance Validation

Status: planned. v1.0 is a freeze release: validation confirms the acceptance
matrix and the contract freeze, not a new operator feature.

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v100.XXXXXX)"
# Run the v1.0 acceptance matrix (the 17 freeze-blocking criteria) end to end,
# then the freeze gate and release gates.
mix allbert.test release.v1
mix precommit
```

Expected behavior:

- all 17 acceptance-matrix criteria pass on macOS, Linux, and Windows/WSL2 as
  applicable.
- the no-docs run is performed by a true non-developer, the consumer-default model
  path downloads from an empty model cache, and the upgrade evidence starts from a
  real `v0.66.0` packaged Home.
- no new feature code or docs landed in v1.0; v0.47 self-improvement behavior is
  frozen rather than expanded.
- Tier-1 contracts have >= 2 documented consumers; Tier-2 carve-outs and
  public-surface upstream-tracking wire details are explicit; release gates and
  docs-only checks pass.
