## Main User/Operator Flows

- Operator reviews the public contract inventory.
- Operator runs final acceptance on a disposable Allbert Home.
- Operator confirms inherited v0.66 evidence and completes the v1.0 DIT evidence for
  the primary local files/notes/memory launch path with a true non-developer operator.
- Maintainer confirms no new feature work is included.
- Maintainer prepares the stability release and tag.

## Data Flow

No new runtime data flow is introduced. The release validates documented public
contracts, release gates, security posture, and operator acceptance evidence.

## Security Eval Rows

The 7 freeze rows are registered in `EvalInventory` (`:v1` milestone) and bound in
`v1_sweep_eval_test.exs` by **exact-name assertions** (plan Freeze Enforcement section):
each row proves a frozen contract still exists by exact name (`function_exported?/3`,
signal literals, `__schema__(:fields)`, Settings-key resolution, `Paths` roots,
`Policy.permission_classes()`). `mix allbert.test release.v1` fails if any row is
prose-only or unbound, or if a frozen symbol was renamed/removed. Additive Tier 2 changes
stay green (rows assert presence, not exhaustive equality).

- `v1-contract-freeze-no-new-features-001`
- `v1-public-contract-docs-001`
- `v1-action-boundary-regression-001`
- `v1-settings-home-layout-freeze-001`
- `v1-public-surface-policy-freeze-001`
- `v1-tier1-tier2-classification-001`
- `v1-adr0021-vocabulary-not-frozen-001`

The install/browser/model/cross-host acceptance-matrix rows are **operator-attested**
(the DIT milestones in `v1.0-handoff.md`), reusing v0.66 evidence; they are not `:v1`
gate rows.

## Operator Manual Validation (DIT closeout)

Status: planned. v1.0 is a freeze release. Verification is two-layer: the
deterministic `mix allbert.test release.v1` gate (checkout-bound; 35 frozen contracts
by exact name) plus the operator-attested **DIT** freeze-prerequisites below, which run
on real hosts/providers. Record each under `docs/validation/v1.0/` — redacted, disposable
or explicitly-declared test Home only, never a real operator Home. **DIT-3** (model-backed
`:kept` recall) and **DIT-6** (source parity) are already attested; DIT-1/2/4/5 remain.
No `PENDING-OPERATOR` row may remain before the freeze. Each install step below ends with
its cleanup; the canonical uninstall reference is
[`docs/operator/install.md`](../operator/install.md) §Uninstall (`brew uninstall` /
`scripts/install/uninstall.sh` — Allbert Home is preserved unless `--purge`).

### Step 0 — deterministic gate (run first)

```sh
cd <repo>
MIX_ENV=test mix compile         # avoid the stale-beam trap before the gates
mix allbert.test release.v1      # freeze gate — GREEN 9/9 (v1_freeze_sweep = 35 frozen contracts)
mix allbert.test release.v066    # GREEN 16/16 (product-RC contract rows)
mix precommit                    # format, credo --strict, dialyzer, tests
```

### DIT-1 — WSL2 cross-host install (on a Windows host, in WSL2)

```sh
curl -fsSL https://raw.githubusercontent.com/lexlapax/allbert-assist/main/scripts/install/install.sh | sh
allbert admin health                       # runtime/web/channels up
curl -fsS http://localhost:4000/health     # {"status":"ok",...}
# save transcript -> docs/validation/v1.0/dit1-wsl2-install.log
# cleanup (docs/operator/install.md §Uninstall) — removes the binary, preserves Home:
sh scripts/install/uninstall.sh
```

### DIT-2 — true non-developer, no-docs first run (clean host, EMPTY model cache)

```sh
brew install lexlapax/allbert/allbert      # or the curl installer above
brew services start allbert                # Linux: systemctl --user start allbert
# Open http://localhost:4000/workspace -> onboarding auto-opens -> "Start QuickStart"
#   -> one-click model download FROM EMPTY (no pre-pulled Ollama model, no API key)
#   -> reach a first useful chat. A real non-developer drives it; no developer docs.
#   QuickStart must enable direct answers before the first real question.
# notes -> docs/validation/v1.0/dit2-no-docs.md (record any stumble as a blocking defect)
# cleanup (docs/operator/install.md §Uninstall) — preserves Home:
brew services stop allbert && brew uninstall allbert
```

### DIT-4 — live advanced-surface regression (configure >=1 provider per class first)

```sh
mix allbert.test external-smoke -- browser_research
mix allbert.test external-smoke -- browser_research_delegate
mix allbert.test external-smoke -- telegram   # + >=1 more channel (>=1 outbound, >=1 inbound)
# Public protocols (v0.51 MCP/OpenAI-compatible/ACP command set) + the v0.44 Plan/Build
# approval smoke per their runbooks.
# -> docs/validation/v1.0/dit4-advanced.md (an unconfigured class = SCOPED-OUT + rationale, else blocking)
```

### DIT-5 — real v0.66.0 -> v1.0 upgrade + uninstall preserves Home

```sh
# On a real v0.66.0 install with data (notes connected, a kept memory):
allbert admin home export /tmp/allbert-v066-home.tar
# On the v1.0 install / a fresh Home:
allbert admin home import /tmp/allbert-v066-home.tar
allbert admin health                       # verify notes/memory preserved
# uninstall the service AND the binary, then verify Home preservation
# (docs/operator/install.md §Uninstall):
allbert admin service uninstall            # removes the per-user OS service (confirmation-gated)
brew uninstall allbert                     # or: sh scripts/install/uninstall.sh  (removes the binary; keeps Home)
ls "$ALLBERT_HOME"                          # Home files still present (uninstall preserves Home; --purge removes it)
# save transcript -> docs/validation/v1.0/dit5-upgrade-uninstall.log
```

### Step N — close Phase A

Flip DIT-1/2/4/5 to `PASS` (or `SCOPED-OUT` + rationale) in
`docs/validation/v1.0/README.md`; recheck the web-usability caveats; commit + push.
Only then proceed to the Release Closeout.

Expected behavior at closeout: all 17 acceptance-matrix criteria pass on macOS, Linux,
and (via DIT-1) Windows/WSL2; the no-docs run is a true non-developer with an empty model
cache; the upgrade starts from a real `v0.66.0` packaged Home; no new feature code/docs
landed and v0.47 self-improvement is frozen; Tier-1 contracts have >= 2 documented
consumers and Tier-2 carve-outs are explicit; all release + docs gates pass.

## Release Closeout (after every DIT row is green)

Line-by-line. The tag is effectively irreversible — pushing it publishes a public GitHub
**Latest** release and moves the Homebrew tap. The tag message must **not** contain
`[skip-artifacts]` (that would skip the packaged build).

```sh
# 1. version bump 0.66.0 -> 1.0.0 (three mix.exs files, all line 7)
sed -i '' 's/version: "0.66.0"/version: "1.0.0"/' mix.exs apps/allbert_assist/mix.exs apps/allbert_assist_web/mix.exs
grep -rn 'version:' mix.exs apps/*/mix.exs           # confirm all three = 1.0.0

# 2. edit docs (no AI/co-author trailers):
#    - CHANGELOG.md: add "## v1.0.0 - Stability Release & Public Contract Freeze" at top
#      (no new features; tiered freeze per docs/developer/public-contract-freeze.md;
#       release.v1 enforces 35 frozen contracts by exact name; DIT closeout complete)
#    - docs/plans/roadmap.md: flip item 67 (## v1.0 ...) status to "Released - tagged v1.0.0"
#    - docs/plans/v1.0-plan.md: Status -> released
#    - homebrew/allbert.rb: bump version + URLs
sed -i '' 's/0\.66\.0/1.0.0/g' homebrew/allbert.rb    # sha256 stay REPLACE_* until step 5

# 3. verify + commit
MIX_ENV=test mix compile
mix allbert.test release.v1 && mix allbert.test release.v066 && mix precommit && mix allbert.test docs
git add -A
git commit -m "v1.0.0 closeout: bump 0.66.0->1.0.0, CHANGELOG, roadmap, formula; DIT closeout"
git push

# 4. tag + publish (CI builds macos-arm64/linux-x64/linux-arm64, publishes Latest, cosign-signs)
git tag -a v1.0.0 -m "v1.0.0 - Stability Release & Public Contract Freeze

Tiered public-contract freeze enforced by mix allbert.test release.v1 over 35 frozen
contracts. No new features. DIT closeout complete."
git push origin v1.0.0
gh run watch "$(gh run list --workflow=release-artifacts.yml -L1 --json databaseId --jq '.[0].databaseId')" --exit-status
gh release view v1.0.0 --json assets --jq '.assets[].name'   # 3 tarballs + SHA256SUMS + cosign.bundle

# 5. Homebrew tap (separate repo lexlapax/homebrew-allbert; CI does NOT push it)
gh release download v1.0.0 --pattern SHA256SUMS --dir /tmp --clobber
gh repo clone lexlapax/homebrew-allbert /tmp/tap && cd /tmp/tap
bash <repo>/homebrew/fill-sha256.sh /tmp/SHA256SUMS Formula/allbert.rb
grep -c REPLACE Formula/allbert.rb                    # must be 0
git commit -am "allbert 1.0.0" && git push

# 6. final verification
gh release list -L1                                   # v1.0.0 = Latest
brew update && brew info lexlapax/allbert/allbert     # shows 1.0.0
```
