# Allbert v0.47 Channel Pack 2: WhatsApp, Signal, iMessage, And Matrix

Request flow: `docs/plans/v0.47-request-flow.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Expand Allbert into mobile and personal-messaging channels after team-channel
patterns are proven.

## Expected Inputs

- v0.43 Discord/Slack channel pack.
- Existing channel adapter and plugin contracts.
- Existing Settings Central secret handling and Approval Handoff.

## Operator-Visible Win

Allbert becomes reachable on common mobile messaging surfaces while keeping
pairing, identity, privacy, retry, and platform limits explicit.

## Planned Interfaces

- WhatsApp channel plugin.
- Signal channel plugin.
- iMessage macOS-only channel plugin.
- Matrix channel plugin.
- Provider-specific setup and pairing flows.

## Security Posture

Each provider gets explicit Settings Central schema, identity mapping, retry
and dedupe policy, and pairing recovery docs. Phone/device identity cannot be
treated as permission authority. SMS remains parked outside 1.0.

## Non-Goals

- No SMS.
- No hidden device/session pairing.
- No channel-specific confirmation model.
- No unsupported platform promise for iMessage outside macOS.

## Milestones

- M1: Matrix and iMessage plugin skeletons.
- M2: WhatsApp and Signal plugin skeletons.
- M3: Pairing/setup UX and identity mapping.
- M4: Delivery/retry/dedupe and confirmation rendering.
- M5: Mobile-channel security evals and docs.

## Documentation Deliverables

- Operator setup docs for each channel.
- Security-hardening notes for mobile identity and pairing.
- Future-features update if SMS is promoted later.

## Definition Of Done

v0.47 is done when each planned channel has simulated inbound/reply coverage,
explicit setup docs, bounded identity mapping, and evals for pairing/session
replay, phone identity confusion, and mobile link handling.
