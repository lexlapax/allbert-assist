# Allbert v0.49 Channel Pack 2 And Public Protocol Interop

Request flow: `docs/plans/v0.49-request-flow.md`
ADR: `docs/adr/0044-public-protocol-exposure.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`.

## Purpose

Expand Allbert into mobile/personal messaging channels and expose public
protocol access without creating parallel authority systems.

## Expected Inputs

- v0.40 MCP client lessons.
- v0.43 Discord/Slack channel pack.
- v0.45 marketplace-lite provenance lessons.
- Existing channel adapter and plugin contracts.
- Existing Settings Central secret handling and Approval Handoff.
- Proposed ADR 0044 accepted before implementation closeout.

## Operator-Visible Win

Allbert becomes reachable on common mobile messaging surfaces and can be driven
from local API/editor/agent clients under the same policy boundary as the
workspace.

## Planned Interfaces

- WhatsApp channel plugin.
- Signal channel plugin.
- iMessage macOS-only channel plugin.
- Matrix channel plugin.
- Provider-specific setup and pairing flows.
- OpenAI-compatible local HTTP endpoint.
- ACP server mode.
- MCP server mode.
- Public AG-UI/A2UI bridge.

## Security Posture

Each channel provider gets explicit Settings Central schema, identity mapping,
retry/dedupe policy, and pairing recovery docs. External protocol clients
never receive more authority than local workspace users. One
auth/rate-limit/CSP/redaction policy covers API, ACP, MCP server, and
AG-UI/A2UI bridge. External clients cannot approve their own confirmations.

## Non-Goals

- No SMS.
- No hidden device/session pairing.
- No channel-specific confirmation model.
- No unsupported platform promise for iMessage outside macOS.
- No hosted multi-user authorization model.
- No cloud sync service.
- No remote UI code execution.
- No MCP Apps iframe execution.
- No external client self-approval of confirmations.

## Milestones

- M1: Matrix and iMessage plugin skeletons.
- M2: WhatsApp and Signal plugin skeletons.
- M3: Pairing/setup UX, identity mapping, delivery/retry/dedupe, and
  confirmation rendering.
- M4: ADR 0044 public protocol policy.
- M5: API, ACP, MCP server, and AG-UI/A2UI bridge surfaces.
- M6: Mobile-channel and protocol security evals, CSP reconciliation, docs,
  and closeout.

## Documentation Deliverables

- Operator setup docs for each channel.
- Developer protocol interop guide.
- Security-hardening notes for mobile identity, pairing, external clients, and
  CSP.
- Future-features update if SMS or MCP Apps iframe support is promoted later.

## Definition Of Done

v0.49 is done when each planned channel has simulated inbound/reply coverage,
public protocol clients can submit work only through Allbert policy, and evals
cover pairing/session replay, phone identity confusion, mobile link handling,
protocol auth, CSP, redaction, and confirmation ownership.
