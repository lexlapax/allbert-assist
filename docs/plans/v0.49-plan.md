# Allbert v0.49 Operator-Supervised Self-Improvement

Request flow: `docs/plans/v0.49-request-flow.md`
ADR: `docs/adr/0045-operator-supervised-self-improvement-trust-tier.md`

Status: planned.

Source: `docs/archives/version-1.0-planning-03.md`, `docs/plans/allbert-jido-vision.md`,
and `docs/plans/future-features.md`.

## Purpose

Add the safe self-improvement precursor before 1.0 without granting Allbert
autonomous authority. v0.49 covers trace-to-skill draft suggestions,
workflow/intention draft suggestions, and dynamic capability review loops that
reuse the v0.36 sandbox/gate runner, v0.37 dynamic integration gate, and v0.38
templated creation substrate.

## Expected Inputs

- v0.48 marketplace-lite and protocol interop accepted.
- v0.21 memory review and retrieval accepted through operator use.
- v0.24 Objective Runtime accepted for multi-step work.
- v0.36 sandbox/gate reports accepted through operator use.
- v0.37 dynamic code/config generation and live integration accepted through
  operator use.
- v0.38 templates accepted as the deterministic creation path.
- v0.44 operator workflow YAML available for workflow drafts.
- v0.48 marketplace-lite trust tiers available for reviewed skill/template
  discovery.

## Operator-Visible Win

Allbert can notice repeated operator patterns and propose useful inert drafts:
new skills, workflow YAML, template inputs, or capability-gap drafts. The
operator remains in charge of review, validation, sandbox evidence, approval,
enablement, and publication.

## Planned Interfaces

- Workspace suggestion queue for trace-to-skill, trace-to-workflow, and
  capability-gap drafts.
- CLI inspection command for suggestions, for example
  `mix allbert.self_improvement list`.
- Registered read-only analysis action for pattern discovery over traces,
  objectives, memory review decisions, and operator-marked examples.
- Registered draft creation action that writes inert drafts only into reviewed
  draft roots.
- Handoff into v0.38 templates for plugin/app/tool/flow scaffolds.
- Handoff into the v0.36/v0.37 sandbox/gate/integration path for code-bearing
  drafts.

## Security Posture

The milestone is advisory and inert by default.

- Trace analysis does not grant permissions.
- Draft suggestions do not enable skills, actions, plugins, workflows, or
  package installs.
- Code-bearing drafts must pass the existing sandbox/gate and trusted
  validation path before any live authority can be requested.
- Instruction-only skill drafts are created disabled and untrusted.
- Workflow YAML drafts are declarative objective inputs only, not an execution
  engine.
- Any enablement, integration, rollback, publish, or marketplace action
  requires normal registered actions, Security Central, durable confirmation,
  traces, and audits.

## Non-Goals

- No autonomous skill creation.
- No auto-enable, auto-publish, or remote install from trace patterns.
- No broad execution permission derived from model output or repeated use.
- No hidden model training, personality distillation, or learned system-memory
  authority.
- No unsupervised self-recompilation, compiler-loop bootstrapping, or runtime
  mutation outside the v0.36/v0.37/v0.38 review path.
- No hosted multi-user or distributed coordination model.
- No release-candidate freeze; v0.50 owns final hardening and RC evidence.

## Milestones

- M3: Read-only pattern discovery action and workspace suggestion queue.
- M4: Inert draft creation path that targets reviewed draft roots only.
- M5: Handoff into v0.38 templates and v0.36/v0.37 gate paths.
- M6: Security evals for suggestion authority, draft enablement, provenance,

## Definition Of Done

v0.49 is done when Allbert can propose useful drafts from repeated operator
patterns while every effectful transition still requires explicit operator
review, registered actions, Security Central, confirmation, traces, and audit
evidence.
