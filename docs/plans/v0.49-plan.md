# Allbert v0.49 Vision And Image Generation

Request flow: `docs/plans/v0.49-request-flow.md`

ADRs: `docs/adr/0051-provider-capability-preferences.md`,
`docs/adr/0042-audio-image-and-media-resource-classes.md`,
`docs/adr/0047-provider-doctor-contract.md`

Status: planned.

Source: current roadmap sequencing plus the v0.48 provider capability handoff.
The older archive source is historical context only.

## Purpose

Add image input, screenshot analysis, and image generation through resource
classes and registered provider-backed actions.

## Expected Inputs

- ADR 0047 provider doctor contract, proven by the v0.39 model-profile doctor.
- ADR 0051 provider capability and preference substrate from v0.48.
- v0.48 audio lessons for media redaction, retention, and display-only
  provider/cost metadata.
- v0.48 profile media metadata, including `input_modalities`,
  `output_modalities`, `transport_modes`, and deployment mode. v0.49 consumes
  this metadata for image/screenshot/image-generation routing only; `video_input`
  remains vocabulary/future metadata unless this plan is explicitly amended.
- v0.43 browser screenshot lessons.
- Existing workspace media rendering and trace redaction.
- Proposed ADR 0042 accepted before implementation closeout.

## Operator-Visible Win

Operators can paste a screenshot or image, ask Allbert about it, and generate
diagrams/images on demand with visible provider/cost metadata and retention
policy.

## Planned Interfaces

- Image resource `image://capture/<id>`.
- Screenshot resource `screen://capture/<id>`.
- Capability-aware model preference resolution for `vision_input` and
  `image_generation`.
- Vision-capable model profile doctor checks.
- Image generation registered action with provider profile.
- Workspace media rendering and cache controls.

## Security Posture

Images and screenshots are resources with size/type/redaction constraints.
Generated image calls are external provider actions with display-only cost
metadata and confirmation where required. Traces redact binary/media content by
default.

## Non-Goals

- No arbitrary remote media fetch.
- No video ingestion or video understanding implementation.
- No video generation.
- No permanent media retention by default.
- No model-generated UI code.

## Milestones

- M1: ADR 0042 image/screenshot portion, ADR 0051 capability consumption, and
  provider doctor checks.
- M2: Image paste/upload and screenshot resources.
- M3: Vision analysis flow and workspace rendering.
- M4: Image generation action and cost/confirmation UX.
- M5: Security evals, docs, and closeout.

## Documentation Deliverables

- Operator vision/image guide.
- Security-hardening notes for media redaction.
- Provider profile docs for vision and image generation.

## Definition Of Done

v0.49 is done when image input and generation work from workspace, provider
capability checks are visible, media is bounded and redacted in traces, and
security evals cover media-resource bypasses.
