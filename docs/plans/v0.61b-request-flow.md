## Cross-Surface Data / Decision Flow (the one path)

```
operator feedback (v0.61 manual validation, 8 items)
  → v0.61b plan Feedback Inventory (triage + Locked Decisions)
    → M0 plan shell-spec/sign-off section + ADR 0080 (Proposed)
      → S2 operator sign-off  ──►  ADR 0080 Accepted (v0.61b)
        → contained fixes M1–M4 (tokens/CSS; registered rename_thread action
          through Runner → PermissionGate(:conversation_write) → Conversations.rename_thread/3,
          identity server-derived, ownership-scoped)
        → shell arc M5 (sidebar consolidation) → M6 (docked pane)
          → M7 (top bars → per-view headers, relocation map)
            → S4 live shell validation (send-backs → named M9.x or tag block)
              → M8 (rail + full-hide collapse, LayoutPrefs persistence)
                → M9 :v061b rows + release.v061b (incl. :v061 regression)
                  → M9.1/M9.2 audit remediation
                    → M9.3/M9.4 operator-handoff/spec cleanup
                  → S6 final validation → version 0.61.1, tag held for operator
```

Reads flow through the existing renderer context (threads, destinations, active
objective) — the consolidated sidebar consumes the same context the retired
submenu column consumed. The only write added anywhere is `rename_thread`, and
it takes the one spine: LiveView event → `ContextBuilder.live_view_context`
(identity pinned server-side) → `Runner.run("rename_thread", %{thread_id,
title}, ctx)` → PermissionGate `:conversation_write` → ownership-scoped
`Conversations.get_thread/2` → `Repo.update`. Params never carry identity.

  no route added/removed; `?destination=workspace:*` deep links keep working.

  `Session.Scratchpad` untouched (v0.58 invariant).
- Direction C held; theme × high-contrast × reduced-motion × OS cells ≥ AA;
  `:v061` proof tests stay green after the plan-named M3/M5 reconciliations
  where they pinned changed literal token values or the pre-M5 nav structure.
- Layout preferences (collapse state, pane width) are client-side localStorage
  only — never settings, never synced authority.
