## Cross-Surface Data / Decision Flow (the one path)

```
operator feedback (v0.61 manual validation, 8 items)
  → v0.61b plan Feedback Inventory (triage + Locked Decisions)
    → M0 consolidated-shell-spec.md + ADR 0080 (Proposed)
      → S2 operator sign-off  ──►  ADR 0080 Accepted (v0.61b)
        → contained fixes M1–M4 (tokens/CSS; registered rename_thread action
          through Runner → PermissionGate(:conversation_write) → Conversations.rename_thread/3,
          identity server-derived, ownership-scoped)
        → shell arc M5 (sidebar consolidation) → M6 (docked pane)
          → M7 (top bars → per-view headers, relocation map)
            → S4 early live shell review (send-backs → M8 window)
              → M8 (rail + full-hide collapse, LayoutPrefs persistence)
                → M9 :v061b rows + release.v061b (incl. :v061 regression)
                  → S6 final validation → version 0.61.1, tag held for operator
```

Reads flow through the existing renderer context (threads, destinations, active
objective) — the consolidated sidebar consumes the same context the retired
submenu column consumed. The only write added anywhere is `rename_thread`, and
it takes the one spine: LiveView event → `ContextBuilder.live_view_context`
(identity pinned server-side) → `Runner.run("rename_thread", %{thread_id,
title}, ctx)` → PermissionGate `:conversation_write` → ownership-scoped
`Conversations.get_thread/2` → `Repo.update`. Params never carry identity.

## Authority Invariants

- No new permission atom, Settings Central key, confirmation-floor change, or
  egress grant relative to v0.61.0. The registry diff is exactly
  `rename_thread` on the existing `:conversation_write` permission.
- The catalog remains the rendering boundary; no model-/data-generated markup;
  no route added/removed; `?destination=workspace:*` deep links keep working.
- No internal rename: `Conversations.Thread` modules/atoms/topics/keys and
  `Session.Scratchpad` untouched (v0.58 invariant).
- Direction C held; theme × high-contrast × reduced-motion × OS cells ≥ AA;
  `:v061` proof tests stay green (regression step in the gate).
- Layout preferences (collapse state, pane width) are client-side localStorage
  only — never settings, never synced authority.

### S3 — Contained Refinements Pass (after M1–M4)
- **Chat type (M1):** open a conversation with both user and Allbert messages;
  in light, dark, and high-contrast: body reads most prominent; "2m ago" is the
  smallest, most muted element; continuity rows unchanged.
- **Link-chip (M2):** with an active objective, the chat-header chip reads
  status + destination (e.g. "Running · Objective"); hover shows link
  affordance; click lands on that objective's view; screen-reader name is
  "View objective — status: …".
- **Dark mode (M3):** cycle system → dark → light; with OS dark + theme
  `system`, palette identical to `dark`. Judge the aesthetic: **is dark now
  subtle enough?** Check dark high-contrast still reads.
- **Rename (M4):** hover a thread row → rename affordance → inline input;
  Enter saves (list + labels update); Esc reverts; double-click also enters
  edit; restart the server — the title persists; a second user's thread cannot
  be renamed (dev-mode check or test evidence).
- **Record:** pass/fail per item; M3 is an explicit operator aesthetic judgment.

### S4 — Early Live Shell Review (after M7; send-backs → M8 window)
- **One sidebar (M5):** on `/workspace`, no second submenu column; Workspace
  section expanded with Conversations/Output/Apps/Workspace nested and the
  active destination highlighted; on `/jobs`, the Workspace section collapses
  to its header; every destination you used before is reachable (spot-check
  `?destination=workspace:settings` deep link).
- **Docked pane (M6):** open Settings/Models/Channels/Trust from the sidebar —
  the panel docks beside chat; chat messages and composer fully visible and
  usable; drag the divider; collapse the pane to the edge; reload — width
  restored.
- **Top bars gone (M7):** no persistent bar on `/workspace`, `/jobs`,
  `/objectives`; per-view header names the view; walk the **relocation map**:
  theme toggle + overflow in the sidebar footer work; objective chips in the
  chat header; nothing you used from the old bars is missing.
- Judge the whole: does the consolidated shell give chat/canvas the space items
  1/3/6/7 asked for?
- **Record:** pass, or **send-backs** (specific, per surface) — send-backs are
  in-scope for M8's polish window; structural rejections reopen S2.

### S5 — Collapse States (after M8)
