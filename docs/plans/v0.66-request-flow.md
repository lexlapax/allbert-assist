## Inputs

v0.64 closes trusted install and repairable first-run. v0.65 makes local files,
notes, and reviewed agent memory the primary launch integration. v0.66 validates
those paths together with every already-implemented advanced surface before the
v1.0 freeze.

## Main User/Operator Flows

- Operator installs Allbert from the packaged path on a clean host.
- Operator starts the persistent service and sees runtime/web/channels health go green.
- Operator completes QuickStart onboarding on the consumer-default path: guided local
  runtime setup if needed, then a one-click in-app curated-local-model download (no manual
  model CLI, no API key), and reaches a first useful chat. The advanced BYOK/custom path
  is validated as a regression surface.
- Operator's natural non-developer prompts route sensibly (no mis-route to a disabled or
  demo capability), and the web workspace passes a non-developer usability audit.
- Operator connects a local notes/files root, searches/reads a note, confirms a
  write, reviews an agent-memory candidate, and recalls it later.
- Operator inspects security/confirmation/event state through web, CLI, and TUI
  surfaces.
- Operator exercises existing advanced surfaces — remote channels, MCP, browser
  research, plan approval, public protocols, and export/import — as regression
  blockers, even though they are not the default first-run path.
- Operator exports/imports or upgrades an Allbert Home and verifies behavior.
- Operator uninstalls service/binary without deleting user data unless requested.
- Maintainer runs final docs, warning, drift, secret-scan, and `release.v066`
  gates.

## Data Flow

No new runtime data flow is introduced. The RC validates the v0.61-v0.65 product
capability releases integrated on the v0.60 product-experience design. It records
redacted evidence from the packaged binary, daemon, web workspace, onboarding,
settings/profile, trusted install, local knowledge/memory, TUI, advanced
surfaces, export/import, and uninstall paths. Evidence must use disposable homes
or explicitly declared test homes, never a real operator home.

## Authority Invariants

No new capability or authority. The RC validates that every product path remains
behind the same action/settings/security spine and that packaged entry points,
onboarding profiles, and web affordances grant nothing by themselves.

## Handles / CLI Surfaces

- Packaged install/uninstall commands.
- Persistent service start/stop, `/health`, `allbert onboard`, `allbert tui`,
  `allbert admin ...`; foreground `allbert serve` remains a diagnostic fallback.
- Web `/`, `/workspace`, `/jobs`, `/objectives`, onboarding, settings/model, and
  operator panels.
- Export/import or upgrade commands.
- Secret/evidence leak scan.
- `mix allbert.test release.v066` deterministic gate.

## Smoke Checklist

- Install binary on a clean host/home.
- Start the persistent service; verify runtime/web/channels health.
- Complete web QuickStart onboarding via guided local runtime setup if needed and the
  consumer-default one-click local-model download (no manual model CLI, no API key) to
  first chat; terminal onboarding smoke.
- Run the natural-prompt routing set and confirm no mis-route to a disabled/demo
  capability; run the non-developer web-UX usability audit against the item-11 bar.
- Recheck the v0.63 onboarding command contract: bare resume,
  `--quickstart`, `--advanced`, confirmed `--reset`, and
  canonical `--non-interactive --authorize` plus the deprecated
  `--accept-risk` alias warning.
- Use browser control to verify `/`, `/workspace`, `/jobs`, `/objectives`,
  onboarding, settings/model, and operator panels.
- Use grouped CLI/TUI admin reads without raw `mix`.
- Exercise the local files/notes/memory launch path.
- Exercise remote channels, MCP, browser research, plan approval, public
  protocols, and export/import as implemented-surface regression blockers.
- Export/import or upgrade a Home; verify behavior and redaction.
- Uninstall service/binary; verify Home preservation unless explicit data removal
  is requested.
- Run leak scans, docs gate, warning gate, and `release.v066`.

## Operator Validation

Status: planned. Commands are illustrative until the milestone lands; verify and
replace with observed output at closeout.

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v066.XXXXXX)"
# Choose the packaged service path for the host:
#   macOS/Homebrew: brew services start allbert
#   Linux service:  systemctl --user start allbert
curl -fsS http://localhost:4000/health
allbert admin health
allbert onboard --quickstart
allbert onboard
allbert onboard --advanced
allbert ask "hello from v0.66 validation"
allbert admin security status
mix allbert.test release.v066
```

Expected behavior:

- install, service start, onboarding, local files/notes/memory, first chat, inspection,
  implemented-surface regression smoke, export/import or upgrade, and uninstall
  all pass with redacted evidence;
- no-docs validation of the primary launch path is recorded;
- web and CLI/TUI surfaces require no raw `mix` for operator work;
- no secret appears in logs, evidence, output, docs, or screenshots;
- `release.v066`, docs, warning, and drift checks pass.
