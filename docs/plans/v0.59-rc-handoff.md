# v0.59 RC-Substrate Handoff

v0.59 is a hardening substrate handoff. It leaves three downstream consumers with
concrete, testable outputs and no new product authority. Packaging, onboarding,
and product-RC work remain in their later milestones.

## Consumer Handoff Matrix

| Consumer | v0.59 output consumed | Evidence owner | Boundary |
| --- | --- | --- | --- |
| v0.61 Packaged-Install Home Layout | Allbert Home export envelope; dry-run import diagnostic; first-class Settings fragment `schema_version`; secret references only; operator/developer portability docs | `release.v059` evidence plus `docs/operator/export-import.md`, `docs/operator/settings-version.md`, and `docs/developer/portability.md` | v0.61 may adapt package layout and Settings Central secret-vault references, but the credential move is a separate operator migration outside ADR 0046. |
| v0.63 Product RC Revalidation | packaged-layout portability revalidation; M5 perf/CSP target evidence; S0-S8 operator validation flow; `release.v059` evidence | v0.63 request-flow and RC evidence | v0.63 validates product readiness on top of this substrate; it does not redefine the v0.59 envelope, version contract, or dry-run-only import behavior. |
| v1.0 Contract Freeze | ADR 0046 settings version contract; ADR 0065 central Runner param seam; Registry action schema; `:invalid_params` shape; redaction and dry-run import invariants | v1.0 freeze checklist and release evidence | v1.0 freezes these Tier-1 contracts only after v0.59 M7 and M8 evidence are green. |

## v0.61 Inputs

v0.61 consumes the portability substrate, not a live migration engine:

- `AllbertAssist.Portability.Export` envelope with explicit `envelope_version`.
- `AllbertAssist.Portability.Import.dry_run/2` diagnostic with `dry_run: true`
  and `applied: false`.
- Per-fragment settings inventory from `Settings.VersionContract`.
- Secret reference status metadata; secret values and secret-store files remain
  excluded from export.
- Operator steps in `docs/plans/v0.59-request-flow.md` S1-S3 and S7.

The v0.61 package layout may choose where Allbert Home is placed and how
Settings Central secret-vault references are resolved. It must not reinterpret
v0.59 dry-run import as an applied import.

## v0.63 Inputs

v0.63 consumes v0.59 as the substrate proof for product RC:

- Re-run export and dry-run import against the packaged Allbert Home layout.
- Re-use the M5 deterministic perf target rule:
  `max(2s, baseline * 1.25)` for export/import and
  `max(50ms, baseline * 1.10)` for boot-check overhead.
- Preserve the reconciled landing/workspace CSP from M5.
- Treat `mix allbert.test release.v059` evidence as the hardening baseline before
  product-RC evidence is collected.

v0.63 can add product readiness checks, but it must not relax v0.59 redaction,
dry-run, inert-import, settings-version, or param-contract requirements.

## v1.0 Inputs

v1.0 consumes v0.59 as freeze-prep evidence:

- ADR 0046 settings version contract: `schema_version` on every registered
  Settings fragment, additive-only schema comparison, and fail-closed forward or
  invalid fragment versions.
- ADR 0065 central Runner param seam: schema-known string-key normalization,
  unknown-key rejection unless explicitly dispositioned, `validate_params/1`
  execution before action bodies, and `validated_params` passed to `run/2`.
- Registry action schema and the redacted `:invalid_params` shape.
- Export/import redaction, dry-run-only behavior, self-improvement inertness, and
  media-capture inertness.

The v1.0 freeze cannot treat advisory model output, package metadata, imported
metadata, or objective IDs as authority. Security Central and registered Jido
actions remain the authority boundary.

## Drift Check

`rc-substrate-no-drift-001` passes only when the handoff remains explicit:

- v0.61, v0.63, and v1.0 are all named.
- Each consumer has at least one concrete v0.59 output.
- Secret-vault credential movement is not folded into ADR 0046.
- Applied import, packaging, onboarding, and product-RC implementation are not
  moved into v0.59.
- No placeholder or vague target wording remains.
