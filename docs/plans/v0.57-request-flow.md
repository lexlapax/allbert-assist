## Locked Decisions
Recorded in `v0.57-plan.md`. Pi-mode is a channel/app on the one spine with no
extra authority; **six default tools** — four effectful boundary actions
(`read`/`write`/`edit`/`bash`) + two read-only search actions (`grep`/`glob`,
unprompted) — through `Actions.Runner.run/3` + Security Central; a named
**local-coding operator** trust tier (ADR 0056 lineage, sandbox **Level 1**; single
trusted operator, main session, `tui`) that is never the default and never for
channel-originated or generated-code sessions; **coder-facing approval modes**
(`default`/`accept-edits`/`plan`/`tier`) + per-repo "always allow" as a
confirmation-cost layer over Security Central (no new authority); `bash` is
tier-restricted **raw shell** (argv-only `run_shell_command` for everyone else),
cwd-jailed + timed out; deterministic acceptance gates for effectful/generated-code
work (the ordinary turn ends when the model stops calling tools); sub-1000-token
**prompt + tool-defs** budget + chunked-read context discipline; **net-new v0.57**
live rendering — assistant-text token streaming + progressive tool-argument diff
streaming over the v0.55 **static** split, plus Esc-to-cancel + queued steering;
a coding **slash set** (`/help`/`/model`/`/clear`/`/init`/`/diff`/`/compact`);
mid-session model switch over req_llm message-history; delegate agents (no
bash-spawned sub-agents).

## Main User / Operator Flows

### Opening a Pi-mode coding session
A single trusted operator, at their own terminal, opens the v0.55 `tui` channel in
the **main session**. They request Pi-mode for the working repo; Security Central
resolves the **local-coding operator** tier (at sandbox **Level 1**) *only* because
the caller is the trusted operator, in the main session, on `tui`. The session
loads a coding system prompt **+ four tool definitions under the combined token
budget**, assembled from the `AGENTS.md` hierarchy + the v0.03 skill loader,
exposing the six default tools (`read`/`write`/`edit`/`bash` + read-only
`grep`/`glob`) and the operator's chosen approval mode. The objective is "pair on
this repo"
(stage 3 trivial); step proposal (stage 4) is trivial; authorization (stage 5)
runs cheaply at the tier — *present, not absent* — and policy/redaction/trace/audit
still run at Level 1.

### The read / write / edit / bash / grep / glob loop at the local-coding tier
The operator types an instruction. The model proposes a tool call as **advisory
output** — it does not execute anything itself. `grep`/`glob` (read-only,
gitignore-aware) run **unprompted**, as in every peer CLI. Each effectful proposal
(`read`/`write`/`edit`/`bash`) becomes a signal → `Actions.Runner.run/3` → Security
Central gate. The operator's **approval mode** sets the confirmation cost:
`default` prompts effects, `accept-edits` auto-approves `write`/`edit` but prompts
`bash`, `plan` is read-only (search/read only — proposals never execute), `tier` is
cheapest for the trusted local caller; a persisted "always allow `<cmd>`" suppresses
a specific prompt per repo. At the local-coding tier the gate runs *cheap* but
**runs** — it still evaluates policy, redacts, and writes the trace at Level 1.
`read`/`write`/`edit` are confined to `coding.workspace.cwd_jail` (normalized path
and symlink checks); `bash` is host execution behind the registered action: raw
shell **only at this tier** (argv-only otherwise), cwd-jailed and timed out (reuse
`skill_script_runner`); the host/container, not OTP/BEAM, is the security boundary.
Context is gathered through **chunked reads** (offset/limit) and file artifacts.

### Streamed diffs + token streaming
`write`/`edit` produce the split tool-result payload (`model_payload` vs.
`surface_payload`, ADR 0067 extending ADR 0029): the model receives the clean
`model_payload`; the terminal renders the `surface_payload` diff through the
unified renderer (ADR 0030) / `owl`. The streaming is **net-new v0.57** work — a
coding-diff live-region renderer that renders the diff by progressively parsing the
tool-call **arguments** as they arrive, **and token-by-token streaming of the
assistant's prose** so the turn feels live. Token streaming needs the runtime to
emit model token deltas (v0.55 is turn-complete). The `surface_payload` never leaks
back into model context.

### Interrupt + steer
The operator can press **Esc to cancel** a running turn: the in-flight model call
and any running action are aborted cleanly, work done so far is kept, and a
**partial-turn trace** is written (`bash` brutal-kill bounds a runaway command). A
**queued correction** lets the operator redirect without killing the turn (applied
on the next turn). No authority path changes.

### Mid-session model switch
The operator invokes the model-switch affordance (a `tui` slash command
`/model <name>`). The session's req_llm message history is serialized and handed to
the new model; cross-provider history conversion is **best-effort** (provider
reasoning/tool-call shapes differ). M5 verifies the concrete API through Context7
when available and otherwise through vendored `deps/req_llm` docs/source. The
conversation continues with **no change to the authority path** — same runner, same
Security Central, same trace.

### Completion
The ordinary interactive turn ends naturally when the model **stops emitting tool
calls** and control returns to the operator prompt — the same REPL norm as Claude
Code / Codex / Gemini. The separate rule, "the model never decides it's *done*,"
governs **deterministic acceptance gates** for effectful or generated-code work
(evidence + confirmation), the same posture Allbert took against Hermes-style
model-confidence completion — not the end of a conversational turn.

## Data Flow
```text
operator terminal input (v0.55 tui channel, supervised GenServer)
  -> normalize -> Identity.resolve (list-shaped channels.tui.identity_map)
  -> Channels.derive_session_id (main session) -> Runtime.submit_user_input
  -> coding loop: model emits advisory tool proposal (6 default tools)
       (sub-1000-token prompt+tool-defs, chunked-read context)
       grep/glob [read-only] -> run unprompted, no gate cost
  -> signal -> Actions.Registry -> Actions.Runner.run/3
  -> Security Central gate  [approval mode + local-coding tier @ Level 1: present-but-cheap]
       mode: default(prompt) | accept-edits(auto write/edit) | plan(read-only) | tier(cheapest)
       + per-repo "always allow <cmd>"  (confirmation cost only; no authority)
       trusted operator + main session + tui  -> cheap gate, raw-shell bash, full trace
       channel-originated / scheduled / generated-code -> full gate, argv-only (no tier)
  -> action executes (read[chunked] | grep/glob[read-only] | write[cwd-jail] | edit[cwd-jail] | bash[Level 1])
  -> split tool-result (ADR 0067/0029): model_payload + surface_payload diff
  -> terminal render via NET-NEW v0.57 live region (ADR 0030 / owl): progressive
       tool-argument diff streaming + assistant-text token streaming; model sees only model_payload
       [operator] Esc -> cancel turn (keep work, partial-turn trace) | queue correction
  -> channel_events (dedupe, redacted summary) + trace + markdown-first memory
  -> [operator] mid-session model switch -> req_llm message-history handoff (no authority change)
```

## How the gates stay present-but-cheap at the tier
The local-coding operator tier changes the *cost* of the gate, never its
*existence* or the *sandbox level*. For the single trusted operator in the main
session on `tui`, Security Central skips the per-action confirmation prompt **but
still runs at Level 1**: it evaluates policy, enforces the `bash` cwd-jail +
timeout, redacts/truncates `channel_events` summaries, and writes the full trace.
The tier is a **named, auditable trust tier** (ADR 0056 lineage), never the default
— every other caller (channel-originated inbound, scheduled jobs, generated-code
sessions) gets the full gate and argv-only `bash` with no tier escalation. This is
the rationale's "stages 3–5 intact; gates present but cheap" made concrete, without
relaxing the sandbox level.

## Authority Invariants
- Pi-mode calls only `Runtime.submit_user_input/1` + `Actions.Runner.run/3`; no
  direct Security Central / confirmation-store / Resource Access access (ADR 0016).
- Advisory model output never authorizes execution; every effect routes through the
  runner + Security Central.
- The local-coding operator tier resolves only for trusted operator + main session
  + `tui` (at sandbox Level 1); never the default; unreachable by
  channel-originated, scheduled, or generated-code sessions.
- `bash` is cwd-jailed + timed out; raw shell resolves only at the tier (argv-only
  otherwise); OTP/BEAM supervision is not an OS security boundary — the
  host/container is.
- `grep`/`glob` are read-only and run unprompted; they have no effect path.
- Approval modes and "always allow" change only confirmation cost — never the
  authority path; `plan` mode executes nothing; no mode grants authority beyond its
  definition.
- Esc-to-cancel aborts the in-flight model call + any running action cleanly and
  writes a partial-turn trace; it changes no authority.
- Acceptance for effectful/generated-code work is deterministic (evidence +
  confirmation); the ordinary turn ends when the model stops calling tools.
- The split-result UI payload never leaks into the model payload (diffs **and**
  streamed assistant tokens render from `surface_payload`/`model_payload` only).
- Mid-session model switch changes the model only; the authority path is unchanged.
- Coding slash commands are non-routable (allowlist only); never intent candidates.
- Delegation uses restricted-registry delegate agents; no bash-spawned sub-agent.

## Smoke Checklist
- Pi-mode opens on `tui` for the trusted operator; a full loop across the six tools
  drives streamed diffs + token-streamed prose and `channel_events` dedupe.
- `grep`/`glob` return repo matches **unprompted** (read-only); no confirmation.
- The local-coding operator tier resolves only for the trusted/main/`tui` case (at
  Level 1, raw-shell `bash`); a simulated channel-originated or generated-code
  caller gets the full gate and argv-only `bash`.
- Approval modes behave: `plan` blocks a `write` (read-only); `accept-edits`
  auto-applies an `edit` but still prompts `bash`; a persisted "always allow `<cmd>`"
  suppresses only that prompt, per repo, and is auditable.
- `write`/`edit` refuse path traversal, symlink escape, and destinations outside
  `coding.workspace.cwd_jail`; the same simulated non-tier callers get the full
  gate for file effects.
- A `write`/`edit` produces a streamed diff (progressive tool-argument render) and
  token-streamed assistant text with no `surface_payload` in model context.
- **Esc** mid-turn cancels cleanly (work kept, partial-turn trace); a queued
  correction applies on the next turn.
- The coding slash set (`/help`/`/model`/`/clear`/`/init`/`/diff`/`/compact`) creates
  no model turn or `channel_events` row.
- A mid-session model switch preserves session history and changes no authority.
