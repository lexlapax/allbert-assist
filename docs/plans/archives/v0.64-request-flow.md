# Allbert v0.64 Request Flow - Trusted Install And Non-Developer First Run

Status: released as `v0.64.3` on 2026-07-10 UTC; version 0.64.3. `mix allbert.test
release.v064` remains the deterministic handoff gate. `v0.64.0` was tagged first, but its release-artifacts workflow
blocked before publish because the pre-publish Linux rehearsal did not create the cosign
bundle required by the fail-closed installer. `v0.64.1` fixes that release-contract gap;
the release-artifacts workflow passed and published GitHub Release assets. Local macOS
validation confirmed fail-closed installer guidance before `cosign` was available, then
after explicit verifier-install approval confirmed full curl-install success,
boot/health/attach/first-run behavior, and uninstall/Home preservation. `v0.64.2` is the
corrective pass for the stale Homebrew tap/formula path, service-first installer copy,
and concurrent first-boot migration race found after the v0.64.1 closeout. `v0.64.3` is
the final-audit corrective pass: it fixes a cross-app version drift
(`allbert_assist_web` had stayed at `0.64.1`), gates that drift class in `release.v064`,
makes the web curated-model pull stream progress live instead of batching, and reconciles
the validation/install docs with observed released behavior. `v0.64.5` is a source/docs
point tag (`[skip-artifacts]`) that corrects remaining release-facing prose and hardens
the docs/source tag gate only; the packaged artifact line stays `v0.64.3`. See
`v0.64-plan.md` M9 and M10.

Implementation details live in `docs/plans/archives/v0.64-plan.md`.

## Main User/Operator Flows

- Operator installs Allbert from Homebrew or curl. The curl installer verifies the signed
  checksum bundle before install; Homebrew verifies the artifact against the trusted
  formula's pinned SHA256.
- Operator starts Allbert as a persistent background service (`brew services start
  allbert` / `systemctl --user`) once and opens the browser workspace -- no repeated
  `serve`.
- Operator completes web-first QuickStart or resumes through `allbert onboard`.
- **Consumer default:** if the supported local runtime is missing, Allbert offers a
  guided confirmed install; once the runtime is healthy, the operator one-click downloads
  a curated local model with in-web progress, no manual `ollama` CLI, and no API key to
  reach first chat.
- **Advanced:** prosumer operator supplies a BYOK hosted key or a custom endpoint.
- Operator sees plain-language repair screens for any blocked first-run state.
- Operator reviews the trust spine and sends a first useful chat.
- Maintainer runs docs, warning, drift, leak-scan, and `release.v064` gates.

## Data Flow

No new runtime data flow is introduced. This release makes existing install,
first-run, model/provider, vault, onboarding, and trust surfaces coherent for a
non-developer. Evidence must use disposable homes or explicitly declared test
homes, never a real operator home.

## Authority Invariants

Installer trust verifies artifacts but grants no runtime authority. Onboarding,
first-run state, profile selection, and repair affordances do not grant
permissions by themselves. Effectful work remains behind registered actions,
Security Central, confirmations, Resource Access, traces, and audits.

## Handles / CLI Surfaces

- Homebrew and curl packaged install paths, including Homebrew tap freshness.
- Persistent service manager commands, `/health`, `allbert admin health`,
  `allbert onboard`, `allbert ask`, and read-only admin status checks.
- `allbert serve` remains a diagnostic/repair fallback, not the happy-path first-run
  command.
- Web `/`, `/workspace`, onboarding, settings/model, and trust-spine surfaces.
- Installer verification, rollback/restore, secret/evidence leak scan.
- `mix allbert.test release.v064` deterministic gate.

## Smoke Checklist

- Install from the packaged path on a clean host/home.
- Verify signature/trust failure messages are understandable and fail closed on the curl path; if the
  verifier is missing, verify the guided verifier-install path completes before artifact
  installation continues.
- Verify the Homebrew tap resolves to the current release version, not an older formula,
  and that `brew test allbert` passes after install.
- Start Allbert as a background service; verify browser-first onboarding opens and the
  operator does not need to re-run `serve`.
- Consumer default: from both runtime-missing and model-missing states, verify guided
  local runtime setup as needed, one-click curated-model download, visible progress,
  retry/resume behavior, and first chat with no manual `ollama` CLI and no API key.
- Advanced: exercise the BYOK/custom path far enough to verify credential storage and
  fallback messaging.
- Verify a set of natural non-developer prompts route sensibly with no mis-route to a
  disabled or demo capability.
- Trigger at least one blocked first-run state and verify one primary repair
  action is shown.
- Confirm first chat works without raw `mix` or hand-edited config.
- Verify no secret appears in output, logs, traces, evidence, docs, or
  screenshots.
- Run docs gate, warning gate, drift checks, and `release.v064`.

## Operator Validation

Status: validated. The packaged install, persistent-service, and first-run behavior were
operator-validated on macOS at v0.64.1 and re-validated for the Homebrew tap and first-boot
migration serialization at v0.64.2 (both recorded in `CHANGELOG.md`). v0.64.3 changes only
the umbrella version, the `release.v064` version-drift gate, the web pull-progress
rendering, and docs — the install/service/first-run path is unchanged from v0.64.2, and the
v0.64.3 web live-progress behavior is proven deterministically by `release.v064` (the
async `workspace_live` progress case). The commands below are the validated operator path;
run them to reproduce on a clean host/Home.

```sh
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v064.XXXXXX)"
# curl install path: missing cosign fails closed; installed cosign verifies the bundle.
# Homebrew path: the public tap installs directly (a newer Homebrew may prompt once to
# trust a third-party tap on first install — approve the prompt if shown).
brew install lexlapax/allbert/allbert
allbert --version                 # 0.64.3
brew test allbert

# Choose the packaged service path for the host:
#   macOS/Homebrew: brew services start lexlapax/allbert/allbert
#   Linux service:  systemctl --user start allbert

curl -fsS http://localhost:4000/health
allbert admin health
allbert admin service status
allbert admin db list-backups
# Open the browser workspace and complete web QuickStart:
# 1. guided verifier/runtime confirmations if shown;
# 2. consumer-default local model pull with progress;
# 3. first useful chat.
allbert ask "hello from v0.64 first run"
mix allbert.test release.v064
```

Expected behavior:

- curl verification succeeds or fails closed with a clear message;
- missing verification tooling is guided, then verification completes before curl install;
- Homebrew resolves the current formula version/URLs and verifies the artifact SHA256;
- concurrent fresh-Home first commands serialize startup migrations and avoid raw
  duplicate-table migration errors;
- backup-before-migrate restore is inspectable through `allbert admin db list-backups`
  and `allbert admin db restore ... --dry-run`;
- consumer-default onboarding reaches first useful chat through guided local runtime
  setup if needed and one-click model pull with progress, without manual `ollama` CLI or
  an API key;
- BYOK/custom remains available as an advanced or fallback path and stores credentials
  through the vault;
- blocked first-run states show one clear repair action;
- web and CLI/TUI operator surfaces require no raw `mix`;
- no secret appears in logs, evidence, output, docs, or screenshots;
- `release.v064`, docs, warning, and drift checks pass.
