# Allbert v0.45.1 Request Flow - Gate Transparency And Precommit Decomposition

Plan: `docs/plans/archives/v0.45.1-plan.md`

ADR amendment: `docs/adr/0049-development-gates-and-test-parallelization.md`

Status: implemented as 0.45.1.

## Flow Summary

v0.45.1 changes developer gate behavior, not assistant runtime
behavior. The request "run the gate" becomes explicit about intent:

- commit gate: fast local confidence for the next commit;
- prepush gate: high-coverage local confidence before sharing;
- release gate: exhaustive release evidence;
- external smoke: explicit machine-dependent validation.

No flow installs or rewrites Git hooks. Operators and agents run the
commands directly unless a later plan adds hook installation.

## Developer Story

### Commit-Time Flow

Input:

```sh
mix allbert.test commit
```

Compatibility input:

```sh
mix precommit
```

Expected behavior:

1. The command prints `commit gate` with timestamp and cwd.
2. The command inspects the working tree and staged tree enough to
   classify the change as docs-only, code, mixed, or unknown.
3. Docs-only changes run the docs gate and stop.
4. Code or mixed changes run static checks and changed/stale-aware
   focused guidance.
5. Any stale/changed-file test execution is labeled non-release
   evidence.
6. The command prints the next recommended gate:
   - focused files for a milestone;
   - `mix allbert.test prepush` before sharing;
   - `mix allbert.test release` before release handoff.

The command must not print "release-ready" or equivalent language.

Failure behavior:

- Static failure stops the gate and prints the phase id.
- Changed-file mapping failure does not fail the gate by itself; it
  prints the focused-test command the developer should run manually.
- Docs-only drift fails on `mix allbert.test docs`.

### Prepush Flow

Input:

```sh
mix allbert.test prepush
```

Expected behavior:

1. Static checks run first.
2. The high-coverage partitioned local gate runs with documented
   default partitions:

```sh
mix allbert.test fast-local --core-lanes --stocksage-lanes --web-lanes --partitions N
```

3. Each lane prints a phase start, phase finish, duration, and status.
4. A JSON timing artifact is written when `ALLBERT_HOME` is set or a
   temp evidence root is created.

Failure behavior:

- Any phase failure stops later phases unless the command is run with a
  future explicit diagnostic flag.
- Empty partition shards are accepted only under the existing v0.41
  empty-partition rules.

### Release Flow

Input:

```sh
mix allbert.test release
```

Expected behavior:

The release gate runs the phases below directly. It must not call
`mix precommit`.

1. `static_compile`
2. `deps_unused`
3. `format`
4. `credo`
5. `core_tests`
6. `web_tests`
7. `stocksage_tests`
8. `channel_plugin_tests`
9. `dialyzer`

Each phase emits:

- `==> release <phase_id> started`
- command cwd and redacted command args
- bounded streaming output or a heartbeat while the child command runs
- `==> release <phase_id> finished in <duration>`
- status and summary
- a full redacted phase log path in release/prepush evidence
- ExUnit seed values when the phase output includes them

The final summary prints all phases in order with durations and statuses.
The command writes JSON evidence under:

```text
<ALLBERT_HOME>/release_evidence/gates/release-<timestamp>.json
```

When `ALLBERT_HOME` is absent, the command creates a temp Allbert Home,
prints the temp evidence path, and uses that path for the run.
Full redacted phase logs are written beside the JSON in a per-run artifact
directory named for the gate and timestamp.

Failure behavior:

- A failing phase stops later phases.
- The failed phase's redacted output tail is included in the summary.
- The failed phase's full redacted log path is included in evidence and in the
  raised Mix error where available.
- Failed Mix test phases snapshot relevant `.mix_test_failures` manifests into
  the same artifact directory.
- The process exits non-zero.
- Any secret-looking value in output tails and full logs must be redacted before
  persistence.

### Version-Specific Release Flow

Input:

```sh
mix allbert.test release.v045
```

Expected behavior:

- Existing deterministic Marketplace Lite release evidence remains.
- v0.45.1 may add timing output, but it must not change the v0.45
  surface invariants or the no-external-network contract.

### External Smoke Flow

Input:

```sh
mix allbert.test external-smoke -- browser_research
mix allbert.test external-smoke -- docker_sandbox
mix allbert.test external-smoke -- docker_full_gate
```

Expected behavior:

- External smokes remain explicit opt-in gates.
- They are not hidden inside `commit`, `prepush`, `precommit`, or
  `release`.
- A plan must name an external smoke before it can be release evidence.

## Evidence JSON Shape

The release and prepush timing artifact uses this minimum shape:

```json
{
  "schema_version": 1,
  "gate": "release",
  "started_at": "2026-06-02T00:00:00Z",
  "finished_at": "2026-06-02T00:00:00Z",
  "duration_ms": 0,
  "status": "passed",
  "phases": [
    {
      "id": "core_tests",
      "cwd": "apps/allbert_assist",
      "args": ["mix", "test"],
      "started_at": "2026-06-02T00:00:00Z",
      "finished_at": "2026-06-02T00:00:00Z",
      "duration_ms": 0,
      "status": "passed",
      "exit_status": 0,
      "summary": {
        "tests": 1335,
        "failures": 0,
        "skipped": 3
      },
      "redacted_output_tail": "..."
    }
  ]
}
```

The artifact is diagnostic evidence only. It grants no runtime
authority, modifies no settings, and is not read by Security Central.

## Help Text Contract

`mix allbert.test` help must list:

```text
mix allbert.test docs
mix allbert.test inventory [--output PATH] [--check-tags]
mix allbert.test focused -- FILE [FILE...]
mix allbert.test commit
mix allbert.test prepush [--partitions N]
mix allbert.test fast-local [--core-lanes] [--stocksage-lanes] [--web-lanes] [--partitions N]
mix allbert.test partition-smoke [--partitions N]
mix allbert.test serial-core --lane LANE [--partitions N]
mix allbert.test release
mix allbert.test release.v042
mix allbert.test release.v043
mix allbert.test release.v044
mix allbert.test release.v045
mix allbert.test external-smoke list
mix allbert.test external-smoke -- browser_research
mix allbert.test external-smoke -- docker_sandbox
mix allbert.test external-smoke -- docker_full_gate
```

The help text must call out that `mix precommit` is a shortcut for the
commit gate after v0.45.1 and not release evidence.

## Test Matrix

Focused tests:

- `MIX_ENV=test mix test apps/allbert_assist/test/mix/tasks/allbert_test.exs`
- New phase-runner tests for:
  - successful phase timing
  - failing phase short-circuit
  - redaction before evidence persistence
  - bounded output tails
  - `release` does not invoke `mix precommit`
  - `mix precommit` routes to the commit gate
  - prepush uses high-coverage partitioned fast-local command shape

Gate tests:

- `MIX_ENV=test mix allbert.test docs`
- `MIX_ENV=test mix allbert.test commit`
- `MIX_ENV=test mix precommit`
- `MIX_ENV=test mix allbert.test prepush`
- `MIX_ENV=test mix allbert.test release.v045`
- `MIX_ENV=test mix allbert.test release`

Docs-only closeout:

- `git diff --check`
