## Locked Decisions

Each is recorded in `v0.50b-plan.md` §"Locked Decisions".

1. A plugin/app `plugins/allbert.artifacts/` with plugin id
   `allbert.artifacts` and app id `:allbert_artifacts`; not core.
2. Read-only through core `:artifact_read` actions; delete via core
   confirmation-gated `delete_artifact`; no direct object-store access.
3. Workspace `:canvas_panels` panel via `App.surfaces/0` +
   `workspace_panel_surfaces/1`; rendered through the host catalog.
4. `/apps/artifacts/<sha>` page LiveView; route in core router, module
   plugin-owned, rendered via host `Surface.Renderer`.
5. `mix allbert.artifacts` CLI as a thin shell over `Actions.Runner.run/3`.
6. Search/filter over the core list/query actions; no new index.

## Main User/Operator Flows

- Operator opens the Artifacts panel in `/workspace` and scans recent artifacts
  (metadata only). Row navigation can be wired after the route and filter
  controls exist.
- Operator deep-links `/apps/artifacts/<sha>` to inspect a specific artifact's
  metadata, provenance (linked threads/messages), retention state, and
  confirmation-gated delete request.
- Operator filters the Artifacts panel by URL/context query params such as
  `artifact_type`, `artifact_origin`, `artifact_thread`, and `artifact_since`.
- Operator runs `mix allbert.artifacts list --type <mime> --origin <origin>
  --thread <id> --since <date>`, `show <sha>`, `threads <sha>` (reverse lookup),
  `doctor`, or `rm <sha>` (confirms).
- From an artifact, the operator follows its `created_by` thread link back to the
  conversation that produced it.

## Data Flow

```text
workspace panel | /apps/artifacts/<sha> page | mix allbert.artifacts
  -> validate params (sha is lowercase 64-char SHA-256 before page/action read)
  -> normalize browser filters (type/origin/thread/since/retention/lifecycle/limit)
  -> Actions.Runner.run("list_artifacts"|"get_artifact"|"artifact_threads"
       |"artifact_doctor", params, %{channel: :workspace|:cli, ...})
  -> :artifact_read (floor :allowed) -> core store/query (v0.50)
  -> redacted metadata + provenance rows (raw bytes never returned)
  -> rendered as host catalog nodes (panel/page) or printed (CLI)

invalid sha
  -> redacted not-found/error state; no store read and no fallback filesystem access

remove -> Actions.Runner.run("delete_artifact", %{sha: ...}, ctx)
  -> :artifact_delete (floor :needs_confirmation) -> core confirm + delete
```

## Authority Invariants

- The browser contributes data only and acquires no authority (ADR 0017); it
  cannot define the scheme, register permissions, add a Home root, or own the
  sensor.
- Every read resolves through `:artifact_read`; a rendered row, content address,
  or thread link grants nothing.
- The plugin never calls the core store, Allbert Home artifact root, metadata
  sidecars, or filesystem APIs directly.
- Delete is the only mutation and routes through the core confirmation-gated
  `delete_artifact`.
- Surfaces render redacted metadata only; raw bytes never enter LiveView
  assigns, the page, or CLI output. No model-generated UI code is executed.

## Smoke Checklist

- Open the Artifacts panel; confirm it lists metadata-only rows.
- Deep-link a detail page by sha; confirm provenance + retention render.
- Try an invalid sha path; confirm it returns a redacted not-found/error state
  without a store read.
- `mix allbert.artifacts list --thread <id>` returns that thread's artifacts.
- `threads <sha>` returns the producing thread; the link grants nothing.
- `rm <sha>` confirms before deleting; output is redacted.
- Type/origin/since filters narrow panel and CLI results without a new index.
