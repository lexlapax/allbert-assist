Status: **approved; M0 in progress.** This is the executable companion to
[v1.0.4-plan.md](v1.0.4-plan.md). It exists because published v1.0.3 omitted
the Playwright package tree and Chromium payload: macOS `browser_doctor`
failed with `Cannot find module 'playwright/package.json'`. The v1.0.3 tag is
immutable; all corrective assets ship as v1.0.4.

## D. Tag, CI, Release, Signature, And Assets

Only after section C passes:

```sh
git push origin main
test "$(git rev-parse HEAD)" = "$(git ls-remote origin refs/heads/main | awk '{print $1}')"
git tag -a "$VERSION" -m "Allbert $EXPECTED_VERSION"
git push origin "$VERSION"
test "$(git rev-parse "$VERSION^{}")" = "$RELEASE_SHA"
test "$(git ls-remote origin "refs/tags/$VERSION^{}" | awk '{print $1}')" = "$RELEASE_SHA"

RUN_ID="$(gh run list --repo "$REPO" --workflow release-artifacts.yml \
  --branch "$VERSION" --limit 1 --json databaseId --jq '.[0].databaseId')"
gh run watch "$RUN_ID" --repo "$REPO" --exit-status
gh run view "$RUN_ID" --repo "$REPO" --json conclusion,headSha,url,jobs \
  > "$EVIDENCE_ROOT/ci-run.json"
test "$(jq -r .headSha "$EVIDENCE_ROOT/ci-run.json")" = "$RELEASE_SHA"
test "$(jq -r .conclusion "$EVIDENCE_ROOT/ci-run.json")" = success

mkdir -p "$EVIDENCE_ROOT/release-assets"
gh release view "$VERSION" --repo "$REPO" \
  --json tagName,isLatest,url,assets > "$EVIDENCE_ROOT/release.json"
gh release download "$VERSION" --repo "$REPO" \
  --dir "$EVIDENCE_ROOT/release-assets"
test "$(find "$EVIDENCE_ROOT/release-assets" -maxdepth 1 \
  -name 'allbert-v1.0.4-*.tar.gz' | wc -l | tr -d ' ')" = 3
test -f "$EVIDENCE_ROOT/release-assets/SHA256SUMS"
test -f "$EVIDENCE_ROOT/release-assets/SHA256SUMS.cosign.bundle"
(cd "$EVIDENCE_ROOT/release-assets" && sha256sum -c SHA256SUMS)
cosign verify-blob \
  --bundle "$EVIDENCE_ROOT/release-assets/SHA256SUMS.cosign.bundle" \
  "$EVIDENCE_ROOT/release-assets/SHA256SUMS" \
  --certificate-identity "https://github.com/lexlapax/allbert-assist/.github/workflows/release-artifacts.yml@refs/tags/$VERSION" \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' \
  | tee "$EVIDENCE_ROOT/cosign-verify.log"
```

PASS: the peeled tag, workflow `headSha`, Release tag, three tarballs,
SHA256SUMS, and mandatory bundle all identify v1.0.4; every CI job, including
all three browser-inclusive artifact smokes, is green.

## E. Fresh Tap Fill

Always clone a new checkout. Do not reuse an installed tap worktree:

```sh
export ALLBERT_ASSIST_CHECKOUT="$(pwd)"
export TAP_CHECKOUT="$(mktemp -d /tmp/homebrew-allbert-v104.XXXXXX)"
git clone https://github.com/lexlapax/homebrew-allbert "$TAP_CHECKOUT"
sh homebrew/fill-sha256.sh \
  "$EVIDENCE_ROOT/release-assets/SHA256SUMS" \
  "$TAP_CHECKOUT/Formula/allbert.rb"

test "$(sed -n 's/^  version "\([^"]*\)"/\1/p' Formula/allbert.rb)" = 1.0.4
test "$(rg -c 'releases/download/v1.0.4' Formula/allbert.rb)" = 3
rg -n 'PLACEHOLDER|REPLACE_' Formula/allbert.rb && exit 1 || true
brew tap lexlapax/allbert "$TAP_CHECKOUT" --custom-remote
brew trust --formula lexlapax/allbert/allbert
brew audit --strict --online --formula allbert \
  | tee "$EVIDENCE_ROOT/tap-audit.log"
git add Formula/allbert.rb
git commit -m "allbert 1.0.4"
git push origin main
git rev-parse HEAD | tee "$EVIDENCE_ROOT/tap-commit.txt"
cd "$ALLBERT_ASSIST_CHECKOUT"
```

PASS: formula version, three URLs, and checksums match the published release;
audit passes and the tap commit is pushed. If `brew tap` already exists, remove
or retarget only that tap registration before this block; never assume `brew
update` refreshed a prior local custom remote.

## F. macOS Packaged Product Validation

Use separate disposable Homes for curl and Homebrew. Refresh the installed tap
before installing:

```sh
brew update
brew tap --repair
brew upgrade lexlapax/allbert/allbert || brew reinstall lexlapax/allbert/allbert
export ALLBERT_HOME="$(mktemp -d /tmp/allbert-v104-macos-home.XXXXXX)"
test "$(allbert --version)" = "allbert 1.0.4"
allbert admin status | tee "$EVIDENCE_ROOT/macos-status.log"
allbert admin settings set browser.enabled true
allbert admin settings set research.enabled true
allbert eval 'Application.ensure_all_started(:allbert_assist); IO.inspect(AllbertAssist.Actions.Runner.run("browser_doctor", %{}, %{actor: "release", channel: :cli}))' \
  | tee "$EVIDENCE_ROOT/macos-browser-doctor.log"
rg 'live_check_status: :ok|"live_check_status":"ok"' \
  "$EVIDENCE_ROOT/macos-browser-doctor.log"
```

Start the browser workspace only for browser validation and use the canonical
origin:

```sh
allbert serve > "$EVIDENCE_ROOT/macos-serve.log" 2>&1 &
DAEMON_PID=$!
trap 'kill "$DAEMON_PID" 2>/dev/null || true' EXIT INT TERM
for _ in $(seq 1 30); do
  curl -fsS http://localhost:4000/health && break
  sleep 1
done
curl -fsS http://localhost:4000/health \
  | tee "$EVIDENCE_ROOT/macos-health.json"
allbert admin status 2>&1 | tee "$EVIDENCE_ROOT/macos-attach.log"
# Perform real configured-provider browser research at
# http://localhost:4000/workspace and record a redacted transcript/screenshot.
kill "$DAEMON_PID"
wait "$DAEMON_PID" 2>/dev/null || true
trap - EXIT INT TERM
```

The daemon must be fully stopped before the warm TUI proof:

```sh
allbert admin settings set channels.tui.identity_map \
  '[{"external_user_id":"default","user_id":"local","enabled":true}]'
allbert admin settings set channels.tui.enabled true
script "$EVIDENCE_ROOT/macos-tui.txt" allbert tui
```

Inside TUI run `/help`, `/status`, `/channels`, a real configured-provider
turn, and `/quit`. Then run the release’s ACP and configured channel-send
blocks and store `$EVIDENCE_ROOT/macos-acp.log` and
`$EVIDENCE_ROOT/macos-channel.log`. Missing credentials/identity are an
operator decision, not PASS.

Repeat installation through the exact-version curl path with another
disposable Home, then uninstall without `--purge` and prove the Home remains.
PASS: Homebrew and curl resolve 1.0.4; browser doctor/research, health/attach,
TUI, ACP, and configured channel all pass; no single-writer error occurs; Home
survives uninstall.
