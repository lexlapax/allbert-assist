## Purpose

Deepen the intent subsystem so a chat-primary conversation surface (v0.55) routes
reliably. Today intent is per-turn proposal infrastructure (`Intent.Engine.decide/1`,
`IntentAgent`, advisory `Classifier`, `Descriptor`/`Handoff`); model output is
advisory only and never grants authority. v0.54 strengthens classification,
multi-turn / clarification handling, and disambiguation so natural chat works
well — without changing the authority boundary.

Sequenced **before** the v0.55 Web UX redo because chat quality depends on intent.

## Scope (to be detailed in Phase B / R2)
- Define concrete "deeper" goals: classification accuracy, multi-turn context,
  clarification UX, disambiguation, follow-up handling for chat-primary use.
- Identify ADR 0019/0034 limits to lift (model output stays advisory; intent
  never authorizes; `active_app` stays ranking context only).
- Acceptance + eval rows + `release.v054` lane.

## Non-Goals (preliminary)
- No change to the Security Central / action-boundary authority model.
- No model-output-as-authority; no intent-driven permission grants.
