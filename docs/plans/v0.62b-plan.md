Request flow: `docs/plans/v0.62b-request-flow.md`

Status: **released** as a source/docs point release, tagged `v0.62.1` (annotated,
`[skip-artifacts]`) on 2026-07-07. v0.62.0 remains the GitHub Latest packaged
product release because it carries the tarballs, latest aliases, and
`SHA256SUMS`; **no packaged GitHub Release is created for v0.62.1** — the
`release-artifacts.yml` `gate` job enforces this (the `[skip-artifacts]` tag

distribution closeout and turns the reusable parts of the v0.62 release
operation into a repeatable operator/developer pattern: Homebrew tap fill,
package-manager install proof, packaged TUI transcript, both Linux Docker
rehearsals, and evidence taxonomy. It does not renumber v0.63, v0.64, or v1.0,
and it does not introduce new authority, permission atoms, Settings keys,
runtime actions, install trust semantics, or product capability.

## Purpose

v0.62 implemented the packaging and entry-point substrate. The final release
operation exposed a separate need: some work is **release-instance work** (exact
tag, exact checksums, exact tap commit), while some work is **reusable release
ops** that future versions should not rediscover from terminal scrollback.

v0.62b closes that loop. It is deliberately narrower than a product milestone:
make the shipped v0.62.0 distribution path inspectable from tap to installed
binary, rehearse the packaged TUI and both Linux artifacts, and capture the
handoff pattern future packaging releases should copy.

## Reuse Assessment

Approximately 70-80% of the remaining release-operation work is reusable and
belongs in durable docs:

- release/tag sanity checks: release exists, `latest` points at the intended
  version, annotated tag peels to the intended commit, no stale RC tag/release
  remains;
- tap fill and validation: download `SHA256SUMS`, update formula checksums,
  audit, install, test, uninstall, and record the tap commit;
- packaged install proof: verify `allbert --version`, `allbert admin status`,
  daemon health, attach-first behavior, uninstall, and Allbert Home preservation;
- packaged TUI transcript: run one TTY-backed session through the packaged
  binary, not a cold Mix task, and keep a redacted transcript;
- Linux rehearsal taxonomy: separate source/CI evidence from real-host service
  and vault evidence, and treat Docker rehearsals as containerized package
  smokes rather than substitutes for all host integration;
- evidence ledger: store command, platform, artifact, result, skipped/best-effort
  reason, evidence path, and follow-up owner in one place.

The remaining 20-30% is v0.62.0-specific and should stay local to this release
record: exact artifact checksums, exact GitHub release id, exact tag object, exact
tap commit, and the v0.62 M8 remediation history.

## Scope

In scope:

- Homebrew tap fill for `lexlapax/homebrew-allbert` from the published v0.62.0
  `SHA256SUMS`.
- Homebrew audit/install/test/uninstall proof on the operator machine.
- Packaged `allbert tui` transcript with disposable `ALLBERT_HOME`.
- Docker-based Linux package rehearsals for **both** Linux artifacts:
  `linux-arm64` native on Apple Silicon Docker, and `linux-x64` under
  `--platform linux/amd64`.
- Evidence taxonomy and reusable command pattern updates in the release
  rehearsal and test-strategy docs.
- Changelog, README, roadmap, and agent-context-map reconciliation so v0.62.0 is
  described as shipped and v0.62b is the follow-up owner for closeout evidence.

Out of scope:

- New runtime behavior, new CLI commands, new TUI commands, new Settings Central
  keys, or new permissions.
- Changing v0.62's trust model. Installer-side signature verification remains a
  v0.64 trusted-install intake item; v0.62/v0.62b keep SHA256 verification plus optional
  out-of-band cosign verification.
- Treating Docker as a replacement for real-host Linux service/vault validation.
  Docker can prove install layout, packaged binary startup, health, attach, and
  uninstall behavior. Secret Service and systemd `--user` may pass in a container
  only when the container explicitly provides those host services; otherwise they
  are recorded as skipped with a reason.
- Creating packaged GitHub Release artifacts for `v0.62.1`. If/when cut,
  v0.62.1 is a source/docs tag; `v0.62.0` remains the packaged product artifact
  and GitHub Latest release for installer/Homebrew resolution.

### M6: Closeout Decision

**Status:** done for implementation and ready to tag, but release is deferred
per operator direction (2026-07-07). v0.62b stages source/docs point-release
version metadata `0.62.1` because the rehearsals produced release-facing
runbook/script fixes: the Docker Linux package smoke must run the packaged
runtime as non-root, and Apple Silicon x64 emulation needs the recorded BEAM JIT
memory flag. No GitHub Release/assets are planned for `v0.62.1`; keeping
`v0.62.0` as GitHub Latest protects `install.sh` `latest` downloads and the
Homebrew artifact contract.

**"No packaged release" is ENFORCED, not just intended (release closeout,
2026-07-07).** `release-artifacts.yml` triggers on every `v*` tag, so a bare
`v0.62.1` tag would otherwise build + publish a packaged release and — being
non-prerelease and newer than v0.62.0 — steal GitHub "Latest", redirecting
`install.sh latest` and Homebrew. A `gate` job now reads the annotated tag
message: a tag whose message contains `[skip-artifacts]` skips build/publish
entirely, so no packaged Release is created and Latest is not moved. The
`v0.62.1` tag carries `[skip-artifacts]`; product-release tags (no marker) build
+ publish unchanged. `install_path_test` asserts the gate; `release-rehearsal.md`
documents the point-release tag form.

Decision: v0.62b staged a source/docs point-release candidate because the
rehearsals produced a release-facing smoke-script guard and durable operator docs.
On operator direction (2026-07-07) release closeout resumed: the `[skip-artifacts]`
gate landed, then `v0.62.1` was tagged + pushed as a source/docs point release
(no packaged GitHub Release). v0.62.0 remains the shipped packaged product
artifact and GitHub Latest.

Acceptance:

- changelog and roadmap state the staged-not-released disposition;
- focused stale-test fixes pass;
- release/docs gates pass or any residual blocker is recorded before commit;
- no tag or GitHub Release is created during this commit-only closeout.

## Vision Check

v0.62b preserves the v0.62 vision rather than adding a new product surface. The
center remains the packaged runtime/action spine: install and launch paths change
how the operator reaches Allbert, not what any surface may do. The TUI rehearsal
must prove the same operator-only slash reads from the packaged binary. Docker and
Homebrew rehearsals are evidence of distribution integrity, not authority grants.
